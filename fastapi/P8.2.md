# P8.2 — Full CI/CD Pipeline

**Phase:** 8 — Deployment & CI/CD
**Topic:** GitHub Actions, PR checks, staging vs production, tag-based deploys
**Type:** Project

---

## Goal

Build a complete CI/CD pipeline with PR quality gates, automatic staging deploys on merge to main, and production deploys triggered by version tags.

## What to Build

- **PR checks**: lint, type check, unit tests (fast, no DB)
- **Merge to main**: full test suite + deploy to staging environment
- **Version tag** (`v*`): build production image + deploy to production
- Environment-specific configs via GitHub Environments
- Slack/Discord notification on deploy success/failure
- Automatic rollback on health check failure post-deploy

## Architecture

```
.github/
├── workflows/
│   ├── pr-checks.yml       # fast checks on every PR
│   ├── staging-deploy.yml  # full tests + staging deploy on main
│   └── production-deploy.yml # production deploy on tag
└── actions/
    └── deploy-and-verify/
        └── action.yml      # reusable deploy action
```

## Implementation

### .github/workflows/pr-checks.yml

```yaml
name: PR Checks

on:
  pull_request:
    branches: [main, develop]

concurrency:
  group: pr-${{ github.event.pull_request.number }}
  cancel-in-progress: true

jobs:
  lint-and-typecheck:
    name: Lint & Type Check
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
          cache: pip
      - run: pip install ruff mypy
      - run: ruff check app/ tests/
      - run: ruff format --check app/ tests/
      - run: mypy app/ --ignore-missing-imports

  unit-tests:
    name: Unit Tests
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
          cache: pip
      - run: pip install -r requirements.txt -r requirements-dev.txt
      - run: pytest tests/unit/ -m unit --tb=short
        env:
          SECRET_KEY: test-secret

  security-scan:
    name: Security Scan
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
          cache: pip
      - run: pip install bandit safety
      - run: bandit -r app/ -ll
      - run: safety check -r requirements.txt
```

### .github/workflows/staging-deploy.yml

```yaml
name: Staging Deploy

on:
  push:
    branches: [main]

jobs:
  full-test-suite:
    name: Full Test Suite
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16-alpine
        env:
          POSTGRES_USER: postgres
          POSTGRES_PASSWORD: postgres
          POSTGRES_DB: test_db
        options: >-
          --health-cmd pg_isready
          --health-interval 5s
          --health-retries 5
        ports: ["5432:5432"]
      redis:
        image: redis:7-alpine
        options: --health-cmd "redis-cli ping" --health-interval 5s
        ports: ["6379:6379"]

    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
          cache: pip
      - run: pip install -r requirements.txt -r requirements-dev.txt
      - name: Run migrations
        run: alembic upgrade head
        env:
          DATABASE_URL: postgresql://postgres:postgres@localhost:5432/test_db
      - run: pytest tests/ --cov=app --cov-report=xml --cov-fail-under=85
        env:
          DATABASE_URL: postgresql+asyncpg://postgres:postgres@localhost:5432/test_db
          REDIS_URL: redis://localhost:6379/0
          SECRET_KEY: ci-secret-key
      - uses: codecov/codecov-action@v4
        with:
          file: coverage.xml

  build-staging-image:
    needs: full-test-suite
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    outputs:
      image-tag: ${{ steps.meta.outputs.tags }}
      image-digest: ${{ steps.build.outputs.digest }}

    steps:
      - uses: actions/checkout@v4
      - uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - id: meta
        uses: docker/metadata-action@v5
        with:
          images: ghcr.io/${{ github.repository }}
          tags: |
            type=sha,prefix=staging-
            type=raw,value=staging-latest

      - id: build
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  deploy-staging:
    needs: build-staging-image
    runs-on: ubuntu-latest
    environment:
      name: staging
      url: https://staging.myapp.com

    steps:
      - uses: actions/checkout@v4
      - name: Deploy to staging
        run: |
          npx @railway/cli@latest up \
            --service myapp-staging \
            --image ghcr.io/${{ github.repository }}:staging-latest
        env:
          RAILWAY_TOKEN: ${{ secrets.RAILWAY_STAGING_TOKEN }}

      - name: Wait for deployment
        run: sleep 30

      - name: Smoke test staging
        run: |
          curl -f https://staging.myapp.com/health | jq '.status'
          curl -f https://staging.myapp.com/ready

      - name: Notify Slack on success
        if: success()
        uses: slackapi/slack-github-action@v1.26
        with:
          payload: |
            {"text": "Staging deploy successful :white_check_mark: — ${{ github.sha }}"}
        env:
          SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}

      - name: Notify Slack on failure
        if: failure()
        uses: slackapi/slack-github-action@v1.26
        with:
          payload: |
            {"text": "Staging deploy FAILED :x: — ${{ github.sha }} — <${{ github.server_url }}/${{ github.repository }}/actions/runs/${{ github.run_id }}|View logs>"}
        env:
          SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}
```

### .github/workflows/production-deploy.yml

```yaml
name: Production Deploy

on:
  push:
    tags:
      - "v[0-9]+.[0-9]+.[0-9]+"

jobs:
  build-production-image:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    outputs:
      version: ${{ github.ref_name }}

    steps:
      - uses: actions/checkout@v4
      - uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: |
            ghcr.io/${{ github.repository }}:${{ github.ref_name }}
            ghcr.io/${{ github.repository }}:latest
          cache-from: type=gha

  deploy-production:
    needs: build-production-image
    runs-on: ubuntu-latest
    environment:
      name: production
      url: https://myapp.com
    concurrency:
      group: production-deploy
      cancel-in-progress: false

    steps:
      - uses: actions/checkout@v4

      - name: Deploy to production
        id: deploy
        run: |
          npx @railway/cli@latest up \
            --service myapp-prod \
            --image ghcr.io/${{ github.repository }}:${{ github.ref_name }}
        env:
          RAILWAY_TOKEN: ${{ secrets.RAILWAY_PROD_TOKEN }}

      - name: Verify production health
        id: health-check
        run: |
          for i in {1..10}; do
            STATUS=$(curl -sf https://myapp.com/health | jq -r '.status' || echo "error")
            if [ "$STATUS" = "healthy" ]; then
              echo "Health check passed"
              exit 0
            fi
            echo "Attempt $i: status=$STATUS, retrying..."
            sleep 10
          done
          echo "Health check failed after 10 attempts"
          exit 1

      - name: Create GitHub Release
        if: success()
        uses: softprops/action-gh-release@v1
        with:
          tag_name: ${{ github.ref_name }}
          generate_release_notes: true

      - name: Notify on success
        if: success()
        uses: slackapi/slack-github-action@v1.26
        with:
          payload: '{"text": "Production deploy ${{ github.ref_name }} successful :rocket:"}'
        env:
          SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}
```

### .github/actions/deploy-and-verify/action.yml

```yaml
name: Deploy and Verify
description: Deploy image and run health checks with rollback on failure

inputs:
  image:
    required: true
  service-name:
    required: true
  health-url:
    required: true
  railway-token:
    required: true

runs:
  using: composite
  steps:
    - name: Deploy
      shell: bash
      run: npx @railway/cli@latest up --service ${{ inputs.service-name }} --image ${{ inputs.image }}
      env:
        RAILWAY_TOKEN: ${{ inputs.railway-token }}

    - name: Health check
      shell: bash
      run: |
        for i in {1..12}; do
          if curl -sf ${{ inputs.health-url }} > /dev/null; then exit 0; fi
          sleep 5
        done
        exit 1
```
