## Goal
Use FastAPI's dependency injection system to share logic across routes, manage resources cleanly, and make code testable.

## Background

### What is Dependency Injection
Dependency injection is a pattern where a function (the route handler) declares what it needs (its dependencies), and a framework (FastAPI) provides those things. Instead of your route function creating a database connection, it declares it needs one, and FastAPI creates and provides it.

The main benefits: reusable logic (declare once, use in many routes), clean resource management (database sessions opened and closed correctly), and testability (swap real dependencies for fake ones in tests).

### How FastAPI Resolves Dependencies
When FastAPI receives a request for a route, it inspects the route function's parameters. Any parameter typed as `Depends(some_function)` triggers FastAPI to call `some_function` first, and inject its return value into the route parameter. Dependencies can have their own dependencies (sub-dependencies), forming a dependency graph. FastAPI resolves the graph automatically, runs each dependency once per request (unless `use_cache=False`), and handles cleanup via `yield`.

### yield Dependencies
Dependencies that use `yield` are like context managers. Code before `yield` runs at the start of the request (setup), the yielded value is injected, and code after `yield` runs when the response is sent (teardown). This is perfect for database sessions: open before, yield the session, close after.

## Tasks

### Task 1: Basic Depends()
```python
# deps.py
from fastapi import Depends, Header, HTTPException
from typing import Annotated

# A simple dependency — gets the current user from a header
async def get_current_user(x_user_id: Annotated[str | None, Header()] = None) -> dict:
    if x_user_id is None:
        raise HTTPException(status_code=401, detail="X-User-Id header required")
    # In real code, you'd look up the user in the database
    return {"id": x_user_id, "name": f"User {x_user_id}"}

# Usage in a route
from fastapi import FastAPI

app = FastAPI()

@app.get("/profile")
async def get_profile(
    current_user: Annotated[dict, Depends(get_current_user)]
):
    return {"profile": current_user}

@app.get("/settings")
async def get_settings(
    current_user: Annotated[dict, Depends(get_current_user)]
):
    return {"settings": {"user": current_user, "theme": "dark"}}
```
The same `get_current_user` dependency is shared across both routes. If the header is missing, both routes automatically return 401.

### Task 2: yield Dependencies (Resource Management)
```python
# database.py
from sqlalchemy.ext.asyncio import AsyncSession, create_async_engine, async_sessionmaker
from typing import AsyncGenerator

DATABASE_URL = "postgresql+asyncpg://user:password@localhost/mydb"

engine = create_async_engine(DATABASE_URL, echo=True)
AsyncSessionLocal = async_sessionmaker(engine, expire_on_commit=False)

async def get_db() -> AsyncGenerator[AsyncSession, None]:
    async with AsyncSessionLocal() as session:
        try:
            yield session
            await session.commit()
        except Exception:
            await session.rollback()
            raise

# In routes:
from fastapi import Depends
from sqlalchemy.ext.asyncio import AsyncSession

DB = Annotated[AsyncSession, Depends(get_db)]

@app.get("/users")
async def list_users(db: DB):
    # db is a live AsyncSession, will be committed/rolled back after this returns
    result = await db.execute(select(User))
    return result.scalars().all()
```

### Task 3: Class-Based Dependencies
```python
# class_deps.py
from fastapi import Depends, Query
from typing import Annotated

class Pagination:
    def __init__(
        self,
        page: Annotated[int, Query(ge=1)] = 1,
        page_size: Annotated[int, Query(ge=1, le=100)] = 20,
    ):
        self.page = page
        self.page_size = page_size
        self.offset = (page - 1) * page_size

    def paginate(self, items: list) -> dict:
        total = len(items)
        return {
            "items": items[self.offset : self.offset + self.page_size],
            "total": total,
            "page": self.page,
            "page_size": self.page_size,
            "pages": (total + self.page_size - 1) // self.page_size,
        }


class Filtering:
    def __init__(
        self,

    ):
        self.search = search
        self.sort_by = sort_by
```

### Task 4: Sub-Dependencies
```python
# sub_deps.py
from fastapi import Depends, HTTPException, Header
from typing import Annotated

# Level 1: Extract raw token
async def get_token(authorization: Annotated[str | None, Header()] = None) -> str:
    if not authorization or not authorization.startswith("Bearer "):
        raise HTTPException(status_code=401, detail="Bearer token required")
    return authorization.removeprefix("Bearer ").strip()


# Level 2: Decode the token (depends on level 1)
async def get_token_data(token: Annotated[str, Depends(get_token)]) -> dict:
    # In real code: decode JWT here
    if token == "invalid":
        raise HTTPException(status_code=401, detail="Invalid token")
    return {"user_id": 42, "roles": ["user"]}


# Level 3: Get current user (depends on level 2)
async def get_current_user(
    token_data: Annotated[dict, Depends(get_token_data)]
) -> dict:
    # In real code: fetch user from DB using token_data["user_id"]
    return {"id": token_data["user_id"], "name": "Alice", "roles": token_data["roles"]}


# Level 4: Require admin (depends on level 3)
async def require_admin(
    current_user: Annotated[dict, Depends(get_current_user)]
) -> dict:
    if "admin" not in current_user.get("roles", []):
        raise HTTPException(status_code=403, detail="Admin access required")
    return current_user


@app.get("/admin/users")
async def admin_list_users(admin: Annotated[dict, Depends(require_admin)]):
    return {"admin": admin, "users": [...]}
```
FastAPI resolves the full chain: `require_admin` → `get_current_user` → `get_token_data` → `get_token`.

### Task 5: Overriding Dependencies in Tests
```python
# test_routes.py
import pytest
from fastapi.testclient import TestClient
from main import app
from deps import get_current_user

# Override the dependency to avoid real auth
def override_get_current_user():
    return {"id": "test-user", "name": "Test User", "roles": ["user"]}

# Test where user is an admin
def override_get_admin_user():
    return {"id": "admin-user", "name": "Admin", "roles": ["admin"]}


@pytest.fixture
def client():
    return TestClient(app)


@pytest.fixture
def authed_client():
    app.dependency_overrides[get_current_user] = override_get_current_user
    client = TestClient(app)
    yield client
    app.dependency_overrides.clear()


def test_profile_requires_auth(client):
    response = client.get("/profile")
    assert response.status_code == 401


def test_profile_with_auth(authed_client):
    response = authed_client.get("/profile")
    assert response.status_code == 200
    assert response.json()["profile"]["name"] == "Test User"
```
