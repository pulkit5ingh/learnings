## Goal
Produce rich, accurate OpenAPI documentation that serves as a contract between your API and its consumers.

## Background

### OpenAPI as a Contract
The OpenAPI specification (formerly Swagger) is a machine-readable description of your REST API. FastAPI generates it automatically from your code. Good OpenAPI docs serve two purposes: interactive exploration (Swagger UI at `/docs`, ReDoc at `/redoc`) and code generation (clients, SDKs, mocks). The goal is that a developer can understand your API from the docs alone.

### What FastAPI Generates Automatically
FastAPI reads your type hints, Pydantic models, and route decorators to generate path definitions, parameter schemas, request body schemas, and response schemas. You enhance this with: `tags` for grouping, `summary` for the endpoint title, `description` for extended explanation, `responses` for documenting all possible responses (not just the success case), and `example` on fields for realistic examples.

### Common Documentation Mistakes
Not documenting error responses (your API can return more than 200). Not providing examples (makes Swagger UI much more useful). Using generic names like `Response` or `Item` that clash across routes. Not using tags to group related endpoints.

### Task 2: Route-Level Documentation
```python
# app/items/router.py
from fastapi import APIRouter, HTTPException
from pydantic import BaseModel, Field

router = APIRouter(prefix="/items", tags=["Items"])

class ItemResponse(BaseModel):
    id: int = Field(examples=[42])
    name: str = Field(examples=["Wireless Keyboard"])
    price: float = Field(examples=[49.99])
    in_stock: bool = Field(examples=[True])

class ErrorResponse(BaseModel):
    detail: str

@router.get(
    "/{item_id}",
    response_model=ItemResponse,
    summary="Get an item by ID",
    description="""
Retrieve a single catalog item by its unique identifier.

Returns the complete item object including current stock status.
The price is in USD and includes any active discounts.
    """,
    responses={
        200: {
            "description": "Item found",
            "content": {
                "application/json": {
                    "example": {"id": 1, "name": "Widget", "price": 9.99, "in_stock": True}
                }
            },
        },
        404: {
            "model": ErrorResponse,
            "description": "Item not found",
            "content": {
                "application/json": {
                    "example": {"detail": "Item with id 999 not found"}
                }
            },
        },
        422: {"description": "Validation error — item_id must be a positive integer"},
    },
)
async def get_item(item_id: int):
    if item_id <= 0:
        raise HTTPException(status_code=422, detail="item_id must be positive")
    if item_id > 100:
        raise HTTPException(status_code=404, detail=f"Item with id {item_id} not found")
    return {"id": item_id, "name": "Widget", "price": 9.99, "in_stock": True}
```

### Task 3: Request Body Examples

# Multiple examples for a request body
from fastapi import Body
from typing import Annotated

class ItemCreate(BaseModel):
    name: str = Field(min_length=1, examples=["Bluetooth Headphones"])
    price: float = Field(gt=0, examples=[89.99])
    category: str = Field(examples=["Electronics"])
    in_stock: bool = Field(default=True)

@router.post(
    "/",
    response_model=ItemResponse,
    status_code=201,
    summary="Create a new item",
)
async def create_item(
    item: Annotated[
        ItemCreate,
        Body(
            openapi_examples={
                "electronics": {
                    "summary": "Electronics example",
                    "description": "A typical electronics item",
                    "value": {
                        "name": "Wireless Mouse",
                        "price": 29.99,
                        "category": "Electronics",

                    "summary": "Clothing example",

                        "name": "Cotton T-Shirt",
                        "price": 19.99,
                        "category": "Clothing",
                        "in_stock": False,
                    },
                },
            }
        ),
    ]
):
    return {"id": 1, **item.model_dump()}

### Task 6: Custom OpenAPI Schema
```python
# Customize the generated OpenAPI schema
from fastapi.openapi.utils import get_openapi

def custom_openapi():
    if app.openapi_schema:
        return app.openapi_schema
    
    openapi_schema = get_openapi(
        title=app.title,
        version=app.version,
        description=app.description,
        routes=app.routes,
    )
    
    # Add custom security scheme
    openapi_schema["components"]["securitySchemes"] = {
        "BearerAuth": {
            "type": "http",
            "scheme": "bearer",
            "bearerFormat": "JWT",
        },
        "ApiKeyAuth": {
            "type": "apiKey",
            "in": "header",
            "name": "X-API-Key",
        },
    }
    
    # Apply security globally to all routes
    openapi_schema["security"] = [{"BearerAuth": []}, {"ApiKeyAuth": []}]
    
    app.openapi_schema = openapi_schema
    return app.openapi_schema

app.openapi = custom_openapi
```
