## Background

### FastAPI Architecture
FastAPI is a modern Python web framework built on top of Starlette (ASGI) and Pydantic. It uses Python type hints to automatically generate input validation, serialization, and OpenAPI documentation. Every route decorator (`@app.get`, `@app.post`, etc.) maps an HTTP method + path to a Python function. FastAPI calls that function, validates inputs, and serializes the output.

### ASGI vs WSGI
Traditional Python frameworks (Django, Flask) use WSGI — synchronous, one request per worker. FastAPI uses ASGI — asynchronous, one event loop handles many concurrent requests. This is why FastAPI endpoints should be `async def`. You can use plain `def` in FastAPI, but FastAPI will run it in a thread pool, which has overhead.

### Path Operations
FastAPI provides decorators for all HTTP methods: `@app.get`, `@app.post`, `@app.put`, `@app.patch`, `@app.delete`. Each takes at minimum a path string. Path parameters are curly-brace tokens in the path that become function parameters. Query parameters are function parameters without a corresponding path token. Request body comes from a Pydantic `BaseModel` parameter.

### Response Models
The `response_model` parameter on a route decorator tells FastAPI what shape the response should be. FastAPI filters the return value through the response model, stripping any fields not in the model (this is how you prevent leaking internal data like passwords). If you return a SQLAlchemy model object, FastAPI will convert it to the response model.

### Task 2: Path and Query Parameters
```python
# routes/items.py
from fastapi import APIRouter, HTTPException, Query
from typing import Annotated

router = APIRouter(prefix="/items", tags=["items"])

ITEMS_DB: dict[int, dict] = {
    1: {"id": 1, "name": "Laptop", "price": 999.99},
    2: {"id": 2, "name": "Mouse", "price": 29.99},
    3: {"id": 3, "name": "Keyboard", "price": 79.99},
}

@router.get("/{item_id}")
async def get_item(item_id: int):
    if item_id not in ITEMS_DB:
        raise HTTPException(status_code=404, detail=f"Item {item_id} not found")
    return ITEMS_DB[item_id]

@router.get("/")
async def list_items(
    skip: Annotated[int, Query(ge=0)] = 0,
    limit: Annotated[int, Query(ge=1, le=100)] = 10,
    name_contains: str | None = None,
):
    items = list(ITEMS_DB.values())
    if name_contains:
        items = [i for i in items if name_contains.lower() in i["name"].lower()]
    return items[skip : skip + limit]
```
`Annotated[int, Query(ge=0)]` attaches validation metadata to the type hint. FastAPI reads it and returns a 422 if the constraint is violated.

### Task 3: Request Body with Pydantic
```python
# schemas.py
from pydantic import BaseModel, Field, field_validator

class ItemCreate(BaseModel):
    name: str = Field(min_length=1, max_length=100)
    price: float = Field(gt=0, description="Price in USD")
    in_stock: bool = True

    @field_validator("name")
    @classmethod
    def name_must_not_be_blank(cls, v: str) -> str:
        if v.strip() == "":
            raise ValueError("Name cannot be blank or whitespace only")
        return v.strip()


class ItemUpdate(BaseModel):
    name: str | None = Field(default=None, min_length=1, max_length=100)
    price: float | None = Field(default=None, gt=0)
    in_stock: bool | None = None


class ItemResponse(BaseModel):
    id: int
    name: str
    price: float
    in_stock: bool
```

```python
# routes/items.py (continued)
from schemas import ItemCreate, ItemUpdate, ItemResponse

_next_id = 4

@router.post("/", response_model=ItemResponse, status_code=201)
async def create_item(item: ItemCreate):
    global _next_id
    new_item = {"id": _next_id, **item.model_dump()}
    ITEMS_DB[_next_id] = new_item
    _next_id += 1
    return new_item

@router.patch("/{item_id}", response_model=ItemResponse)
async def update_item(item_id: int, updates: ItemUpdate):
    if item_id not in ITEMS_DB:
        raise HTTPException(status_code=404, detail="Item not found")
    # Only update fields that were explicitly provided
    update_data = updates.model_dump(exclude_unset=True)
    ITEMS_DB[item_id].update(update_data)
    return ITEMS_DB[item_id]

@router.delete("/{item_id}", status_code=204)
async def delete_item(item_id: int):
    if item_id not in ITEMS_DB:
        raise HTTPException(status_code=404, detail="Item not found")
    del ITEMS_DB[item_id]
    # Return None for 204 No Content
```

### Task 4: Response Models and Status Codes
```python
# Understanding response_model filtering
from pydantic import BaseModel

class UserInternal(BaseModel):
    id: int
    email: str
    hashed_password: str  # NEVER expose this

class UserResponse(BaseModel):
    id: int
    email: str
    # hashed_password is intentionally absent

# In your route:
@app.get("/users/{user_id}", response_model=UserResponse)
async def get_user(user_id: int):
    # Even if you return UserInternal with hashed_password,
    # FastAPI strips fields not in UserResponse
    return UserInternal(id=1, email="a@b.com", hashed_password="$2b$secret")
```

Common status codes to know:
- 200 OK — default for GET, PUT, PATCH
- 201 Created — use for POST that creates a resource
- 204 No Content — use for DELETE (return None)
- 400 Bad Request — malformed request (not validation error)
- 401 Unauthorized — not authenticated
- 403 Forbidden — authenticated but not allowed
- 404 Not Found — resource doesn't exist
- 409 Conflict — e.g., duplicate resource
- 422 Unprocessable Entity — FastAPI validation error (automatic)
- 500 Internal Server Error — unhandled exception
