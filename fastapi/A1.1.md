## Goal

## Background

### The Event Loop
Python's `asyncio` module provides a single-threaded concurrency model based on an event loop. The event loop runs coroutines — functions defined with `async def` — by switching between them whenever one yields control with `await`. This means you can handle many concurrent I/O operations (HTTP requests, DB queries, file reads) without threads.

The key insight is that `await` does not block the thread. It tells the event loop: "I'm waiting for this operation. Go run something else, and come back to me when it's done." This is cooperative multitasking.

### Coroutines vs Threads
A regular Python thread blocks when doing I/O. With 1000 concurrent requests, you would need 1000 threads — expensive. With async, you have one thread and one event loop, and the event loop efficiently switches between 1000 coroutines while they wait for I/O. FastAPI uses this model exclusively.

### When to Use Sync vs Async
If a function does CPU-heavy work (image processing, large computations), make it sync and run it in a thread pool executor. If it does I/O (DB queries, HTTP calls, file operations), make it async. Mixing incorrectly — calling a blocking function inside `async def` — will stall your entire server.

### Task 2: Sequential vs Concurrent
```python
# task2.py
import asyncio
import time

async def fetch_data(source: str, delay: float) -> str:
    print(f"Start fetching from {source}")
    await asyncio.sleep(delay)
    print(f"Done fetching from {source}")
    return f"Data from {source}"

async def sequential():
    start = time.perf_counter()
    r1 = await fetch_data("db", 1.0)
    r2 = await fetch_data("api", 1.5)
    r3 = await fetch_data("cache", 0.5)
    elapsed = time.perf_counter() - start
    print(f"Sequential took {elapsed:.2f}s")  # ~3.0s
    return [r1, r2, r3]

async def concurrent():
    start = time.perf_counter()
    results = await asyncio.gather(
        fetch_data("db", 1.0),
        fetch_data("api", 1.5),
        fetch_data("cache", 0.5),
    )
    elapsed = time.perf_counter() - start
    print(f"Concurrent took {elapsed:.2f}s")  # ~1.5s
    return results

asyncio.run(sequential())
asyncio.run(concurrent())
```
Understand why concurrent is faster. `asyncio.gather` schedules all coroutines at once and waits for all to finish.

### Task 3: asyncio.Task
```python
# task3.py
import asyncio

async def worker(name: str, duration: float) -> str:
    print(f"{name}: starting")
    await asyncio.sleep(duration)
    print(f"{name}: done after {duration}s")
    return f"{name} result"

async def main():
    # Create tasks explicitly — they start running immediately
    task_a = asyncio.create_task(worker("A", 2.0))
    task_b = asyncio.create_task(worker("B", 1.0))
    
    print("Tasks created, doing other work...")
    await asyncio.sleep(0.1)
    print("Back to waiting for tasks")
    
    # Wait for both
    result_a = await task_a
    result_b = await task_b
    print(result_a, result_b)

asyncio.run(main())
```
Notice B finishes before A even though A was created first.

### Task 4: Error Handling in Concurrent Code
```python
# task4.py
import asyncio

async def might_fail(name: str, should_fail: bool) -> str:
    await asyncio.sleep(0.5)
    if should_fail:
        raise ValueError(f"{name} failed!")
    return f"{name} succeeded"

async def gather_with_errors():
    # return_exceptions=True: exceptions are returned as values, not raised
    results = await asyncio.gather(
        might_fail("task1", False),
        might_fail("task2", True),
        might_fail("task3", False),
        return_exceptions=True,
    )
    for result in results:
        if isinstance(result, Exception):
            print(f"Error: {result}")
        else:
            print(f"Success: {result}")

async def taskgroup_example():
    # Python 3.11+: TaskGroup (preferred for structured concurrency)
    results = []
    try:
        async with asyncio.TaskGroup() as tg:
            t1 = tg.create_task(might_fail("task1", False))
            t2 = tg.create_task(might_fail("task2", True))  # will fail
    except* ValueError as eg:
        print(f"Got errors: {eg.exceptions}")

asyncio.run(gather_with_errors())
asyncio.run(taskgroup_example())
```

### Task 5: Blocking Code in Async Context
```python
# task5.py
import asyncio
import time
from concurrent.futures import ThreadPoolExecutor

def cpu_heavy(n: int) -> int:
    """Blocking CPU work — must NOT be called directly in async context."""
    result = sum(range(n))
    time.sleep(0.5)  # simulating slow computation
    return result

async def wrong_way():
    # BAD: blocks the event loop for 0.5s per call
    result = cpu_heavy(1_000_000)
    return result

async def right_way():
    # GOOD: offload to thread pool
    loop = asyncio.get_running_loop()
    result = await loop.run_in_executor(None, cpu_heavy, 1_000_000)
    return result

async def main():
    # These run concurrently because run_in_executor doesn't block the loop
    r1, r2 = await asyncio.gather(right_way(), right_way())
    print(r1, r2)

asyncio.run(main())
```

### Task 6: Async Context Managers and Iterators
```python
# task6.py
import asyncio

class AsyncDB:
    """Simulates an async database connection."""

    async def __aenter__(self):
        print("Connecting to DB...")
        await asyncio.sleep(0.1)
        return self

    async def __aexit__(self, exc_type, exc_val, exc_tb):
        print("Closing DB connection")
        return False

    async def query(self, sql: str) -> list[dict]:
        await asyncio.sleep(0.2)
        return [{"id": 1, "name": "Alice"}, {"id": 2, "name": "Bob"}]


async def async_row_generator(rows: list[dict]):
    for row in rows:
        await asyncio.sleep(0.05)  # simulate processing per row
        yield row


async def main():
    async with AsyncDB() as db:
        rows = await db.query("SELECT * FROM users")
        async for row in async_row_generator(rows):
            print(row)

asyncio.run(main())
```
