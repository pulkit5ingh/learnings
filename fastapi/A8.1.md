## Goal
Build a minimal, secure Docker image for a FastAPI application with multi-stage builds and production-ready server configuration.

## Background

### Multi-Stage Builds
A multi-stage Dockerfile uses multiple `FROM` instructions. Early stages (builder) install build dependencies and compile packages. The final stage copies only the built artifacts — no build tools, compilers, or dev dependencies. This significantly reduces the final image size (often from 1GB to under 100MB) and reduces the attack surface.

### uvicorn vs gunicorn
Uvicorn is an ASGI server that runs a single-process event loop. For production, you run multiple uvicorn workers managed by Gunicorn (a WSGI process manager). Gunicorn acts as a process supervisor: it starts N worker processes, monitors them, and restarts crashed workers. The `uvicorn.workers.UvicornWorker` class is the bridge. Each worker handles many concurrent requests via the event loop.

Workers = number of CPU cores. For I/O-heavy FastAPI apps, `(2 × CPU cores) + 1` is the standard formula, but profile your specific app.

### Non-Root User
Running as root in a container is a security risk. If an attacker exploits your application, they have root access in the container. Always create a non-root user and switch to it before the CMD.

## Tasks

### Task 1: Minimal Dockerfile
```dockerfile
# Dockerfile
FROM python:3.11-slim

WORKDIR /app

# Install dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application
COPY . .

# Run
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```
This is a starting point — we'll improve it in subsequent tasks.

### Task 2: Multi-Stage Build with uv
```dockerfile
# Dockerfile (multi-stage with uv)
# Stage 1: Builder — installs dependencies
FROM python:3.11-slim AS builder

# Install uv
RUN pip install uv

WORKDIR /app

# Copy only dependency files first (for layer caching)
COPY pyproject.toml uv.lock ./

# Install dependencies into a virtual environment
RUN uv sync --frozen --no-dev

# Stage 2: Runtime — only the application
FROM python:3.11-slim AS runtime

WORKDIR /app

# Copy the virtual environment from builder
COPY --from=builder /app/.venv /app/.venv

# Copy application code
COPY app/ ./app/
COPY alembic/ ./alembic/
COPY alembic.ini .

# Create non-root user
RUN adduser --disabled-password --gecos "" --uid 1001 appuser
USER appuser

# Ensure .venv/bin is in PATH
ENV PATH="/app/.venv/bin:$PATH"

EXPOSE 8000

# Health check
HEALTHCHECK --interval=30s --timeout=10s --start-period=40s --retries=3 \
    CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')"

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### Task 4: Gunicorn + Uvicorn Workers for Production
```dockerfile
# Production Dockerfile with Gunicorn
FROM python:3.11-slim AS builder
RUN pip install uv
WORKDIR /app
COPY pyproject.toml uv.lock ./
RUN uv sync --frozen --no-dev

FROM python:3.11-slim AS runtime
WORKDIR /app
COPY --from=builder /app/.venv /app/.venv
COPY app/ ./app/
COPY alembic/ ./alembic/
COPY alembic.ini .

RUN adduser --disabled-password --gecos "" --uid 1001 appuser && \
    chown -R appuser:appuser /app
USER appuser

ENV PATH="/app/.venv/bin:$PATH"
ENV PYTHONPATH="/app"
ENV PYTHONDONTWRITEBYTECODE=1
ENV PYTHONUNBUFFERED=1

EXPOSE 8000

HEALTHCHECK --interval=30s --timeout=10s --start-period=40s --retries=3 \
    CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')"

# gunicorn config
ENV WEB_CONCURRENCY=4  # number of worker processes

CMD ["gunicorn", "app.main:app", \
     "--worker-class", "uvicorn.workers.UvicornWorker", \
     "--workers", "4", \
     "--bind", "0.0.0.0:8000", \
     "--timeout", "120", \
     "--keepalive", "5", \
     "--max-requests", "1000", \
     "--max-requests-jitter", "100"]
```

### Task 5: Gunicorn Config File
```python
# gunicorn.conf.py
import multiprocessing
import os

# Number of workers
workers = int(os.getenv("WEB_CONCURRENCY", multiprocessing.cpu_count() * 2 + 1))

# Worker class
worker_class = "uvicorn.workers.UvicornWorker"

# Bind
bind = f"0.0.0.0:{os.getenv('PORT', '8000')}"

# Timeouts
timeout = 120
keepalive = 5

# Prevent memory leaks by recycling workers
max_requests = 1000
max_requests_jitter = 100

# Logging
accesslog = "-"   # stdout
errorlog = "-"    # stderr
loglevel = "info"

# Preload app before forking workers (saves memory with copy-on-write)
preload_app = True
```

```dockerfile
# In Dockerfile CMD:
CMD ["gunicorn", "app.main:app", "--config", "gunicorn.conf.py"]
```

### Task 6: Health Check Endpoint
```python
# app/main.py
from fastapi import FastAPI
from app.database import engine
from sqlalchemy import text

app = FastAPI()

@app.get("/health")
async def health_check():
    """Kubernetes/Docker health check endpoint."""
    return {"status": "ok"}

@app.get("/health/ready")
async def readiness_check():
    """Readiness probe: checks if app can handle traffic (DB connected)."""
    try:
        async with engine.connect() as conn:
            await conn.execute(text("SELECT 1"))
        return {"status": "ready", "db": "connected"}
    except Exception as e:
        from fastapi import HTTPException
        raise HTTPException(status_code=503, detail=f"Database unavailable: {e}")

@app.get("/health/live")
async def liveness_check():
    """Liveness probe: checks if app process is alive."""
    return {"status": "alive"}
```
