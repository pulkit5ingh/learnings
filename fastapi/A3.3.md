**Phase:** 3 — Authentication & Authorization
**Topic:** roles table, permission checking, Depends() for role guards, admin vs user vs guest
**Type:** Assignment

## Goal
Implement a flexible RBAC system where roles are stored in the database and enforced via reusable FastAPI dependencies.

## Background

### RBAC vs ABAC
Role-Based Access Control (RBAC) assigns permissions to roles, and roles to users. It is simple and works well for most applications: "admin can do everything, user can manage their own data, guest can only read public data." Attribute-Based Access Control (ABAC) is more complex — permissions depend on attributes of the user, resource, and environment. Start with RBAC; move to ABAC only if RBAC becomes too rigid.

### Storing Roles
Roles can be stored as an enum column on the user table (simple, but inflexible) or as a separate roles table with a user_roles join table (flexible, supports multiple roles per user). For most applications, a simple string or enum field per user is sufficient. For complex applications with per-resource roles (e.g., "admin of organization X"), you need a roles table.

### Roles in JWT
Embedding roles in the JWT (as a claim) allows role checking without a database query on every request. The tradeoff: if you revoke a role, the change doesn't take effect until the JWT expires. For most applications, this is acceptable. If immediate revocation is needed, check roles in the database on every request.

import enum
from sqlalchemy import Enum as SAEnum, String, Integer, Boolean, ForeignKey

class UserRole(str, enum.Enum):

class Base(DeclarativeBase):
    pass

### Task 2: Role-Checking Dependencies
```python
# app/auth/rbac.py
from fastapi import Depends, HTTPException, status
from typing import Annotated
from app.models import User, UserRole
from app.auth.dependencies import get_current_user

def require_role(*roles: UserRole):
    """
    Factory function that returns a dependency requiring one of the given roles.
    Usage: Depends(require_role(UserRole.ADMIN, UserRole.MODERATOR))
    """
    async def role_checker(
        current_user: Annotated[User, Depends(get_current_user)]
    ) -> User:
        if current_user.role not in roles:
            raise HTTPException(
                status_code=status.HTTP_403_FORBIDDEN,
                detail=f"Role '{current_user.role}' is not allowed. Required: {[r.value for r in roles]}",
            )
        return current_user
    return role_checker

# Convenient pre-built guards
RequireAdmin = Annotated[User, Depends(require_role(UserRole.ADMIN))]
RequireModerator = Annotated[User, Depends(require_role(UserRole.ADMIN, UserRole.MODERATOR))]
RequireUser = Annotated[User, Depends(require_role(UserRole.ADMIN, UserRole.MODERATOR, UserRole.USER))]
```

### Task 3: Resource Ownership Check
```python
# app/auth/ownership.py
from fastapi import HTTPException, status
from app.models import User, UserRole

def check_ownership_or_admin(resource_owner_id: int, current_user: User) -> None:
    """
    Raise 403 if the current user is neither the owner nor an admin.
    Use this for endpoints where users can only modify their own resources.
    """
    if current_user.role == UserRole.ADMIN:
        return  # admins can do anything
    if current_user.id != resource_owner_id:
        raise HTTPException(
            status_code=status.HTTP_403_FORBIDDEN,
            detail="You do not have permission to access this resource",
        )
```

### Task 4: Routes with Role Guards
```python
# app/admin/router.py
from fastapi import APIRouter, Depends
from sqlalchemy.ext.asyncio import AsyncSession
from sqlalchemy import select
from typing import Annotated

from app.deps import get_db
from app.models import User, UserRole
from app.auth.rbac import RequireAdmin, RequireModerator, require_role
from pydantic import BaseModel

router = APIRouter(prefix="/admin", tags=["Admin"])

class UserListItem(BaseModel):
    id: int
    email: str
    full_name: str
    role: UserRole
    is_active: bool

class RoleUpdateRequest(BaseModel):
    role: UserRole

# Only admins can list all users
@router.get("/users", response_model=list[UserListItem])
async def list_all_users(
    admin: RequireAdmin,
    db: Annotated[AsyncSession, Depends(get_db)],
):
    result = await db.execute(select(User))
    return result.scalars().all()

# Only admins can change roles
@router.patch("/users/{user_id}/role", response_model=UserListItem)
async def update_user_role(
    user_id: int,
    data: RoleUpdateRequest,
    admin: RequireAdmin,
    db: Annotated[AsyncSession, Depends(get_db)],
):
    from fastapi import HTTPException
    user = await db.get(User, user_id)
    if user is None:
        raise HTTPException(status_code=404, detail="User not found")
    user.role = data.role
    await db.flush()
    return user

# Moderators and admins can access moderation features
@router.get("/flagged-content")
async def get_flagged_content(
    moderator: RequireModerator,
):
    return {"flagged": []}
```

### Task 5: Content with Ownership + Role Check
```python
# app/posts/router.py
from fastapi import APIRouter, Depends, HTTPException
from sqlalchemy.ext.asyncio import AsyncSession
from typing import Annotated

from app.deps import get_db
from app.models import User, Post
from app.auth.dependencies import CurrentUser
from app.auth.ownership import check_ownership_or_admin

router = APIRouter(prefix="/posts", tags=["Posts"])
DB = Annotated[AsyncSession, Depends(get_db)]

@router.delete("/{post_id}", status_code=204)
async def delete_post(post_id: int, current_user: CurrentUser, db: DB):
    post = await db.get(Post, post_id)
    if post is None:
        raise HTTPException(status_code=404, detail="Post not found")
    
    # Only the author or an admin can delete a post
    check_ownership_or_admin(post.user_id, current_user)
    
    await db.delete(post)

@router.patch("/{post_id}")
async def update_post(post_id: int, data: dict, current_user: CurrentUser, db: DB):
    post = await db.get(Post, post_id)
    if post is None:
        raise HTTPException(status_code=404, detail="Post not found")
    
    check_ownership_or_admin(post.user_id, current_user)
    
    for key, value in data.items():
        setattr(post, key, value)
    return post
```

### Task 6: Role in JWT Claims
```python
# Embed role in the JWT so routes don't need a DB query for role checks
from app.auth.tokens import create_access_token

def create_tokens_with_role(user: User) -> dict:
    access_token = create_access_token(
        subject=user.id,
        additional_claims={"role": user.role.value},
    )
    return {"access_token": access_token, "token_type": "bearer"}

# Then in get_current_user, extract role from token (no DB needed):
async def get_current_user_fast(token: str) -> dict:
    payload = decode_access_token(token)
    return {
        "id": int(payload["sub"]),
        "role": payload.get("role", "user"),
    }
```
