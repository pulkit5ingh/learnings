## Goal
Measure and improve test coverage, organize tests into a clear hierarchy, and run tests automatically in CI.

## Background

### What Coverage Measures
Code coverage measures what percentage of your code lines are executed during tests. 90% coverage means 90% of lines are hit. It does NOT mean the code is bug-free — you can have 100% line coverage with meaningless assertions. Coverage is a useful signal for finding untested code paths, not a quality metric on its own. Branch coverage (testing both `if` and `else` paths) is more meaningful than line coverage.

### Test Hierarchy
- **Unit tests**: Test a single function or class in isolation. All dependencies are mocked. Fast (milliseconds). Should be the majority of your tests.
- **Integration tests**: Test multiple components working together, typically with a real database. Slower (seconds). Test that your SQL queries are correct, that your service layer interacts correctly with the repository, etc.
- **End-to-end (e2e) tests**: Test the entire system from HTTP request to database response. Slowest. Test the most critical user journeys.

### Test Pyramid
Write many unit tests, fewer integration tests, and very few e2e tests. This keeps the test suite fast and maintainable. A common mistake is writing only e2e tests — they catch real bugs but are slow, brittle, and hard to debug.

## Tasks

### Task 1: Setting Up Coverage
```bash
uv add pytest-cov
```

```toml
# pyproject.toml
[tool.pytest.ini_options]
testpaths = ["tests"]
asyncio_mode = "auto"
markers = [
    "unit: unit tests",
    "integration: integration tests",
    "e2e: end-to-end tests",
    "slow: slow tests",
]

[tool.coverage.run]
source = ["app"]
omit = [
    "app/migrations/*",
    "app/__init__.py",
    "*/conftest.py",
]
branch = true  # measure branch coverage too

[tool.coverage.report]
fail_under = 85  # fail if coverage drops below 85%
show_missing = true
exclude_lines = [
    "pragma: no cover",
    "if TYPE_CHECKING:",
    "def __repr__",
    "raise NotImplementedError",
]
```

```bash
# Run tests with coverage
pytest --cov=app --cov-report=term-missing --cov-report=html

# Open the HTML report
open htmlcov/index.html

# Fail if coverage is below threshold
pytest --cov=app --cov-fail-under=85
```

### Task 3: Unit Tests (No DB, No HTTP)
```python
# tests/unit/test_auth_service.py
import pytest
from unittest.mock import AsyncMock, MagicMock
from fastapi import HTTPException
from app.auth.service import AuthService
from app.models import User, UserRole
from app.auth.password import hash_password

@pytest.fixture
def mock_db():
    return AsyncMock()

@pytest.fixture
def auth_service(mock_db):
    return AuthService(mock_db)

@pytest.mark.unit
async def test_authenticate_success(auth_service, mock_db):
    fake_user = User(
        id=1,
        email="alice@test.com",
        hashed_password=hash_password("GoodPass123!"),
        is_active=True,
        role=UserRole.USER,
    )
    # Mock the DB query to return our fake user
    mock_result = MagicMock()
    mock_result.scalars().first.return_value = fake_user
    mock_db.execute = AsyncMock(return_value=mock_result)
    
    user = await auth_service.authenticate("alice@test.com", "GoodPass123!")
    assert user.email == "alice@test.com"

@pytest.mark.unit
async def test_authenticate_wrong_password(auth_service, mock_db):
    fake_user = User(
        id=1,
        email="alice@test.com",
        hashed_password=hash_password("GoodPass123!"),
        is_active=True,
        role=UserRole.USER,
    )
    mock_result = MagicMock()
    mock_result.scalars().first.return_value = fake_user
    mock_db.execute = AsyncMock(return_value=mock_result)
    
    with pytest.raises(HTTPException) as exc_info:
        await auth_service.authenticate("alice@test.com", "WrongPass!")
    assert exc_info.value.status_code == 401

@pytest.mark.unit
async def test_authenticate_user_not_found(auth_service, mock_db):
    mock_result = MagicMock()
    mock_result.scalars().first.return_value = None
    mock_db.execute = AsyncMock(return_value=mock_result)
    
    with pytest.raises(HTTPException) as exc_info:
        await auth_service.authenticate("nobody@test.com", "Any")
    assert exc_info.value.status_code == 401
```

### Task 4: Integration Tests (Real DB)
```python
# tests/integration/test_items_repository.py
import pytest
from app.items.repository import ItemRepository
from tests.factories import ItemFactory

@pytest.mark.integration
async def test_create_and_fetch_item(db_session):
    repo = ItemRepository(db_session)
    
    item = await repo.create(name="Test Item", price=9.99)
    assert item.id is not None
    
    fetched = await repo.get(item.id)
    assert fetched is not None
    assert fetched.name == "Test Item"
    assert fetched.price == 9.99

@pytest.mark.integration
async def test_list_items_pagination(db_session):
    repo = ItemRepository(db_session)
    
    # Create 15 items
    for i in range(15):
        await repo.create(name=f"Item {i}", price=float(i))
    
    page1 = await repo.get_all(limit=10, offset=0)
    page2 = await repo.get_all(limit=10, offset=10)
    
    assert len(page1) == 10
    assert len(page2) == 5

@pytest.mark.integration
async def test_delete_item(db_session):
    repo = ItemRepository(db_session)
    item = await repo.create(name="To Delete", price=1.0)
    
    deleted = await repo.delete(item.id)
    assert deleted == True
    
    fetched = await repo.get(item.id)
    assert fetched is None
```

### Task 5: E2E Tests (Full HTTP Flow)
```python
# tests/e2e/test_items_flow.py
import pytest

@pytest.mark.e2e
async def test_full_items_crud_flow(client, authed_client):
    # Create
    create_resp = await authed_client.post(
        "/items/",
        json={"name": "E2E Item", "price": 99.99},
    )
    assert create_resp.status_code == 201
    item_id = create_resp.json()["id"]
    
    # Read
    get_resp = await authed_client.get(f"/items/{item_id}")
    assert get_resp.status_code == 200
    assert get_resp.json()["name"] == "E2E Item"
    
    # Update
    update_resp = await authed_client.patch(
        f"/items/{item_id}",
        json={"price": 79.99},
    )
    assert update_resp.status_code == 200
    assert update_resp.json()["price"] == 79.99
    
    # Delete
    delete_resp = await authed_client.delete(f"/items/{item_id}")
    assert delete_resp.status_code == 204
    
    # Verify deleted
    get_after_delete = await authed_client.get(f"/items/{item_id}")
    assert get_after_delete.status_code == 404
```

### Task 6: GitHub Actions CI
```yaml
# .github/workflows/test.yml
name: Test

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest

    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_USER: testuser
          POSTGRES_PASSWORD: testpass
          POSTGRES_DB: testdb
        ports:
          - 5432:5432
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

    steps:
      - uses: actions/checkout@v4

      - uses: astral-sh/setup-uv@v3
        with:
          version: "latest"

      - name: Install dependencies
        run: uv sync --all-extras

      - name: Lint
        run: uv run ruff check .

      - name: Type check
        run: uv run mypy app/

      - name: Run tests
        env:
          DATABASE_URL: postgresql+asyncpg://testuser:testpass@localhost/testdb
          SECRET_KEY: test-secret-key-for-ci
        run: |
          uv run pytest -v \
            --cov=app \
            --cov-report=term-missing \
            --cov-fail-under=85 \
            -m "not slow"
```

## Deliverables
- [ ] `pytest --cov=app --cov-report=term-missing` shows coverage percentages per file.
- [ ] HTML coverage report generated in `htmlcov/` and shows branch coverage.
- [ ] Test directory has `unit/`, `integration/`, and `e2e/` subdirectories.
- [ ] Unit tests use mocks and run without a database (fast, < 1 second total).
- [ ] Integration tests use the `db_session` fixture with rollback isolation.
- [ ] `pytest -m unit` runs only unit tests, `pytest -m integration` runs integration tests.
- [ ] GitHub Actions workflow runs lint, type check, and tests with a PostgreSQL service.
- [ ] `--cov-fail-under=85` fails the pipeline if coverage drops below threshold.
