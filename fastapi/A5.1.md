## Goal
Offload slow or non-critical work from request handlers using FastAPI background tasks and Celery for heavy async processing.

## Background

### FastAPI BackgroundTasks vs Celery
FastAPI's `BackgroundTasks` runs a function in the same process after the response is sent. It is lightweight and requires no extra infrastructure — good for quick tasks like sending a welcome email or updating a counter. It cannot retry on failure, cannot be monitored, and runs in the same process (so heavy CPU work still blocks other requests). Celery is a separate worker process with a message broker (Redis/RabbitMQ). Tasks are queued, distributed across workers, retried on failure, and can be monitored. Use Celery for anything that takes > 1 second, needs retry, or must survive server restarts.

### Celery Architecture
Producer (FastAPI app) sends a task message to the broker (Redis). Worker process(es) pick up the message, execute the task, and optionally store the result in the result backend. Celery Beat is a scheduler that sends tasks on a cron schedule (e.g., "generate report every morning at 8 AM").

### Task Serialization
Celery serializes task arguments to JSON before sending them. This means you cannot pass SQLAlchemy objects as arguments. Pass simple types: IDs, strings, dicts. The worker then looks up the object from the database using the ID.

### Task 1: FastAPI BackgroundTasks
```python
# app/main.py
from fastapi import FastAPI, BackgroundTasks
from pydantic import BaseModel
import logging

logger = logging.getLogger(__name__)
app = FastAPI()

def send_welcome_email(email: str, name: str) -> None:
    """Runs after the response is sent. If it fails, user never knows."""
    import time
    time.sleep(2)  # simulate slow email sending
    logger.info(f"Welcome email sent to {email} ({name})")

def update_user_analytics(user_id: int, action: str) -> None:
    logger.info(f"Analytics: user {user_id} performed {action}")

class UserCreate(BaseModel):
    email: str
    name: str

@app.post("/users/", status_code=201)
async def create_user(
    user_data: UserCreate,
    background_tasks: BackgroundTasks,
):
    # Create user in DB (fast)
    new_user = {"id": 1, "email": user_data.email, "name": user_data.name}
    
    # Schedule background work (runs AFTER response is sent)
    background_tasks.add_task(send_welcome_email, user_data.email, user_data.name)
    background_tasks.add_task(update_user_analytics, new_user["id"], "register")
    
    # Response is sent immediately — user doesn't wait for emails
    return new_user
```
Test: `POST /users/` returns instantly. The email log appears 2 seconds later.

```python
# app/celery_app.py
from celery import Celery
import os

REDIS_URL = os.getenv("REDIS_URL", "redis://localhost:6379/0")

celery = Celery(
    "app",
    broker=REDIS_URL,
    backend=REDIS_URL,
    include=["app.tasks.email_tasks", "app.tasks.report_tasks"],
)

celery.conf.update(
    task_serializer="json",
    accept_content=["json"],
    result_serializer="json",
    timezone="UTC",
    enable_utc=True,
    task_track_started=True,
    task_acks_late=True,  # only ack after task completes (important for reliability)
    worker_prefetch_multiplier=1,  # process one task at a time per worker
    task_soft_time_limit=60,   # task gets SoftTimeLimitExceeded at 60s
    task_time_limit=120,        # task is killed at 120s
    result_expires=3600,        # results expire after 1 hour
)
```

### Task 4: Calling Tasks from FastAPI
```python
# app/users/router.py
from fastapi import APIRouter
from app.tasks.email_tasks import send_welcome_email, send_password_reset_email
from celery.result import AsyncResult

router = APIRouter(prefix="/users")

@router.post("/", status_code=201)
async def create_user(data: dict):
    user = {"id": 42, "email": data["email"], "name": data["name"]}
    
    # .delay() is shorthand for .apply_async()
    task = send_welcome_email.delay(user["id"], user["email"], user["name"])
    
    # The task is now in the Redis queue
    # task.id can be used to check status later
    return {**user, "task_id": task.id}

@router.get("/tasks/{task_id}")
async def get_task_status(task_id: str):
    result = AsyncResult(task_id)
    return {
        "task_id": task_id,
        "status": result.status,  # PENDING, STARTED, SUCCESS, FAILURE, RETRY
        "result": result.result if result.ready() else None,
    }

@router.post("/password-reset")
async def request_password_reset(email: str):
    token = "reset-token-123"  # generate real token
    
    # Schedule with countdown (send in 5 seconds)
    send_password_reset_email.apply_async(
        args=[email, token],
        countdown=5,
    )
    return {"message": "Reset email sent"}
```

### Task 5: Celery Beat Scheduling
```python
# app/celery_app.py (add beat schedule)
from celery.schedules import crontab

celery.conf.beat_schedule = {
    "generate-daily-report": {
        "task": "generate_daily_report",
        "schedule": crontab(hour=8, minute=0),  # 8:00 AM UTC daily
        "args": [],  # args passed to the task (date filled at runtime)
        "kwargs": {},
    },
    "cleanup-expired-tokens": {
        "task": "cleanup_expired_tokens",
        "schedule": crontab(minute="*/30"),  # every 30 minutes
    },
    "health-check-ping": {
        "task": "health_ping",
        "schedule": 60.0,  # every 60 seconds (simple interval)
    },
}
```
