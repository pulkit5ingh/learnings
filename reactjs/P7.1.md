# P7.1 — Performance Audit

**Phase:** 7 — Performance
**Topic:** React DevTools Profiler, unnecessary re-renders, React.memo, useMemo, useCallback
**Type:** Project

---

## Goal

Take a deliberately slow React app, profile it with React DevTools, identify at least 3 sources of unnecessary re-renders, apply the correct fixes (React.memo, useMemo, useCallback, state co-location), and document before/after metrics.

---

## What to Build

A "slow" Product Dashboard app with known performance issues, then the fixed version with documented improvements:

- Product list with filtering (100+ items)
- Cart sidebar
- Search input
- Sort controls
- Stats bar (total items, total price, filtered count)

The initial version has intentional anti-patterns; you will find and fix them.

## Implementation

### The slow version — intentional anti-patterns

```tsx
// BAD: AppContext.tsx — everything in one context, all consumers re-render on any change
import { createContext, useContext, useState } from 'react';
import type { Product } from '../data/products';

interface AppState {
  products: Product[];
  cart: Product[];
  search: string;
  sort: 'price-asc' | 'price-desc' | 'name';
  setSearch: (s: string) => void;
  setSort: (s: AppState['sort']) => void;
  addToCart: (p: Product) => void;
}
// This causes the entire tree to re-render when search changes
export const AppContext = createContext<AppState>({} as AppState);
```


### Step 1 — Profile and document the problems

Use React DevTools Profiler:
1. Click "Record"
2. Type a character in the search box
3. Click "Stop recording"
4. Observe which components re-rendered and why

Expected findings:
- `ProductItem` re-renders for all 100 items when typing (filter doesn't change most items)
- `CartSidebar` re-renders when search changes (they share context)
- `StatsBar` re-renders when cart changes even if stats didn't change
- `ExpensiveChart` re-renders on every search keystroke

### Step 3 — Fix 2: Memoize filtered list

```tsx
// FIXED: ProductList.tsx
import { useMemo, useCallback } from 'react';
import { useSearchContext } from '../context/SearchContext';
import { useCartContext } from '../context/CartContext';

function ProductList() {
  const { products, search, sort } = useSearchContext();
  const { addToCart } = useCartContext();

  // Only recomputes when search or sort changes — not on cart changes
  const filtered = useMemo(() => {
    return products
      .filter(p => p.name.toLowerCase().includes(search.toLowerCase()))
      .sort((a, b) => {
        if (sort === 'price-asc') return a.price - b.price;
        if (sort === 'price-desc') return b.price - a.price;
        return a.name.localeCompare(b.name);
      });
  }, [products, search, sort]);

  // Stable reference — ProductItem won't re-render because of this
  const handleAdd = useCallback((product: Product) => {
    addToCart(product);
  }, [addToCart]);

  return (
    <ul>
      {filtered.map(p => (
        <ProductItem key={p.id} product={p} onAdd={handleAdd} />
      ))}
    </ul>
  );
}
```

### Step 4 — Fix 3: React.memo on ProductItem

```tsx
// FIXED: ProductItem.tsx
import { memo } from 'react';
import type { Product } from '../data/products';

interface Props {
  product: Product;
  onAdd: (p: Product) => void;
}

// Now only re-renders when product or onAdd reference changes
export const ProductItem = memo(function ProductItem({ product, onAdd }: Props) {
  return (
    <div className="flex justify-between items-center p-3 border-b">
      <div>
        <p className="font-medium">{product.name}</p>
        <p className="text-sm text-gray-500">${product.price.toFixed(2)}</p>
      </div>
      <button
        onClick={() => onAdd(product)}
        className="px-3 py-1 bg-blue-500 text-white rounded text-sm"
      >
        Add
      </button>
    </div>
  );
});
```

### Step 5 — Fix 4: Memoize expensive derived value

```tsx
// FIXED: StatsBar.tsx
import { useMemo } from 'react';
import { useCartContext } from '../context/CartContext';
import { useSearchContext } from '../context/SearchContext';

export const StatsBar = memo(function StatsBar() {
  const { cart } = useCartContext();
  const { filteredCount, totalProducts } = useSearchContext();

  // Only recomputes when cart changes
  const cartTotal = useMemo(
    () => cart.reduce((sum, item) => sum + item.price, 0),
    [cart]
  );

  return (
    <div className="flex gap-6 p-3 bg-gray-50 border-b text-sm">
      <span>Total products: {totalProducts}</span>
      <span>Showing: {filteredCount}</span>
      <span>Cart total: ${cartTotal.toFixed(2)}</span>
    </div>
  );
});
```

### Fix 5 — Prevent ExpensiveChart re-renders

```tsx
// ExpensiveChart is slow because it simulates expensive computation
// BEFORE: it re-renders whenever parent re-renders
function ExpensiveChart({ data }: { data: number[] }) {
  // Simulate expensive computation
  const result = data.reduce((sum, n) => sum + n * Math.sqrt(n), 0);
  return <div>Chart total: {result.toFixed(2)}</div>;
}

// AFTER: wrap in memo + memoize the data prop
export const ExpensiveChart = memo(function ExpensiveChart({ data }: { data: number[] }) {
  const result = useMemo(
    () => data.reduce((sum, n) => sum + n * Math.sqrt(n), 0),
    [data]
  );
  return <div>Chart total: {result.toFixed(2)}</div>;
});

// In parent: stabilise the data array
const chartData = useMemo(() => products.map(p => p.price), [products]);
<ExpensiveChart data={chartData} />
```

### PERFORMANCE_NOTES.md template

```markdown
# Performance Audit Notes

## Profiling Setup
- React DevTools Profiler, React 18 development build
- 120 products in the list
- Profiled: typing one character in the search box

## Issues Found

### Issue 1: All items re-render on every search keystroke
- **Root cause:** Inline arrow function `() => onAdd(product)` creates a new reference every render
- **Fix:** `useCallback` for `onAdd` + `React.memo` on `ProductItem`
- **Before:** 120 component renders per keystroke
- **After:** 0–3 renders (only items entering/leaving the filtered list)

### Issue 2: CartSidebar re-renders on search changes
- **Root cause:** Single AppContext — any state change triggers all consumers
- **Fix:** Split into CartContext and SearchContext
- **Before:** CartSidebar renders on every search keystroke
- **After:** CartSidebar only renders when cart changes

### Issue 3: Filter + sort runs on every cart update
- **Root cause:** Filtering logic in render without `useMemo`
- **Fix:** `useMemo` with `[products, search, sort]` dependencies
- **Before:** O(n log n) sort runs on every cart add/remove
- **After:** Sort only runs when search or sort order changes

### Issue 4: ExpensiveChart re-renders unnecessarily
- **Root cause:** `data` prop is a new array reference every render
- **Fix:** `useMemo` for `chartData` in parent + `React.memo` on component
- **Before:** Chart recomputes on every render (~40 ms)
- **After:** Chart only recomputes when products change

## Metrics
| Metric | Before | After |
|--------|--------|-------|
| Renders per keystroke (ProductItem) | 120 | 2–5 |
| Renders per cart add (CartSidebar) | Always | Only on cart change |
| Filter computation per keystroke | Always | Only on search/sort change |
| Interaction to Next Paint (INP) | ~180 ms | ~20 ms |
```
