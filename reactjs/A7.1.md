## Goal
Understand React's render cycle, identify unnecessary re-renders with the DevTools Profiler, and prevent them selectively with React.memo.

## Background

### When Does a Component Re-render?
A component re-renders when: (1) its own state changes via setState/dispatch, (2) a context it consumes changes value, (3) its parent re-renders. The third case is the most important to understand — when a parent re-renders, ALL its children re-render by default, even if their props didn't change. This is by design and usually fine. It only becomes a problem when re-rendering a child is expensive (long render, deep subtree) and the parent re-renders frequently.

### React.memo
`React.memo(Component)` wraps a component and memoizes its last rendered output. Before re-rendering, it shallowly compares old and new props. If they're equal, it skips the render and reuses the previous output. "Shallow comparison" means primitive values are compared by value, but objects and arrays are compared by reference. This is why passing an inline object `prop={{ key: 'val' }}` to a memo'd component defeats memoization.

### The Profiler
React DevTools includes a Profiler tab. Click Record, interact with your app, click Stop, and see a flame graph of which components rendered and how long each took. The "Why did this render?" feature (in settings: "Record why each component rendered") tells you whether a component re-rendered due to props change, state change, or a hook change.

### Task 1 — Demonstrating Unnecessary Re-renders

```tsx
// src/components/RerenderDemo.tsx
import { useState, memo } from 'react'

// This component logs every render
function RegularChild({ name }: { name: string }) {
  console.log(`RegularChild(${name}) rendered`)
  return (
    <div style={{ padding: '0.5rem', border: '1px solid #e5e7eb', margin: '0.25rem 0', borderRadius: 4 }}>
      Regular: {name}
    </div>
  )
}

// Memo'd version — skips render if name prop is unchanged
const MemoizedChild = memo(function MemoizedChild({ name }: { name: string }) {
  console.log(`MemoizedChild(${name}) rendered`)
  return (
    <div style={{ padding: '0.5rem', border: '1px solid #bfdbfe', margin: '0.25rem 0', borderRadius: 4, background: '#eff6ff' }}>
      Memoized: {name}
    </div>
  )
})

export function RerenderDemo() {
  const [count, setCount] = useState(0)

  // Open console and click "Increment count"
  // RegularChild re-renders on every click (parent state changed)
  // MemoizedChild skips re-render because its props didn't change

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(c => c + 1)}>Increment count</button>
      <p style={{ fontSize: 13, color: '#6b7280' }}>Open console to see render logs</p>

      <RegularChild name="Alice" />
      <RegularChild name="Bob" />
      <MemoizedChild name="Carol" />
      <MemoizedChild name="Dave" />
    </div>
  )
}
```

### Task 2 — The Object Reference Problem

```tsx
// src/components/ObjectProblem.tsx
import { useState, memo, useCallback, useMemo } from 'react'

interface Config {
  theme: string
  size: number
}

const ExpensiveComponent = memo(function ExpensiveComponent({ config, onAction }: {
  config: Config
  onAction: () => void
}) {
  console.log('ExpensiveComponent rendered')
  return (
    <div style={{ padding: '1rem', border: '1px solid #e5e7eb' }}>
      <p>Theme: {config.theme}, Size: {config.size}</p>
      <button onClick={onAction}>Action</button>
    </div>
  )
})

export function ObjectProblem() {
  const [count, setCount] = useState(0)
  const [theme, setTheme] = useState('light')

  // PROBLEM: new object created every render → memo is defeated
  const badConfig = { theme, size: 16 }
  const badHandler = () => console.log('action')

  // FIX: memoize the object
  const goodConfig = useMemo(() => ({ theme, size: 16 }), [theme])
  const goodHandler = useCallback(() => console.log('action'), [])

  return (
    <div>
      <p>Count: {count} (changing this should not re-render the component below)</p>
      <button onClick={() => setCount(c => c + 1)}>Increment</button>
      <button onClick={() => setTheme(t => t === 'light' ? 'dark' : 'light')}>Toggle theme</button>

      <h4>With bad (inline) config — re-renders on every count change:</h4>
      <ExpensiveComponent config={badConfig} onAction={badHandler} />

      <h4>With good (memoized) config — re-renders only when theme changes:</h4>
      <ExpensiveComponent config={goodConfig} onAction={goodHandler} />
    </div>
  )
}
```

### Task 3 — Custom Comparison Function

```tsx
// src/components/CustomComparison.tsx
import { memo } from 'react'

interface User {
  id: number
  name: string
  email: string
  lastSeen: Date  // changes every render but we don't display it
}

// Custom comparison: ignore lastSeen changes
const UserCard = memo(
  function UserCard({ user }: { user: User }) {
    console.log(`UserCard(${user.id}) rendered`)
    return (
      <div style={{ padding: '0.75rem', border: '1px solid #e5e7eb', borderRadius: 6 }}>
        <p style={{ margin: 0, fontWeight: 500 }}>{user.name}</p>
        <p style={{ margin: '2px 0 0', fontSize: 13, color: '#6b7280' }}>{user.email}</p>
      </div>
    )
  },
  // Second argument: custom comparison function
  // Return true = same props, skip render
  // Return false = different props, re-render
  (prevProps, nextProps) =>
    prevProps.user.id === nextProps.user.id &&
    prevProps.user.name === nextProps.user.name &&
    prevProps.user.email === nextProps.user.email
    // lastSeen is intentionally excluded
)

export { UserCard }
```

### Task 4 — Profiler API (Programmatic)

```tsx
// src/components/ProfilerWrapper.tsx
import { Profiler, type ProfilerOnRenderCallback, type ReactNode } from 'react'

export function DebugProfiler({ id, children }: { id: string; children: ReactNode }) {
  if (process.env.NODE_ENV !== 'development') return <>{children}</>

  const onRender: ProfilerOnRenderCallback = (
    id,          // component id
    phase,       // "mount" or "update"
    actualDuration,   // time spent rendering
    baseDuration,     // estimated time without memo
    startTime,
    commitTime
  ) => {
    if (actualDuration > 16) {  // log slow renders (>1 frame at 60fps)
      console.warn(`Slow render: ${id} (${phase}) took ${actualDuration.toFixed(1)}ms`)
    }
  }

  return <Profiler id={id} onRender={onRender}>{children}</Profiler>
}
```
