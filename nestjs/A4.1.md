## Goal
Use built-in NestJS pipes for parsing and validation, and write custom pipes for business-specific transformations.

## Background

Pipes in NestJS sit between route params/body extraction and the handler. They serve two purposes: **validation** (throw an exception if input is invalid) and **transformation** (convert input to the expected type). They implement the `PipeTransform` interface with a single `transform(value, metadata)` method.

`ValidationPipe` is the most important built-in pipe. It uses `class-validator` and `class-transformer` to validate and transform incoming data against DTO classes. Key options:
- `whitelist: true` — strips any properties not in the DTO (security best practice).
- `forbidNonWhitelisted: true` — throws an error if extra properties are sent.
- `transform: true` — automatically transforms plain JS objects to DTO class instances and converts primitive types (string `"1"` → number `1`).
- `transformOptions.enableImplicitConversion: true` — type-coercion without explicit `@Type()` decorators.

`ParseIntPipe` and `ParseUUIDPipe` validate that URL params are valid integers/UUIDs before reaching your handler.

Custom pipes are useful for: parsing pagination query params into an object, trimming whitespace, parsing comma-separated strings into arrays, or loading an entity from the database by ID (param pipe pattern).

### 2. Enable global ValidationPipe

**`src/main.ts`**
```typescript
import { ValidationPipe } from '@nestjs/common';
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  app.useGlobalPipes(
    new ValidationPipe({
      whitelist: true,               // strip unknown properties
      forbidNonWhitelisted: true,    // throw if unknown props present
      transform: true,               // auto-transform to DTO instances
      transformOptions: {
        enableImplicitConversion: true, // coerce types from metadata
      },
    }),
  );

  await app.listen(3000);
}
bootstrap();
```

### 3. DTO with class-validator decorators

**`src/users/dto/create-user.dto.ts`**
```typescript
import {
  IsEmail,
  IsString,
  IsNotEmpty,
  MinLength,
  MaxLength,
  IsInt,
  Min,
  Max,
  IsOptional,
  IsEnum,
  Matches,
} from 'class-validator';
import { Role } from '../../common/enums/role.enum';

export class CreateUserDto {
  @IsString()
  @IsNotEmpty()
  @MinLength(2)
  @MaxLength(100)
  name: string;

  @IsEmail()
  email: string;

  @IsString()
  @MinLength(8)
  @Matches(/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)/, {
    message: 'Password must contain uppercase, lowercase, and number',
  })
  password: string;

  @IsOptional()
  @IsInt()
  @Min(13)
  @Max(120)
  age?: number;

  @IsOptional()
  @IsEnum(Role)
  role?: Role;
}
```

### 4. Nested DTO validation

```typescript
import { ValidateNested, IsArray, ArrayMinSize } from 'class-validator';
import { Type } from 'class-transformer';

class AddressDto {
  @IsString()
  @IsNotEmpty()
  street: string;

  @IsString()
  city: string;

  @IsString()
  @Matches(/^\d{5}$/, { message: 'Postal code must be 5 digits' })
  postalCode: string;
}

export class CreateOrderDto {
  @ValidateNested()
  @Type(() => AddressDto)    // class-transformer needs this for nested objects
  shippingAddress: AddressDto;

  @IsArray()
  @ArrayMinSize(1)
  @ValidateNested({ each: true })
  @Type(() => OrderItemDto)
  items: OrderItemDto[];
}
```

### 6. Custom pipe — PaginationPipe

**`src/common/pipes/pagination.pipe.ts`**
```typescript
import { BadRequestException, Injectable, PipeTransform } from '@nestjs/common';

export interface PaginationParams {
  page: number;
  limit: number;
  skip: number;
}

@Injectable()
export class PaginationPipe implements PipeTransform {
  transform(value: { page?: string; limit?: string }): PaginationParams {
    const page = parseInt(value.page ?? '1', 10);
    const limit = parseInt(value.limit ?? '20', 10);

    if (isNaN(page) || page < 1) {
      throw new BadRequestException('page must be a positive integer');
    }
    if (isNaN(limit) || limit < 1 || limit > 100) {
      throw new BadRequestException('limit must be between 1 and 100');
    }

    return { page, limit, skip: (page - 1) * limit };
  }
}
```

Usage:
```typescript
@Get()
findAll(@Query(PaginationPipe) pagination: PaginationParams) {
  return this.usersService.paginate(pagination);
}
```

### 7. Custom pipe — ParseArrayPipe for comma-separated values

**`src/common/pipes/parse-tags.pipe.ts`**
```typescript
import { Injectable, PipeTransform, BadRequestException } from '@nestjs/common';

@Injectable()
export class ParseTagsPipe implements PipeTransform<string, string[]> {
  transform(value: string): string[] {
    if (!value) return [];
    const tags = value.split(',').map(t => t.trim()).filter(Boolean);
    if (tags.length > 10) {
      throw new BadRequestException('Maximum 10 tags allowed');
    }
    return tags;
  }
}
```

Usage:
```typescript
@Get()
findByTags(@Query('tags', ParseTagsPipe) tags: string[]) {
  // GET /posts?tags=nestjs,typescript,backend
  return this.postsService.findByTags(tags);
}
```
