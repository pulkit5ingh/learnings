**Phase:** 2 — Database with TypeORM & Prisma  
**Topic:** TypeORM migration:generate, migration:run, migration:revert, entity sync, seeders  
**Type:** Assignment

---

## Goal
Replace `synchronize: true` with proper migration-based schema management and write a seeder to populate initial data.

## Background

`synchronize: true` automatically alters the database schema to match your entities on every startup. It is dangerous in production because it can drop columns, lose data, and run without a review process. **Migrations** are the production-safe alternative: TypeORM generates a timestamped migration file with `UP` and `DOWN` SQL, you review it, commit it, and run it explicitly.

The workflow is:
1. Make changes to entity files (add column, rename, add relation).
2. Run `migration:generate` — TypeORM compares the current database schema to your entities and produces a migration file with the diff.
3. Review the generated SQL.
4. Run `migration:run` — executes all pending migrations (tracked in the `migrations` table).
5. If something went wrong, run `migration:revert` to roll back the last migration.

**Seeders** are not built into TypeORM 0.3+, but the common approach is to write a standalone script (or a NestJS command via `nestjs-command` or a simple `ts-node` script) that uses the same `DataSource` to insert seed data. Seeders should be idempotent — running them twice should not create duplicate records.

## Tasks

### 1. Configure TypeORM for migrations

**`src/database/data-source.ts`** (used by the TypeORM CLI):
```typescript
import { DataSource } from 'typeorm';
import { User } from '../users/entities/user.entity';
import { Post } from '../posts/entities/post.entity';
import { Tag } from '../posts/entities/tag.entity';

export const AppDataSource = new DataSource({
  type: 'postgres',
  host: process.env.DB_HOST ?? 'localhost',
  port: Number(process.env.DB_PORT) || 5432,
  username: process.env.DB_USER ?? 'nestuser',
  password: process.env.DB_PASS ?? 'nestpass',
  database: process.env.DB_NAME ?? 'nestdb',
  entities: [User, Post, Tag],
  migrations: ['src/database/migrations/*.ts'],
  synchronize: false,   // NEVER true in migration workflow
});
```

**`package.json`** — add migration scripts:
```json
{
  "scripts": {
    "typeorm": "ts-node -r tsconfig-paths/register ./node_modules/typeorm/cli.js --dataSource src/database/data-source.ts",
    "migration:generate": "npm run typeorm migration:generate src/database/migrations/",
    "migration:run": "npm run typeorm migration:run",
    "migration:revert": "npm run typeorm migration:revert",
    "migration:show": "npm run typeorm migration:show"
  }
}
```

Install the CLI helper:
```bash
npm install ts-node tsconfig-paths --save-dev
```

### 2. Generate the initial migration

```bash
npm run migration:generate -- -n InitialSchema
```

This creates something like:
```
src/database/migrations/1700000000000-InitialSchema.ts
```

**Example generated migration:**
```typescript
import { MigrationInterface, QueryRunner } from 'typeorm';

export class InitialSchema1700000000000 implements MigrationInterface {
  name = 'InitialSchema1700000000000';

  public async up(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.query(`
      CREATE TABLE "tags" (
        "id" SERIAL NOT NULL,
        "name" character varying NOT NULL,
        CONSTRAINT "UQ_tags_name" UNIQUE ("name"),
        CONSTRAINT "PK_tags" PRIMARY KEY ("id")
      )
    `);

    await queryRunner.query(`
      CREATE TABLE "users" (
        "id" SERIAL NOT NULL,
        "name" character varying(100) NOT NULL,
        "email" character varying(255) NOT NULL,
        "isActive" boolean NOT NULL DEFAULT true,
        "createdAt" TIMESTAMP NOT NULL DEFAULT now(),
        "updatedAt" TIMESTAMP NOT NULL DEFAULT now(),
        "deletedAt" TIMESTAMP,
        CONSTRAINT "UQ_users_email" UNIQUE ("email"),
        CONSTRAINT "PK_users" PRIMARY KEY ("id")
      )
    `);

    await queryRunner.query(`
      CREATE TABLE "posts" (
        "id" SERIAL NOT NULL,
        "title" character varying NOT NULL,
        "content" text NOT NULL,
        "published" boolean NOT NULL DEFAULT false,
        "createdAt" TIMESTAMP NOT NULL DEFAULT now(),
        "authorId" integer,
        CONSTRAINT "PK_posts" PRIMARY KEY ("id")
      )
    `);

    await queryRunner.query(`
      ALTER TABLE "posts"
      ADD CONSTRAINT "FK_posts_authorId"
      FOREIGN KEY ("authorId") REFERENCES "users"("id")
      ON DELETE CASCADE
    `);
  }

  public async down(queryRunner: QueryRunner): Promise<void> {
    await queryRunner.query(`ALTER TABLE "posts" DROP CONSTRAINT "FK_posts_authorId"`);
    await queryRunner.query(`DROP TABLE "posts"`);
    await queryRunner.query(`DROP TABLE "users"`);
    await queryRunner.query(`DROP TABLE "tags"`);
  }
}
```

### 4. Add a column via migration

Add `bio` column to the `users` table. Edit the entity first:
```typescript
// In User entity, add:
@Column({ nullable: true, length: 500 })
bio?: string;
```

Generate the addColumn migration:
```bash
npm run migration:generate -- -n AddBioToUser
```

Generated diff migration:
```typescript
public async up(queryRunner: QueryRunner): Promise<void> {
  await queryRunner.query(`
    ALTER TABLE "users" ADD "bio" character varying(500)
  `);
}

public async down(queryRunner: QueryRunner): Promise<void> {
  await queryRunner.query(`
    ALTER TABLE "users" DROP COLUMN "bio"
  `);
}
```

```bash
npm run migration:run
npm run migration:revert  # undo the last migration
npm run migration:run     # re-apply it
```

### 5. Write a seeder

**`src/database/seed.ts`**
```typescript
import { AppDataSource } from './data-source';
import { User } from '../users/entities/user.entity';
import { Tag } from '../posts/entities/tag.entity';

async function seed() {
  await AppDataSource.initialize();
  console.log('Database connected. Seeding...');

  const userRepo = AppDataSource.getRepository(User);
  const tagRepo = AppDataSource.getRepository(Tag);

  // Idempotent: skip if already exists
  const existingUser = await userRepo.findOne({ where: { email: 'admin@example.com' } });
  if (!existingUser) {
    const admin = userRepo.create({ name: 'Admin', email: 'admin@example.com' });
    await userRepo.save(admin);
    console.log('Created admin user');
  } else {
    console.log('Admin user already exists, skipping');
  }

  const defaultTags = ['nestjs', 'typescript', 'postgresql', 'backend'];
  for (const name of defaultTags) {
    const existing = await tagRepo.findOne({ where: { name } });
    if (!existing) {
      await tagRepo.save(tagRepo.create({ name }));
      console.log(`Created tag: ${name}`);
    }
  }

  await AppDataSource.destroy();
  console.log('Seeding complete');
}

seed().catch(err => {
  console.error('Seeding failed:', err);
  process.exit(1);
});
```

**`package.json`** — add seed script:
```json
{
  "scripts": {
    "seed": "ts-node -r tsconfig-paths/register src/database/seed.ts"
  }
}
```

```bash
npm run seed
npm run seed  # run twice — should skip existing records
```

### 6. AppModule with migrations (production-safe)

```typescript
TypeOrmModule.forRoot({
  type: 'postgres',
  host: process.env.DB_HOST,
  port: Number(process.env.DB_PORT),
  username: process.env.DB_USER,
  password: process.env.DB_PASS,
  database: process.env.DB_NAME,
  autoLoadEntities: true,
  synchronize: false,          // never true
  migrationsRun: true,         // auto-run pending migrations on app start
  migrations: ['dist/database/migrations/*.js'],
})
```
