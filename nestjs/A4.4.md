# A4.4 — class-validator & class-transformer

**Phase:** 4 — Pipes, Interceptors & Filters  
**Topic:** Decorators (@IsString, @IsEmail, @Min, @Max, @IsOptional), nested validation, custom validators, plainToInstance  
**Type:** Assignment

---

## Goal
Master class-validator and class-transformer together to validate input deeply and transform plain objects to typed class instances with controlled serialisation.

## Background

`class-validator` works by attaching metadata to class properties via decorators. When `validate(instance)` is called (or `ValidationPipe` does it automatically), it reads the metadata and checks each constraint, returning an array of `ValidationError` objects with the property path and failed constraints.

`class-transformer` converts plain JavaScript objects (from JSON.parse, database results, etc.) to class instances via `plainToInstance(Cls, plain)`, and converts instances back to plain objects via `instanceToPlain(instance)`. This matters because `class-validator` decorators only work on class instances, not plain objects.

**`@Exclude()` and `@Expose()`** from class-transformer control serialisation. When you annotate a class with `@Exclude()`, only properties explicitly marked with `@Expose()` are included in the output. This is essential for hiding passwords, internal fields, and computed properties.

**Custom validators** let you implement domain-specific rules that the built-in decorators can't express, such as "password must not contain the username" or "endDate must be after startDate".

## Tasks

### 1. Core validation decorators

**`src/users/dto/create-user.dto.ts`**
```typescript
import {
  IsEmail, IsEnum, IsInt, IsNotEmpty, IsOptional,
  IsPhoneNumber, IsString, IsUrl, Matches, Max,
  Min, MinLength, MaxLength, IsDateString,
} from 'class-validator';
import { Transform, Type } from 'class-transformer';

export class CreateUserDto {
  @IsString()
  @IsNotEmpty({ message: 'Name is required' })
  @MinLength(2, { message: 'Name too short' })
  @MaxLength(100)
  @Transform(({ value }) => (value as string).trim())
  name: string;

  @IsEmail({}, { message: 'Invalid email address' })
  @Transform(({ value }) => (value as string).toLowerCase().trim())
  email: string;

  @IsString()
  @MinLength(8)
  @Matches(/(?=.*[A-Z])/, { message: 'Must contain uppercase' })
  @Matches(/(?=.*[0-9])/, { message: 'Must contain a number' })
  @Matches(/(?=.*[!@#$%^&*])/, { message: 'Must contain special character' })
  password: string;

  @IsOptional()
  @IsInt()
  @Min(13)
  @Max(120)
  @Type(() => Number)   // coerce string "25" → number 25
  age?: number;

  @IsOptional()
  @IsPhoneNumber('IN')  // ISO 3166-1 country code
  phone?: string;

  @IsOptional()
  @IsUrl()
  avatarUrl?: string;

  @IsOptional()
  @IsDateString()
  birthDate?: string;
}
```

### 2. class-transformer for response serialisation

**`src/users/entities/user.entity.ts`** — with transform annotations:
```typescript
import { Exclude, Expose, Transform } from 'class-transformer';
import { Entity, PrimaryGeneratedColumn, Column } from 'typeorm';

@Entity('users')
export class User {
  @PrimaryGeneratedColumn()
  @Expose()
  id: number;

  @Column()
  @Expose()
  name: string;

  @Column({ unique: true })
  @Expose()
  email: string;

  @Column()
  @Exclude()   // NEVER serialised to JSON
  passwordHash: string;

  @Column({ default: true })
  @Exclude()   // internal field, not shown to clients
  isActive: boolean;

  @Column({ nullable: true })
  @Exclude()
  refreshTokenHash?: string;

  @Column({ type: 'simple-array', default: 'user' })
  @Exclude()   // roles returned only in admin contexts
  roles: string[];

  @Expose()
  @Transform(({ obj }) => `${(obj as User).name.charAt(0).toUpperCase()}.`)
  initials?: string;  // computed, not a DB column

  @Column()
  createdAt: Date;

  @Expose()
  @Transform(({ value }) => (value as Date).toISOString().split('T')[0])
  get createdDate(): string {
    return this.createdAt?.toISOString().split('T')[0] ?? '';
  }
}
```

In your controller/service, use `ClassSerializerInterceptor` to apply `@Exclude`/`@Expose` automatically:
```typescript
// main.ts or module level
import { ClassSerializerInterceptor } from '@nestjs/common';
import { Reflector } from '@nestjs/core';
app.useGlobalInterceptors(new ClassSerializerInterceptor(app.get(Reflector)));
```

Or use `plainToInstance` explicitly:
```typescript
import { plainToInstance } from 'class-transformer';

const userInstance = plainToInstance(User, rawUserFromDb, {
  excludeExtraneousValues: true,  // only @Expose() fields returned
});
```

### 3. Cross-field custom validator

**`src/common/validators/is-after-date.validator.ts`**
```typescript
import {
  ValidatorConstraint,
  ValidatorConstraintInterface,
  ValidationArguments,
  registerDecorator,
  ValidationOptions,
} from 'class-validator';

@ValidatorConstraint({ name: 'isAfterDate', async: false })
export class IsAfterDateConstraint implements ValidatorConstraintInterface {
  validate(endDate: string, args: ValidationArguments): boolean {
    const [startDateField] = args.constraints as string[];
    const object = args.object as Record<string, string>;
    const startDate = object[startDateField];
    if (!startDate || !endDate) return true;  // let other validators handle missing fields
    return new Date(endDate) > new Date(startDate);
  }

  defaultMessage(args: ValidationArguments): string {
    const [startDateField] = args.constraints as string[];
    return `${args.property} must be after ${startDateField}`;
  }
}

export function IsAfterDate(property: string, options?: ValidationOptions) {
  return (object: object, propertyName: string) => {
    registerDecorator({
      target: object.constructor,
      propertyName,
      options,
      constraints: [property],
      validator: IsAfterDateConstraint,
    });
  };
}
```

Usage:
```typescript
export class CreateEventDto {
  @IsDateString()
  startDate: string;

  @IsDateString()
  @IsAfterDate('startDate', { message: 'End date must be after start date' })
  endDate: string;
}
```

### 4. Async custom validator (database check)

```typescript
import { Injectable } from '@nestjs/common';
import {
  ValidatorConstraint,
  ValidatorConstraintInterface,
  ValidationArguments,
  registerDecorator,
} from 'class-validator';
import { UsersService } from '../../users/users.service';

@ValidatorConstraint({ name: 'isEmailUnique', async: true })
@Injectable()
export class IsEmailUniqueConstraint implements ValidatorConstraintInterface {
  constructor(private readonly usersService: UsersService) {}

  async validate(email: string): Promise<boolean> {
    const user = await this.usersService.findByEmail(email).catch(() => null);
    return !user;  // return true = valid (email not taken)
  }

  defaultMessage(): string {
    return 'Email $value is already in use';
  }
}

export function IsEmailUnique(options?: ValidationOptions) {
  return (object: object, propertyName: string) => {
    registerDecorator({
      target: object.constructor,
      propertyName,
      options,
      async: true,
      validator: IsEmailUniqueConstraint,
    });
  };
}
```

Register the constraint as a provider and use `useContainer` to allow injection:
```typescript
// main.ts
import { useContainer } from 'class-validator';
useContainer(app.select(AppModule), { fallbackOnErrors: true });
```

### 5. Groups for conditional validation

```typescript
export class UpdateUserDto {
  @IsOptional()
  @IsString()
  @MinLength(2)
  name?: string;

  @IsOptional()
  @IsEmail()
  email?: string;

  // Only validated when group 'admin' is used
  @IsOptional()
  @IsEnum(Role)
  @Validate(SomeConstraint, { groups: ['admin'] })
  role?: Role;
}
```

In controller, specify groups:
```typescript
@UseGuards(JwtAuthGuard, RolesGuard)
@Roles(Role.ADMIN)
@Patch(':id/role')
async updateRole(
  @Param('id', ParseIntPipe) id: number,
  @Body(new ValidationPipe({ groups: ['admin'] })) dto: UpdateUserDto,
) { ... }
```

### 6. plainToInstance with nested exclusion

```typescript
import { plainToInstance, instanceToPlain } from 'class-transformer';

class AddressDto {
  @Expose() street: string;
  @Expose() city: string;
  @Exclude() internalCode: string;
}

class UserResponseDto {
  @Expose() id: number;
  @Expose() name: string;
  @Exclude() passwordHash: string;

  @Expose()
  @Type(() => AddressDto)
  address: AddressDto;
}

// Convert raw DB object to safe response:
const safeUser = plainToInstance(UserResponseDto, rawUserFromDb, {
  excludeExtraneousValues: true,
});

// Convert back to plain object for JSON serialisation:
const json = instanceToPlain(safeUser);
```

## Deliverables

- [ ] `@Transform(({ value }) => value.trim())` strips whitespace from name before validation
- [ ] `@Exclude()` prevents `passwordHash` from appearing in any serialised response
- [ ] `@IsAfterDate('startDate')` throws a validation error when endDate ≤ startDate
- [ ] Async `IsEmailUniqueConstraint` checks the database and rejects duplicate emails
- [ ] `useContainer` is set up so that async constraint providers can inject `UsersService`
- [ ] `plainToInstance` with `excludeExtraneousValues: true` returns only `@Expose()` fields
