# A6.3 — Task Scheduling

**Phase:** 6 — Caching, Queues & Performance  
**Topic:** @nestjs/schedule, @Cron, @Interval, @Timeout, dynamic cron jobs  
**Type:** Assignment

---

## Goal
Schedule recurring tasks using cron expressions and intervals, manage dynamic cron jobs at runtime, and integrate scheduled tasks with queues for reliable execution.

## Background

`@nestjs/schedule` wraps the `node-cron` library with a NestJS-friendly decorator API. Three types of scheduled work:
- `@Cron(expression)` — runs on a specific schedule defined by a cron expression.
- `@Interval(milliseconds)` — runs every N milliseconds.
- `@Timeout(milliseconds)` — runs once after N milliseconds delay.

Cron expressions follow the standard `* * * * * *` format (second, minute, hour, day-of-month, month, day-of-week). NestJS provides the `CronExpression` enum for common patterns.

**Important:** In a multi-instance (horizontally scaled) deployment, every instance runs `@Cron` jobs independently, which means the job runs N times instead of once. Solutions:
- Use a distributed lock (e.g., Redis lock via `redlock`) before the job body.
- Move the job logic into a Bull queue — add the job once (via a "trigger" cron) and Bull ensures exactly-once processing.

**Dynamic jobs** via `SchedulerRegistry` allow adding, deleting, and stopping jobs at runtime — useful for user-configured reminders or scheduled reports.

```typescript
import { Injectable, Logger } from '@nestjs/common';
import {
  Cron, CronExpression,
  Interval, Timeout,
  SchedulerRegistry,
} from '@nestjs/schedule';
import { CronJob } from 'cron';

@Injectable()
export class ScheduledTasksService {
  private readonly logger = new Logger(ScheduledTasksService.name);

  constructor(private readonly schedulerRegistry: SchedulerRegistry) {}

  // Every day at midnight
  @Cron(CronExpression.EVERY_DAY_AT_MIDNIGHT)
  async cleanupExpiredTokens(): Promise<void> {
    this.logger.log('Running: cleanup expired tokens');
    // await this.authService.deleteExpiredRefreshTokens();
  }

  // Every Monday at 9 AM
  @Cron('0 9 * * 1', { name: 'weekly-report', timeZone: 'Asia/Kolkata' })
  async generateWeeklyReport(): Promise<void> {
    this.logger.log('Generating weekly report...');
    // await this.reportService.generateAndEmail();
  }

  // Every 5 minutes
  @Cron('*/5 * * * *')
  async syncExternalData(): Promise<void> {
    this.logger.debug('Syncing external data...');
  }

  // Every 30 seconds
  @Interval('health-ping', 30_000)
  async pingHealthCheck(): Promise<void> {
    this.logger.debug('Health ping');
  }

  // Run once, 10 seconds after startup
  @Timeout('startup-warmup', 10_000)
  async onStartupWarmup(): Promise<void> {
    this.logger.log('Running startup warmup tasks...');
    // e.g., pre-warm caches, sync DB state
  }
}
```

### 4. Dynamic cron jobs via SchedulerRegistry

```typescript
@Injectable()
export class DynamicSchedulerService {
  private readonly logger = new Logger(DynamicSchedulerService.name);

  constructor(
    private readonly schedulerRegistry: SchedulerRegistry,
    private readonly reportService: ReportService,
  ) {}

  addUserReport(userId: number, cronExpression: string): void {
    const jobName = `user-report-${userId}`;

    // Remove existing job if it exists
    if (this.schedulerRegistry.doesExist('cron', jobName)) {
      this.schedulerRegistry.deleteCronJob(jobName);
    }

    const job = new CronJob(cronExpression, async () => {
      this.logger.log(`Running scheduled report for user ${userId}`);
      await this.reportService.generateForUser(userId);
    });

    this.schedulerRegistry.addCronJob(jobName, job);
    job.start();

    this.logger.log(`Scheduled report for user ${userId}: ${cronExpression}`);
  }

  removeUserReport(userId: number): void {
    const jobName = `user-report-${userId}`;
    if (this.schedulerRegistry.doesExist('cron', jobName)) {
      this.schedulerRegistry.deleteCronJob(jobName);
      this.logger.log(`Removed scheduled report for user ${userId}`);
    }
  }

  listAllJobs(): string[] {
    return Array.from(this.schedulerRegistry.getCronJobs().keys());
  }

  pauseJob(name: string): void {
    const job = this.schedulerRegistry.getCronJob(name);
    job.stop();
    this.logger.log(`Paused job: ${name}`);
  }

  resumeJob(name: string): void {
    const job = this.schedulerRegistry.getCronJob(name);
    job.start();
    this.logger.log(`Resumed job: ${name}`);
  }
}
```

### 5. Queue-backed cron (distributed-safe pattern)

For multi-node deployments, use a cron job only to trigger a Bull queue job:

```typescript
@Injectable()
export class SafeScheduledTasksService {
  constructor(
    @InjectQueue('cleanup') private readonly cleanupQueue: Queue,
    @InjectQueue('reports') private readonly reportsQueue: Queue,
  ) {}

  // This cron runs on all instances, but Bull ensures the job is processed once
  @Cron(CronExpression.EVERY_DAY_AT_MIDNIGHT)
  async triggerDailyCleanup(): Promise<void> {
    await this.cleanupQueue.add('daily-cleanup', {}, {
      jobId: `daily-cleanup-${new Date().toISOString().split('T')[0]}`,
      // Unique jobId per day → Bull deduplicates; second instance's add() is a no-op
    });
  }

  @Cron('0 8 * * 1')  // Every Monday at 8 AM
  async triggerWeeklyReport(): Promise<void> {
    const week = this.getWeekId();
    await this.reportsQueue.add('weekly', { week }, {
      jobId: `weekly-report-${week}`,
    });
  }

  private getWeekId(): string {
    const now = new Date();
    const start = new Date(now.getFullYear(), 0, 1);
    const week = Math.ceil(((now.getTime() - start.getTime()) / 86400000 + start.getDay() + 1) / 7);
    return `${now.getFullYear()}-W${week}`;
  }
}
```
