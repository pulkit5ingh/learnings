# A8.2 — Redis Transport

**Phase:** 8 — Microservices  
**Topic:** @nestjs/microservices with Redis, pub/sub, microservice-to-microservice communication  
**Type:** Assignment

---

## Goal
Replace TCP transport with Redis transport for microservice communication, use Redis pub/sub for event broadcasting, and enable services to communicate with each other (not just gateway-to-service).

## Background

The Redis transport has significant advantages over TCP:
- **Resilient** — if a microservice goes down, messages queue up in Redis and are processed when it restarts.
- **Multi-instance** — multiple instances of a microservice can share the same Redis channel; Redis delivers each message to exactly one instance (competing consumers pattern).
- **Cross-cutting events** — Redis pub/sub lets many services subscribe to the same event channel.

`@nestjs/microservices` with Redis uses `ioredis` under the hood. The `transport: Transport.REDIS` option switches the transport layer — the `@MessagePattern` and `@EventPattern` decorators work identically.

**Service-to-service communication** means Service A calls Service B directly, without going through the HTTP gateway. For example, when an order is created, the Order Service calls the Inventory Service to reserve stock.

### 2. Redis transport setup

**Users microservice `main.ts`:**
```typescript
import { NestFactory } from '@nestjs/core';
import { MicroserviceOptions, Transport } from '@nestjs/microservices';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.createMicroservice<MicroserviceOptions>(AppModule, {
    transport: Transport.REDIS,
    options: {
      host: process.env.REDIS_HOST ?? 'localhost',
      port: Number(process.env.REDIS_PORT) || 6379,
      password: process.env.REDIS_PASSWORD,
      // Redis transport uses separate pub/sub channels per service
    },
  });

  await app.listen();
  console.log('Users microservice connected to Redis');
}
bootstrap();
```

**Orders microservice with hybrid server (HTTP + microservice):**
```typescript
import { NestFactory } from '@nestjs/core';
import { Transport } from '@nestjs/microservices';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);  // HTTP server

  // Also connect as a microservice consumer
  app.connectMicroservice<MicroserviceOptions>({
    transport: Transport.REDIS,
    options: { host: 'localhost', port: 6379 },
  });

  await app.startAllMicroservices();
  await app.listen(3002);
  console.log('Orders service: HTTP on 3002, Redis consumer active');
}
bootstrap();
```

### 3. ClientsModule with Redis in AppModule

**`src/app.module.ts`**
```typescript
import { ClientsModule, Transport } from '@nestjs/microservices';
import { ConfigModule, ConfigService } from '@nestjs/config';

@Module({
  imports: [
    ClientsModule.registerAsync([
      {
        name: 'USERS_SERVICE',
        imports: [ConfigModule],
        inject: [ConfigService],
        useFactory: (config: ConfigService) => ({
          transport: Transport.REDIS,
          options: {
            host: config.get('REDIS_HOST', 'localhost'),
            port: config.get<number>('REDIS_PORT', 6379),
          },
        }),
      },
      {
        name: 'INVENTORY_SERVICE',
        imports: [ConfigModule],
        inject: [ConfigService],
        useFactory: (config: ConfigService) => ({
          transport: Transport.REDIS,
          options: {
            host: config.get('REDIS_HOST', 'localhost'),
            port: config.get<number>('REDIS_PORT', 6379),
          },
        }),
      },
    ]),
  ],
})
export class AppModule {}
```

### 4. Service-to-service communication in Order Service

**`src/orders/orders.service.ts`** (in the Orders microservice):
```typescript
import { Injectable, Inject } from '@nestjs/common';
import { ClientProxy } from '@nestjs/microservices';
import { lastValueFrom } from 'rxjs';
import { timeout } from 'rxjs/operators';

@Injectable()
export class OrdersService {
  constructor(
    @Inject('INVENTORY_SERVICE') private readonly inventoryClient: ClientProxy,
    @Inject('USERS_SERVICE') private readonly usersClient: ClientProxy,
  ) {}

  async createOrder(userId: number, items: { productId: number; quantity: number }[]) {
    // 1. Verify user exists (call users-ms)
    const user = await lastValueFrom(
      this.usersClient.send({ cmd: 'find_user' }, { id: userId }).pipe(timeout(5000)),
    );

    // 2. Reserve stock for each item (call inventory-ms)
    for (const item of items) {
      await lastValueFrom(
        this.inventoryClient
          .send({ cmd: 'reserve_stock' }, { productId: item.productId, quantity: item.quantity })
          .pipe(timeout(5000)),
      );
    }

    // 3. Create the order
    const order = { id: Date.now(), userId, items, status: 'confirmed' };

    // 4. Emit an event for other services to react to
    this.inventoryClient.emit('order_created', { order, user });

    return order;
  }
}
```

### 5. Broadcast events via Redis pub/sub

When you use `@EventPattern`, any service subscribed to that pattern receives the event. This enables broadcasting:

**Order Service emits:**
```typescript
this.redisClient.emit('order_shipped', {
  orderId: 123,
  trackingNumber: 'TRK-456',
  estimatedDelivery: '2024-01-20',
});
```

**Email Service subscribes:**
```typescript
@EventPattern('order_shipped')
async sendShippingConfirmation(@Payload() data: { orderId: number; trackingNumber: string }) {
  await this.emailService.sendShippingConfirmation(data);
}
```

**Analytics Service also subscribes (both receive the same event):**
```typescript
@EventPattern('order_shipped')
async recordShipmentEvent(@Payload() data: any) {
  await this.analyticsService.record('order_shipped', data);
}
```

With Redis transport, **both** the Email Service and Analytics Service receive the `order_shipped` event.

### 7. Handling Redis connection failures

```typescript
// In the gateway, handle microservice unavailability:
import { catchError, timeout } from 'rxjs/operators';
import { throwError, TimeoutError } from 'rxjs';
import { ServiceUnavailableException } from '@nestjs/common';

async callUsersService(cmd: string, data: any) {
  return lastValueFrom(
    this.usersClient.send({ cmd }, data).pipe(
      timeout(5000),
      catchError(err => {
        if (err instanceof TimeoutError) {
          return throwError(() => new ServiceUnavailableException('Users service unavailable'));
        }
        return throwError(() => err);
      }),
    ),
  );
}
```

### 8. ClientProxy lifecycle

```typescript
// In app.module.ts or main.ts, connect clients on startup:
async onModuleInit() {
  await this.usersClient.connect();
  await this.inventoryClient.connect();
}

async onModuleDestroy() {
  await this.usersClient.close();
  await this.inventoryClient.close();
}
```

Or register with `APP_INITIALIZER` to connect all clients:
```typescript
{
  provide: APP_INITIALIZER,
  useFactory: (usersClient: ClientProxy, inventoryClient: ClientProxy) =>
    async () => {
      await usersClient.connect();
      await inventoryClient.connect();
    },
  inject: ['USERS_SERVICE', 'INVENTORY_SERVICE'],
  multi: true,
}
```
