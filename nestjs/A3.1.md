# A3.1 — Guards

**Phase:** 3 — Authentication & Guards  
**Topic:** CanActivate interface, ExecutionContext, Reflector, global guards, route-level guards  
**Type:** Assignment

---

## Goal
Build custom guards that use the execution context to protect routes, read route metadata via Reflector, and apply guards at different scopes.

## Background

Guards in NestJS implement the `CanActivate` interface and decide whether a request should proceed. They run **after middleware but before interceptors and pipes**. Guards have access to the full `ExecutionContext` — they can inspect the underlying HTTP request, WebSocket client, gRPC call, or microservice message depending on the transport.

`ExecutionContext` extends `ArgumentsHost` and adds `getClass()` (the controller class) and `getHandler()` (the route handler method). These are critical for reading metadata set by decorators using `SetMetadata`.

`Reflector` is a helper service provided by NestJS that reads metadata from classes and handlers. The pattern `reflector.getAllAndOverride<T>(key, [context.getHandler(), context.getClass()])` is the idiomatic way to read metadata, allowing handler-level metadata to override class-level metadata.

Guards return `true` (allow) or `false` (deny, throws 403 Forbidden by default). They can also throw exceptions directly for custom HTTP status codes (e.g., 401 Unauthorized for missing tokens vs 403 Forbidden for insufficient permissions).

**Scope levels:**
- Route-level: `@UseGuards(GuardClass)` on a method
- Controller-level: `@UseGuards(GuardClass)` on the class
- Global: `app.useGlobalGuards(new Guard())` or `APP_GUARD` provider (preferred — allows DI)

## Tasks

### 1. Basic guard

**`src/guards/api-key.guard.ts`**
```typescript
import {
  CanActivate,
  ExecutionContext,
  Injectable,
  UnauthorizedException,
} from '@nestjs/common';
import { Request } from 'express';

@Injectable()
export class ApiKeyGuard implements CanActivate {
  private readonly validKey = process.env.API_KEY ?? 'secret-key-123';

  canActivate(context: ExecutionContext): boolean {
    const request = context.switchToHttp().getRequest<Request>();
    const apiKey = request.headers['x-api-key'];

    if (!apiKey || apiKey !== this.validKey) {
      throw new UnauthorizedException('Invalid or missing API key');
    }
    return true;
  }
}
```

### 3. Guard that reads metadata (Reflector)

**`src/guards/auth.guard.ts`**
```typescript
import {
  CanActivate,
  ExecutionContext,
  Injectable,
  UnauthorizedException,
} from '@nestjs/common';
import { Reflector } from '@nestjs/core';
import { IS_PUBLIC_KEY } from '../decorators/public.decorator';
import { Request } from 'express';

@Injectable()
export class AuthGuard implements CanActivate {
  constructor(private readonly reflector: Reflector) {}

  canActivate(context: ExecutionContext): boolean {
    // Check if the route is marked @Public()
    const isPublic = this.reflector.getAllAndOverride<boolean>(IS_PUBLIC_KEY, [
      context.getHandler(),   // method-level metadata wins
      context.getClass(),     // falls back to class-level
    ]);

    if (isPublic) return true;

    const request = context.switchToHttp().getRequest<Request>();
    const token = this.extractToken(request);

    if (!token) throw new UnauthorizedException('Authentication token required');

    // In a real app, verify the JWT here. For this exercise, accept any non-empty token.
    (request as any).userId = 1; // attach user to request
    return true;
  }

  private extractToken(request: Request): string | null {
    const auth = request.headers.authorization;
    if (!auth?.startsWith('Bearer ')) return null;
    return auth.slice(7);
  }
}
```

### 4. Roles decorator and guard

**`src/decorators/roles.decorator.ts`**
```typescript
import { SetMetadata } from '@nestjs/common';

export enum Role {
  USER = 'user',
  ADMIN = 'admin',
  MODERATOR = 'moderator',
}

export const ROLES_KEY = 'roles';
export const Roles = (...roles: Role[]) => SetMetadata(ROLES_KEY, roles);
```

**`src/guards/roles.guard.ts`**
```typescript
import {
  CanActivate,
  ExecutionContext,
  ForbiddenException,
  Injectable,
} from '@nestjs/common';
import { Reflector } from '@nestjs/core';
import { Role, ROLES_KEY } from '../decorators/roles.decorator';
import { Request } from 'express';

@Injectable()
export class RolesGuard implements CanActivate {
  constructor(private readonly reflector: Reflector) {}

  canActivate(context: ExecutionContext): boolean {
    const requiredRoles = this.reflector.getAllAndOverride<Role[]>(ROLES_KEY, [
      context.getHandler(),
      context.getClass(),
    ]);

    // No @Roles() decorator → allow any authenticated user
    if (!requiredRoles || requiredRoles.length === 0) return true;

    const request = context.switchToHttp().getRequest<Request>();
    const user = (request as any).user; // set by AuthGuard earlier in the chain

    if (!user) throw new ForbiddenException('User not authenticated');

    const hasRole = requiredRoles.some(role => user.roles?.includes(role));
    if (!hasRole) {
      throw new ForbiddenException(
        `Requires one of: ${requiredRoles.join(', ')}`,
      );
    }

    return true;
  }
}
```

### 5. Apply guards at different scopes

**Controller-level** (all routes in this controller require authentication):
```typescript
@UseGuards(AuthGuard)
@Controller('admin')
export class AdminController {
  @Get('dashboard')
  @Roles(Role.ADMIN)
  @UseGuards(RolesGuard)
  getDashboard() {
    return { data: 'admin only' };
  }

  @Get('public-info')
  @Public()  // overrides controller-level AuthGuard
  getPublicInfo() {
    return { data: 'everyone can see this' };
  }
}
```

**Global guard via APP_GUARD** (preferred — allows DI, unlike `app.useGlobalGuards`):
```typescript
// In AppModule providers:
{
  provide: APP_GUARD,
  useClass: AuthGuard,
}
```

### 6. Guard execution order with multiple guards

When multiple guards are stacked, they run left-to-right at the decorator level, outer-to-inner at the class level. Test this:
```typescript
@UseGuards(FirstGuard, SecondGuard)
@Get('test')
testOrder() {}
// FirstGuard runs first, then SecondGuard
// If FirstGuard returns false, SecondGuard never runs
```

### 7. Async guard

Guards can be async — useful when you need to validate against a database:
```typescript
@Injectable()
export class TokenRevocationGuard implements CanActivate {
  constructor(private readonly tokenService: TokenService) {}

  async canActivate(context: ExecutionContext): Promise<boolean> {
    const request = context.switchToHttp().getRequest<Request>();
    const token = extractToken(request);
    const isRevoked = await this.tokenService.isRevoked(token);
    if (isRevoked) throw new UnauthorizedException('Token has been revoked');
    return true;
  }
}
```
