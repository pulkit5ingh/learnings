# A5.1 — ConfigModule

**Phase:** 5 — Configuration & Environment  
**Topic:** @nestjs/config, .env files, ConfigService, Joi validation, namespaced config  
**Type:** Assignment

---

## Goal
Replace hardcoded values with environment variables, validate them at startup with Joi, and organise them into namespaced typed config objects.

## Background

`@nestjs/config` is the official NestJS configuration module. It reads environment variables from `.env` files (using dotenv under the hood) and makes them available via `ConfigService`. When you add a `validationSchema` (Joi), NestJS validates all environment variables at startup and throws a clear error message if any required variable is missing or has the wrong type — catching configuration errors before the app ever handles a request.

**Namespaced config** (also called "config namespaces" or "typed config") lets you group related env vars into config objects. You define a factory function decorated with `@nestjs/config`'s `registerAs` and inject the resulting `ConfigType<typeof configFactory>` in your services. This gives you compile-time type safety for config values.

**Config loading order** (from lowest to highest priority):
1. `.env` file
2. Environment variables in the OS shell
3. Any additional env files specified in `envFilePath`

### 3. App-level ConfigModule with Joi validation

**`src/config/config.module.ts`**
```typescript
import { Module } from '@nestjs/common';
import { ConfigModule as NestConfigModule } from '@nestjs/config';
import * as Joi from 'joi';
import { appConfig } from './app.config';
import { databaseConfig } from './database.config';
import { jwtConfig } from './jwt.config';

@Module({
  imports: [
    NestConfigModule.forRoot({
      isGlobal: true,
      cache: true,       // cache env reads — important for performance
      expandVariables: true,  // allows ${VAR} interpolation in .env
      load: [appConfig, databaseConfig, jwtConfig],
      validationSchema: Joi.object({
        NODE_ENV: Joi.string()
          .valid('development', 'production', 'test')
          .default('development'),
        PORT: Joi.number().integer().min(1).max(65535).default(3000),
        APP_NAME: Joi.string().required(),

        DB_HOST: Joi.string().required(),
        DB_PORT: Joi.number().integer().default(5432),
        DB_USER: Joi.string().required(),
        DB_PASS: Joi.string().required(),
        DB_NAME: Joi.string().required(),

        JWT_SECRET: Joi.string().min(32).required(),
        JWT_EXPIRES_IN: Joi.string().default('15m'),
        JWT_REFRESH_SECRET: Joi.string().min(32).required(),
      }),
      validationOptions: {
        allowUnknown: true,   // allow extra env vars not in schema
        abortEarly: false,    // report all validation errors, not just the first
      },
    }),
  ],
})
export class ConfigModule {}
```

### 4. Namespaced config factories

**`src/config/app.config.ts`**
```typescript
import { registerAs } from '@nestjs/config';

export const appConfig = registerAs('app', () => ({
  name: process.env.APP_NAME ?? 'api',
  port: parseInt(process.env.PORT ?? '3000', 10),
  env: process.env.NODE_ENV ?? 'development',
  isDevelopment: process.env.NODE_ENV === 'development',
  isProduction: process.env.NODE_ENV === 'production',
}));

export type AppConfig = ReturnType<typeof appConfig>;
```

**`src/config/database.config.ts`**
```typescript
import { registerAs } from '@nestjs/config';

export const databaseConfig = registerAs('database', () => ({
  host: process.env.DB_HOST!,
  port: parseInt(process.env.DB_PORT ?? '5432', 10),
  username: process.env.DB_USER!,
  password: process.env.DB_PASS!,
  database: process.env.DB_NAME!,
  synchronize: process.env.NODE_ENV === 'development',
  ssl: process.env.NODE_ENV === 'production'
    ? { rejectUnauthorized: false }
    : false,
}));
```

**`src/config/jwt.config.ts`**
```typescript
import { registerAs } from '@nestjs/config';

export const jwtConfig = registerAs('jwt', () => ({
  secret: process.env.JWT_SECRET!,
  expiresIn: process.env.JWT_EXPIRES_IN ?? '15m',
  refreshSecret: process.env.JWT_REFRESH_SECRET!,
  refreshExpiresIn: process.env.JWT_REFRESH_EXPIRES_IN ?? '7d',
}));
```

### 5. Injecting typed config in a service

```typescript
import { Injectable } from '@nestjs/common';
import { ConfigType } from '@nestjs/config';
import { Inject } from '@nestjs/common';
import { appConfig } from '../config/app.config';
import { jwtConfig } from '../config/jwt.config';

@Injectable()
export class AppInfoService {
  constructor(
    @Inject(appConfig.KEY)
    private readonly app: ConfigType<typeof appConfig>,

    @Inject(jwtConfig.KEY)
    private readonly jwt: ConfigType<typeof jwtConfig>,
  ) {}

  getInfo() {
    return {
      name: this.app.name,
      env: this.app.env,
      jwtExpiry: this.jwt.expiresIn,
      // jwt.secret is not returned — never expose secrets
    };
  }
}
```

### 6. Using ConfigService (non-typed)

```typescript
import { Injectable } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';

@Injectable()
export class SomeService {
  constructor(private readonly config: ConfigService) {}

  getDbHost(): string {
    return this.config.get<string>('database.host')!;
    // dot notation for namespaced config
  }

  getPort(): number {
    return this.config.get<number>('app.port', 3000);
    // second argument is the default value
  }
}
```

### 7. main.ts using config

**`src/main.ts`**
```typescript
import { NestFactory } from '@nestjs/core';
import { ConfigService } from '@nestjs/config';
import { ValidationPipe } from '@nestjs/common';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);

  const config = app.get(ConfigService);
  const port = config.get<number>('app.port', 3000);
  const env = config.get<string>('app.env');

  app.useGlobalPipes(new ValidationPipe({ whitelist: true, transform: true }));

  await app.listen(port);
  console.log(`App running on port ${port} in ${env} mode`);
}
bootstrap();
```

- [ ] `DB_PORT=abc` (non-numeric) causes startup to fail with a type error message
- [ ] Namespaced `appConfig` is injectable via `@Inject(appConfig.KEY)` with full TypeScript types
- [ ] `ConfigService.get('database.host')` returns the correct value using dot notation
- [ ] `isGlobal: true` means you don't need to import `ConfigModule` in every feature module
- [ ] `cache: true` means env reads are fast even when ConfigService is injected in many places
