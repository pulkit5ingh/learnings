## Goal
Build a GraphQL API using NestJS's code-first approach, implement typed resolvers with authentication, and solve the N+1 query problem using DataLoader.

## Background

NestJS supports two GraphQL approaches:
- **Code-first** — you write TypeScript classes with decorators (`@ObjectType`, `@Resolver`, `@Query`), and NestJS generates the GraphQL schema automatically. Recommended.
- **Schema-first** — you write the `.graphql` schema file manually and generate TypeScript types from it.

**Key decorators:**
- `@ObjectType()` — marks a class as a GraphQL type (equivalent to `type` in SDL).
- `@InputType()` — marks a class as a GraphQL input type (used in mutations).
- `@Resolver(of => Entity)` — marks a class as a resolver for the given type.
- `@Query(returns => Type)` — defines a query field.
- `@Mutation(returns => Type)` — defines a mutation field.
- `@ResolveField()` — resolves a field on a type (used for computed fields and relations).
- `@Context()` — injects the GraphQL context (contains `req`, user, DataLoader instances).

**DataLoader** solves the N+1 problem: when fetching 10 posts with their authors, a naive resolver makes 10 separate database calls. DataLoader batches these into a single call using a key-value cache per request.

### 4. User resolver

**`src/users/users.resolver.ts`**
```typescript
import {
  Args, Context, Int, Mutation,
  Parent, Query, ResolveField, Resolver,
} from '@nestjs/graphql';
import { UseGuards } from '@nestjs/common';
import { User } from './models/user.model';
import { Post } from '../posts/models/post.model';
import { UsersService } from './users.service';
import { CreateUserInput } from './dto/create-user.input';
import { UpdateUserInput } from './dto/update-user.input';
import { GqlAuthGuard } from '../auth/guards/gql-auth.guard';
import { CurrentUser } from '../auth/decorators/current-user.decorator';

@Resolver(() => User)
export class UsersResolver {
  constructor(private readonly usersService: UsersService) {}

  @Query(() => [User])
  async users(): Promise<User[]> {
    return this.usersService.findAll();
  }

  @Query(() => User, { nullable: true })
  async user(@Args('id', { type: () => Int }) id: number): Promise<User | null> {
    return this.usersService.findOne(id);
  }

  @Query(() => User)
  @UseGuards(GqlAuthGuard)
  me(@CurrentUser() user: User): User {
    return user;
  }

  @Mutation(() => User)
  async createUser(@Args('input') input: CreateUserInput): Promise<User> {
    return this.usersService.create(input);
  }

  @Mutation(() => User)
  @UseGuards(GqlAuthGuard)
  async updateUser(@Args('input') input: UpdateUserInput): Promise<User> {
    return this.usersService.update(input.id, input);
  }

  @Mutation(() => Boolean)
  @UseGuards(GqlAuthGuard)
  async deleteUser(
    @Args('id', { type: () => Int }) id: number,
  ): Promise<boolean> {
    await this.usersService.softDelete(id);
    return true;
  }

  // ResolveField uses DataLoader to solve N+1
  @ResolveField(() => [Post])
  async posts(
    @Parent() user: User,
    @Context() ctx: { req: any; postsByUserLoader: DataLoader<number, Post[]> },
  ): Promise<Post[]> {
    return ctx.postsByUserLoader.load(user.id);
  }
}
```

### 5. DataLoader setup

**`src/posts/loaders/posts-by-user.loader.ts`**
```typescript
import * as DataLoader from 'dataloader';
import { Injectable, Scope } from '@nestjs/common';
import { InjectRepository } from '@nestjs/typeorm';
import { Repository, In } from 'typeorm';
import { Post } from '../entities/post.entity';

@Injectable({ scope: Scope.REQUEST })  // new loader per request
export class PostsByUserLoader {
  constructor(
    @InjectRepository(Post)
    private readonly postRepo: Repository<Post>,
  ) {}

  createLoader(): DataLoader<number, Post[]> {
    return new DataLoader<number, Post[]>(async (userIds: readonly number[]) => {
      const posts = await this.postRepo.find({
        where: { authorId: In([...userIds]) },
        order: { createdAt: 'DESC' },
      });

      // Group posts by userId
      const postsByUser = new Map<number, Post[]>();
      userIds.forEach(id => postsByUser.set(id, []));
      posts.forEach(post => {
        const list = postsByUser.get(post.authorId) ?? [];
        list.push(post);
        postsByUser.set(post.authorId, list);
      });

      return userIds.map(id => postsByUser.get(id) ?? []);
    });
  }
}
```

**Inject the DataLoader into the GraphQL context:**
```typescript
// In GraphQLModule.forRoot:
context: ({ req }) => {
  const postsByUserLoader = postsByUserLoaderService.createLoader();
  return { req, postsByUserLoader };
},
```

Or use a context factory with NestJS DI:
```typescript
// Add to providers in GraphQLModule
{
  provide: APP_INTERCEPTOR,
  useClass: DataLoaderInterceptor,
}
```

### 6. GraphQL authentication guard

**`src/auth/guards/gql-auth.guard.ts`**
```typescript
import { ExecutionContext, Injectable } from '@nestjs/common';
import { AuthGuard } from '@nestjs/passport';
import { GqlExecutionContext } from '@nestjs/graphql';

@Injectable()
export class GqlAuthGuard extends AuthGuard('jwt') {
  getRequest(context: ExecutionContext) {
    // GraphQL context contains the request differently than HTTP
    const ctx = GqlExecutionContext.create(context);
    return ctx.getContext().req;
  }
}
```

**`src/auth/decorators/current-user.decorator.ts`**
```typescript
import { createParamDecorator, ExecutionContext } from '@nestjs/common';
import { GqlExecutionContext } from '@nestjs/graphql';

export const CurrentUser = createParamDecorator((data: unknown, ctx: ExecutionContext) => {
  const gqlCtx = GqlExecutionContext.create(ctx);
  return gqlCtx.getContext().req.user;
});
```

### 7. Example queries and mutations

```graphql
# Register
mutation {
  createUser(input: { name: "Alice", email: "alice@example.com", password: "Password1!" }) {
    id
    name
    email
  }
}

# Get all users with their posts (DataLoader prevents N+1)
query {
  users {
    id
    name
    posts {
      id
      title
      published
    }
  }
}

# Get current user (requires auth)
query {
  me {
    id
    email
    name
  }
}
```

## Deliverables

- [ ] `autoSchemaFile` generates `schema.gql` from decorators — readable and correct SDL
- [ ] `query { users { id name posts { title } } }` resolves without N+1 (single SQL for posts)
- [ ] `mutation { createUser(...) }` validates input via `class-validator` decorators on `CreateUserInput`
- [ ] `@UseGuards(GqlAuthGuard)` on `me` query returns 401 without a Bearer token
- [ ] `DataLoader` batches 10 author lookups into a single `WHERE authorId IN (...)` query
- [ ] `@ResolveField()` receives the DataLoader instance from `@Context()`
