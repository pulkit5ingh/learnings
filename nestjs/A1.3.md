# A1.3 — Providers & Services

**Phase:** 1 — NestJS Fundamentals  
**Topic:** @Injectable, constructor DI, custom providers (useClass/useValue/useFactory), scope (DEFAULT/REQUEST/TRANSIENT)  
**Type:** Assignment

---

## Goal
Master the full provider system: standard services, custom providers using all three factory forms, and provider scopes that affect instance lifetime.

## Background

A **provider** is anything registered in a module's `providers` array. The DI container identifies each provider by a **token** — by default the class itself, but you can use a string or `Symbol`. The container creates providers lazily (on first injection) and caches them according to their **scope**.

**Three provider scopes:**
- `DEFAULT` (singleton) — one instance per application lifetime. This is the 99% case. Safe for stateless services, database connections, and caches.
- `REQUEST` — one instance per incoming HTTP request. Use when you need request-scoped data (e.g., the current user) accessible across multiple injected services without passing it explicitly.
- `TRANSIENT` — a new instance every time it is injected. Rare; use for providers that must not share state between consumers.

**Custom provider forms** allow you to decouple the token from the implementation:
- `useClass` — standard; can swap implementations by environment.
- `useValue` — inject a pre-built object (useful for config objects, constants, mocks in tests).
- `useFactory` — inject using a factory function that can be async and can itself receive injected dependencies.

## Tasks

### 1. Standard provider (recap)

```typescript
// Standard shorthand (token = class reference)
@Module({ providers: [CatsService] })

// Equivalent long form:
@Module({
  providers: [{ provide: CatsService, useClass: CatsService }],
})
```

Both are identical. The long form becomes useful when you want to swap implementations.

### 2. useValue provider — inject a config object

**`src/config/database.config.ts`**
```typescript
export interface DatabaseConfig {
  host: string;
  port: number;
  name: string;
}

export const DATABASE_CONFIG = 'DATABASE_CONFIG';

export const databaseConfig: DatabaseConfig = {
  host: 'localhost',
  port: 5432,
  name: 'nestdb',
};
```

Register in module:
```typescript
import { Module } from '@nestjs/common';
import { DATABASE_CONFIG, databaseConfig } from './config/database.config';

@Module({
  providers: [
    {
      provide: DATABASE_CONFIG,
      useValue: databaseConfig,
    },
  ],
  exports: [DATABASE_CONFIG],
})
export class ConfigModule {}
```

Inject in a service using `@Inject`:
```typescript
import { Injectable, Inject } from '@nestjs/common';
import { DATABASE_CONFIG, DatabaseConfig } from '../config/database.config';

@Injectable()
export class ReportService {
  constructor(
    @Inject(DATABASE_CONFIG) private readonly dbConfig: DatabaseConfig,
  ) {}

  getConnectionString(): string {
    return `postgres://${this.dbConfig.host}:${this.dbConfig.port}/${this.dbConfig.name}`;
  }
}
```

### 3. useClass provider — swap implementations

Define an interface and two implementations:
```typescript
// src/notifications/notification.interface.ts
export interface NotificationService {
  send(message: string): void;
}

export const NOTIFICATION_SERVICE = 'NOTIFICATION_SERVICE';
```

```typescript
// src/notifications/email-notification.service.ts
import { Injectable } from '@nestjs/common';
import { NotificationService } from './notification.interface';

@Injectable()
export class EmailNotificationService implements NotificationService {
  send(message: string): void {
    console.log(`[EMAIL] ${message}`);
  }
}
```

```typescript
// src/notifications/sms-notification.service.ts
import { Injectable } from '@nestjs/common';
import { NotificationService } from './notification.interface';

@Injectable()
export class SmsNotificationService implements NotificationService {
  send(message: string): void {
    console.log(`[SMS] ${message}`);
  }
}
```

Register with useClass:
```typescript
const isProduction = process.env.NODE_ENV === 'production';

@Module({
  providers: [
    EmailNotificationService,
    SmsNotificationService,
    {
      provide: NOTIFICATION_SERVICE,
      useClass: isProduction ? SmsNotificationService : EmailNotificationService,
    },
  ],
})
export class NotificationsModule {}
```

### 4. useFactory provider — async initialization

```typescript
// src/database/database.provider.ts
import { DataSource } from 'typeorm';

export const DATABASE_CONNECTION = 'DATABASE_CONNECTION';

export const databaseProviders = [
  {
    provide: DATABASE_CONNECTION,
    useFactory: async (): Promise<DataSource> => {
      const dataSource = new DataSource({
        type: 'sqlite',
        database: ':memory:',
        entities: [],
        synchronize: true,
      });
      return dataSource.initialize();
    },
  },
];
```

A factory can also receive injected dependencies:
```typescript
{
  provide: 'ADVANCED_SERVICE',
  useFactory: (configService: ConfigService, logger: Logger) => {
    return new AdvancedService(configService.get('API_KEY'), logger);
  },
  inject: [ConfigService, Logger],
}
```

### 5. Provider scopes

```typescript
import { Injectable, Scope } from '@nestjs/common';

// Singleton (default) - shared across all requests
@Injectable()
export class SingletonService {
  private readonly id = Math.random();
  getId() { return this.id; }
}

// Request-scoped - new instance per HTTP request
@Injectable({ scope: Scope.REQUEST })
export class RequestScopedService {
  private readonly id = Math.random();
  getId() { return this.id; }
}

// Transient - new instance per injection
@Injectable({ scope: Scope.TRANSIENT })
export class TransientService {
  private readonly id = Math.random();
  getId() { return this.id; }
}
```

To demonstrate scope behaviour, create a controller that injects all three and returns their IDs — hit the endpoint twice and observe that only the REQUEST and TRANSIENT IDs change:

```typescript
@Controller('scope-demo')
export class ScopeDemoController {
  constructor(
    private readonly singleton: SingletonService,
    private readonly requestScoped: RequestScopedService,
    private readonly transient: TransientService,
  ) {}

  @Get()
  getIds() {
    return {
      singleton: this.singleton.getId(),
      requestScoped: this.requestScoped.getId(),
      transient: this.transient.getId(),
    };
  }
}
```

Note: when a REQUEST-scoped provider is injected into a controller, the controller itself becomes REQUEST-scoped too — NestJS propagates the scope upward.

### 6. Inject REQUEST object in REQUEST-scoped service

```typescript
import { Injectable, Scope, Inject } from '@nestjs/common';
import { REQUEST } from '@nestjs/core';
import { Request } from 'express';

@Injectable({ scope: Scope.REQUEST })
export class CurrentUserService {
  constructor(@Inject(REQUEST) private readonly request: Request) {}

  getUserAgent(): string {
    return this.request.headers['user-agent'] ?? 'unknown';
  }
}
```
