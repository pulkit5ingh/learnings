# P8.2 — LLM Cost & Quality Monitoring Dashboard

**Phase:** 8 — Production & MLOps  
**Topic:** Observability, Monitoring, Analytics  
**Type:** Project

---

## Goal
Build a monitoring dashboard that tracks token usage, cost, latency, and quality scores across all LLM calls in your apps. Every team running an LLM product needs this to avoid surprise bills and catch quality regressions.

---

## What to Build

### 1. Instrumentation Library
Write a Python package `llm_monitor` that wraps any LLM call:
```python
from llm_monitor import track

# Usage — wrap any LLM call
@track(project="rag-api", tags={"user_id": user_id, "question_type": "factual"})
def ask_claude(messages):
    return anthropic.messages.create(...)

# Or use the context manager
with track("rag-retrieval") as span:
    result = retrieve(question)
    span.set_metric("chunks_retrieved", len(result))
```

The `track` decorator automatically captures:
- Prompt tokens, completion tokens
- Latency (time to first token, total time)
- Model used
- Cost (calculated from token counts × model pricing)
- Any errors

### 2. Storage Backend
PostgreSQL database:
```sql
CREATE TABLE llm_calls (
    id UUID PRIMARY KEY,
    project TEXT,
    model TEXT,
    prompt_tokens INT,
    completion_tokens INT,
    cost_usd DECIMAL(10, 6),
    latency_ms INT,
    ttft_ms INT,          -- time to first token
    tags JSONB,
    error TEXT,
    created_at TIMESTAMP
);

CREATE TABLE quality_scores (
    call_id UUID REFERENCES llm_calls(id),
    metric TEXT,          -- "faithfulness", "relevancy", "user_rating"
    score DECIMAL(4, 3),
    scored_at TIMESTAMP
);
```

### 3. Dashboard — Grafana or Custom Streamlit
Build a dashboard with these pages:

**Overview Page**:
- Total spend: today / this week / this month (with sparklines)
- Total requests: today / this week
- Average latency: P50, P95, P99
- Error rate
- Cache hit rate

**Cost Breakdown**:
- Cost by model (pie chart)
- Cost by project (bar chart)
- Cost over time (line chart, daily/weekly view)
- Most expensive individual calls (table)
- Projected monthly spend (based on current usage rate)

**Quality Page**:
- Faithfulness score over time (line chart)
- Answer relevancy over time
- User ratings distribution (thumbs up/down ratio)
- Lowest quality calls (table, sorted by score)
- Quality vs cost scatter plot (are expensive calls higher quality?)

**Latency Page**:
- P50, P95, P99 latency over time
- Latency by model comparison
- Slowest calls (table)
- Time to first token distribution

**Alerts Page**:
- Alert rules: cost > $X/day, quality < threshold, error rate > X%
- Alert history
- Email notifications via SendGrid (or just log to console)

### 4. Real-Time Updates
- Dashboard auto-refreshes every 60 seconds
- Show a "last updated" timestamp
- Highlight metrics that changed significantly in the last hour (red/green arrows)

### 5. API for External Data Push
Allow other apps to push metrics:
```
POST /api/metrics
Body: { "project": "...", "model": "...", "tokens": {...}, "latency_ms": 234, "tags": {} }

POST /api/quality
Body: { "call_id": "...", "metric": "faithfulness", "score": 0.87 }
```

### 6. Export
- Download CSV: any time range, any project
- Weekly report email: auto-generated markdown summary of the week's usage and quality

---

## Tech Stack
- **Backend**: FastAPI + PostgreSQL (TimescaleDB extension for time-series)
- **Dashboard**: Streamlit or Grafana (with PostgreSQL datasource)
- **Instrumentation**: Python package installable via `pip install llm-monitor`
