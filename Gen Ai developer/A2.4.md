# A2.4 — Dropout & Batch Normalization From Scratch

**Phase:** 2 — Machine Learning Core  
**Topic:** Deep Learning Internals  
**Type:** Assignment

---

## Goal
Implement Dropout and Batch Normalization as custom PyTorch modules. Measure their effect on training stability and overfitting. This forces you to understand exactly what these layers do mathematically.

---

## Task

### Part 1 — Custom Dropout Layer
Implement `class CustomDropout(nn.Module)`:
- During training: randomly zero out neurons with probability `p`. Scale remaining by `1/(1-p)` (inverted dropout).
- During eval: pass through unchanged
- Verify your output matches `nn.Dropout` exactly on the same input with the same random seed

### Part 2 — Custom Batch Normalization
Implement `class CustomBatchNorm1d(nn.Module)`:
- Compute mean and variance across the batch dimension
- Normalize: `x_hat = (x - mean) / sqrt(var + epsilon)`
- Apply learnable scale `gamma` and shift `beta`: `y = gamma * x_hat + beta`
- Track running mean/variance for inference
- Verify your output matches `nn.BatchNorm1d` exactly

### Part 3 — Overfitting Experiment
- Build a simple 4-layer MLP for MNIST classification
- Train 3 versions:
  1. No regularization
  2. With `nn.Dropout(0.5)` after each hidden layer
  3. With `nn.BatchNorm1d` after each hidden layer
- Use a small training set (only 1000 samples) to force overfitting
- Track training accuracy and validation accuracy per epoch

### Part 4 — Results
- Plot training vs validation accuracy for all 3 models on one graph
- Write a 5-sentence comparison: which worked better and why?
