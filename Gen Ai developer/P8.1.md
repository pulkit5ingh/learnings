## Goal
Take an existing RAG system and make it truly production-ready: streaming, auth, rate limiting, monitoring, automated evals, Docker, and a live deployment. This is the complete production AI backend.

---

## What to Build

### 1. Core API (FastAPI)
```
POST /v1/ask               — ask a question, get streaming answer
POST /v1/documents/ingest  — upload and index a document
GET  /v1/documents         — list all indexed documents
DELETE /v1/documents/{id}  — remove a document from the index
GET  /v1/health            — health check (returns: db ok, vector store ok, llm ok)
```

All endpoints require Bearer token auth.

### 2. Streaming with SSE
The `/ask` endpoint streams the response using Server-Sent Events:
```python
async def ask_stream(question: str, session_id: str):
    chunks = retrieve(question)
    async for token in claude.stream_message(question, chunks):
        yield f"data: {json.dumps({'token': token})}\n\n"
    yield f"data: {json.dumps({'done': True, 'sources': [...]})}\n\n"
```

### 3. Caching
- **Semantic cache**: before retrieval, check if a very similar question was asked recently
  - Embed the question, find nearest cached answer (similarity > 0.97)
  - Return cached answer instantly (log as "cache hit")
- Cache stored in Redis with 1-hour TTL
- Track cache hit rate in metrics

### 4. Rate Limiting
- 100 questions/hour per user
- 10 document uploads/day per user
- Return `Retry-After` header when rate limit exceeded

### 5. LangSmith Observability
- Trace every request: retrieval → reranking → generation
- Tag traces with: user_id, session_id, question_type, cache_hit
- Set up a LangSmith dataset for your eval questions
- Connect LangSmith feedback: users can rate answers 👍/👎

### 6. Automated Eval Suite
Write `eval/regression_eval.py` with 50 curated question-answer pairs:
```python
EVAL_SET = [
    {"question": "...", "expected_contains": ["..."], "expected_not_contains": ["hallucinated fact"]},
    ...
]
```
Run after every deploy. Gate deployment if:
- Pass rate < 0.85
- P95 latency > 4000ms
- Hallucination rate > 0.10

### 7. Docker Compose Stack
```yaml
services:
  api:        # FastAPI app
  chromadb:   # Vector database
  redis:      # Cache + rate limiting
  postgres:   # Request logs, user data
  nginx:      # Reverse proxy with SSL termination
```

### 8. Deployment
Deploy to Railway, Fly.io, or Render (free tier):
- Set environment variables as secrets
- Configure auto-deploy from main branch
- Custom domain (optional)
- SSL certificate auto-provisioned

### 9. Monitoring Dashboard
Simple Streamlit admin dashboard:
- Total questions today / this week
- Cache hit rate
- Average latency (P50, P95)
- Top 10 most asked questions
- Eval pass rate over time (line chart)
- Active users count
