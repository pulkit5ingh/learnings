## RAGAS Metrics Explained

| Metric | What it measures |
|--------|-----------------|
| **Faithfulness** | Is the answer grounded in the retrieved context? (no hallucination) |
| **Answer Relevancy** | Does the answer actually address the question? |
| **Context Precision** | Are the retrieved chunks relevant to the question? |
| **Context Recall** | Does the retrieved context contain everything needed to answer? |

## Task

### Part 1 — Create Evaluation Dataset
Build a dataset of 30 test examples:
```python
from datasets import Dataset

eval_data = {
    "question": ["What is RAG?", ...],          # 30 questions
    "answer": ["RAG stands for...", ...],        # your RAG's answers
    "contexts": [["chunk1", "chunk2"], ...],     # retrieved chunks per question
    "ground_truth": ["The correct answer...", ...],  # you write these
}
dataset = Dataset.from_dict(eval_data)
```
- Write at least 10 questions where faithfulness should be low (questions the doc doesn't cover → watch model hallucinate)
- Write at least 10 questions well-covered by the document

### Part 2 — Run RAGAS
```python
from ragas import evaluate
from ragas.metrics import faithfulness, answer_relevancy, context_precision, context_recall

results = evaluate(dataset, metrics=[faithfulness, answer_relevancy, context_precision, context_recall])
print(results)
```

- For each: what irrelevant chunks were retrieved?

### Part 4 — Improve and Re-Evaluate

- Re-run RAGAS on the same 30 questions
- Show the before/after scores for all 4 metrics

### Part 5 — Automated Eval in CI
