# P4.2 — Domain-Specific Chatbot (Fine-tuned LLM)

**Phase:** 4 — Large Language Models  
**Topic:** Fine-tuning, Deployment, Evaluation  
**Type:** Project

---

## Goal
Fine-tune an open-source LLM on a domain-specific dataset, deploy it locally with Ollama, and benchmark it rigorously against the base model. This end-to-end project covers the full fine-tuning lifecycle.

---

## Domain Options (pick one)
- **Medical Q&A**: Answer patient questions about symptoms, medications (non-diagnostic)
- **Legal FAQ**: Explain legal concepts, Indian law, common legal procedures
- **Customer Support**: Answer product-specific questions for a fictional company
- **Coding Assistant**: Specialized in a niche language or framework (e.g., Rust, Flutter)

---

## What to Build

### 1. Dataset Collection & Curation
- Collect or find 1,000–5,000 instruction-response pairs in your domain
- Format as JSON: `{"instruction": "...", "input": "...", "output": "..."}`
- Data quality check:
  - Remove duplicates
  - Remove examples where output is <10 words
  - Manually review 50 random examples and flag bad ones
- Split: 80% train, 10% val, 10% test — freeze test set and never train on it

### 2. Fine-tuning (QLoRA)
- Base model: `mistralai/Mistral-7B-Instruct-v0.2` or `Llama-3.1-8B-Instruct`
- Train with QLoRA (see A4.3 for setup)
- Track: training loss, validation loss per epoch
- Use early stopping based on validation loss

### 3. Evaluation Suite
Create 50 test questions for your domain. For each, evaluate base model vs fine-tuned model:

**Automated metrics:**
- ROUGE-L score on test set
- BERTScore on test set

**LLM-as-judge:**
- Use Claude/GPT-4 to score both responses 1-5 on: accuracy, helpfulness, safety
- Prompt: "You are an expert in {domain}. Rate this response 1-5 for accuracy and helpfulness."

**Report:**
- Win/tie/loss count: fine-tuned vs base
- Average LLM judge score: fine-tuned vs base

### 4. Local Deployment with Ollama
- Merge LoRA adapters into base model
- Convert to GGUF format (Q4_K_M quantization)
- Create a custom Modelfile for Ollama with your system prompt
- `ollama create my-domain-bot -f Modelfile`
- Test via `ollama run my-domain-bot`

### 5. Chat Interface
Build a simple Streamlit chat app:
- Connect to Ollama API
- Chat history display
- Domain-specific welcome message and suggested starter questions
- Show token count and response time per message
- "Clear conversation" button

### 6. Safety Guardrails
- Add a pre-response check: if user asks something outside your domain, respond with a polite redirect
- Implement using a fast classifier (keyword-based or small LLM call)
