# A3.4 — Fine-tune T5 for Text Summarization

**Phase:** 3 — Natural Language Processing  
**Topic:** Seq2Seq Models, Evaluation  
**Type:** Assignment

---

## Goal
Fine-tune a T5 model for abstractive text summarization. Learn seq2seq training, beam search, and how to evaluate generation quality with ROUGE scores.

---

## Dataset
```python
from datasets import load_dataset
dataset = load_dataset("cnn_dailymail", "3.0.0")
```
Use 10,000 training samples and 1,000 validation samples (full dataset is large).

---

## Task

### Part 1 — Data Preparation
- Load `t5-small` tokenizer
- Prefix inputs with `"summarize: "` (T5 requires task prefix)
- Tokenize: articles with `max_length=512`, summaries with `max_length=128`
- Inspect 2 examples: show the tokenized input and target

### Part 2 — Fine-tune T5
- Load `T5ForConditionalGeneration` with `t5-small`
- Use `Seq2SeqTrainer` with `Seq2SeqTrainingArguments`:
  - `predict_with_generate=True`
  - `num_train_epochs=3`
  - `per_device_train_batch_size=8`
  - `per_device_eval_batch_size=8`

### Part 3 — Evaluation with ROUGE
- Install `rouge_score`
- Compute ROUGE-1, ROUGE-2, ROUGE-L on 500 validation examples
- Generate summaries using:
  - Greedy decoding
  - Beam search (num_beams=4)
  - Compare ROUGE scores of both

### Part 4 — Compare to Baseline
- Zero-shot: use the pretrained `t5-small` (no fine-tuning) and measure ROUGE
- Report the improvement from fine-tuning

### Part 5 — Manual Analysis
- Show 5 article → reference summary → generated summary comparisons
- Identify 2 cases where the model hallucinated (added facts not in the article)
