# A6.1 — ReAct Agent From Scratch

**Phase:** 6 — AI Agents  
**Topic:** Agent Fundamentals, Tool Use  
**Type:** Assignment

---

## Goal
Build a ReAct (Reasoning + Acting) agent using only the raw LLM API — no LangChain agents, no AutoGen. Understand the thought-action-observation loop from the ground up.

---

## Tools to Give the Agent
1. **Calculator**: evaluates a math expression string, returns the result
2. **Wikipedia**: fetches the first 3 paragraphs of a Wikipedia article by topic
3. **Web Search**: uses the Tavily API or DuckDuckGo to search the web

---

## Task

### Part 1 — ReAct Prompt Format
Implement the ReAct loop using this prompt structure:
```
You have access to the following tools:
- calculator(expression: str) -> float
- wikipedia(topic: str) -> str
- web_search(query: str) -> str

To use a tool, output:
Thought: [your reasoning]
Action: tool_name(argument)

When you have the final answer:
Thought: I now have enough information.
Final Answer: [your answer]

Begin!
Question: {question}
```

### Part 2 — Parse Action, Execute Tool, Feed Back Observation
```python
def run_agent(question, max_steps=10):
    history = f"Question: {question}\n"
    for step in range(max_steps):
        response = llm(history)
        history += response
        if "Final Answer:" in response:
            return extract_final_answer(response)
        action = parse_action(response)  # extract tool_name and argument
        observation = execute_tool(action)  # call the real tool
        history += f"\nObservation: {observation}\n"
    return "Max steps reached"
```

Test questions that require multi-step reasoning:

### Part 4 — Failure Analysis
- Find 3 questions where the agent fails (loops, wrong answer, tool misuse)
- Document exactly where the reasoning breaks down
- Propose one fix for each failure

### Part 5 — Tool Error Handling
- Modify `execute_tool` to handle: API timeout, invalid argument, tool returns empty result
- The agent should gracefully retry or try a different approach rather than crashing
