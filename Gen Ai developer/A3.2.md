## Goal
Fine-tune a BERT model on a binary sentiment classification task using the Hugging Face ecosystem. Understand tokenization, model loading, and the Trainer API.

---

## Dataset
Use **IMDb Movie Reviews** dataset:
```python
from datasets import load_dataset
dataset = load_dataset("imdb")
```
50,000 reviews, balanced positive/negative labels.

## Task

### Part 1 — Tokenization
- Load `bert-base-uncased` tokenizer
- Tokenize the dataset with `max_length=256`, `truncation=True`, `padding="max_length"`
- Inspect 3 examples: print raw text, token IDs, attention mask, and decoded tokens
- Answer: what are `[CLS]` and `[SEP]` tokens for?

### Part 2 — Model Setup
- Load `BertForSequenceClassification` with `num_labels=2`
- Print total parameter count vs trainable parameter count
- Freeze all layers except the last 2 transformer blocks + classifier head
- Print trainable param count now

### Part 3 — Training with Trainer API
```python
training_args = TrainingArguments(
    output_dir="./bert-sentiment",
    num_train_epochs=3,
    per_device_train_batch_size=16,
    per_device_eval_batch_size=32,
    evaluation_strategy="epoch",
    save_strategy="epoch",
    load_best_model_at_end=True,
    metric_for_best_model="f1",
)
```
- Define compute_metrics returning accuracy, F1, precision, recall
- Train on 20,000 samples, eval on 5,000 (use a subset to keep it fast)

### Part 4 — Evaluation
- Final test accuracy and F1 on 5,000 test samples
- Plot the confusion matrix
- Show 5 examples where the model is wrong — what went wrong?

### Part 5 — Compare
- Run the same evaluation with `distilbert-base-uncased` (faster, smaller)
- Compare: accuracy, F1, inference speed (seconds per 1000 samples)
