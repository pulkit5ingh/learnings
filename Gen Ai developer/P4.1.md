# P4.1 — AI Code Reviewer CLI Tool

**Phase:** 4 — Large Language Models  
**Topic:** LLM APIs, Developer Tools  
**Type:** Project

---

## Goal
Build a CLI tool that reads a git diff (or GitHub PR) and generates structured, actionable code review comments using Claude or GPT-4. This mirrors real tools used at engineering teams.

---

## What to Build

### 1. Core CLI
```bash
# Usage examples
ai-review --diff ./changes.diff
ai-review --pr 123 --repo owner/repo
ai-review --staged          # review currently staged git changes
ai-review --branch main     # review current branch vs main
```

### 2. Diff Parsing
- Parse unified diff format into structured chunks: `file → hunk → lines changed`
- For each changed file, extract: filename, language, added lines, removed lines, surrounding context (5 lines)
- Filter out: lock files, generated files, binary files, files >500 lines changed

### 3. Review Generation
For each modified file, send to Claude/GPT-4 with:
```
System: You are a senior software engineer doing code review. 
Be concise, specific, and constructive. Focus on real issues.

User: Review this diff for {filename}:
[DIFF CONTENT]

Respond as JSON:
{
  "issues": [
    {
      "line": 42,
      "severity": "error|warning|suggestion",
      "category": "bug|security|performance|style|test",
      "message": "...",
      "suggestion": "..."
    }
  ],
  "summary": "One sentence overall assessment"
}
```

### 4. Output Formats
Support multiple output formats via `--format` flag:
- `--format terminal` — colored terminal output with line numbers
- `--format markdown` — GitHub-flavored markdown
- `--format json` — raw JSON
- `--format github-comment` — formatted for posting as a PR review comment

### 5. GitHub Integration
- Use GitHub API (`gh` CLI or `PyGithub`) to:
  - Fetch PR diff
  - Post review comments inline on the PR (each issue as a comment on the specific line)
  - Post a summary review comment

### 6. Configuration
Support a `.ai-review.yml` config file:
```yaml
model: claude-3-5-sonnet-20241022
focus: [security, performance]   # categories to prioritize
ignore_files: ["*.lock", "migrations/*"]
max_files_per_review: 10
severity_threshold: warning      # only show warnings and above
```

### 7. Cost Tracking
- Track token usage per review
- Print: "Review cost: ~$0.04 (2,400 input tokens + 800 output tokens)"
