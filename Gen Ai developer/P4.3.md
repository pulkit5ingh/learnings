# P4.3 — Prompt Playground Dashboard

**Phase:** 4 — Large Language Models  
**Topic:** LLM APIs, Developer Tools, Observability  
**Type:** Project

---

## Goal
Build a developer tool for A/B testing prompts side by side, tracking token usage and cost, and saving prompt versions. Every LLM product team needs something like this.

---

## What to Build

### 1. Core Prompt Editor
- Split-pane interface: left side is Prompt A, right side is Prompt B
- Each pane has:
  - System prompt text area
  - User message text area
  - Model selector dropdown (GPT-4o, GPT-4o-mini, Claude 3.5 Sonnet, Claude 3 Haiku)
  - Temperature slider (0.0 to 2.0)
  - Max tokens input
- "Run Both" button: sends both prompts simultaneously, displays responses side by side

### 2. Response Display
For each response show:
- Response text with markdown rendering
- Token usage: prompt tokens, completion tokens, total
- Cost: calculated from token usage × current model pricing
- Latency: time to first token (streaming), time to full response
- "Copy" button

### 3. Prompt Versioning
- Save prompts with a name, version number, and notes
- Backend: SQLite database with table `prompts(id, name, version, system_prompt, user_template, model, params, created_at)`
- Version history: click a prompt name to see all past versions
- "Restore" button to load a previous version into the editor

### 4. Variable Templating
- Support `{{variable}}` syntax in prompts
- Users define variables in a key-value table below the editor
- Example: `"Summarize this article in {{language}}: {{article_text}}"`
- Variables get substituted before sending

### 5. Test Suite Mode
- Upload a CSV with columns: `input`, `expected_output`
- Run your current prompt against all rows in the CSV
- Score each row with a pass/fail rubric (exact match, contains, LLM judge)
- Show results table with pass rate

### 6. Cost & Usage Dashboard
- Track all API calls in the SQLite database
- Dashboard page showing:
  - Total spend (today, this week, this month)
  - Cost breakdown by model
  - Average cost per prompt run
  - Most expensive prompts
  - Usage over time (line chart)

### 7. Export
- Export prompt + response as markdown
- Export test suite results as CSV
- Share a prompt via a shareable link (serialize to URL params)

---

## Tech Stack
- **Frontend**: Streamlit (fast) or Next.js (polished)
- **Backend**: FastAPI + SQLite
- **LLM**: OpenAI SDK + Anthropic SDK (both)
