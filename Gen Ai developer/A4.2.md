# A4.2 — Chain-of-Thought vs Direct Prompting

**Phase:** 4 — Large Language Models  
**Topic:** Prompt Engineering, Reasoning  
**Type:** Assignment

---

## Goal
Empirically measure how chain-of-thought (CoT) prompting improves an LLM's accuracy on multi-step reasoning problems. Understand the "think step by step" effect.

---

## Dataset
Use 20 math word problems from the **GSM8K** dataset:
```python
from datasets import load_dataset
ds = load_dataset("gsm8k", "main")
# Use ds["test"] — sample 20 problems
```

---

## Task

### Part 1 — Direct Prompting (Baseline)
For each of the 20 problems:
- Prompt: `"Answer this math problem: {question}\nAnswer:"`
- Record the answer and whether it is correct
- Calculate accuracy

### Part 2 — Chain-of-Thought Prompting
For each of the same 20 problems:
- Prompt: `"Solve this step by step: {question}\nLet's think through this:"`
- Record the full reasoning chain and final answer
- Calculate accuracy

### Part 3 — Few-Shot CoT
- Create 3 worked examples (problem → step-by-step solution → answer)
- Prepend these to each of the 20 problems
- Measure accuracy again

### Part 4 — Self-Consistency
- For 5 problems where CoT was wrong, run CoT 5 times (temperature=0.7)
- Take the majority vote answer
- Does majority voting fix any of the wrong answers?

### Part 5 — Analysis
- Build a results table:

| Problem | Direct | CoT | Few-shot CoT | Self-consistency |
|---------|--------|-----|--------------|-----------------|
| 1 | ✗ | ✓ | ✓ | ✓ |

- Write observations:
  - How much did CoT improve accuracy?
  - What types of problems benefit most from CoT?
  - When does CoT still fail?
