# P5.3 — Research Paper Assistant

**Phase:** 5 — Retrieval-Augmented Generation  
**Topic:** RAG, Multi-Document, Academic AI  
**Type:** Project

---

## Goal
Build a chat interface that lets you search and query across 50+ research papers in a domain. Ask cross-paper questions like "What are the different approaches to RAG evaluation?" and get synthesized answers with citations.

---

## Domain Options (pick one)
- RAG & Retrieval (papers on arXiv: cs.IR, cs.CL)
- Diffusion models
- LLM alignment & safety
- Multimodal models
- Any research topic you want to go deep on

---

## What to Build

### 1. Paper Collection
- Find 50+ papers on your topic (arXiv, Semantic Scholar, Google Scholar)
- Download PDFs automatically using arXiv API:
  ```python
  import arxiv
  search = arxiv.Search(query="retrieval augmented generation", max_results=60)
  for paper in search.results():
      paper.download_pdf(dirpath="./papers/")
  ```
- Save metadata: title, authors, abstract, year, arXiv ID for each paper

### 2. Structured Ingestion
Extract and preserve document structure:
- Paper title
- Abstract (always kept as one chunk — never split)
- Section headings
- Content under each section
- References

Store metadata per chunk: `paper_title`, `authors`, `year`, `section`, `arxiv_id`

### 3. Query Types to Support
Your system must handle these distinct query types:

**Type 1 — Factual**: "What is the RAGAS framework?"
→ Retrieve specific sections, cite one paper

**Type 2 — Comparative**: "How do REALM and RAG differ in their retrieval approach?"
→ Must retrieve from multiple papers, synthesize a comparison

**Type 3 — Landscape**: "What are the main approaches to chunking in RAG systems?"
→ Survey multiple papers, list approaches with citations

**Type 4 — Deep Dive**: "Summarize the key findings of the paper 'Dense Passage Retrieval for Open-Domain QA'"
→ Retrieve from a specific named paper

### 4. Citation System
Every answer must include:
- Inline citations: [Smith et al., 2023]
- Reference list at the bottom: paper title, authors, year, arXiv link
- "Jump to paper" button in the UI

### 5. Paper-Level Features
- `/papers` command: list all 50 indexed papers with title, year, abstract
- `/summary arxiv_id` command: generate a structured summary of one paper (background, method, results, limitations)
- `/compare paper1 paper2` command: generate a structured comparison

### 6. Search UI (Streamlit)
- Chat interface with citation support
- Sidebar with paper list (filterable by year, first author)
- Click any paper → opens its summary
- "Related papers" suggestions based on the current query topic
