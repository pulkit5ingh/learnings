# P3.1 — Named Entity Recognition (NER) API

**Phase:** 3 — Natural Language Processing  
**Topic:** NER, Hugging Face, FastAPI  
**Type:** Project

---

## Goal
Fine-tune a BERT-based NER model and deploy it as a production-style REST API. NER is used heavily in legal, finance, healthcare, and news domains.

---

## Dataset
CoNLL-2003 English NER dataset:
```python
from datasets import load_dataset
dataset = load_dataset("conll2003")
```
Entities: PER (person), ORG (organization), LOC (location), MISC (miscellaneous)

---

## What to Build

### 1. Model Fine-tuning
- Load `bert-base-cased` (casing matters for NER)
- Fine-tune with `TokenClassificationTrainer`
- Handle subword tokenization: align labels to first subword token, ignore rest
- Train for 5 epochs
- Evaluate with seqeval metrics: entity-level F1, precision, recall per entity type

### 2. Inference Pipeline
- Build a post-processing function that:
  - Converts subword predictions back to word-level
  - Groups consecutive tokens of the same entity type (B-PER, I-PER → one span)
  - Returns: `[{"text": "Elon Musk", "type": "PER", "start": 0, "end": 9}, ...]`

### 3. FastAPI Service
```
POST /ner
Body: { "text": "Elon Musk founded Tesla in California." }
Response: {
  "entities": [
    {"text": "Elon Musk", "type": "PER", "start": 0, "end": 9},
    {"text": "Tesla", "type": "ORG", "start": 18, "end": 23},
    {"text": "California", "type": "LOC", "start": 27, "end": 37}
  ],
  "processing_time_ms": 45
}

POST /ner/batch
Body: { "texts": ["...", "...", "..."] }
Response: list of entity results
```

### 4. Performance
- Batch inference: process up to 32 texts in one forward pass
- Benchmark: measure latency for 1, 8, 32 texts
- Add response caching with `cachetools` for repeated identical inputs

### 5. Test Suite
- Write `pytest` tests for:
  - Basic entity extraction (5 examples)
  - Edge cases: empty string, very long text (>512 tokens), numbers-only text
  - Batch endpoint
