# A7.1 — CLIP Image Search Engine

**Phase:** 7 — Multimodal AI  
**Topic:** Vision + Language, CLIP  
**Type:** Assignment

---

## Goal
Use OpenAI's CLIP model to build a text-to-image and image-to-image search engine. Understand how CLIP maps images and text into a shared embedding space.

---

## Task

### Part 1 — Understand CLIP
- Load `openai/clip-vit-base-patch32` from Hugging Face
- Embed 5 different images and their corresponding text descriptions
- Compute cosine similarity between each image embedding and all 5 text embeddings
- Visualize as a 5×5 heatmap: diagonal should be highest (correct image-text pairs)

### Part 2 — Build the Image Library
- Collect 1000 images (use CIFAR-10 test set or download from Unsplash API)
- Embed all images using CLIP
- Store embeddings in a numpy array + a JSON file mapping index → image path

### Part 3 — Text-to-Image Search
Implement `search_by_text(query: str, top_k=10) -> List[image_paths]`:
- Embed the query text with CLIP text encoder
- Compute cosine similarity vs all image embeddings
- Return top-k most similar images

Build a Streamlit app:
- Text input: "a red sports car at night"
- Shows top-10 matching images in a grid

### Part 4 — Image-to-Image Search
Implement `search_by_image(image_path: str, top_k=10) -> List[image_paths]`:
- Embed the query image
- Find the most similar images in the library
- Useful for "find more like this" features

### Part 5 — Cross-Modal Analysis
- Pick 10 diverse text queries
- For each, compare CLIP results vs keyword filename search
- Document 3 cases where CLIP understands semantics that filenames can't capture
- Also find 3 CLIP failures (wrong results)

---

## Deliverables
- [ ] CLIP similarity heatmap (5×5 image-text matrix)
- [ ] Streamlit text-to-image search app with 1000-image library
- [ ] Image-to-image search working
- [ ] 10-query comparison: CLIP vs filename search
- [ ] 3 CLIP successes and 3 CLIP failures documented
