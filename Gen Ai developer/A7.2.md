# A7.2 — Podcast Transcription & Intelligence Pipeline

**Phase:** 7 — Multimodal AI  
**Topic:** Audio, Whisper, LLMs  
**Type:** Assignment

---

## Goal
Build a complete audio intelligence pipeline: transcribe a podcast with Whisper, then use an LLM to extract structure, insights, and summaries.

---

## Task

### Part 1 — Transcription with Whisper
- Download a 10-15 minute podcast episode (any public podcast — use `yt-dlp` to download audio from YouTube)
- Transcribe with OpenAI Whisper (`openai-whisper` package, model size: `base` or `small`):
  ```python
  import whisper
  model = whisper.load_model("base")
  result = model.transcribe("podcast.mp3", verbose=True)
  ```
- Save the transcript with word-level timestamps: `[{"word": "Hello", "start": 0.0, "end": 0.3}, ...]`

### Part 2 — Speaker Diarization (Who Said What)
- Use `pyannote.audio` for speaker diarization:
  ```python
  from pyannote.audio import Pipeline
  pipeline = Pipeline.from_pretrained("pyannote/speaker-diarization-3.1")
  diarization = pipeline("podcast.mp3")
  ```
- Align Whisper timestamps with diarization to produce:
  ```
  [00:00:15] Speaker 1: Welcome to the podcast, today we're talking about...
  [00:00:42] Speaker 2: Thanks for having me, I'm excited to discuss...
  ```

### Part 3 — LLM Post-Processing
Using the timestamped, diarized transcript, extract:

1. **Chapter timestamps** — identify natural topic shifts, generate chapter markers:
   ```
   [00:00:00] Introduction and guest background
   [00:03:15] Main topic: State of AI in 2025
   [00:08:30] Discussion: Key challenges
   [00:12:00] Closing thoughts
   ```

2. **Key takeaways** — 5 bullet points summarizing the main insights

3. **Quotes** — top 3 most quotable sentences (good for social media)

4. **Action items** — any tasks or recommendations mentioned

5. **Entities** — people, companies, tools mentioned (use NER or LLM extraction)

### Part 4 — Output Formats
Generate 3 output files from one podcast:
- `transcript.md` — full diarized transcript with chapters
- `summary.md` — takeaways, quotes, entities
- `show_notes.html` — formatted for podcast website with chapter timestamps as links

### Part 5 — Quality Check
- Compare auto-generated chapters with your own manual segmentation of the same podcast
- Calculate: how many of the LLM-generated chapters match your manual ones (within 30 seconds)?

---

## Deliverables
- [ ] Diarized transcript with speaker labels and timestamps
- [ ] Chapter markers generated
- [ ] Summary with takeaways, quotes, entities
- [ ] All 3 output files generated
- [ ] Chapter quality comparison documented
