## Goal
Add a reranking step to your RAG pipeline and measure the improvement in retrieval precision. Reranking is used in production RAG at companies like Notion, Perplexity, and LinkedIn.

---

## Background
Two-stage retrieval:
1. **Stage 1 (Recall)**: Fast approximate nearest neighbor search → returns top-20 candidates
2. **Stage 2 (Precision)**: Slower but accurate cross-encoder reranker → reranks top-20 into final top-5

### Part 2 — Add Cross-Encoder Reranker
```python
from sentence_transformers import CrossEncoder
reranker = CrossEncoder("cross-encoder/ms-marco-MiniLM-L-6-v2")

def rerank(query, candidates, top_k=5):
    pairs = [(query, doc) for doc in candidates]
    scores = reranker.predict(pairs)
    ranked = sorted(zip(scores, candidates), reverse=True)
    return [doc for _, doc in ranked[:top_k]]
```
- Retrieve top-20 with bi-encoder, then rerank to top-5 with cross-encoder
- Calculate Precision@1, Precision@3, Precision@5 after reranking
