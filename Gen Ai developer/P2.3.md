# P2.3 — Time Series Forecasting with LSTM vs ARIMA

**Phase:** 2 — Machine Learning Core  
**Topic:** Deep Learning, RNNs, Time Series  
**Type:** Project

---

## Goal
Build an LSTM to forecast a real-world time series. Compare it against a classical statistical model (ARIMA). Learn to handle temporal data, evaluate forecasts, and visualize results.

---

## Dataset Options (pick one)
- **Stock prices** — Yahoo Finance via `yfinance` library (e.g., Apple, NVDA)
- **Weather data** — Open-Meteo API (temperature, precipitation)
- **Energy consumption** — UCI ML Repository, "Individual household electric power consumption"
- **Air quality** — Beijing PM2.5 dataset

---

## What to Build

### 1. Data Preparation
- Load at least 2 years of daily or hourly data
- Plot the raw time series
- Check for stationarity (ADF test)
- Create lag features for LSTM: use last 30 time steps to predict the next 1
- Split: train on 80%, test on last 20% (no shuffle — time order matters)
- Normalize with MinMaxScaler fit only on training data

### 2. ARIMA Baseline
- Use `pmdarima.auto_arima` to automatically find best (p, d, q) parameters
- Fit on training data, forecast on test period
- Calculate RMSE and MAE on test set

### 3. LSTM Model
```
Input: (batch, 30, 1) → LSTM(hidden=64, layers=2) → Linear(64, 1)
```
- Train for 50 epochs, batch size 32
- Use MSE loss, Adam optimizer
- Apply early stopping (patience=10)
- Log training loss per epoch

### 4. Evaluation & Comparison
- Denormalize predictions back to original scale
- Calculate RMSE, MAE, MAPE for both models on the test set
- Plot: actual values vs ARIMA predictions vs LSTM predictions on one chart

### 5. Multi-step Forecast
- Extend the LSTM to forecast the next 30 days (iterative prediction)
- Plot the 30-day forecast with confidence interval (use dropout at inference for uncertainty)
