# A8.3 — LangSmith Observability & Failure Analysis

**Phase:** 8 — Production & MLOps  
**Topic:** LLM Observability, LangSmith  
**Type:** Assignment

---

## Goal
Instrument an existing LLM app (RAG or agent) with LangSmith tracing, then use the trace data to identify performance bottlenecks and failure patterns.

---

## Setup
```bash
pip install langsmith langchain
export LANGCHAIN_API_KEY="ls__..."
export LANGCHAIN_TRACING_V2="true"
export LANGCHAIN_PROJECT="my-rag-eval"
```

---

## Task

### Part 1 — Instrument Your RAG App
Take your RAG app from Phase 5 and add LangSmith tracing:
```python
from langsmith import traceable

@traceable(name="RAG Pipeline")
def answer_question(question: str) -> str:
    chunks = retrieve(question)        # auto-traced if using LangChain
    answer = generate(question, chunks)
    return answer
```

For custom code (non-LangChain), use the decorator pattern to manually trace:
- `retrieve()` → trace as "Retrieval"
- `rerank()` → trace as "Reranking"
- `generate()` → trace as "Generation"

Tag each run with: `{"question_type": "factual", "doc_source": "product_manual"}`

### Part 2 — Generate 100 Runs
Run 100 diverse questions through your RAG pipeline:
- 25 factual questions (specific facts from documents)
- 25 comparative questions (compare two things)
- 25 summary questions (summarize a section)
- 25 out-of-scope questions (not in the documents)

Record: question, retrieved context, answer, latency for each.

### Part 3 — LangSmith Analysis
In the LangSmith dashboard:
- Find the **3 slowest runs** — which step is the bottleneck?
- Find the **3 runs with most tokens used** — are they the complex questions?
- Filter by tag `question_type=out-of-scope` — does the model correctly say "I don't know"?
- Export the traces as CSV

### Part 4 — Failure Identification
Using the trace data, identify:
- **Retrieval failures**: runs where the correct chunk was NOT retrieved (score based on your manual check of 20 samples)
- **Hallucination**: runs where the answer contains info not in the retrieved chunks (check 20 samples)
- **Latency outliers**: runs that took >3x the median time

For each category, find 3 specific examples and explain why they failed.

### Part 5 — Fix One Issue
Based on your analysis, implement one improvement:
- If retrieval is the problem: add reranking
- If hallucination is the problem: tighten the system prompt
- If latency is the problem: cache common queries

Re-run the same 100 questions and compare metrics before/after.
