# A6.2 — Stateful Agent with Long-Term Memory

**Phase:** 6 — AI Agents  
**Topic:** Agent Memory  
**Type:** Assignment

---

## Goal
Build an agent that remembers facts from past conversations and can reference them in future sessions. Implement both short-term (in-context) and long-term (vector store) memory.

---

## Task

### Part 1 — Short-Term Memory (In-Context)
Build a basic conversational agent that:
- Maintains a sliding window of the last N messages in the prompt
- When the window is full, summarizes the oldest messages and replaces them with the summary
- Implement `compress_history(messages, keep_last=10)`:
  - Summarize messages beyond the window with a single LLM call
  - Return: `[summary_message] + last_10_messages`

Test: have a 30-turn conversation about a complex topic. Verify the agent still references early facts after compression.

### Part 2 — Long-Term Memory with Vector Store
Design a memory system with two operations:

**Memory storage** (after each turn):
- Extract factual statements from the conversation: use LLM to extract `["User's name is Pulkit", "User works at a startup", ...]`
- Embed each fact and store in ChromaDB with timestamp

**Memory retrieval** (before each response):
- Embed the current user message
- Retrieve top-5 most relevant past facts
- Inject them into the system prompt: `"Relevant memories: [fact1, fact2, ...]"`

### Part 3 — Memory-Aware Agent
Combine both:
- Short-term: last 10 messages in context
- Long-term: retrieve relevant facts from all past sessions
- The agent can also explicitly save a memory: if user says "Remember that I prefer Python over JavaScript", immediately store this

### Part 4 — Multi-Session Test
Run 3 separate conversation sessions:
- **Session 1**: Tell the agent your name, job, 3 technical preferences, and 2 ongoing projects
- **Session 2**: Ask about your projects without reminding the agent who you are
- **Session 3**: Ask the agent to recommend a tool based on your preferences

Verify: does the agent correctly recall facts across all 3 sessions?

### Part 5 — Memory Decay & Relevance
- Add a timestamp to each memory
- Implement memory scoring: `score = relevance_score * recency_weight`
  - `recency_weight = exp(-days_old / 30)` (memories decay over 30 days)
- Test: create old vs recent memories on the same topic, see which wins
