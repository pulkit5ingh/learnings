# P7.2 — Visual RAG App (Text + Images)

**Phase:** 7 — Multimodal AI  
**Topic:** Multimodal RAG, Vision LLMs  
**Type:** Project

---

## Goal
Build a RAG system that understands both text and images in documents. When a user asks a question, the system retrieves both relevant text chunks and relevant images, then uses a vision LLM to answer with full context.

---

## Problem It Solves
Standard RAG ignores images in PDFs. A technical manual might have a diagram that shows exactly what the user is asking about — standard text RAG would miss it entirely. Visual RAG retrieves both.

---

## What to Build

### 1. Document Processing Pipeline
For each uploaded PDF:

**Text extraction**:
- Extract text per page using `pdfplumber`
- Apply semantic chunking

**Image extraction**:
- Extract all images from PDF using `PyMuPDF`:
  ```python
  import fitz  # PyMuPDF
  doc = fitz.open("manual.pdf")
  for page_num, page in enumerate(doc):
      for img_index, img in enumerate(page.get_images()):
          pix = fitz.Pixmap(doc, img[0])
          pix.save(f"page{page_num}_img{img_index}.png")
  ```
- Caption each extracted image using GPT-4V or Claude Vision: "Describe this image in 2 sentences for search purposes"
- Store captions as text embeddings alongside the image path

**Table extraction**:
- Detect and extract tables using `pdfplumber.extract_tables()`
- Convert tables to markdown text for indexing

### 2. Unified Index
ChromaDB with 3 collections:
- `text_chunks`: page text chunks + metadata (page, section)
- `image_captions`: AI-generated captions + image path
- `tables`: markdown tables + page number

### 3. Multimodal Retrieval
```python
def retrieve_multimodal(query: str, top_k=5):
    text_results = text_collection.query(query_embedding, n_results=top_k)
    image_results = image_collection.query(query_embedding, n_results=3)
    table_results = table_collection.query(query_embedding, n_results=2)
    return text_results, image_results, table_results
```

### 4. Vision LLM Answer Generation
Build the prompt for Claude Vision or GPT-4V:
- Include retrieved text chunks as text
- Include retrieved images as image attachments
- Include retrieved tables as markdown

```python
messages = [
    {"role": "user", "content": [
        {"type": "text", "text": f"Context text:\n{text_chunks}\n\nContext tables:\n{tables}\n\nQuestion: {question}"},
        {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{img1_b64}"}},
        {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{img2_b64}"}},
    ]}
]
```

### 5. Streamlit UI
- Upload PDF button → processing status bar
- Chat interface with rich display:
  - Answer text
  - "Sources used" expander showing: text snippets | thumbnail images | tables
  - "Which page?" → each source shows its page number
- Side panel: page-by-page thumbnail view of the PDF

### 6. Test Documents
Test with documents that have rich visuals:
- A technical product manual (machinery, electronics)
- A research paper with methodology diagrams
- An annual report with charts and tables

For each, ask 10 questions that require understanding both text and images.
