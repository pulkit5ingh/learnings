# P3.2 — Semantic Search Engine with Streamlit UI

**Phase:** 3 — Natural Language Processing  
**Topic:** Embeddings, Semantic Search  
**Type:** Project

---

## Goal
Build a semantic search engine over a large text corpus using Sentence Transformers. Unlike keyword search, semantic search understands meaning — "happy dog" finds "joyful puppy" results.

---

## Corpus Options (pick one)
- 10,000 Wikipedia article intros (use `wikipedia` Python library)
- 5,000 research paper abstracts from ArXiv
- All Stack Overflow Q&A posts for a specific tag (via Data Explorer)
- Any large collection of text you find interesting

---

## What to Build

### 1. Embedding Pipeline
- Load `sentence-transformers/all-MiniLM-L6-v2` (fast, good quality)
- Embed all documents (batch size 64)
- Store embeddings in a numpy array, save to disk with `np.save`
- Benchmark: how long does embedding 10,000 docs take?

### 2. Search Engine
Implement `search(query, top_k=10)`:
- Embed the query
- Compute cosine similarity against all stored embeddings
- Return top-k results with similarity scores
- Time the search: it should be under 50ms for 10,000 docs

### 3. Streamlit UI
Build a web app with:
- Search bar at the top
- Results below: title, snippet (first 200 chars), similarity score badge
- "Load more" button to see next 10 results
- Search history in the sidebar (last 10 queries)

### 4. Keyword Search Baseline
- Implement TF-IDF search using `sklearn.TfidfVectorizer`
- Run the same 20 queries on both semantic and keyword search
- Compare the top-3 results for each query

### 5. Hybrid Search
- Combine semantic score and TF-IDF score: `final = 0.7 * semantic + 0.3 * tfidf`
- Compare hybrid vs semantic-only on your 20 test queries
- Document 3 cases where hybrid beats pure semantic search
