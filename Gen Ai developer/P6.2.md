# P6.2 — AI Data Analyst Agent

**Phase:** 6 — AI Agents  
**Topic:** Agents, Tool Use, Data Analysis  
**Type:** Project

---

## Goal
Build an agent connected to a database that answers natural language questions by writing SQL, executing it, interpreting results, and generating charts. This is one of the most commercially valuable AI agent use cases.

---

## What to Build

### 1. Database Setup
Use a realistic dataset — pick one:
- **E-commerce**: orders, products, customers, reviews (generate with Faker)
- **HR Analytics**: employees, departments, salaries, performance (generate with Faker)
- **Financial**: transactions, accounts, merchants (use a public finance dataset)

Create a PostgreSQL database with at least 5 tables and 10,000+ rows.

### 2. Schema-Aware Tool Design
The agent gets these tools:
```python
tools = [
    get_schema() -> str,           # returns table names + column descriptions
    run_sql(query: str) -> DataFrame,  # executes SQL, returns result as pandas DF
    plot_chart(df, chart_type, x, y, title) -> image_path,  # matplotlib chart
    get_sample_rows(table: str, n=5) -> DataFrame,  # show sample data
    explain_query(sql: str) -> str,  # EXPLAIN the query plan
]
```

### 3. Agent Logic
```python
# Agent system prompt includes:
# - Database schema with column descriptions
# - SQL dialect (PostgreSQL)  
# - Rules: never use SELECT *, always add LIMIT, prefer CTEs for complex queries
# - Output format: answer + chart if visual, SQL shown for transparency
```

### 4. Query Types to Handle
Test with these question categories:

**Aggregation**: "What are the top 10 products by revenue this quarter?"
**Trend**: "Show me monthly sales growth over the last year"
**Comparison**: "Which sales rep closed the most deals vs their quota?"
**Anomaly**: "Which customers have unusually high return rates?"
**Segmentation**: "Break down customers by lifetime value tier"
**Prediction question** (require agent to say it can't do ML from SQL): "Will sales increase next month?"

### 5. Streamlit Chat Interface
- Chat input: type natural language question
- Response area shows:
  - The generated SQL (collapsible code block)
  - Result table (first 20 rows)
  - Chart (if relevant — bar, line, pie, scatter)
  - Plain English interpretation: "Your top 3 products generated 65% of total revenue..."
- Query history in sidebar
- "Export results as CSV" button

### 6. Error Handling & Retry
- If SQL has a syntax error → send error back to agent, it retries (max 3 times)
- If query returns 0 rows → agent suggests why and tries a broader query
- If query is too slow (>5 seconds) → agent adds LIMIT or simplifies

### 7. Safety
- Block dangerous queries: DELETE, DROP, UPDATE, INSERT
- Read-only database user for the agent connection
- Log all executed SQL to an audit table
