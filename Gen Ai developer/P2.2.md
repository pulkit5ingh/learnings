# P2.2 — Image Classifier API with Transfer Learning

**Phase:** 2 — Machine Learning Core  
**Topic:** Deep Learning, CNNs, FastAPI  
**Type:** Project

---

## Goal
Fine-tune a pretrained CNN on a custom image dataset and deploy it as a REST API. This is the standard workflow for vision AI at companies.

---

## Dataset
Collect or download a 5-class image dataset (at least 200 images per class). Ideas:
- 5 dog breeds
- 5 types of food
- 5 types of skin conditions (use HAM10000 dataset)
- 5 Indian landmarks

---

## What to Build

### 1. Data Pipeline
- Organize images into `train/`, `val/`, `test/` folders by class
- Use `torchvision.transforms` for augmentation:
  - Train: RandomResizedCrop, RandomHorizontalFlip, ColorJitter, Normalize
  - Val/Test: Resize(224), CenterCrop(224), Normalize
- Use `ImageFolder` dataset + `DataLoader`

### 2. Model — Transfer Learning
- Load pretrained ResNet-18 from `torchvision.models`
- Freeze all layers except the final fully connected layer
- Replace the final `fc` layer with `Linear(512, 5)` for your 5 classes
- Train only the new layer for 5 epochs (feature extraction)
- Then unfreeze all layers and fine-tune end-to-end for 10 more epochs with lr=1e-4

### 3. Evaluation
- Final test accuracy, per-class F1 score
- Confusion matrix
- Grad-CAM visualization: show what regions of the image the model focuses on for 5 example images

### 4. FastAPI Endpoint
```
POST /predict
Body: multipart/form-data with image file
Response: { "class": "labrador", "confidence": 0.94, "all_scores": {...} }
```
- Load the model once at startup
- Return top-1 prediction and all class probabilities
- Add basic error handling (wrong file type, empty file)

### 5. Test the API
- Use `curl` or a Python `requests` script to send 10 test images
- Report accuracy on these 10
