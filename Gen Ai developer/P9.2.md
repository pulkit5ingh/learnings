## What to Build (Core Components)

### 1. Codebase Indexer
Parse and embed your entire codebase:
```python
def index_codebase(repo_path: str):
    for file in find_code_files(repo_path):  # .py, .ts, .js, .go, etc.
        content = read_file(file)
        chunks = chunk_code(content, file)   # split by functions/classes
        embed_and_store(chunks, metadata={
            "file": file,
            "language": detect_language(file),
            "functions": extract_function_names(content),
            "imports": extract_imports(content),
        })
```

Code-aware chunking — split at function/class boundaries, not arbitrary character counts:
```python
import ast

def chunk_python(code: str, file: str) -> List[dict]:
    tree = ast.parse(code)
    chunks = []
    for node in ast.walk(tree):
        if isinstance(node, (ast.FunctionDef, ast.AsyncFunctionDef, ast.ClassDef)):
            chunk_text = ast.get_source_segment(code, node)
            chunks.append({"text": chunk_text, "name": node.name, "type": type(node).__name__})
    return chunks
```

### 2. Context-Aware RAG
When user asks a question:
1. Retrieve relevant code chunks from the index
2. Also retrieve: related files (imports), recent git changes
3. Build context: relevant code + project structure summary

### 3. Specialized Commands

**`codeai explain <file>`**
- Retrieves the file + related files it imports
- Sends to Claude with prompt: "Explain this code, its purpose, key design decisions, and how it fits into the larger codebase"

**`codeai test <file>`**
- Retrieves existing test files for examples of the testing style
- Retrieves the implementation file
- Generates comprehensive tests following your project's conventions

**`codeai ask <question>`**
- Retrieves top-5 most relevant code chunks
- Answers in the context of your actual code

**`codeai refactor <file> <instruction>`**
- Retrieves the file + its tests
- Applies the refactor
- Shows git diff, asks for approval before writing

**`codeai similar <description>`**
- Semantic search: finds code in your codebase that does something similar
- Returns: file path, function name, similarity score, snippet

### 4. Project Awareness
Load project context once at startup:
- `README.md` and `CLAUDE.md` / project docs
- `package.json` or `pyproject.toml` for dependencies
- Directory tree (summarized)
- Recent git log (last 20 commits)

### 5. Streaming Output
All responses stream to the terminal (or VS Code panel) in real-time.

### 6. Watch Mode
```bash
codeai watch  # watches for file changes
# When you save a file, automatically:
# - Update the index for that file
# - Check if any tests need updating
# - Alert if the change breaks any known patterns
```
