## Goal
Serve a local LLM at multiple quantization levels and benchmark throughput, latency, memory usage, and quality. This is a real trade-off evaluation every MLOps engineer needs to do.

## Setup
Use Ollama for easy model management:
```bash

ollama pull llama3.1:8b-q8_0     # 8-bit
ollama pull llama3.1:8b-fp16     # full precision (if GPU memory allows)
```
Or use `llama.cpp` directly for more control.

## Task

### Part 1 — Setup & Baseline
- Install Ollama and pull all 3 model variants
- Verify each is working: `ollama run llama3.1:8b "Hello, world!"`
- Check GPU memory usage with `nvidia-smi` for each variant

### Part 2 — Throughput Benchmark
For each quantization level, measure tokens per second:
```python
import time, ollama

def benchmark_throughput(model, prompt, n_runs=5):
    times = []
    token_counts = []
    for _ in range(n_runs):
        start = time.time()
        response = ollama.generate(model=model, prompt=prompt)
        elapsed = time.time() - start
        tokens = response["eval_count"]
        times.append(elapsed)
        token_counts.append(tokens)
    return {
        "avg_tokens_per_second": sum(token_counts) / sum(times),
        "avg_latency_seconds": sum(times) / n_runs
    }
```

Use 3 prompt types:
- Short (50 tokens output): "What is RAG?"
- Medium (200 tokens output): "Explain transformers in 5 paragraphs"
- Long (500 tokens output): "Write a detailed tutorial on building a REST API"

### Part 3 — Memory Usage
- Use `nvidia-smi` or `torch.cuda.memory_allocated()` to measure peak VRAM usage for each model
- Also measure RAM usage (for CPU offloading)

### Part 4 — Quality Evaluation
For each model variant, run 20 questions across categories:
- Factual recall (5 questions)
- Code generation (5 questions — generate simple Python functions)
- Reasoning (5 questions — simple math/logic)
- Instruction following (5 questions — structured output tasks)

Score each answer 1-3. Calculate average quality score per category.
