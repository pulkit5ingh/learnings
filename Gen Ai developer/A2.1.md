# A2.1 — Random Forest vs XGBoost with Hyperparameter Tuning

**Phase:** 2 — Machine Learning Core  
**Topic:** Classical ML  
**Type:** Assignment

---

## Goal
Train and compare two powerful ensemble models. Tune hyperparameters systematically using Optuna and understand feature importance.

---

## Dataset
Use the **California Housing** dataset (`sklearn.datasets.fetch_california_housing`) or any Kaggle tabular dataset you like.

---

## Task

### Part 1 — Baseline Models
- Train a `RandomForestRegressor` with default hyperparameters
- Train an `XGBRegressor` with default hyperparameters
- Evaluate both on a held-out test set using RMSE and R² score
- Report results in a comparison table

### Part 2 — Feature Importances
- Extract feature importances from both models
- Plot a horizontal bar chart for each model showing top 10 important features
- Do both models agree on the top features?

### Part 3 — Hyperparameter Tuning with Optuna
- Use Optuna to tune `XGBRegressor`:
  - `n_estimators`: 100–1000
  - `max_depth`: 3–10
  - `learning_rate`: 0.01–0.3
  - `subsample`: 0.5–1.0
- Run 50 trials, use 5-fold cross-validation as the objective
- Report the best params and improvement over baseline

### Part 4 — Learning Curves
- Plot learning curves (training score vs validation score) as training set size increases
- Diagnose: is the model underfitting or overfitting?

---

## Deliverables
- [ ] Comparison table: baseline RF vs baseline XGBoost vs tuned XGBoost
- [ ] Feature importance plots for both models
- [ ] Optuna study visualization (optimization history plot)
- [ ] Learning curve plot with diagnosis
- [ ] Written answer: which model won and why?
