# P5.1 — Personal Knowledge Base Chat App

**Phase:** 5 — Retrieval-Augmented Generation  
**Topic:** RAG, Full-Stack AI App  
**Type:** Project

---

## Goal
Build a production-ready chat app over your own notes, PDFs, or documents. Ask questions across all your knowledge and get cited answers. This is the most useful personal AI tool you can build.

---

## What to Build

### 1. Document Ingestion Pipeline
Support multiple file types:
- PDF (`pdfplumber`)
- Markdown / text files
- Word documents (`python-docx`)
- Web URLs (scrape with `BeautifulSoup`)

CLI ingestion command:
```bash
kb add ./my-notes/
kb add https://some-article-url.com
kb add ./research-paper.pdf
kb status  # show indexed documents and chunk counts
```

Pipeline: load → chunk (semantic chunking) → embed → store in ChromaDB.

### 2. Chat Interface (Streamlit)
- Chat UI with message history
- Sidebar: list of all ingested documents with document count and date added
- Each assistant message shows: answer + 3 collapsible source citations (document name, page/section, relevant snippet)
- "New Chat" button to clear history
- Export conversation as markdown

### 3. RAG Backend
- Multi-turn conversation: include last 3 exchanges in the prompt for context continuity
- Hybrid retrieval: BM25 (keyword) + ChromaDB (semantic), combine scores
- Reranking with cross-encoder before sending to LLM
- System prompt: "Answer using the provided sources. Always cite which document the information comes from. Say 'I don't have information about this' if the answer is not in your knowledge base."

### 4. Multi-Document Query
- User can filter by document: "Only search in research-paper.pdf"
- User can search across all documents (default)
- Show which documents were searched in the response metadata

### 5. Smart Features
- **Follow-up awareness**: detect pronouns referring to previous answers ("What else does it say?" → resolve "it" from context)
- **No-answer detection**: if retrieval scores are all low (<0.5), respond "I couldn't find relevant information" instead of hallucinating
- **Document summary**: `/summarize research-paper.pdf` command generates a 5-bullet summary of any ingested document

### 6. Deployment
- Package as a Docker container
- Deploy to Railway or Fly.io (free tier)
- Add basic password auth to protect your personal knowledge
