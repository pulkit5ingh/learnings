# P6.3 — Multi-Agent Content Creation Pipeline

**Phase:** 6 — AI Agents  
**Topic:** Multi-Agent Systems, CrewAI  
**Type:** Project

---

## Goal
Build a 4-agent pipeline that takes a topic and produces a publication-ready blog post. Each agent has a distinct persona, skills, and responsibilities. This mirrors how AI-assisted content workflows work at companies.

---

## The Four Agents

| Agent | Role | Responsibilities |
|-------|------|-----------------|
| Researcher | Senior Research Analyst | Find facts, statistics, expert opinions on the topic |
| Writer | Expert Blog Writer | Write engaging, structured content using the research |
| Editor | Senior Editor | Fix grammar, improve flow, ensure logical structure |
| SEO Specialist | SEO Expert | Optimize title, meta description, keywords, headings |

---

## What to Build

### 1. CrewAI Setup
```python
from crewai import Agent, Task, Crew, Process

researcher = Agent(
    role="Senior Research Analyst",
    goal="Find accurate, current, and compelling information about {topic}",
    backstory="You are an expert researcher with 10 years of experience finding reliable sources...",
    tools=[web_search_tool, wikipedia_tool],
    llm=claude_or_gpt4,
    verbose=True
)

# Define writer, editor, seo_specialist similarly
```

### 2. Task Chain
```python
research_task = Task(
    description="Research {topic}. Find: key statistics, expert quotes, recent developments, common misconceptions. Output structured markdown with sources.",
    agent=researcher,
    expected_output="Structured research brief with 5+ findings, each with a source URL"
)

write_task = Task(
    description="Using the research brief, write a 1200-word blog post on {topic}. Include: catchy intro, 4 main sections with H2 headers, conclusion with CTA.",
    agent=writer,
    context=[research_task],  # gets researcher's output as input
    expected_output="A 1200-word blog post in markdown"
)

# edit_task and seo_task similarly
```

### 3. Quality Control Between Agents
- After each agent, score the output (1-10) with a quick LLM judge call
- If score < 6, retry with feedback (max 2 retries)
- Log all scores to track pipeline quality

### 4. CLI Interface
```bash
python pipeline.py --topic "The Future of RAG in Enterprise AI" --tone professional --length 1200 --output ./output/

# Output files:
```
