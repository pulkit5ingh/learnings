# P8.3 — Self-Hosted LLM Gateway with Semantic Cache

**Phase:** 8 — Production & MLOps  
**Topic:** LLM Infrastructure, Caching, Fallback  
**Type:** Project

---

## Goal
Deploy an open-source LLM on a cloud GPU and build a smart gateway in front of it. The gateway handles: semantic caching, routing, fallback to OpenAI, and cost tracking. This is a real infrastructure pattern at AI-first companies.

---

## Architecture
```
Client → Gateway API → [Cache Check]
                              ↓ (miss)
                       [Route Decision]
                         ↙         ↘
              Local LLM           OpenAI/Claude
              (Modal/RunPod)      (fallback)
                         ↘         ↙
                       [Response]
                       [Cache Store]
                       [Log to DB]
                              ↓
                          Client
```

## What to Build

### 1. Deploy Local LLM on Modal
```python
# modal_app.py
import modal

app = modal.App("llm-server")
image = modal.Image.debian_slim().pip_install("vllm")

@app.function(gpu="A10G", image=image)
@modal.web_endpoint(method="POST")
def generate(request: dict):
    from vllm import LLM, SamplingParams
    llm = LLM(model="mistralai/Mistral-7B-Instruct-v0.2")
    params = SamplingParams(temperature=request.get("temperature", 0.7))
    outputs = llm.generate([request["prompt"]], params)
    return {"text": outputs[0].outputs[0].text}
```

Deploy with: `modal deploy modal_app.py`

### 2. Gateway API
FastAPI gateway that all clients connect to:

```
POST /v1/chat/completions   — OpenAI-compatible endpoint
GET  /v1/models             — list available models
GET  /v1/stats              — cache stats, usage, cost
POST /admin/cache/clear     — clear the semantic cache
```

### 4. Routing Logic
```python
def route_request(request: ChatRequest) -> str:
    # Use local model if:
    # - question is simple (under 100 tokens)
    # - user selected "local" model explicitly
    # - within cost budget for local compute
    
    # Use OpenAI if:
    # - local model is down / slow (>10s timeout)
    # - request requires very long context (>8K tokens)
    # - user selected "gpt-4" explicitly
    
    return "local" or "openai"
```

### 5. Fallback & Retry
- If local LLM returns error or times out (10s): automatically retry with OpenAI
- Log which model was actually used for each request
- Alert if local model fails >5 times in 10 minutes

### 6. Cost Tracking
Track cost for every request:
- Local GPU: Modal charges per GPU-second (A10G = ~$0.10/hr) → calculate cost from latency
- OpenAI: calculate from token counts × pricing
- Report: "This month: $12.40 local + $3.20 OpenAI = $15.60 total"
