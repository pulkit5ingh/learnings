# P2.1 — Kaggle Tabular Competition

**Phase:** 2 — Machine Learning Core  
**Topic:** Classical ML, Feature Engineering, Ensembles  
**Type:** Project

---

## Goal
Compete on a real Kaggle tabular competition. Build a full end-to-end ML pipeline and hit the top 30% of the leaderboard. This mimics real industry data science work.

---

## Recommended Competitions (pick one)
- **House Prices: Advanced Regression Techniques** (beginner-friendly)
- **Titanic: Machine Learning from Disaster** (classic)
- **Playground Series Season 4** (current, rotating monthly problems)
- **AmEx Default Prediction** (finance domain)

---

## Pipeline to Build

### 1. EDA
- Analyze the training data thoroughly (see P1.2 for EDA structure)
- Identify the most important features visually

### 2. Feature Engineering
- Create at least 5 new features from existing ones (ratios, interactions, date parts, aggregations)
- Handle categorical features: target encoding, frequency encoding, or one-hot
- Handle missing values with domain-informed logic
- Log-transform skewed numerical features

### 3. Baseline Model
- Train an XGBoost or LightGBM baseline with default params
- Log the CV score and leaderboard score

### 4. Advanced Models
- Train at minimum: LightGBM, XGBoost, CatBoost
- Use 5-fold cross-validation for all models
- Use Optuna to tune the best-performing model

### 5. Stacking Ensemble
- Build a stacking ensemble: use out-of-fold predictions from the 3 models as features
- Train a simple linear model (Ridge) as the meta-learner
- Compare stacking CV score vs individual models

### 6. Submission
- Generate predictions on the test set
- Submit to Kaggle
- Document your public leaderboard score and percentile
