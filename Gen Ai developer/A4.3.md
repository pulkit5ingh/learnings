# A4.3 — Fine-tune Llama with QLoRA

**Phase:** 4 — Large Language Models  
**Topic:** Fine-tuning, LoRA, PEFT  
**Type:** Assignment

---

## Goal
Fine-tune an open-source LLM using QLoRA (4-bit quantization + LoRA adapters). Learn how to prepare instruction datasets, run PEFT training, and evaluate before/after.

---

## Model
`meta-llama/Llama-3.1-8B-Instruct` or `mistralai/Mistral-7B-Instruct-v0.2`  
(Use Google Colab with A100 GPU or Kaggle GPU — free tier works)

---

## Dataset
Pick a domain-specific instruction dataset (500-2000 examples):
- Medical Q&A: `medalpaca/medical_meadow_medqa`
- Code: `iamtarun/python_code_instructions_18k_alpaca` (sample 1000)
- Customer support: create your own from a company's FAQ
- Indian laws/Constitution: build manually or scrape

Format each example as:
```json
{
  "instruction": "What is the capital of France?",
  "input": "",
  "output": "The capital of France is Paris."
}
```

---

## Task

### Part 1 — Baseline Evaluation (Before Fine-tuning)
- Load the base model in 4-bit with bitsandbytes
- Run 20 held-out test prompts from your domain
- Score each output: 1 (wrong/irrelevant), 2 (partial), 3 (correct)
- Record average baseline score

### Part 2 — QLoRA Fine-tuning Setup
```python
# Key hyperparameters
lora_config = LoraConfig(
    r=16,           # rank
    lora_alpha=32,
    target_modules=["q_proj", "v_proj"],
    lora_dropout=0.05,
    task_type="CAUSAL_LM"
)
```
- Use `SFTTrainer` from TRL
- Training: 3 epochs, batch size 4, gradient accumulation 4 steps
- Learning rate: 2e-4 with cosine schedule
- Log training loss every 10 steps

### Part 3 — After Fine-tuning Evaluation
- Run the same 20 test prompts on the fine-tuned model
- Score each output with the same rubric
- Compare before/after average scores

### Part 4 — LoRA Rank Experiment
- Fine-tune 3 small versions: r=4, r=16, r=64 (use 200 samples to make it fast)
- Compare their test scores and training time
- Answer: does higher rank always mean better performance?

### Part 5 — Merge and Export
- Merge LoRA weights into the base model
- Save the merged model in GGUF format using llama.cpp
- Test it locally with Ollama
