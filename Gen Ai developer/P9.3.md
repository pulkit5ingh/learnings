## Goal
Build a complex Q&A system that combines retrieval-augmented generation with a reasoning model (o1-mini or DeepSeek-R1) for multi-hop questions — questions that require connecting information across multiple documents.

---

## What Multi-Hop Means
**Simple (1-hop)**: "Who is the CEO of OpenAI?" → one document has the answer  
**Multi-hop (2+ hop)**: "What company does the author of the GPT-3 paper work at now?" 
→ need to: (1) find who authored GPT-3, (2) find where they work now

Standard RAG retrieves once. Multi-hop RAG must iterate.

### 2. Multi-Hop Retrieval — Iterative Approach
```python
def multi_hop_retrieve(question: str, max_hops: int = 3) -> List[str]:
    retrieved = []
    current_question = question
    
    for hop in range(max_hops):
        # Retrieve for the current sub-question
        new_chunks = retrieve(current_question, top_k=3)
        retrieved.extend(new_chunks)
        
        # Ask the reasoning model: do we have enough? What's missing?
        assessment = reasoning_model(f"""
        Question: {question}
        Retrieved so far: {retrieved}
        
        Can we answer the question from this information?
        If yes, say DONE.
        If no, what specific information do we still need? 
        Write a focused sub-query to find that information.
        """)
        
        if "DONE" in assessment:

    return retrieved
```

```python
def reason_and_answer(question: str, contexts: List[str]) -> str:
```
