# P3.3 — Text Summarizer Web App

**Phase:** 3 — Natural Language Processing  
**Topic:** Seq2Seq, Abstractive Summarization  
**Type:** Project

---

## Goal
Deploy a text summarization model as a polished web app. Users paste any article and get a concise summary. This is a production-grade NLP app.

---

## What to Build

### 1. Backend — FastAPI
- Load `facebook/bart-large-cnn` (best out-of-the-box summarizer from Hugging Face)
- Endpoint:
```
POST /summarize
Body: { "text": "...", "max_length": 150, "min_length": 40 }
Response: { "summary": "...", "original_word_count": 800, "summary_word_count": 120, "compression_ratio": 0.15 }
```
- Handle long texts: chunk texts >1024 tokens into segments, summarize each, then summarize the summaries (hierarchical summarization)
- Async endpoint for long articles (return a job ID, poll for result)

### 2. Frontend — Streamlit or React
Build a clean UI with:
- Large text area for pasting the article
- Slider: summary length (short/medium/long)
- "Summarize" button with spinner while loading
- Result displayed with: summary text, compression stats, time taken
- "Copy to clipboard" button

### 3. Multi-Model Support
Support switching between models:
- `facebook/bart-large-cnn` (best quality, slower)
- `sshleifer/distilbart-cnn-12-6` (faster, slightly worse)
- `t5-small` (your fine-tuned model from A3.4)
- Show a side-by-side comparison mode

### 4. URL Summarizer
- Accept a URL instead of pasted text
- Use `newspaper3k` library to scrape and extract article text
- Summarize the scraped content

### 5. Evaluation Mode
- Add a mode where users can paste a reference summary
- Show ROUGE-1, ROUGE-2, ROUGE-L scores comparing generated vs reference
