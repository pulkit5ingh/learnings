# P1.1 — NumPy Neural Network on MNIST

**Phase:** 1 — Foundations  
**Topic:** Python for AI, Mathematics  
**Type:** Project

---

## Goal
Build a fully connected 2-layer neural network using only NumPy — no PyTorch, no TensorFlow. Train it on MNIST handwritten digits. This forces you to deeply understand forward pass, backpropagation, and weight updates.

---

## Architecture
```
Input (784) → Hidden Layer (128, ReLU) → Output Layer (10, Softmax)
```

---

## What to Build

### Core Components (all in NumPy)
- `relu(x)` and `relu_derivative(x)`
- `softmax(x)`
- `cross_entropy_loss(predictions, labels)`
- `forward_pass(X, W1, b1, W2, b2)`
- `backward_pass(...)` — compute gradients for W1, b1, W2, b2
- `update_weights(...)` — gradient descent step

### Training Loop
- Mini-batch gradient descent (batch size = 32)
- Train for 20 epochs
- Track training loss and validation accuracy per epoch
- Learning rate: start with 0.01

### Data
- Load MNIST using `keras.datasets.mnist` or `torchvision` (only for loading)
- Normalize pixel values to [0, 1]
- One-hot encode labels

### Evaluation
- Final test accuracy (target: >95%)
- Plot training loss curve
- Show 10 random test predictions with the actual image

---

## Stretch Goals

- Implement L2 regularization and observe its effect on overfitting
