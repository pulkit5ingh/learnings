# A9.4 — Reasoning Models: o1 vs Standard CoT

**Phase:** 9 — Advanced Topics  
**Topic:** Reasoning Models, Test-Time Compute  
**Type:** Assignment

---

## Goal
Empirically compare a reasoning model (o1-mini or DeepSeek-R1) against a standard model with chain-of-thought prompting. Understand when extended thinking provides genuine value and when it's overkill.

---

## Task

### Part 1 — Test Set Design
Create 30 problems across 4 categories (7-8 per category):

**Category 1 — Math / Logic**
- Multi-step word problems requiring 5+ steps
- Example: "A train leaves at 2 PM going 60 mph. Another train leaves 30 min later going 90 mph. When do they meet if the track is 200 miles?"

**Category 2 — Code Generation**
- Algorithmic challenges: dynamic programming, graph problems
- Example: "Write a function to find the minimum number of coins to make change"

**Category 3 — Complex Reasoning**
- Multi-constraint satisfaction problems
- Ambiguous questions requiring disambiguation
- Example: "Schedule 5 meetings for 4 people with 10 conflicting constraints"

**Category 4 — Creative + Open-Ended**
- Essay questions, strategy questions
- Example: "Design a system to rank news articles by credibility"

### Part 2 — Models to Compare
For each of the 30 problems, collect responses from:
1. **GPT-4o** with direct prompting (no CoT instructions)
2. **GPT-4o** with chain-of-thought: "Think step by step before answering"
3. **o1-mini** (or DeepSeek-R1 via API — free)

### Part 3 — Evaluation
For each problem, score all 3 responses on:
- **Correctness** (0/1 for math/code, 1-3 for open-ended)
- **Reasoning quality** (1-3): was the thinking process sound?
- **Conciseness** (1-3): did it avoid unnecessary verbosity?

Also measure:
- Tokens used (input + output)
- Latency (seconds to response)
- Estimated cost

### Part 5 — Deep Dive: 3 Interesting Cases
Find 3 problems where the results were surprising:
1. A problem where CoT matched reasoning model quality (reasoning model not needed)
2. A problem where reasoning model dramatically outperformed CoT
3. A problem where the reasoning model was wrong but CoT was right

For each: show all 3 responses and explain what happened.

### Part 6 — Decision Framework
Based on your experiments, write a decision guide:
- When to use a reasoning model (test-time compute justified)
- When CoT on a standard model is sufficient
- When neither is needed (direct prompt is fine)
- Cost-quality tradeoff recommendation per use case
