## Goal
Dockerize an LLM app and build a GitHub Actions CI pipeline that automatically runs quality evaluations on every PR. Bad quality = blocked merge. This is how real AI teams prevent regressions.

### Part 3 — GitHub Actions Workflow
Write `.github/workflows/eval.yml`:
```yaml
name: LLM Quality Gate

on:
  pull_request:
    branches: [main]

jobs:
  eval:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Build and start services
        run: docker-compose up -d --build
        
      - name: Wait for services to be ready
        run: |
          for i in {1..30}; do
            curl -sf http://localhost:8000/health && break
            sleep 2
          done
      
      - name: Run evaluations
        run: python eval/run_evals.py
        env:
          OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
      
      - name: Publish eval results
        if: always()
        uses: actions/upload-artifact@v4
        with:
          name: eval-results
          path: eval/results.json
```

### Part 4 — Eval Thresholds
Implement threshold checks for 3 metrics:
- `pass_rate >= 0.80` — 80% of questions answered correctly
- `avg_latency_ms <= 3000` — median response under 3 seconds  
- `hallucination_rate <= 0.10` — less than 10% hallucinations

If any threshold fails: the workflow fails and the PR cannot be merged.

### Part 5 — Regression Test
- Merge a change that you know will break the RAG (e.g., corrupt the prompt, change retrieval to top-1 instead of top-5)
- Verify CI catches it and blocks the merge
- Fix the issue, verify CI passes again
