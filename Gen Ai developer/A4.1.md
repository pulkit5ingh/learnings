# A4.1 — Prompt Engineering Experiments

**Phase:** 4 — Large Language Models  
**Topic:** Prompt Engineering  
**Type:** Assignment

---

## Goal
Systematically experiment with different prompting strategies on the same task. Build intuition for what makes a prompt effective.

---

## Task: Code Review Assistant

You will prompt an LLM (Claude or GPT-4) to review a buggy Python function. Use the same function for all experiments.

```python
# Test function — intentionally has bugs and issues
def calculate_discount(price, discount, is_member):
    if is_member = True:
        discount = discount + 0.1
    final = price - price * discount
    if final < 0:
        final = 0
    return final
```

---

### Part 1 — Zero-Shot Variations (write 5 different prompts)
1. **Vague prompt**: "Review this code."
2. **Role prompt**: "You are a senior Python engineer. Review this code."
3. **Structured prompt**: "Review this code for: (1) bugs, (2) style issues, (3) edge cases. Use bullet points."
4. **Output format prompt**: Same as above but specify: "Return a JSON with keys: bugs, style_issues, edge_cases, severity (1-5)."
5. **Chain-of-Thought prompt**: "Think step by step. First identify what the function is supposed to do, then find any issues."

### Part 2 — Few-Shot Prompting
- Provide 2 examples of (buggy code → good review) before asking it to review the test function
- Measure: does few-shot improve consistency of the output format?

### Part 3 — System Prompt Design
- Design a system prompt that makes the LLM always respond as a strict code reviewer
- Test it on 5 different code snippets (write your own)
- Evaluate: does the persona hold across all 5?

### Part 4 — Temperature Experiments
- Run the same structured prompt with temperatures: 0.0, 0.3, 0.7, 1.0
- Compare the 4 outputs: do higher temperatures produce more creative or less consistent reviews?

### Part 5 — Analysis
- Build a scoring rubric (1-5) for each output: accuracy, completeness, format adherence
- Score all outputs from Part 1
- Write a 1-page guide: "What I learned about effective prompting"
