## Goal
Train the same model with different precision settings and memory optimizations. Measure GPU memory reduction and training speed. This knowledge is required for training large models on limited hardware.

## Task

### Part 1 — Baseline: Full Precision (FP32)
Train a medium-sized model in full precision:
- Model: ResNet-50 or a small BERT (for classification)
- Dataset: CIFAR-100 or IMDb sentiment (whichever your GPU handles)
- Train for 5 epochs
- Measure:
  - Peak GPU memory (VRAM)
  - Time per epoch
  - Final validation accuracy

```python
import torch
model = YourModel().cuda()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-4)
# Normal training loop — FP32 by default
```

### Part 2 — AMP (Automatic Mixed Precision — FP16)
```python
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for batch in dataloader:
    optimizer.zero_grad()
    with autocast():  # forward pass in FP16
        output = model(batch)
        loss = criterion(output, labels)
    scaler.scale(loss).backward()  # backward in FP32
    scaler.step(optimizer)
    scaler.update()
```
- Same model, same data, same epochs
- Measure the same 3 metrics

### Part 3 — BF16
Replace `autocast()` with `autocast(dtype=torch.bfloat16)`:
- BF16 has the same range as FP32 but lower precision than FP16
- Works better on newer GPUs (A100, RTX 4090)
- Measure the same 3 metrics

### Part 4 — Gradient Checkpointing
Enable gradient checkpointing (trades compute for memory):

- Combine with AMP

### Part 6 — Gradient Accumulation
Simulate a larger batch size with limited memory:
```python
ACCUMULATION_STEPS = 4  # effective batch = 4x actual batch

for i, batch in enumerate(dataloader):
    loss = model(batch) / ACCUMULATION_STEPS
    loss.backward()
    if (i + 1) % ACCUMULATION_STEPS == 0:
        optimizer.step()
        optimizer.zero_grad()
```
