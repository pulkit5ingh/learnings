# A3.2 — Docker Compose: Multi-Service Application

**Phase:** 3 — Containers (Docker)  
**Topic:** Docker Compose, multi-service orchestration  
**Type:** Assignment

---

## Goal
Use Docker Compose to run a multi-service application locally — the standard tool for local development environments.

---

## Task

### Part 1 — Basic Compose File
```yaml
# docker-compose.yml
version: '3.9'

services:
  api:
    build:
      context: .
      dockerfile: Dockerfile
      target: runtime
    ports:
      - "8080:8080"
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/appdb
      - REDIS_URL=redis://cache:6379/0
    depends_on:
      db:
        condition: service_healthy
      cache:
        condition: service_healthy
    restart: unless-stopped
    volumes:
      - ./src:/app/src:ro  # mount source for development

  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: appdb
    volumes:
      - postgres_data:/var/lib/postgresql/data
      - ./sql/init.sql:/docker-entrypoint-initdb.d/init.sql
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U user -d appdb"]
      interval: 10s
      timeout: 5s
      retries: 5

  cache:
    image: redis:7-alpine
    command: redis-server --maxmemory 256mb --maxmemory-policy allkeys-lru
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 3s
      retries: 5

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/certs:/etc/nginx/certs:ro
    depends_on:
      - api

volumes:
  postgres_data:
```

### Part 2 — Environment-Specific Compose Files
Use multiple compose files for different environments:
```bash
# Base: docker-compose.yml (shared config)
# Development: docker-compose.dev.yml (hot reload, debug)
# Test: docker-compose.test.yml (in-memory db, coverage)
# Production: docker-compose.prod.yml (no mounts, resource limits)
```

```yaml
# docker-compose.dev.yml (overrides)
services:
  api:
    environment:
      - DEBUG=true
      - LOG_LEVEL=debug
    volumes:
      - ./src:/app/src  # writable for hot reload
    command: python -m watchdog  # hot reload
  
  mailhog:
    image: mailhog/mailhog  # fake SMTP for dev
    ports:
      - "8025:8025"  # web UI
```

```bash
# Run dev environment
docker compose -f docker-compose.yml -f docker-compose.dev.yml up

# Run tests
docker compose -f docker-compose.yml -f docker-compose.test.yml up --exit-code-from test
```

### Part 3 — Networking
```yaml
services:
  frontend:
    networks:
      - public
  
  api:
    networks:
      - public
      - internal

  db:
    networks:
      - internal  # not accessible from frontend

networks:
  public:
    driver: bridge
  internal:
    driver: bridge
    internal: true  # no external access
```

Test network isolation:
```bash
# Verify frontend cannot reach db directly
docker compose exec frontend ping db  # should fail
docker compose exec api ping db       # should succeed
```

### Part 4 — Compose for Running Tests
```yaml
# docker-compose.test.yml
services:
  test:
    build:
      context: .
      target: builder  # includes test dependencies
    environment:
      - DATABASE_URL=postgresql://user:pass@test-db:5432/testdb
    command: pytest tests/ -v --tb=short
    depends_on:
      test-db:
        condition: service_healthy

  test-db:
    image: postgres:16-alpine
    tmpfs: /var/lib/postgresql/data  # in-memory for speed
    environment:
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
      POSTGRES_DB: testdb
    healthcheck:
      test: ["CMD-SHELL", "pg_isready"]
      interval: 5s
      retries: 5
```

```bash
docker compose -f docker-compose.test.yml up --exit-code-from test
# Exit code will be the test process exit code
```

### Part 5 — Resource Limits and Profiles
```yaml
services:
  api:
    deploy:
      resources:
        limits:
          cpus: '0.5'
          memory: 512M
        reservations:
          cpus: '0.1'
          memory: 128M
    profiles:
      - api

  worker:
    build: .
    command: celery -A app worker
    profiles:
      - worker
```

```bash
# Start only specific profiles
docker compose --profile api up
docker compose --profile api --profile worker up
```

### Part 6 — Useful Commands
```bash
# Start all services in background
docker compose up -d

# View logs (follow)
docker compose logs -f api

# Execute command in running container
docker compose exec api bash
docker compose exec db psql -U user appdb

# Scale a service
docker compose up -d --scale api=3

# See resource usage
docker compose stats

# Rebuild and restart one service
docker compose up -d --no-deps --build api

# Tear down including volumes
docker compose down -v
```
