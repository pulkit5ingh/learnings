# P3.2 — Private Container Registry with Automated CI/CD Push

**Phase:** 3 — Containers (Docker)  
**Topic:** Container registries, automated image building  
**Type:** Project

---

## Goal
Set up a private container registry and a fully automated pipeline that builds, scans, signs, and publishes images — the foundation of a container-based deployment workflow.

---

## What to Build
1. A private container registry (self-hosted or GitHub Container Registry)
2. Automated build pipeline that triggers on every commit
3. Vulnerability scanning before publishing
4. Image signing with Cosign
5. A simple web UI to browse images

## Option A: Self-Hosted Registry with Harbor
Harbor is a production-grade registry with scanning, signing, and RBAC.

```yaml

services:

      - /data/registry:/storage

    ports:
      - "80:8080"

  core:
    image: goharbor/harbor-core:v2.10.0
    # ... (Harbor has many services, use official installer)
```

Or use the official Harbor installer:
```bash
curl -L https://github.com/goharbor/harbor/releases/download/v2.10.0/harbor-online-installer-v2.10.0.tgz | tar xz
cd harbor
cp harbor.yml.tmpl harbor.yml
# Edit harbor.yml: set hostname, admin password
./install.sh --with-trivy
```

## Option B: GitHub Container Registry (Simpler)
Use GHCR for free — skip running a self-hosted registry and focus on the pipeline:
```
ghcr.io/your-username/your-image:tag
```

## The Automated Pipeline

### Build Triggers
```yaml
# .github/workflows/build-image.yml
on:
  push:
    branches: [main]
    paths:
      - 'src/**'
      - 'Dockerfile'
      - 'requirements.txt'
  schedule:
    - cron: '0 2 * * 1'  # weekly rebuild for base image updates
```

### Multi-Arch Build (x86_64 + ARM64)
```yaml
- name: Set up QEMU (for ARM emulation)
  uses: docker/setup-qemu-action@v3

- name: Set up Docker Buildx
  uses: docker/setup-buildx-action@v3

- name: Build and push multi-arch
  uses: docker/build-push-action@v5
  with:
    platforms: linux/amd64,linux/arm64
    push: true
    tags: |
      ghcr.io/${{ github.repository }}:latest
      ghcr.io/${{ github.repository }}:${{ github.sha }}
    cache-from: type=gha
    cache-to: type=gha,mode=max
```

### Security Pipeline
```yaml
jobs:
  build:
    outputs:
      digest: ${{ steps.build.outputs.digest }}
    steps:
      - ... (build and push)

  scan:
    needs: build
    steps:
      - name: Scan with Trivy
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: ghcr.io/${{ github.repository }}@${{ needs.build.outputs.digest }}
          severity: CRITICAL
          exit-code: 1  # fail on critical vulns

  sign:
    needs: scan  # only sign if scan passes
    permissions:
      id-token: write
      packages: write
    steps:
      - name: Sign with Cosign (keyless)
        uses: sigstore/cosign-installer@v3
      - run: |
          cosign sign --yes \
            ghcr.io/${{ github.repository }}@${{ needs.build.outputs.digest }}

  notify:
    needs: [sign]
    if: always()
    steps:
      - name: Slack notification
        run: |
          STATUS="${{ needs.sign.result }}"
          if [ "$STATUS" = "success" ]; then
            MSG="✅ New image published: ghcr.io/${{ github.repository }}:${{ github.sha }}"
          else
            MSG="❌ Image build failed for ${{ github.sha }}"
          fi
          curl -X POST ${{ secrets.SLACK_WEBHOOK }} -d "{\"text\": \"$MSG\"}"
```

## Image Tagging Strategy
```bash
# Tag with:
# 1. Git SHA (immutable, unique per commit)
ghcr.io/myorg/myapp:sha-a1b2c3d

# 2. Branch name (mutable, for development)
ghcr.io/myorg/myapp:main
ghcr.io/myorg/myapp:feature-login

# 3. Semantic version (for releases)
ghcr.io/myorg/myapp:1.2.3
ghcr.io/myorg/myapp:1.2
ghcr.io/myorg/myapp:1

# 4. latest (mutable, latest stable)
ghcr.io/myorg/myapp:latest

# NEVER: use only :latest in production (can't roll back)
```

## Image Retention Policy
```bash
# Clean up old images — keep last 10 per tag pattern
# GitHub has built-in retention, or use:
docker image prune -f --filter "until=720h"  # delete images older than 30 days

# For GHCR, use the package versions API:
gh api /user/packages/container/myapp/versions | jq '.[] | select(.metadata.container.tags | length == 0)'
# Delete untagged images
```
