## Task

### Part 1 — Role Structure
Roles are reusable units of Ansible automation:
```bash
# Create a role structure
ansible-galaxy role init nginx
ansible-galaxy role init app-server
ansible-galaxy role init postgresql
```

Each role gets this structure:
```
nginx/
├── tasks/
│   └── main.yml       # tasks to run
├── handlers/
│   └── main.yml       # handlers (triggered by notify)
├── templates/
│   └── nginx.conf.j2
├── files/
│   └── index.html     # static files (not templated)
├── vars/
│   └── main.yml       # role-internal variables (high precedence)
├── defaults/
│   └── main.yml       # default variables (low precedence, override-able)
├── meta/
│   └── main.yml       # role dependencies
└── README.md
```

### Part 2 — nginx Role
```yaml
# nginx/defaults/main.yml
nginx_port: 80
nginx_worker_processes: auto
nginx_worker_connections: 1024
nginx_server_name: "{{ ansible_hostname }}"
nginx_root: /var/www/html
nginx_enable_ssl: false

# nginx/tasks/main.yml
---
- name: Install nginx
  package:
    name: nginx
    state: present

- name: Configure nginx
  template:
    src: nginx.conf.j2
    dest: /etc/nginx/sites-available/default
  notify: Reload nginx

- name: Enable nginx site
  file:
    src: /etc/nginx/sites-available/default
    dest: /etc/nginx/sites-enabled/default
    state: link

- name: Start and enable nginx
  service:
    name: nginx
    state: started
    enabled: true

# nginx/handlers/main.yml
---
- name: Reload nginx
  service:
    name: nginx
    state: reloaded

- name: Restart nginx
  service:
    name: nginx
    state: restarted
```

### Part 3 — app-server Role with Dependencies
```yaml
# app-server/meta/main.yml
dependencies:
  - role: nginx
    vars:
      nginx_port: 80
  - role: common   # a base role that sets up the server
```

```yaml
# app-server/tasks/main.yml
---
- name: Create app user
  user:
    name: "{{ app_user }}"
    system: true
    shell: /sbin/nologin

- name: Deploy application
  git:
    repo: "{{ app_repo }}"
    dest: "{{ app_dir }}"
    version: "{{ app_version }}"
  become_user: "{{ app_user }}"
  notify: Restart app service

- name: Install Python dependencies
  pip:
    requirements: "{{ app_dir }}/requirements.txt"
    virtualenv: "{{ app_dir }}/venv"
  become_user: "{{ app_user }}"

- name: Deploy systemd service
  template:
    src: app.service.j2
    dest: "/etc/systemd/system/{{ app_name }}.service"
  notify:
    - Reload systemd
    - Restart app service
```

### Part 4 — Playbook Using Roles
```yaml
# site.yml
---
- name: Configure all servers
  hosts: all
  become: true
  roles:
    - common

- name: Configure web servers
  hosts: webservers
  become: true
  roles:
    - nginx
    - role: app-server
      vars:
        app_user: webapp
        app_repo: https://github.com/myorg/myapp
        app_version: main
        app_dir: /opt/webapp

- name: Configure database servers
  hosts: databases
  become: true
  roles:
    - postgresql
```

### Part 5 — Dynamic Inventory from AWS
Instead of static inventory files, query AWS directly:

```bash
# Install the AWS dynamic inventory plugin
pip install boto3 botocore

# Create aws_ec2.yml inventory file
```

```yaml
# aws_ec2.yml
plugin: amazon.aws.aws_ec2
regions:
  - us-east-1
filters:
  instance-state-name: running
  tag:ManagedBy: ansible
keyed_groups:
  - key: tags.Role      # group by "Role" tag
    prefix: role
  - key: tags.Environment
    prefix: env
hostnames:
  - private-ip-address  # use private IP
compose:
  ansible_host: private_ip_address
```

```bash
# Test dynamic inventory
ansible-inventory -i aws_ec2.yml --list
ansible-inventory -i aws_ec2.yml --graph

# Run against hosts discovered from AWS
ansible-playbook site.yml -i aws_ec2.yml
```

### Part 6 — Ansible Vault for Secrets
```bash
# Encrypt a file
ansible-vault encrypt group_vars/all/secrets.yml

# Edit an encrypted file
ansible-vault edit group_vars/all/secrets.yml

# Encrypt a string value (embed in a playbook)
ansible-vault encrypt_string 'my_secret_password' --name 'db_password'

# Run a playbook with vault password
ansible-playbook site.yml --ask-vault-pass
# or use a password file (never commit this)
ansible-playbook site.yml --vault-password-file ~/.vault_pass
```

```yaml
# group_vars/all/secrets.yml (encrypted)
db_password: !vault |
  $ANSIBLE_VAULT;1.1;AES256
  66386439...
```

### Part 7 — Ansible Galaxy
```bash
# Install community roles from Galaxy
ansible-galaxy install geerlingguy.nginx
ansible-galaxy install geerlingguy.postgresql
ansible-galaxy install geerlingguy.docker

# Install from requirements file
cat requirements.yml
```

```yaml
# requirements.yml
roles:
  - name: geerlingguy.nginx
    version: 3.2.0
  - name: geerlingguy.postgresql
collections:
  - name: amazon.aws
    version: ">=6.0"
  - name: community.docker
```

```bash
ansible-galaxy install -r requirements.yml
```
