# P9.1 — GitOps Platform: Multi-Environment with ArgoCD App of Apps

**Phase:** 9 — Advanced: GitOps, SRE & Platform Engineering  
**Topic:** GitOps, ArgoCD, multi-cluster  
**Type:** Project

---

## Goal
Build a complete GitOps platform that manages multiple applications across multiple environments — all driven from a single Git repository, with no manual kubectl in production.

---

## What to Build
A GitOps platform managing:
- 4 applications (user-service, product-service, order-service, frontend)
- 3 environments (dev, staging, production)
- Progressive promotion (auto-promote dev→staging, manual approve staging→production)
- ArgoCD notifications to Slack
- Config drift detection and auto-remediation

## ArgoCD App of Apps Pattern
```yaml
# apps/root-dev.yaml — Root app manages all dev apps
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: root-dev
  namespace: argocd
  finalizers:
    - resources-finalizer.argocd.argoproj.io
spec:
  project: dev
  source:
    repoURL: https://github.com/myorg/gitops-platform
    targetRevision: HEAD
    path: environments/dev  # watches this folder
  destination:
    server: https://kubernetes.default.svc
    namespace: argocd
  syncPolicy:
    automated:
      prune: true
      selfHeal: true

---
# environments/dev/user-service.yaml — Individual app
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: user-service-dev
  namespace: argocd
spec:
  project: dev
  source:
    repoURL: https://github.com/myorg/gitops-platform
    targetRevision: HEAD
    path: manifests/user-service/overlays/dev
  destination:
    server: https://kubernetes.default.svc
    namespace: dev
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
      allowEmpty: false
    syncOptions:
      - CreateNamespace=true
      - PrunePropagationPolicy=foreground
      - ApplyOutOfSyncOnly=true
```

## ArgoCD Projects (RBAC)
```yaml
# argocd/projects/prod-project.yaml
apiVersion: argoproj.io/v1alpha1
kind: AppProject
metadata:
  name: production
  namespace: argocd
spec:
  description: Production environment
  
  sourceRepos:
    - 'https://github.com/myorg/gitops-platform'
  
  destinations:
    - namespace: production
      server: https://kubernetes.default.svc
  
  # Only allow images from our registry
  sourceNamespaces:
    - argocd
  
  # Deny dangerous operations
  clusterResourceWhitelist:
    - group: ''
      kind: Namespace
  
  namespaceResourceWhitelist:
    - group: 'apps'
      kind: Deployment
    - group: 'apps'
      kind: StatefulSet
    - group: ''
      kind: Service
    - group: 'networking.k8s.io'
      kind: Ingress

  roles:
    - name: production-deployer
      policies:
        - p, proj:production:production-deployer, applications, sync, production/*, allow
        - p, proj:production:production-deployer, applications, get, production/*, allow
      groups:
        - myorg:platform-team
```

## Automated Image Promotion Pipeline
```bash
# scripts/promote.sh — promote a service from dev to staging
#!/usr/bin/env bash
set -euo pipefail

SERVICE=$1
ENV_FROM=${2:-dev}
ENV_TO=${3:-staging}

# Get the current image tag from dev
CURRENT_TAG=$(kubectl get deployment "$SERVICE" -n "$ENV_FROM" \
    -o jsonpath='{.spec.template.spec.containers[0].image}' | cut -d: -f2)

echo "Promoting $SERVICE from $ENV_FROM to $ENV_TO with tag $CURRENT_TAG"

# Update the kustomization overlay
cd "manifests/$SERVICE/overlays/$ENV_TO"
kustomize edit set image "myorg/$SERVICE=ghcr.io/myorg/$SERVICE:$CURRENT_TAG"

# Commit and push
git add kustomization.yaml
git commit -m "promote($ENV_TO): $SERVICE to $CURRENT_TAG [skip ci]"
git push

echo "Promoted. ArgoCD will pick up the change within 3 minutes."
```

```yaml
# CI: auto-promote to staging after successful dev deployment
- name: Promote to staging
  if: github.ref == 'refs/heads/main' && github.event_name == 'push'
  run: ./scripts/promote.sh user-service dev staging
```

## ArgoCD Notifications to Slack
```yaml
# argocd/notifications.yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: argocd-notifications-cm
  namespace: argocd
data:
  service.slack: |
    token: $slack-token

  template.deployment-success: |
    slack:
      blocks: |
        [{
          "type": "section",
          "text": {
            "type": "mrkdwn",
            "text": "✅ *{{.app.metadata.name}}* deployed successfully in `{{.app.spec.destination.namespace}}`\nRevision: {{.app.status.sync.revision}}"
          }
        }]

  template.deployment-failed: |
    slack:
      blocks: |
        [{
          "type": "section",
          "text": {
            "type": "mrkdwn",
            "text": "❌ *{{.app.metadata.name}}* deployment FAILED in `{{.app.spec.destination.namespace}}`\nError: {{.app.status.conditions[0].message}}"
          }
        }]

  trigger.on-deployed: |
    - when: app.status.operationState.phase in ['Succeeded'] and app.status.health.status == 'Healthy'
      send: [deployment-success]
      oncePer: app.status.sync.revision

  trigger.on-health-degraded: |
    - when: app.status.health.status == 'Degraded'
      send: [deployment-failed]

  subscriptions: |
    - recipients: [slack:#deployments]
      triggers: [on-deployed, on-health-degraded]
```
