## Task

### Part 1 — Docker Network Types
```bash
# List existing networks
docker network ls

# Inspect the default bridge network
docker network inspect bridge
```

**Bridge Network (default)**
```bash
# Two containers on the same custom bridge network can communicate by name
docker network create mynet

docker run -d --name server --network mynet nginx
docker run -it --rm --network mynet busybox sh
# Inside busybox:
wget -qO- http://server  # works! DNS resolution by container name

# Default bridge network: containers can communicate by IP, NOT by name
docker run -d --name server2 nginx  # on default bridge
docker run -it --rm busybox sh
# wget -qO- http://server2  # FAILS — no DNS on default bridge
```

**Host Network**
```bash
# Container shares the host's network namespace
docker run -d --network host nginx
# nginx is now on port 80 of the HOST directly
# No port mapping needed — but no isolation either
```

**None Network**
```bash
# Completely isolated — no network access
docker run -it --rm --network none busybox sh
ping 8.8.8.8  # fails — no network
```

**Macvlan Network**
```bash
# Container gets its own MAC address and appears as a physical device
# Useful for: legacy apps that need to be directly on the LAN
docker network create -d macvlan \
    --subnet=192.168.1.0/24 \
    --gateway=192.168.1.1 \
    -o parent=eth0 mylan
```

### Part 2 — Container DNS and Service Discovery
```bash
# Create a network with multiple services
docker network create app-net

docker run -d --name redis --network app-net redis:alpine
docker run -d --name postgres --network app-net \
    -e POSTGRES_PASSWORD=secret postgres:alpine

# Any container on app-net can reach redis and postgres by name
docker run -it --rm --network app-net busybox sh
nslookup redis    # resolves to container IP
nslookup postgres
wget -qO- http://redis:6379  # (would fail on HTTP but shows DNS works)

# Docker's embedded DNS server is at 127.0.0.11
cat /etc/resolv.conf  # inside a container
```

### Part 3 — Volume Types
```bash
# 1. Named volumes (managed by Docker, persisted across containers)
docker volume create mydata
docker run -d --name app -v mydata:/app/data nginx
docker volume inspect mydata  # shows actual path on host

# 2. Bind mounts (mount a specific host directory)
docker run -d \
    -v $(pwd)/config:/etc/nginx/conf.d:ro \
    -v $(pwd)/html:/usr/share/nginx/html \
    nginx

# 3. tmpfs (in-memory, not persisted)
docker run -d --tmpfs /tmp:rw,noexec,nosuid,size=100m nginx

# 4. Volume in Docker Compose
volumes:
  postgres_data:
    driver: local
    driver_opts:
      type: none
      device: /data/postgres
      o: bind
```

### Part 4 — Volume Backup and Restore
```bash
# Backup a named volume
docker run --rm \
    -v mydata:/source:ro \
    -v $(pwd):/backup \
    alpine tar czf /backup/mydata-backup.tar.gz -C /source .

# Restore a volume
docker volume create mydata-restored
docker run --rm \
    -v mydata-restored:/target \
    -v $(pwd):/backup:ro \
    alpine sh -c "cd /target && tar xzf /backup/mydata-backup.tar.gz"

# Copy files between container and host
docker cp container_name:/app/logs/error.log ./error.log
docker cp ./config.json container_name:/app/config.json
```

### Part 5 — Practical Exercise: Multi-Tier Networking
Build this architecture:
```
Internet → nginx (public net) → api (public + private net) → postgres (private net only)
```

```bash
docker network create public-net
docker network create private-net --internal  # no internet access

# Database — private only
docker run -d --name postgres \
    --network private-net \
    -v pgdata:/var/lib/postgresql/data \
    -e POSTGRES_PASSWORD=secret \
    postgres:alpine

# API — on both networks
docker run -d --name api \
    --network public-net \
    myapp:latest

docker network connect private-net api

# Nginx — public only
docker run -d --name nginx \
    --network public-net \
    -p 80:80 \
    nginx:alpine
```

Verify:
```bash
# postgres is unreachable from nginx
docker exec nginx ping postgres  # should fail

# api can reach postgres
docker exec api ping postgres    # should succeed

# nginx can reach api
docker exec nginx wget -qO- http://api:8080/health
```
