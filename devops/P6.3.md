# P6.3 — Multi-Region Active-Passive Disaster Recovery

**Phase:** 6 — Cloud Platforms  
**Topic:** AWS multi-region, DR, failover  
**Type:** Project

---

## Goal
Implement a multi-region disaster recovery setup with automated failover — so that when a region fails, the application recovers automatically within minutes.

---

## DR Targets
- **RTO (Recovery Time Objective)**: < 15 minutes — time to restore service
- **RPO (Recovery Point Objective)**: < 5 minutes — maximum data loss acceptable

## Architecture

```
Normal Operation:
  Users → Route 53 → us-east-1 (Primary)
                        ├── ALB + ECS
                        ├── RDS Primary
                        └── S3 (with replication)
                               ↓ (replication)
                         us-west-2 (Standby)
                              ├── ECS (0 tasks, warm)
                              ├── RDS Read Replica
                              └── S3 (replicated)

During Failover:
  Users → Route 53 (health check fails us-east-1)
              → us-west-2 (Standby promoted to Primary)
                    ├── ECS (scale up tasks)
                    ├── RDS Read Replica (promoted to Primary)
                    └── S3 (already replicated, ready)
```

## Setup

### 1. RDS Cross-Region Read Replica
```bash
# Create read replica in us-west-2
aws rds create-db-instance-read-replica \
    --db-instance-identifier myapp-dr-replica \
    --source-db-instance-identifier arn:aws:rds:us-east-1:ACCOUNT:db:myapp-prod-db \
    --db-instance-class db.t3.medium \
    --region us-west-2 \
    --multi-az \
    --publicly-accessible false

# Enable automated backups on the read replica
aws rds modify-db-instance \
    --db-instance-identifier myapp-dr-replica \
    --backup-retention-period 1 \
    --region us-west-2
```

### 4. Automated Failover Lambda
```python
# lambda/failover/handler.py — triggered by CloudWatch alarm via SNS
import boto3

rds = boto3.client('rds', region_name='us-west-2')
ecs = boto3.client('ecs', region_name='us-west-2')
sns = boto3.client('sns')

def handler(event, context):
    print("Starting failover procedure...")
    
    # 1. Promote RDS read replica to standalone primary
    rds.promote_read_replica(
        DBInstanceIdentifier='myapp-dr-replica'
    )
    
    # Wait for promotion
    waiter = rds.get_waiter('db_instance_available')
    waiter.wait(DBInstanceIdentifier='myapp-dr-replica')
    
    # 2. Scale up ECS tasks in DR region
    ecs.update_service(
        cluster='myapp-dr-cluster',
        service='myapp-service',
        desiredCount=4
    )
    
    # 3. Send notification
    sns.publish(
        TopicArn='arn:aws:sns:us-east-1:ACCOUNT:operations',
        Subject='FAILOVER: Switched to DR region us-west-2',
        Message=f'Failover initiated. DR region us-west-2 is now primary.'
    )
    
    print("Failover complete. Check us-west-2.")
```

### 5. DR Runbook
```markdown
# DR Failover Runbook

## When to trigger
- AWS publishes us-east-1 outage
- Route 53 health check fails for > 10 minutes
- RDS Primary goes unreachable

## Automatic Failover
Route 53 automatically routes to us-west-2 when health check fails.

## Manual Steps if Auto-Failover Didn't Work
1. Verify primary is truly down (not a false alarm)
2. Run failover Lambda:
   `aws lambda invoke --function-name failover-trigger --region us-east-1 output.json`
3. Update application config to point to DR database
4. Verify DR is serving traffic: `curl https://myapp.com/health`
5. Communicate status to stakeholders

## Failback (after primary recovers)
1. Re-establish RDS replication from us-west-2 (new primary) to us-east-1
2. Let data sync completely (check replication lag)
3. Switch Route 53 back to primary
4. Scale down us-west-2 ECS tasks
```
