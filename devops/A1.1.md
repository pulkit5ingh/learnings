## Task

### Part 1 — File Permissions
Create a directory structure and set permissions:
```bash
mkdir -p /tmp/devops-lab/{public,private,shared}

# Set permissions:
# public  → readable by everyone, writable only by owner
# private → readable/writable only by owner, no access for others
# shared  → readable/writable by owner and group, no access for others

chmod 644 /tmp/devops-lab/public
chmod 600 /tmp/devops-lab/private
chmod 660 /tmp/devops-lab/shared

# Verify with:
ls -la /tmp/devops-lab/
```

Explain in a comment what each octal digit means.

### Part 2 — User and Group Management
```bash
# Create a group called "devteam"
sudo groupadd devteam

# Create two users: alice and bob
sudo useradd -m -s /bin/bash alice
sudo useradd -m -s /bin/bash bob

# Add both to devteam
sudo usermod -aG devteam alice
sudo usermod -aG devteam bob

# Verify
cat /etc/group | grep devteam
id alice
```

### Part 4 — SUID, SGID, and Sticky Bit
Research and explain each in comments:
```bash
# SUID (Set User ID) — example: /usr/bin/passwd
ls -la /usr/bin/passwd  # notice the 's' in permissions

# SGID on a directory — files inherit the group
mkdir /tmp/team-shared
sudo chown :devteam /tmp/team-shared
sudo chmod 2775 /tmp/team-shared

# Sticky bit — only owner can delete their own files
sudo chmod +t /tmp/team-shared

# Test: as alice, create a file in team-shared
# Test: as bob, try to delete alice's file (should fail)
```

### Part 5 — ACLs (Access Control Lists)
```bash
# Standard permissions are 3-entity (owner/group/other)
# ACLs allow per-user permissions

sudo apt install acl  # or yum install acl

# Give charlie read access to a specific file without changing group
setfacl -m u:charlie:r /tmp/devops-lab/private/report.txt
getfacl /tmp/devops-lab/private/report.txt
```
