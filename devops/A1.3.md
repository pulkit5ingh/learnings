### Part 1 — TCP/IP Fundamentals
Answer these from observation (run commands and explain the output):

```bash
# 1. What is your server's IP address?
ip addr show

cat /etc/resolv.conf
```

Document each command's output and explain what it shows.

### Part 2 — DNS Debugging
```bash
# Basic lookup
nslookup github.com
dig github.com

# Look at the full DNS chain
dig +trace github.com

# Reverse DNS lookup (IP → hostname)
dig -x 8.8.8.8

# Check specific record types
dig github.com MX   # mail records
dig github.com TXT  # text records (SPF, DKIM)
dig github.com AAAA # IPv6 records

# Test with a specific DNS server
dig @1.1.1.1 github.com
```

Write a script `dns_check.sh` that:
1. Takes a domain as argument
2. Reports: A record, MX record, TTL, authoritative nameserver
3. Compares resolution from 8.8.8.8 (Google) vs 1.1.1.1 (Cloudflare)

### Part 3 — TCP Connection Debugging
```bash
# List all open ports and listening services
ss -tlnp      # modern replacement for netstat
netstat -tlnp  # classic

# Check if a specific port is open on a remote host
nc -zv github.com 443   # netcat
telnet github.com 80    # old school

# Watch for connection states (TIME_WAIT, CLOSE_WAIT, etc.)
ss -s   # summary statistics

# Capture network traffic
sudo tcpdump -i eth0 -n port 80 -c 20
# Capture HTTP traffic and decode it
sudo tcpdump -i eth0 -A -n port 80
```

Explain what `TIME_WAIT` and `CLOSE_WAIT` TCP states mean and why they matter for high-traffic servers.

### Part 4 — HTTP Debugging
```bash
# Detailed HTTP request with headers
curl -v https://github.com

# Follow redirects and show each step
curl -L -v https://github.com 2>&1 | grep -E "^[><]"

# Time each phase of an HTTP request
curl -w "@curl_timing.txt" -o /dev/null -s https://github.com
# Create curl_timing.txt with these variables:
# time_namelookup, time_connect, time_appconnect, time_pretransfer, time_total

# Check TLS certificate
openssl s_client -connect github.com:443 </dev/null 2>/dev/null | openssl x509 -noout -dates -subject

# Send a POST request
curl -X POST https://httpbin.org/post \
  -H "Content-Type: application/json" \
  -d '{"key": "value"}'
```

### Part 5 — SSH Hardening
Configure SSH on a local VM or cloud instance:
```bash
# Edit /etc/ssh/sshd_config:
# PermitRootLogin no
# PasswordAuthentication no
# PubkeyAuthentication yes
# Port 2222  (non-standard port)
# AllowUsers alice bob
# MaxAuthTries 3
# ClientAliveInterval 300

# Generate and deploy key pair
ssh-keygen -t ed25519 -C "devops-lab"
ssh-copy-id -i ~/.ssh/id_ed25519.pub alice@server

# Test connection
ssh -p 2222 alice@server -v  # verbose to see handshake
```

Write a checklist of 10 SSH hardening steps with the reason for each.

### Part 6 — Firewall (iptables / ufw)
```bash
# Using ufw (simpler):
sudo ufw enable
sudo ufw allow 2222/tcp   # SSH on custom port
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw deny 23/tcp      # block telnet
sudo ufw status verbose

# Check with iptables directly:
sudo iptables -L -n -v
```
