## Task

### Part 1 — Inventory
```ini
# inventory.ini
[webservers]
web1 ansible_host=10.0.1.10 ansible_user=ubuntu
web2 ansible_host=10.0.1.11 ansible_user=ubuntu

[databases]
db1 ansible_host=10.0.1.20 ansible_user=ubuntu

[all:vars]
ansible_ssh_private_key_file=~/.ssh/id_rsa
ansible_python_interpreter=/usr/bin/python3
```

```yaml
# inventory.yml (YAML format — more powerful)
all:
  children:
    webservers:
      hosts:
        web1:
          ansible_host: 10.0.1.10
          nginx_port: 80
        web2:
          ansible_host: 10.0.1.11
          nginx_port: 80
      vars:
        nginx_worker_processes: auto
    databases:
      hosts:
        db1:
          ansible_host: 10.0.1.20
```

```bash
# Test connectivity
ansible all -m ping -i inventory.yml

# Run a command on all webservers
ansible webservers -m command -a "uptime" -i inventory.yml

# Gather facts about a host
ansible web1 -m setup -i inventory.yml
```

### Part 2 — First Playbook: Install nginx
```yaml
# install-nginx.yml
---
- name: Configure web servers
  hosts: webservers
  become: true  # sudo

  vars:
    nginx_version: "1.24"
    html_content: "<h1>Configured by Ansible</h1>"

  tasks:
    - name: Update apt cache
      apt:
        update_cache: true
        cache_valid_time: 3600

    - name: Install nginx
      apt:
        name: nginx
        state: present

    - name: Deploy nginx configuration
      template:
        src: templates/nginx.conf.j2
        dest: /etc/nginx/sites-available/default
        owner: root
        group: root
        mode: '0644'
      notify: Restart nginx

    - name: Deploy index.html
      copy:
        content: "{{ html_content }}"
        dest: /var/www/html/index.html
        owner: www-data
        group: www-data
        mode: '0644'

    - name: Ensure nginx is started and enabled
      service:
        name: nginx
        state: started
        enabled: true

  handlers:
    - name: Restart nginx
      service:
        name: nginx
        state: restarted
```

```bash
# Run the playbook
ansible-playbook install-nginx.yml -i inventory.yml

# Dry run (check mode)
ansible-playbook install-nginx.yml -i inventory.yml --check

# Run with verbose output
ansible-playbook install-nginx.yml -i inventory.yml -v
```

### Part 3 — Jinja2 Templates
```jinja2
{# templates/nginx.conf.j2 #}
server {
    listen {{ nginx_port | default(80) }};
    server_name {{ ansible_hostname }};
    root /var/www/html;

    access_log /var/log/nginx/{{ ansible_hostname }}_access.log;
    error_log  /var/log/nginx/{{ ansible_hostname }}_error.log;

    location / {
        index index.html;
        try_files $uri $uri/ =404;
    }

    {% if enable_ssl | default(false) %}
    listen 443 ssl;
    ssl_certificate     /etc/ssl/certs/server.crt;
    ssl_certificate_key /etc/ssl/private/server.key;
    {% endif %}
}
```

### Part 4 — Variables, Facts, and Conditionals
```yaml
tasks:
  # Ansible facts (collected automatically)
  - name: Print OS info
    debug:
      msg: "{{ ansible_distribution }} {{ ansible_distribution_version }}"

  # Register task output for later use
  - name: Check if nginx is installed
    command: which nginx
    register: nginx_check
    ignore_errors: true

  - name: Install nginx only if missing
    apt:
      name: nginx
    when: nginx_check.rc != 0

  # Loop over a list
  - name: Create required directories
    file:
      path: "{{ item }}"
      state: directory
      mode: '0755'
    loop:
      - /var/www/html
      - /var/log/myapp
      - /etc/myapp

  # Loop with dict
  - name: Set file permissions
    file:
      path: "{{ item.path }}"
      owner: "{{ item.owner }}"
      mode: "{{ item.mode }}"
    loop:
      - { path: /var/www/html, owner: www-data, mode: '0755' }
      - { path: /etc/nginx, owner: root, mode: '0700' }
```

### Part 5 — Deploy a Complete Application
Write a playbook that:
1. Installs nginx, python3, pip
2. Creates a dedicated app user
3. Deploys a Python WSGI app using a systemd service
4. Configures nginx as a reverse proxy
5. Opens the firewall port
6. Verifies the app is responding with a health check task
