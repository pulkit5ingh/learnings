# P6.2 — Serverless Event-Driven Backend

**Phase:** 6 — Cloud Platforms  
**Topic:** AWS Lambda, serverless architecture  
**Type:** Project

---

## Goal
Build a fully serverless image processing pipeline using Lambda, SQS, S3, and DynamoDB — scale from 0 to millions of requests with no servers to manage.

---

## The Application: Image Processing Service
Users upload images → images are processed (resize, watermark, convert) → processed images stored in S3 → metadata stored in DynamoDB → user notified via SNS

```
User → API Gateway → Upload Lambda
                          ↓
                   S3 (raw uploads)
                          ↓
              S3 Event → SQS Queue
                          ↓
              Process Lambda (triggered by SQS)
              ├── Resize to 3 sizes (thumb, medium, large)
              ├── Add watermark
              ├── Convert to WebP
              └── Store in S3 (processed/)
                          ↓
              DynamoDB (image metadata)
                          ↓
              SNS → Email notification
```

## Lambda Functions

### 1. Upload Lambda (API Gateway → S3)
```python
# lambda/upload/handler.py
import boto3, json, uuid, os

s3 = boto3.client('s3')
BUCKET = os.environ['UPLOAD_BUCKET']

def handler(event, context):
    # Generate a pre-signed URL for direct S3 upload
    file_id = str(uuid.uuid4())
    file_name = json.loads(event['body']).get('filename', 'image.jpg')
    
    presigned_url = s3.generate_presigned_post(
        Bucket=BUCKET,
        Key=f"raw/{file_id}/{file_name}",
        Fields={
            "Content-Type": "image/jpeg",
            "x-amz-meta-user-id": event['requestContext']['identity']['sourceIp']
        },
        Conditions=[
            {"Content-Type": "image/jpeg"},
            ["content-length-range", 1, 10_000_000]  # max 10MB
        ],
        ExpiresIn=300
    )
    
    return {
        'statusCode': 200,
        'body': json.dumps({
            'file_id': file_id,
            'upload_url': presigned_url['url'],
            'upload_fields': presigned_url['fields']
        })
    }
```

### 2. Process Lambda (SQS → Processed Images)
```python
# lambda/process/handler.py
import boto3, json, os
from PIL import Image
import io

s3 = boto3.client('s3')
dynamodb = boto3.resource('dynamodb')
sns = boto3.client('sns')

table = dynamodb.Table(os.environ['METADATA_TABLE'])
PROCESSED_BUCKET = os.environ['PROCESSED_BUCKET']
SNS_TOPIC = os.environ['NOTIFICATION_TOPIC']

SIZES = {
    'thumbnail': (150, 150),
    'medium': (800, 600),
    'large': (1920, 1080)
}

def handler(event, context):
    for record in event['Records']:
        s3_event = json.loads(record['body'])
        bucket = s3_event['Records'][0]['s3']['bucket']['name']
        key = s3_event['Records'][0]['s3']['object']['key']
        
        process_image(bucket, key)

def process_image(bucket, key):
    file_id = key.split('/')[1]
    
    # Download original
    response = s3.get_object(Bucket=bucket, Key=key)
    image_data = response['Body'].read()
    img = Image.open(io.BytesIO(image_data))
    
    processed_keys = {}
    
    for size_name, dimensions in SIZES.items():
        resized = img.copy()
        resized.thumbnail(dimensions, Image.LANCZOS)
        
        # Convert to WebP
        buffer = io.BytesIO()
        resized.save(buffer, format='WebP', quality=85, optimize=True)
        buffer.seek(0)
        
        output_key = f"processed/{file_id}/{size_name}.webp"
        s3.put_object(
            Bucket=PROCESSED_BUCKET,
            Key=output_key,
            Body=buffer,
            ContentType='image/webp',
            Metadata={'original-key': key, 'size': size_name}
        )
        processed_keys[size_name] = output_key
    
    # Store metadata
    table.put_item(Item={
        'file_id': file_id,
        'original_key': key,
        'processed_keys': processed_keys,
        'status': 'completed',
        'created_at': int(time.time())
    })
    
    # Notify
    sns.publish(
        TopicArn=SNS_TOPIC,
        Message=json.dumps({
            'file_id': file_id,
            'status': 'processed',
            'urls': {k: f"https://cdn.myapp.com/{v}" for k, v in processed_keys.items()}
        }),
        Subject='Image Processing Complete'
    )
```

## Serverless Framework / SAM Template
```yaml
# template.yaml (AWS SAM)
AWSTemplateFormatVersion: '2010-09-09'
Transform: AWS::Serverless-2016-10-31

Globals:
  Function:
    Runtime: python3.11
    Timeout: 30
    MemorySize: 256
    Environment:
      Variables:
        UPLOAD_BUCKET: !Ref UploadBucket
        PROCESSED_BUCKET: !Ref ProcessedBucket
        METADATA_TABLE: !Ref MetadataTable
        NOTIFICATION_TOPIC: !Ref NotificationTopic

Resources:
  UploadFunction:
    Type: AWS::Serverless::Function
    Properties:
      CodeUri: lambda/upload/
      Handler: handler.handler
      Events:
        Api:
          Type: Api
          Properties:
            Path: /upload
            Method: POST
      Policies:
        - S3WritePolicy:
            BucketName: !Ref UploadBucket

  ProcessFunction:
    Type: AWS::Serverless::Function
    Properties:
      CodeUri: lambda/process/
      Handler: handler.handler
      MemorySize: 1024  # more memory for image processing
      Timeout: 300
      Events:
        SQSEvent:
          Type: SQS
          Properties:
            Queue: !GetAtt ProcessingQueue.Arn
            BatchSize: 5
      Policies:
        - S3ReadPolicy:
            BucketName: !Ref UploadBucket
        - S3WritePolicy:
            BucketName: !Ref ProcessedBucket
        - DynamoDBWritePolicy:
            TableName: !Ref MetadataTable
        - SNSPublishMessagePolicy:
            TopicName: !GetAtt NotificationTopic.TopicName

  UploadBucket:
    Type: AWS::S3::Bucket
    Properties:
      NotificationConfiguration:
        QueueConfigurations:
          - Event: 's3:ObjectCreated:*'
            Filter:
              S3Key:
                Rules:
                  - Name: prefix
                    Value: raw/
            Queue: !GetAtt ProcessingQueue.Arn

  ProcessingQueue:
    Type: AWS::SQS::Queue
    Properties:
      VisibilityTimeout: 360
      RedrivePolicy:
        deadLetterTargetArn: !GetAtt DLQ.Arn
        maxReceiveCount: 3

  DLQ:
    Type: AWS::SQS::Queue

  MetadataTable:
    Type: AWS::DynamoDB::Table
    Properties:
      BillingMode: PAY_PER_REQUEST
      AttributeDefinitions:
        - AttributeName: file_id
          AttributeType: S
      KeySchema:
        - AttributeName: file_id
          KeyType: HASH
      TimeToLiveSpecification:
        AttributeName: expires_at
        Enabled: true
```
