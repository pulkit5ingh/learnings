# A6.4 — VPC Design: Subnets, NAT, ALB, Security Groups

**Phase:** 6 — Cloud Platforms  
**Topic:** AWS networking, load balancing  
**Type:** Assignment

---

## Goal
Design and implement a production-grade VPC with proper network segmentation, NAT, and an Application Load Balancer — the networking foundation for every AWS application.

---

## Task

### Part 1 — Production VPC Architecture
Design this architecture:
```
Internet
    │
    ├── Application Load Balancer (public subnets, AZ-a, AZ-b)
    │       │
    │       ├── EC2 web servers (private subnets, AZ-a, AZ-b)
    │               │
    │               ├── RDS PostgreSQL (database subnets, AZ-a, AZ-b)
    │               └── ElastiCache Redis (database subnets)
    │
    └── Bastion Host (public subnet, AZ-a) — for SSH access
```

IP allocation:
```
VPC: 10.0.0.0/16

Public subnets:
  10.0.1.0/24  (AZ-a) — ALB, NAT Gateway, Bastion
  10.0.2.0/24  (AZ-b) — ALB

Private subnets (app tier):
  10.0.11.0/24 (AZ-a) — Web servers
  10.0.12.0/24 (AZ-b) — Web servers

Database subnets:
  10.0.21.0/24 (AZ-a) — RDS, ElastiCache
  10.0.22.0/24 (AZ-b) — RDS, ElastiCache
```

### Part 2 — Security Groups (Stateful Firewall)
Security group rules:
```
alb-sg:
  Inbound:  0.0.0.0/0 → 80, 443
  Outbound: web-sg → 8080

bastion-sg:
  Inbound:  YOUR_IP/32 → 22
  Outbound: web-sg → 22

web-sg:
  Inbound:  alb-sg → 8080 (only from ALB, not internet)
            bastion-sg → 22
  Outbound: db-sg → 5432
            0.0.0.0/0 → 443 (outbound HTTPS for updates)

db-sg:
  Inbound:  web-sg → 5432 (only from web servers)
  Outbound: (none needed)
```

```bash
# Create security groups referencing each other
ALB_SG=$(aws ec2 create-security-group \
    --group-name alb-sg \
    --description "ALB security group" \
    --vpc-id $VPC_ID \
    --query 'GroupId' --output text)

WEB_SG=$(aws ec2 create-security-group \
    --group-name web-sg \
    --description "Web server security group" \
    --vpc-id $VPC_ID \
    --query 'GroupId' --output text)

# ALB → Web servers (reference by security group ID, not CIDR)
aws ec2 authorize-security-group-ingress \
    --group-id $WEB_SG \
    --protocol tcp --port 8080 \
    --source-group $ALB_SG

# ALB can only get outbound to web servers
aws ec2 authorize-security-group-egress \
    --group-id $ALB_SG \
    --protocol tcp --port 8080 \
    --source-group $WEB_SG
```

### Part 3 — Application Load Balancer
```bash
# Create ALB
ALB_ARN=$(aws elbv2 create-load-balancer \
    --name my-app-alb \
    --subnets $PUBLIC_SUBNET_A $PUBLIC_SUBNET_B \
    --security-groups $ALB_SG \
    --scheme internet-facing \
    --type application \
    --query 'LoadBalancers[0].LoadBalancerArn' --output text)

# Create target group (where ALB sends traffic)
TG_ARN=$(aws elbv2 create-target-group \
    --name web-servers-tg \
    --protocol HTTP \
    --port 8080 \
    --vpc-id $VPC_ID \
    --target-type instance \
    --health-check-path /health \
    --health-check-interval-seconds 30 \
    --healthy-threshold-count 2 \
    --unhealthy-threshold-count 3 \
    --query 'TargetGroups[0].TargetGroupArn' --output text)

# Register instances in target group
aws elbv2 register-targets \
    --target-group-arn $TG_ARN \
    --targets Id=$INSTANCE_1 Id=$INSTANCE_2

# Create listener
aws elbv2 create-listener \
    --load-balancer-arn $ALB_ARN \
    --protocol HTTP \
    --port 80 \
    --default-actions Type=forward,TargetGroupArn=$TG_ARN

# Check ALB DNS name
aws elbv2 describe-load-balancers \
    --load-balancer-arns $ALB_ARN \
    --query 'LoadBalancers[0].DNSName' --output text
```

### Part 4 — ALB Path-Based and Host-Based Routing
```bash
# Path-based routing
# /api/* → api-service-tg
# /static/* → static-tg
# /* → frontend-tg

aws elbv2 create-rule \
    --listener-arn $LISTENER_ARN \
    --priority 10 \
    --conditions '[{"Field":"path-pattern","Values":["/api/*"]}]' \
    --actions "Type=forward,TargetGroupArn=$API_TG_ARN"

# Host-based routing
# api.myapp.com → api-tg
# myapp.com → frontend-tg

aws elbv2 create-rule \
    --listener-arn $LISTENER_ARN \
    --priority 5 \
    --conditions '[{"Field":"host-header","Values":["api.myapp.com"]}]' \
    --actions "Type=forward,TargetGroupArn=$API_TG_ARN"
```

### Part 5 — HTTPS with ACM
```bash
# Request a certificate from ACM (free)
CERT_ARN=$(aws acm request-certificate \
    --domain-name myapp.com \
    --subject-alternative-names "*.myapp.com" \
    --validation-method DNS \
    --query 'CertificateArn' --output text)

# Get DNS validation records
aws acm describe-certificate \
    --certificate-arn $CERT_ARN \
    --query 'Certificate.DomainValidationOptions[*].ResourceRecord'

# Add CNAME records to Route53, wait for validation
aws acm wait certificate-validated --certificate-arn $CERT_ARN

# Create HTTPS listener
aws elbv2 create-listener \
    --load-balancer-arn $ALB_ARN \
    --protocol HTTPS \
    --port 443 \
    --certificates CertificateArn=$CERT_ARN \
    --ssl-policy ELBSecurityPolicy-TLS13-1-2-2021-06 \
    --default-actions Type=forward,TargetGroupArn=$TG_ARN

# Redirect HTTP to HTTPS
aws elbv2 modify-listener \
    --listener-arn $HTTP_LISTENER_ARN \
    --default-actions 'Type=redirect,RedirectConfig={Protocol=HTTPS,Port=443,StatusCode=HTTP_301}'
```

### Part 6 — Network ACLs (Stateless Firewall)
```bash
# Network ACLs are stateless — need explicit inbound AND outbound rules
# They operate at the subnet level (additional layer after security groups)

# Create NACL for database subnets (extra protection)
NACL_ID=$(aws ec2 create-network-acl \
    --vpc-id $VPC_ID \
    --query 'NetworkAcl.NetworkAclId' --output text)

# Allow PostgreSQL from web subnets only
aws ec2 create-network-acl-entry \
    --network-acl-id $NACL_ID \
    --ingress --rule-number 100 \
    --protocol tcp --port-range From=5432,To=5432 \
    --cidr-block 10.0.11.0/24 --rule-action allow

aws ec2 create-network-acl-entry \
    --network-acl-id $NACL_ID \
    --ingress --rule-number 110 \
    --protocol tcp --port-range From=5432,To=5432 \
    --cidr-block 10.0.12.0/24 --rule-action allow

# Deny everything else
aws ec2 create-network-acl-entry \
    --network-acl-id $NACL_ID \
    --ingress --rule-number 200 \
    --protocol -1 \
    --cidr-block 0.0.0.0/0 --rule-action deny
```
