# P4.1 — Deploy a Microservices App to Kubernetes

**Phase:** 4 — Container Orchestration (Kubernetes)  
**Topic:** Kubernetes deployment, microservices  
**Type:** Project

---

## Goal
Deploy a real microservices application to Kubernetes with proper service discovery, configuration, scaling, and networking.

---

## The Application: E-Commerce Platform
Three microservices:
- **user-service**: manages users and authentication (Python/FastAPI)
- **product-service**: manages products and inventory (Python/FastAPI)
- **order-service**: creates and tracks orders, calls user and product services (Python/FastAPI)
- **api-gateway**: nginx that routes external traffic to services

```
External → Ingress → nginx gateway → user-service
                               ↘ product-service
                               ↘ order-service → user-service
                                              ↘ product-service
```

## Key Manifest: Order Service Deployment
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: order-service
  namespace: production
  labels:
    app: order-service
    version: "1.0"
    team: backend
spec:
  replicas: 2
  selector:
    matchLabels:
      app: order-service
  strategy:
    type: RollingUpdate
    rollingUpdate:
      maxSurge: 1
      maxUnavailable: 0
  template:
    metadata:
      labels:
        app: order-service
        version: "1.0"
    spec:
      serviceAccountName: order-service-sa
      securityContext:
        runAsNonRoot: true
        runAsUser: 1001
      containers:
        - name: order-service
          image: ghcr.io/myorg/order-service:v1.0.0
          ports:
            - containerPort: 8080
          env:
            - name: USER_SERVICE_URL
              value: "http://user-service:8080"
            - name: PRODUCT_SERVICE_URL
              value: "http://product-service:8080"
            - name: DATABASE_URL
              valueFrom:
                secretKeyRef:
                  name: db-credentials
                  key: order-service-db-url
          resources:
            requests:
              cpu: 100m
              memory: 128Mi
            limits:
              cpu: 500m
              memory: 256Mi
          livenessProbe:
            httpGet:
              path: /health
              port: 8080
            initialDelaySeconds: 10
            periodSeconds: 30
          readinessProbe:
            httpGet:
              path: /ready
              port: 8080
            initialDelaySeconds: 5
            periodSeconds: 10
          securityContext:
            allowPrivilegeEscalation: false
            readOnlyRootFilesystem: true
```

## Service Discovery
Services communicate via Kubernetes DNS:
- `user-service:8080` → resolves to user-service ClusterIP
- Full name: `user-service.production.svc.cluster.local:8080`

## RBAC for Microservices
```yaml
# Each service gets its own ServiceAccount
apiVersion: v1
kind: ServiceAccount
metadata:
  name: order-service-sa
  namespace: production

---
# Grant only necessary permissions (least privilege)
apiVersion: rbac.authorization.k8s.io/v1
kind: Role
metadata:
  name: order-service-role
  namespace: production
rules:
  - apiGroups: [""]
    resources: ["configmaps"]
    verbs: ["get", "list"]
    resourceNames: ["order-service-config"]

---
apiVersion: rbac.authorization.k8s.io/v1
kind: RoleBinding
metadata:
  name: order-service-binding
  namespace: production
subjects:
  - kind: ServiceAccount
    name: order-service-sa
roleRef:
  kind: Role
  name: order-service-role
  apiGroup: rbac.authorization.k8s.io
```

## Network Policies (Zero-Trust)
```yaml
# Only order-service can call user-service and product-service
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: user-service-ingress
  namespace: production
spec:
  podSelector:
    matchLabels:
      app: user-service
  policyTypes:
    - Ingress
  ingress:
    - from:
        - podSelector:
            matchLabels:
              app: order-service
        - podSelector:
            matchLabels:
              app: api-gateway
      ports:
        - protocol: TCP
          port: 8080
```

## Deliverables
- [ ] All 4 services running in `production` namespace
- [ ] `kubectl get pods -n production` — all pods Running and Ready
- [ ] Services discover each other via DNS (order-service can call user-service)
- [ ] Ingress routing: different paths to different services
- [ ] Each service has its own ServiceAccount and RBAC
- [ ] NetworkPolicy: services only accept traffic from authorized sources
- [ ] HPA configured: each service scales 2-10 replicas
- [ ] Rolling update: update user-service image with zero downtime
- [ ] Sealed Secrets: DB credentials encrypted and committed to git
- [ ] Load test: k6 or locust hitting the system while HPA scales
