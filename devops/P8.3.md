## Top CIS Findings and Remediation

### Finding 1: CloudTrail Not Enabled in All Regions
**CIS 3.1**: Ensure CloudTrail is enabled in all regions

```bash
# Check current state
aws cloudtrail describe-trails --query 'trailList[*].[Name,IsMultiRegionTrail]'

# Remediate: create multi-region trail
aws cloudtrail create-trail \
    --name myorg-audit-trail \
    --s3-bucket-name myorg-cloudtrail-logs \
    --is-multi-region-trail \
    --enable-log-file-validation \
    --include-global-service-events

aws cloudtrail start-logging --name myorg-audit-trail
```

### Finding 2: Root Account Has Active Access Keys
**CIS 1.4**: Ensure no root account access key exists

```bash
# Check for root access keys
aws iam get-account-summary | jq '.SummaryMap.AccountAccessKeysPresent'
# Should be 0

# Delete root access keys (must be done as root in console)
# Cannot be done via CLI if root keys exist
```

### Finding 3: MFA Not Enabled for Root
**CIS 1.5**: Ensure MFA is enabled for root account

```bash
# Check
aws iam get-account-summary | jq '.SummaryMap.AccountMFAEnabled'
# 1 = enabled, 0 = not enabled

# Enable via console: Security Credentials → MFA → Assign MFA device
```

### Finding 4: IAM Password Policy Too Weak
**CIS 1.8-1.11**: Password policy requirements

```bash
# Check current policy
aws iam get-account-password-policy

# Remediate
aws iam update-account-password-policy \
    --minimum-password-length 14 \
    --require-symbols \
    --require-numbers \
    --require-uppercase-characters \
    --require-lowercase-characters \
    --allow-users-to-change-password \
    --max-password-age 90 \
    --password-reuse-prevention 24 \
    --hard-expiry false
```

### Finding 5: S3 Buckets Publicly Accessible
**CIS 2.1.5**: Ensure S3 buckets don't allow public access

```bash
# Find all public buckets
aws s3api list-buckets --query 'Buckets[*].Name' --output text | tr '\t' '\n' | while read bucket; do
    public=$(aws s3api get-public-access-block --bucket "$bucket" 2>/dev/null | \
        jq '.PublicAccessBlockConfiguration | any(.[]; . == false)')
    if [ "$public" = "true" ]; then
        echo "PUBLIC BUCKET: $bucket"
    fi
done

# Remediate: block all public access at account level
aws s3control put-public-access-block \
    --account-id $(aws sts get-caller-identity --query Account --output text) \
    --public-access-block-configuration \
        BlockPublicAcls=true,\
        IgnorePublicAcls=true,\
        BlockPublicPolicy=true,\
        RestrictPublicBuckets=true
```

### Finding 6: VPC Flow Logs Not Enabled
**CIS 3.9**: Ensure VPC flow logging is enabled in all VPCs

```bash
# Find VPCs without flow logs
aws ec2 describe-vpcs --query 'Vpcs[*].VpcId' --output text | tr '\t' '\n' | while read vpc_id; do
    logs=$(aws ec2 describe-flow-logs \
        --filter Name=resource-id,Values=$vpc_id \
        --query 'FlowLogs | length(@)')
    if [ "$logs" = "0" ]; then
        echo "NO FLOW LOGS: $vpc_id"
        
        # Remediate
        aws ec2 create-flow-logs \
            --resource-ids $vpc_id \
            --resource-type VPC \
            --traffic-type ALL \
            --log-group-name /aws/vpc/flowlogs \
            --deliver-logs-permission-arn arn:aws:iam::ACCOUNT:role/FlowLogsRole
    fi
done
```

## AWS Config for Continuous Compliance
```bash
# Enable AWS Config in all regions
aws configservice put-configuration-recorder \
    --configuration-recorder '{
        "name": "default",
        "roleARN": "arn:aws:iam::ACCOUNT:role/AWSConfigRole",
        "recordingGroup": {
            "allSupported": true,
            "includeGlobalResourceTypes": true
        }
    }'

# Enable Security Hub (consolidates findings from Config, GuardDuty, etc.)
aws securityhub enable-security-hub \
    --enable-default-standards

# Enable GuardDuty (threat detection)
aws guardduty create-detector --enable

# Enable CIS AWS Foundations standard in Security Hub
aws securityhub batch-enable-standards \
    --standards-subscription-requests \
    '[{"StandardsArn":"arn:aws:securityhub:::ruleset/cis-aws-foundations-benchmark/v/1.2.0"}]'
```

## Compliance Dashboard in Grafana
```python
# fetch-compliance.py — Lambda that fetches Security Hub findings and publishes to CloudWatch
import boto3, json

securityhub = boto3.client('securityhub')
cloudwatch = boto3.client('cloudwatch')

def handler(event, context):
    # Get aggregated finding counts
    response = securityhub.get_findings(
        Filters={
            'RecordState': [{'Value': 'ACTIVE', 'Comparison': 'EQUALS'}],
            'WorkflowStatus': [{'Value': 'NEW', 'Comparison': 'EQUALS'}]
        }
    )
    
    critical = sum(1 for f in response['Findings'] if f['Severity']['Label'] == 'CRITICAL')
    high = sum(1 for f in response['Findings'] if f['Severity']['Label'] == 'HIGH')
    
    cloudwatch.put_metric_data(
        Namespace='Security/Compliance',
        MetricData=[
            {'MetricName': 'CriticalFindings', 'Value': critical, 'Unit': 'Count'},
            {'MetricName': 'HighFindings', 'Value': high, 'Unit': 'Count'}
        ]
    )
```
