**Phase:** 8 — DevSecOps & Compliance  
**Topic:** Supply chain security, SBOM, image signing  
**Type:** Assignment

## Goal
Secure the container supply chain — from building to deploying. Supply chain attacks (like SolarWinds) are among the most damaging in recent history.

---

## Task

### Part 1 — Understanding Supply Chain Attacks
A supply chain attack compromises the software you depend on, not you directly:
- Malicious package uploaded to PyPI/npm with similar name (typosquatting)
- Compromised base image in Docker Hub
- Backdoored open-source library
- Compromised CI/CD system injecting code at build time

Defense strategy: **SLSA framework** (Supply-chain Levels for Software Artifacts):
- Level 1: Documented build process
- Level 2: Tamper-evident provenance
- Level 3: Hardened builds (no tampering possible)
- Level 4: Hermetic builds, reproducible

### Part 2 — Software Bill of Materials (SBOM)
An SBOM is an inventory of all components in your software:

```bash
# Generate SBOM for a container image using Syft
brew install syft  # or download binary

# Generate SBOM in SPDX format
syft ghcr.io/myorg/myapp:latest -o spdx-json > sbom.spdx.json

# Generate in CycloneDX format (more widely supported)
syft ghcr.io/myorg/myapp:latest -o cyclonedx-json > sbom.cyclonedx.json

# Generate SBOM for local directory
syft dir:./src -o spdx-json > sbom.spdx.json

# Inspect the SBOM
cat sbom.spdx.json | jq '.packages[] | {name: .name, version: .versionInfo}'
```

Scan the SBOM for vulnerabilities with Grype:
```bash
brew install grype

# Scan image directly
grype ghcr.io/myorg/myapp:latest

# Scan from SBOM
grype sbom:./sbom.spdx.json

# Only show HIGH and CRITICAL
grype ghcr.io/myorg/myapp:latest --fail-on high
```

### Part 3 — Image Signing with Cosign
```bash
# Install Cosign
brew install sigstore/tap/cosign

# Generate a key pair (for key-based signing)
cosign generate-key-pair
# Creates: cosign.key (private, never commit) and cosign.pub (public, safe to commit)

# Sign an image
cosign sign --key cosign.key ghcr.io/myorg/myapp:sha-abc123

# Verify the signature
cosign verify --key cosign.pub ghcr.io/myorg/myapp:sha-abc123

# Sign with keyless (uses OIDC identity — best for CI)
# In GitHub Actions, GITHUB_TOKEN provides the OIDC token
cosign sign ghcr.io/myorg/myapp:sha-abc123
# No key needed — identity is verified via Sigstore's transparency log (Rekor)

# Verify keyless signature
cosign verify \
    --certificate-identity=https://github.com/myorg/myapp/.github/workflows/build.yml@refs/heads/main \
    --certificate-oidc-issuer=https://token.actions.githubusercontent.com \
    ghcr.io/myorg/myapp:sha-abc123
```

### Part 4 — Attach SBOM to Image
```bash
# Attach SBOM as an attestation to the image
cosign attest \
    --key cosign.key \
    --predicate sbom.spdx.json \
    --type spdx \
    ghcr.io/myorg/myapp:sha-abc123

# Verify and retrieve the attestation
cosign verify-attestation \
    --key cosign.pub \
    --type spdx \
    ghcr.io/myorg/myapp:sha-abc123 | jq '.payload | @base64d | fromjson'
```

### Part 5 — Full CI/CD Pipeline with Supply Chain Security
```yaml
# .github/workflows/secure-build.yml
name: Secure Build

permissions:
  id-token: write   # for keyless signing
  contents: read
  packages: write

jobs:
  build-and-sign:
    runs-on: ubuntu-latest
    outputs:
      digest: ${{ steps.build.outputs.digest }}

    steps:
      - uses: actions/checkout@v4

      - name: Install Cosign
        uses: sigstore/cosign-installer@v3

      - name: Install Syft
        uses: anchore/sbom-action/download-syft@v0

      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build and push
        id: build
        uses: docker/build-push-action@v5
        with:
          push: true
          tags: ghcr.io/${{ github.repository }}:${{ github.sha }}

      - name: Generate SBOM
        uses: anchore/sbom-action@v0
        with:
          image: ghcr.io/${{ github.repository }}@${{ steps.build.outputs.digest }}
          output-file: sbom.spdx.json

      - name: Scan SBOM for vulnerabilities
        uses: anchore/scan-action@v3
        with:
          sbom: sbom.spdx.json
          fail-build: true
          severity-cutoff: critical

      - name: Sign image (keyless)
        run: |
          cosign sign --yes \
            ghcr.io/${{ github.repository }}@${{ steps.build.outputs.digest }}

      - name: Attest SBOM
        run: |
          cosign attest --yes \
            --predicate sbom.spdx.json \
            --type spdx \
            ghcr.io/${{ github.repository }}@${{ steps.build.outputs.digest }}
```

### Part 6 — Policy Enforcement at Deployment Time
Use Kyverno to enforce: "only deploy signed images"
```yaml
# kyverno-policy.yaml
apiVersion: kyverno.io/v1
kind: ClusterPolicy
metadata:
  name: require-signed-images
spec:
  validationFailureAction: Enforce
  background: false
  rules:
    - name: check-image-signature
      match:
        any:
          - resources:
              kinds: [Pod]
      verifyImages:
        - imageReferences:
            - "ghcr.io/myorg/*"
          attestors:
            - count: 1
              entries:
                - keyless:
                    subject: "https://github.com/myorg/*"
                    issuer: "https://token.actions.githubusercontent.com"
```
