# P5.2 — Configuration Management Pipeline with Ansible + Packer

**Phase:** 5 — Infrastructure as Code  
**Topic:** Ansible, Packer, golden AMIs  
**Type:** Project

---

## Goal
Build a configuration management pipeline: use Packer to bake golden machine images and Ansible to configure them — the "immutable infrastructure" approach used by Netflix and others.

---

## The Concept: Immutable Infrastructure
Traditional approach:
- Deploy servers → update them in place → servers drift → "works on my machine"

Immutable approach:
- Bake everything into an image → deploy new servers from image → discard old ones
- Servers are never modified after deployment — replaced, not updated

## What to Build
```
Git Push (Ansible roles changed)
    ↓
Packer + Ansible: bake new AMI
    ↓
Run Ansible tests on the AMI
    ↓
Publish AMI ID to Parameter Store
    ↓
Terraform reads the AMI ID
    ↓
Terraform replaces old ASG launch config with new AMI
    ↓
Rolling replacement of EC2 instances
```

## Ansible Roles Called by Packer
```yaml
# ansible/site.yml
---
- name: Bake application image
  hosts: all
  become: true

  roles:
    - common           # OS hardening, packages, users
    - security         # SSH hardening, fail2ban, ufw
    - python           # install correct Python version + virtualenv
    - myapp            # install app code and dependencies
    - nginx            # install and configure nginx as reverse proxy
    - cloudwatch-agent # AWS CloudWatch agent for logs and metrics
    - verify           # run verification checks
```

## Verify Role
```yaml
# ansible/roles/verify/tasks/main.yml
---
- name: Run verification suite
  block:
    - name: Check nginx is installed
      command: nginx -v
      changed_when: false

    - name: Check app service file exists
      stat:
        path: /etc/systemd/system/myapp.service
      register: service_file
      failed_when: not service_file.stat.exists

    - name: Check app binary is executable
      stat:
        path: /opt/myapp/venv/bin/gunicorn
      register: binary
      failed_when: not binary.stat.executable

    - name: Test app starts successfully
      command: /opt/myapp/scripts/start-test.sh
      changed_when: false

    - name: Test app health endpoint
      uri:
        url: http://localhost:8080/health
        return_content: true
      register: health
      failed_when: health.status != 200
```

## CI/CD Pipeline for Image Baking
```yaml
# .github/workflows/bake-ami.yml
name: Bake AMI

on:
  push:
    branches: [main]
    paths:
      - 'ansible/**'
      - 'app/**'
      - 'packer/**'

jobs:
  bake:
    runs-on: ubuntu-latest
    environment: production
    outputs:
      ami-id: ${{ steps.get-ami.outputs.ami-id }}

    steps:
      - uses: actions/checkout@v4

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: arn:aws:iam::ACCOUNT:role/PackerRole
          aws-region: us-east-1

      - name: Get latest base AMI
        id: base-ami
        run: |
          AMI_ID=$(aws ec2 describe-images \
            --owners amazon \
            --filters "Name=name,Values=ubuntu/images/hvm-ssd/ubuntu-jammy-22.04-amd64*" \
            --query 'sort_by(Images, &CreationDate)[-1].ImageId' \
            --output text)
          echo "ami-id=$AMI_ID" >> $GITHUB_OUTPUT

      - name: Install Packer and Ansible
        run: |
          pip install ansible
          curl -fsSL https://releases.hashicorp.com/packer/1.10.0/packer_1.10.0_linux_amd64.zip | funzip > /usr/local/bin/packer
          chmod +x /usr/local/bin/packer
          packer plugins install github.com/hashicorp/amazon
          packer plugins install github.com/hashicorp/ansible

      - name: Bake AMI
        env:
          BASE_AMI_ID: ${{ steps.base-ami.outputs.ami-id }}
          APP_VERSION: ${{ github.sha }}
        run: |
          packer build packer/myapp.pkr.hcl
          cat packer-manifest.json

      - name: Get new AMI ID
        id: get-ami
        run: |
          AMI_ID=$(cat packer-manifest.json | jq -r '.builds[-1].artifact_id' | cut -d: -f2)
          echo "ami-id=$AMI_ID" >> $GITHUB_OUTPUT
          echo "New AMI: $AMI_ID"

      - name: Publish AMI ID to Parameter Store
        run: |
          aws ssm put-parameter \
            --name "/myapp/production/ami-id" \
            --value "${{ steps.get-ami.outputs.ami-id }}" \
            --type String \
            --overwrite

  deploy:
    needs: bake
    runs-on: ubuntu-latest
    environment: production
    steps:
      - name: Trigger Terraform to use new AMI
        run: |
          cd infra/environments/production
          terraform init
          terraform apply -auto-approve \
            -var="ami_id=${{ needs.bake.outputs.ami-id }}"
```
