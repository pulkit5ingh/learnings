# P6.1 — Three-Tier Web Application on AWS

**Phase:** 6 — Cloud Platforms  
**Topic:** AWS architecture, multi-tier applications  
**Type:** Project

---

## Goal
Deploy a production-grade three-tier application on AWS — the standard architecture for scalable, resilient web applications.

---

## Architecture
```
Tier 1 (Presentation): Users → Route 53 → CloudFront → ALB → EC2 (Auto Scaling)
Tier 2 (Application):  EC2 (private subnet) → RDS PostgreSQL (Multi-AZ)
Tier 3 (Data):         RDS PostgreSQL + ElastiCache Redis
```

## What to Build
Deploy the Blog application from P3.1 on AWS with:
- **Route 53**: custom domain pointing to CloudFront
- **CloudFront**: CDN for static assets + HTTPS
- **ALB**: load balancing across EC2 instances
- **Auto Scaling Group**: EC2 instances in private subnets
- **RDS PostgreSQL Multi-AZ**: primary + standby in different AZs
- **ElastiCache Redis**: session cache and API caching
- **S3**: static asset storage + application logs
- **CloudWatch**: metrics, logs, alarms

### 2. Launch Template for EC2
```bash
# Create launch template with:
aws ec2 create-launch-template \
    --launch-template-name myapp-template \
    --launch-template-data '{
        "ImageId": "ami-xxx",
        "InstanceType": "t3.small",
        "IamInstanceProfile": {"Name": "MyAppInstanceProfile"},
        "SecurityGroupIds": ["sg-xxx"],
        "UserData": "base64-encoded-startup-script"
    }'
```

User data script:
```bash
#!/bin/bash
# Install and configure the app from S3
aws s3 cp s3://myapp-artifacts/myapp-latest.tar.gz /opt/
tar xzf /opt/myapp-latest.tar.gz -C /opt/myapp/
cd /opt/myapp && pip install -r requirements.txt
systemctl enable myapp
systemctl start myapp
```

### 3. Auto Scaling Group
```bash
aws autoscaling create-auto-scaling-group \
    --auto-scaling-group-name myapp-asg \
    --launch-template "LaunchTemplateName=myapp-template" \
    --min-size 2 \
    --max-size 10 \
    --desired-capacity 2 \
    --vpc-zone-identifier "subnet-private-a,subnet-private-b" \
    --target-group-arns $TG_ARN \
    --health-check-type ELB \
    --health-check-grace-period 120

# Scaling policies
aws autoscaling put-scaling-policy \
    --auto-scaling-group-name myapp-asg \
    --policy-name scale-on-cpu \
    --policy-type TargetTrackingScaling \
    --target-tracking-configuration '{
        "PredefinedMetricSpecification": {
            "PredefinedMetricType": "ASGAverageCPUUtilization"
        },
        "TargetValue": 70.0,
        "DisableScaleIn": false
    }'
```

### 5. RDS Multi-AZ
```bash
aws rds create-db-instance \
    --db-instance-identifier myapp-prod-db \
    --db-instance-class db.t3.medium \
    --engine postgres \
    --engine-version 16.1 \
    --master-username myapp \
    --master-user-password $(aws secretsmanager get-secret-value --secret-id myapp/db-password --query SecretString --output text) \
    --db-name myappdb \
    --db-subnet-group-name myapp-db-subnet-group \
    --vpc-security-group-ids $DB_SG \
    --multi-az \
    --backup-retention-period 7 \
    --deletion-protection \
    --storage-encrypted \
    --allocated-storage 100 \
    --storage-type gp3
```

### 6. CloudWatch Alarms
```bash
# CPU alarm → SNS → Auto Scaling
aws cloudwatch put-metric-alarm \
    --alarm-name myapp-high-cpu \
    --metric-name CPUUtilization \
    --namespace AWS/EC2 \
    --dimensions Name=AutoScalingGroupName,Value=myapp-asg \
    --statistic Average \
    --period 300 \
    --threshold 80 \
    --comparison-operator GreaterThanThreshold \
    --evaluation-periods 2 \
    --alarm-actions arn:aws:sns:us-east-1:ACCOUNT:myapp-alerts

# RDS connection count
aws cloudwatch put-metric-alarm \
    --alarm-name myapp-db-connections \
    --metric-name DatabaseConnections \
    --namespace AWS/RDS \
    --dimensions Name=DBInstanceIdentifier,Value=myapp-prod-db \
    --threshold 100 \
    --comparison-operator GreaterThanThreshold
```

## Disaster Recovery Test
```bash
# Simulate AZ failure: terminate all EC2 instances in AZ-a
aws ec2 terminate-instances --instance-ids $(
    aws ec2 describe-instances \
        --filters "Name=availability-zone,Values=us-east-1a" \
                  "Name=tag:aws:autoscaling:groupName,Values=myapp-asg" \
        --query 'Reservations[].Instances[].InstanceId' \
        --output text
)

# Watch ASG launch replacements in AZ-b
aws autoscaling describe-auto-scaling-groups \
    --auto-scaling-group-names myapp-asg \
    --query 'AutoScalingGroups[0].Instances[*].[InstanceId,AvailabilityZone,HealthStatus]'

# Is the app still working?
curl https://myapp.com/health
```
