## Goal
Operate AWS core services from the command line. The CLI is faster than the console and scriptable — every DevOps engineer uses it daily.

## Task

### Part 1 — AWS CLI Setup and Profiles
```bash
# Configure default profile
aws configure
# AWS Access Key ID: ...
# AWS Secret Access Key: ...
# Default region: us-east-1
# Default output format: json

# Multiple profiles
aws configure --profile staging
aws configure --profile production

# Use a specific profile
aws --profile staging s3 ls

# Set via environment variables (for CI/CD)
export AWS_ACCESS_KEY_ID=...
export AWS_SECRET_ACCESS_KEY=...
export AWS_DEFAULT_REGION=us-east-1

# Verify identity
aws sts get-caller-identity
```

### Part 2 — S3
```bash
# Create a bucket
aws s3 mb s3://my-devops-lab-bucket-$(date +%s) --region us-east-1

# Enable versioning
aws s3api put-bucket-versioning \
    --bucket my-bucket \
    --versioning-configuration Status=Enabled

# Upload files
aws s3 cp file.txt s3://my-bucket/
aws s3 sync ./local-dir s3://my-bucket/prefix/

# List objects
aws s3 ls s3://my-bucket/ --recursive --human-readable

# Download
aws s3 cp s3://my-bucket/file.txt ./downloaded.txt

# Pre-signed URL (share private file temporarily)
aws s3 presign s3://my-bucket/secret-file.txt --expires-in 3600

# Bucket policy: make specific prefix public
aws s3api put-bucket-policy --bucket my-bucket --policy '{
  "Version": "2012-10-17",
  "Statement": [{
    "Effect": "Allow",
    "Principal": "*",
    "Action": "s3:GetObject",
    "Resource": "arn:aws:s3:::my-bucket/public/*"
  }]
}'

# Lifecycle rule: delete old files automatically
aws s3api put-bucket-lifecycle-configuration \
    --bucket my-bucket \
    --lifecycle-configuration file://lifecycle.json
```

### Part 3 — EC2
```bash
# List instances
aws ec2 describe-instances \
    --query 'Reservations[*].Instances[*].[InstanceId,State.Name,PublicIpAddress,Tags[?Key==`Name`].Value|[0]]' \
    --output table

# Launch an instance
aws ec2 run-instances \
    --image-id ami-0c02fb55956c7d316 \
    --instance-type t3.micro \
    --key-name my-key \
    --security-group-ids sg-xxx \
    --subnet-id subnet-xxx \
    --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=test-server}]' \
    --user-data file://user-data.sh

# Stop, start, terminate
aws ec2 stop-instances --instance-ids i-xxx
aws ec2 start-instances --instance-ids i-xxx
aws ec2 terminate-instances --instance-ids i-xxx

# Create a snapshot
aws ec2 create-snapshot \
    --volume-id vol-xxx \
    --description "Backup before upgrade"

# Create an AMI from a running instance
aws ec2 create-image \
    --instance-id i-xxx \
    --name "myapp-$(date +%Y%m%d)" \
    --no-reboot
```

### Part 4 — VPC CLI Operations
```bash
# Create VPC
VPC_ID=$(aws ec2 create-vpc --cidr-block 10.0.0.0/16 --query 'Vpc.VpcId' --output text)
aws ec2 create-tags --resources $VPC_ID --tags Key=Name,Value=my-vpc

# Enable DNS hostnames
aws ec2 modify-vpc-attribute --vpc-id $VPC_ID --enable-dns-hostnames

# Create subnet
SUBNET_ID=$(aws ec2 create-subnet \
    --vpc-id $VPC_ID \
    --cidr-block 10.0.1.0/24 \
    --availability-zone us-east-1a \
    --query 'Subnet.SubnetId' --output text)

# Internet gateway
IGW_ID=$(aws ec2 create-internet-gateway --query 'InternetGateway.InternetGatewayId' --output text)
aws ec2 attach-internet-gateway --vpc-id $VPC_ID --internet-gateway-id $IGW_ID

# Route table
RT_ID=$(aws ec2 create-route-table --vpc-id $VPC_ID --query 'RouteTable.RouteTableId' --output text)
aws ec2 create-route --route-table-id $RT_ID --destination-cidr-block 0.0.0.0/0 --gateway-id $IGW_ID
aws ec2 associate-route-table --subnet-id $SUBNET_ID --route-table-id $RT_ID

# Security group
SG_ID=$(aws ec2 create-security-group \
    --group-name web-sg \
    --description "Web server security group" \
    --vpc-id $VPC_ID \
    --query 'GroupId' --output text)

aws ec2 authorize-security-group-ingress \
    --group-id $SG_ID \
    --protocol tcp --port 80 --cidr 0.0.0.0/0

aws ec2 authorize-security-group-ingress \
    --group-id $SG_ID \
    --protocol tcp --port 22 --cidr $(curl -s ifconfig.me)/32
```

### Part 5 — CLI Output Formatting and Querying
```bash
# JMESPath queries with --query
aws ec2 describe-instances \
    --query 'Reservations[*].Instances[?State.Name==`running`].[InstanceId,PublicIpAddress]' \
    --output text

# JSON output + jq for complex transformations
aws ec2 describe-instances | jq '.Reservations[].Instances[] | {id: .InstanceId, ip: .PublicIpAddress, state: .State.Name}'

# Table output
aws s3api list-buckets --query 'Buckets[*].[Name,CreationDate]' --output table

# Wait for instance to be running
aws ec2 wait instance-running --instance-ids i-xxx
echo "Instance is now running"
```

### Part 6 — CLI Script: Launch and Configure a Server
Write `launch-server.sh` that:
1. Launches an EC2 instance
2. Waits for it to be in `running` state
3. Waits for SSH to be available
4. SSHes in and installs nginx
5. Prints the URL to access it
