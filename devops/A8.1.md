## Goal
Integrate automated security testing into your CI pipeline. Security bugs caught in CI cost 10x less to fix than ones found in production.

---

## Task

### Part 1 — SAST with Semgrep
Semgrep is a fast, open-source static analysis tool that finds security bugs:

```bash
# Install
pip install semgrep

# Run against your code with OWASP rules
semgrep --config=p/owasp-top-ten src/

# Run with Python security rules
semgrep --config=p/python src/

# Output as JSON for CI
semgrep --config=p/owasp-top-ten src/ --json > semgrep-results.json
```

Common findings it detects:
- SQL injection via string formatting
- Hardcoded secrets
- Insecure deserialization
- Command injection
- Improper exception handling
- Missing authentication checks

```python
# Example: Semgrep catches this SQL injection
def get_user(username):
    query = f"SELECT * FROM users WHERE username = '{username}'"  # UNSAFE
    cursor.execute(query)

# Semgrep recommends:
def get_user(username):
    cursor.execute("SELECT * FROM users WHERE username = %s", (username,))  # SAFE
```

### Part 2 — SonarQube (for deeper analysis)
```yaml

services:
  sonarqube:

      - db

    image: postgres:16
    environment:
      POSTGRES_USER: sonar
      POSTGRES_PASSWORD: sonar
      POSTGRES_DB: sonar
```

```bash
# Run SonarScanner
docker run --rm \
    -e SONAR_HOST_URL=http://sonarqube:9000 \
    -e SONAR_TOKEN=your-token \
    -v $(pwd):/usr/src \
    sonarsource/sonar-scanner-cli
```

SonarQube gives you:
- Security hotspots and vulnerabilities
- Code smells and maintainability issues
- Test coverage tracking over time
- Quality gates (fail build if quality drops)

### Part 4 — DAST with OWASP ZAP
DAST tests a running application — finds issues that SAST misses:

```bash
# Run ZAP against a local app (passive scan)
docker run -t owasp/zap2docker-stable zap-baseline.py \
    -t http://host.docker.internal:8080 \
    -r zap-report.html

# Full active scan (more thorough, takes longer)
docker run -t owasp/zap2docker-stable zap-full-scan.py \
    -t http://host.docker.internal:8080 \
    -r zap-full-report.html \
    -I  # don't fail on warnings
```

ZAP finds:
- XSS vulnerabilities
- SQL injection
- Missing security headers
- Sensitive data exposure
- Insecure configurations

### Part 5 — Security Headers Check
```bash
# Check security headers
curl -I https://your-app.com

# Required headers:
# X-Content-Type-Options: nosniff
# X-Frame-Options: DENY
# Strict-Transport-Security: max-age=31536000; includeSubDomains
# Content-Security-Policy: default-src 'self'
# Referrer-Policy: strict-origin-when-cross-origin
# Permissions-Policy: camera=(), microphone=()
```

Add to your nginx config:
```nginx
add_header X-Content-Type-Options nosniff always;
add_header X-Frame-Options DENY always;
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline'" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
```

### Part 6 — Full Security Pipeline in GitHub Actions
```yaml
# .github/workflows/security.yml
name: Security Scan

on:
  push:
    branches: [main, develop]
  pull_request:

jobs:
  sast:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Semgrep SAST
        uses: semgrep/semgrep-action@v1
        with:
          config: >-
            p/owasp-top-ten
            p/python
          generateSarif: true

      - uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: semgrep.sarif

      - name: Bandit Python security scan
        run: |
          pip install bandit
          bandit -r src/ -f json -o bandit.json -ll  # only HIGH and above
          
      - name: pip-audit dependency scan
        run: |
          pip install pip-audit
          pip-audit -r requirements.txt --format json -o audit.json

  dast:
    runs-on: ubuntu-latest
    needs: []  # run in parallel
    steps:
      - uses: actions/checkout@v4

      - name: Start application
        run: docker compose up -d
        
      - name: Wait for app to be ready
        run: |
          for i in $(seq 1 30); do
            curl -sf http://localhost:8080/health && break
            sleep 2
          done

      - name: OWASP ZAP Baseline Scan
        uses: zaproxy/action-baseline@v0.12.0
        with:
          target: 'http://localhost:8080'
          fail_action: false

      - uses: actions/upload-artifact@v4
        with:
          name: zap-report
          path: report_html.html
```
