# P8.1 — Secure CI/CD Pipeline with Automated Security Gates

**Phase:** 8 — DevSecOps & Compliance  
**Topic:** DevSecOps, security in CI/CD  
**Type:** Project

---

## Goal
Build a DevSecOps pipeline where security is enforced automatically at every stage — code, dependencies, containers, infrastructure, and deployment.

---

## The Security Gates Model
```
Code Push
    ↓
[Gate 1] Secret Detection (gitleaks) — BLOCK on any detected secret
    ↓
[Gate 2] SAST (Semgrep + Bandit) — BLOCK on HIGH/CRITICAL
    ↓
[Gate 3] Dependency Scan (pip-audit) — BLOCK on CRITICAL CVE
    ↓
[Gate 4] Container Build
    ↓
[Gate 5] Image Vulnerability Scan (Trivy) — BLOCK on CRITICAL CVE
    ↓
[Gate 6] Container Policy Check (Conftest) — BLOCK on violations
    ↓
[Gate 7] Image Signing (Cosign) — only sign if all gates pass
    ↓
[Gate 8] Deploy to Staging
    ↓
[Gate 9] DAST (OWASP ZAP) — BLOCK if new HIGH vulnerabilities found
    ↓
[Gate 10] IaC Security Scan (Checkov/tfsec) — REPORT on misconfigs
    ↓
Manual Approval for Production
```

## Full Pipeline YAML
```yaml
# .github/workflows/devsecops.yml
name: DevSecOps Pipeline

on:
  push:
    branches: [main]
  pull_request:

permissions:
  contents: read
  packages: write
  id-token: write
  security-events: write  # for SARIF upload

env:
  IMAGE: ghcr.io/${{ github.repository }}
  TAG: ${{ github.sha }}

jobs:
  # ──── Gate 1: Secrets ────
  secrets-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0  # full history for secret scanning
      - uses: gitleaks/gitleaks-action@v2
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

  # ──── Gate 2: SAST ────
  sast:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Semgrep
        uses: semgrep/semgrep-action@v1
        with:
          config: p/owasp-top-ten
          generateSarif: true
        continue-on-error: false  # BLOCKING
      
      - name: Bandit
        run: |
          pip install bandit
          bandit -r src/ -ll -f sarif -o bandit.sarif
          # exit 1 if any HIGH severity
          bandit -r src/ -ll --exit-zero false

      - uses: github/codeql-action/upload-sarif@v3
        if: always()
        with:
          sarif_file: bandit.sarif

  # ──── Gate 3: Dependencies ────
  dependency-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: pip-audit
        run: |
          pip install pip-audit
          pip-audit -r requirements.txt --format json -o audit.json
          # fail if any CRITICAL
          python -c "
          import json, sys
          data = json.load(open('audit.json'))
          crits = [v for pkg in data for v in pkg.get('vulns', []) 
                   if v.get('fix_versions')]
          if crits:
              print(f'CRITICAL vulnerabilities: {crits}')
              sys.exit(1)
          "

  # ──── Gate 4+5: Build + Scan ────
  build-and-scan:
    needs: [secrets-scan, sast, dependency-scan]
    runs-on: ubuntu-latest
    outputs:
      digest: ${{ steps.build.outputs.digest }}
    steps:
      - uses: actions/checkout@v4

      - name: Login to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build
        id: build
        uses: docker/build-push-action@v5
        with:
          push: true
          tags: ${{ env.IMAGE }}:${{ env.TAG }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Trivy scan (Gate 5)
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: ${{ env.IMAGE }}:${{ env.TAG }}
          severity: CRITICAL
          exit-code: '1'  # BLOCKING

      - name: Conftest policy check (Gate 6)
        run: |
          docker run --rm conftest conftest pull \
            ghcr.io/${{ github.repository_owner }}/policies:latest
          conftest test Dockerfile --policy policies/

  # ──── Gate 7: Sign ────
  sign:
    needs: build-and-scan
    runs-on: ubuntu-latest
    steps:
      - uses: sigstore/cosign-installer@v3
      - name: Sign image
        run: |
          cosign sign --yes ${{ env.IMAGE }}@${{ needs.build-and-scan.outputs.digest }}

  # ──── Gate 8+9: Deploy + DAST ────
  deploy-staging:
    needs: sign
    runs-on: ubuntu-latest
    environment: staging
    steps:
      - name: Deploy to staging
        run: ./scripts/deploy.sh staging ${{ env.TAG }}

      - name: Wait for healthy
        run: |
          for i in $(seq 30); do
            curl -sf ${{ secrets.STAGING_URL }}/health && break
            sleep 10
          done

      - name: DAST with OWASP ZAP (Gate 9)
        uses: zaproxy/action-full-scan@v0.10.0
        with:
          target: ${{ secrets.STAGING_URL }}
          fail_action: true
          cmd_options: '-z "-config alert.maxInstances=10"'

  # ──── Gate 10: IaC ────
  iac-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Checkov Terraform scan
        uses: bridgecrewio/checkov-action@master
        with:
          directory: infra/
          framework: terraform
          output_format: sarif
          output_file_path: checkov.sarif
          soft_fail: true  # REPORT only (not blocking for now)

      - uses: github/codeql-action/upload-sarif@v3
        with:
          sarif_file: checkov.sarif
```

## Security Baseline Policy
Document and enforce your security baseline:
```markdown
# Security Baseline for All Services

## Container Requirements
- [ ] Non-root user
- [ ] Read-only root filesystem
- [ ] No privileged mode
- [ ] No capability escalation
- [ ] Resource limits set
- [ ] Health check configured
- [ ] No :latest tag

## Code Requirements
- [ ] No secrets in code (use environment variables)
- [ ] Input validation on all API endpoints
- [ ] SQL queries use parameterized statements
- [ ] Auth required on all non-public endpoints
- [ ] Rate limiting on authentication endpoints

## CI/CD Requirements
- [ ] All gates pass before production deployment
- [ ] Manual approval required for production
- [ ] Images signed before deployment
- [ ] Deployment audited in CloudTrail
```

## Deliverables
- [ ] All 10 security gates implemented in CI/CD
- [ ] Each gate is actually blocking (not just reporting)
- [ ] Secret introduced to code: gitleaks blocks the PR
- [ ] Vulnerable dependency added: pip-audit blocks the PR
- [ ] Container with root user: Trivy/Conftest blocks
- [ ] Signed image: verify signature before deploying
- [ ] ZAP report: no new HIGH vulnerabilities
- [ ] Security baseline documented
- [ ] Security dashboard in Grafana: gate pass/fail over time
- [ ] Written: where in the pipeline is it cheapest to find a security bug?
