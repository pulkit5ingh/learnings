### Part 1 — Process Management
```bash
# View all processes
ps aux             # snapshot
top                # interactive (press 1 for per-CPU, M for memory sort)
htop               # better interactive (install if not present)

# Find a specific process
pgrep nginx
pgrep -la nginx    # with command line
pidof nginx

# Process tree
pstree -p

# Send signals
kill -SIGTERM <pid>   # graceful shutdown
kill -SIGKILL <pid>   # force kill
kill -SIGHUP <pid>    # reload config (nginx uses this)
kill -SIGSTOP <pid>   # pause process
kill -SIGCONT <pid>   # resume process

# Kill by name
pkill nginx
killall nginx
```

Write a script `find_resource_hog.sh` that:
- Finds the top 5 processes by CPU usage
- Finds the top 5 processes by memory usage
- Alerts if any process is using >80% CPU or >2GB RAM
- Outputs a formatted report

### Part 2 — systemd Fundamentals
```bash
# Basic systemd commands
systemctl status nginx
systemctl start nginx
systemctl stop nginx
systemctl restart nginx
systemctl reload nginx      # reload config without downtime
systemctl enable nginx      # start at boot
systemctl disable nginx     # don't start at boot

# View all services
systemctl list-units --type=service --state=running

# Journal (logs for all services)
journalctl -u nginx                     # nginx logs
journalctl -u nginx --since "1 hour ago"
journalctl -u nginx -f                  # follow (like tail -f)
journalctl -p err -u nginx              # errors only
journalctl --disk-usage                 # how much space logs use
```

### Part 3 — Write a systemd Service
Create a simple Python HTTP health endpoint:
```python
# /opt/healthcheck/app.py
from http.server import HTTPServer, BaseHTTPRequestHandler
import json, datetime

class Handler(BaseHTTPRequestHandler):
    def do_GET(self):
        if self.path == "/health":
            payload = json.dumps({
                "status": "ok",
                "timestamp": datetime.datetime.utcnow().isoformat()
            }).encode()
            self.send_response(200)
            self.send_header("Content-Type", "application/json")
            self.end_headers()
            self.wfile.write(payload)
        else:
            self.send_response(404)
            self.end_headers()

HTTPServer(("0.0.0.0", 8080), Handler).serve_forever()
```

Now create the systemd unit file:
```ini
# /etc/systemd/system/healthcheck.service
[Unit]
Description=Health Check HTTP Server
After=network.target
Wants=network-online.target

[Service]
Type=simple
User=nobody
Group=nogroup
ExecStart=/usr/bin/python3 /opt/healthcheck/app.py
Restart=on-failure
RestartSec=5
StandardOutput=journal
StandardError=journal
# Security hardening:
NoNewPrivileges=true
PrivateTmp=true
ProtectSystem=strict
ReadWritePaths=/var/log/healthcheck

[Install]
WantedBy=multi-user.target
```

Enable and start it. Verify with:
```bash
systemctl daemon-reload
systemctl enable healthcheck
systemctl start healthcheck
systemctl status healthcheck
curl http://localhost:8080/health
journalctl -u healthcheck -f
```

Explain each line in the unit file, especially the security options.

### Part 4 — cron and Scheduled Tasks
```bash
# Edit crontab
crontab -e

# Cron syntax: min hour dom month dow command
# Run disk_report.sh every day at 7 AM
0 7 * * * /opt/scripts/disk_report.sh >> /var/log/disk_report.log 2>&1

# Run backup every Sunday at 2 AM
0 2 * * 0 /opt/scripts/backup.sh /data /backups >> /var/log/backup.log 2>&1

# List current cron jobs
crontab -l

# System-wide cron
ls /etc/cron.{hourly,daily,weekly,monthly}/
```

Create a systemd timer as an alternative to cron:
```ini
# /etc/systemd/system/disk-report.timer
[Unit]
Description=Run disk report daily

[Timer]
OnCalendar=daily
Persistent=true

[Install]
WantedBy=timers.target
```

Explain the advantages of systemd timers over cron.

### Part 5 — Resource Limits with cgroups
```bash
# Set memory limit on a service
# In the [Service] section of a unit file:
MemoryMax=512M
CPUQuota=50%    # max 50% of one CPU
TasksMax=100    # max 100 processes/threads

# Check current cgroup limits
systemctl show healthcheck | grep -E "Memory|CPU|Tasks"

# View resource usage per service
systemd-cgtop
```
