# P4.3 вЂ” Helm Chart for a Production-Grade Application

**Phase:** 4 вЂ” Container Orchestration (Kubernetes)  
**Topic:** Helm, Kubernetes packaging  
**Type:** Project

---

## Goal
Author a production-grade Helm chart for the microservices application from P4.1 вЂ” the kind of chart you would publish to an internal Helm repository for your organization to reuse.

---

## What to Build
A Helm chart called `ecommerce` that deploys the entire microservices stack with:
- All 4 services as sub-charts (or a single chart with feature flags)
- Built-in Prometheus metrics scraping
- Configurable autoscaling
- Ingress with TLS
- Optional dependencies (PostgreSQL, Redis from Bitnami)
- Sensible defaults, production-tunable values

## Chart.yaml
```yaml
apiVersion: v2
name: ecommerce
description: E-commerce platform Helm chart
type: application
version: 1.2.0        # chart version
appVersion: "2.0.0"   # application version

dependencies:
  - name: postgresql
    version: "12.x.x"
    repository: "https://charts.bitnami.com/bitnami"
    condition: postgresql.enabled
    alias: postgres
  - name: redis
    version: "17.x.x"
    repository: "https://charts.bitnami.com/bitnami"
    condition: redis.enabled

maintainers:
  - name: Platform Team
    email: platform@myorg.com
```

## Advanced Template: ServiceMonitor for Prometheus
```yaml
# templates/user-service/servicemonitor.yaml
{{- if and .Values.userService.enabled .Values.userService.metrics.enabled }}
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: {{ include "ecommerce.fullname" . }}-user-service
  labels:
    {{- include "ecommerce.labels" . | nindent 4 }}
spec:
  selector:
    matchLabels:
      app: {{ include "ecommerce.fullname" . }}-user-service
  endpoints:
    - port: metrics
      path: {{ .Values.userService.metrics.path }}
      interval: 30s
{{- end }}
```
