# P2.1 — Full CI/CD Pipeline for a Python App

**Phase:** 2 — Version Control & CI/CD Basics  
**Topic:** GitHub Actions, CI/CD  
**Type:** Project

---

## Goal
Build a complete CI/CD pipeline for a real Python application — from code push to deployed, tested, and verified in production.

---

## What to Build
Take a Python FastAPI application and build a full pipeline:

```
Code Push
    ↓
Lint + Type Check
    ↓
Unit + Integration Tests (with Postgres)
    ↓
Security Scan (SAST + deps)
    ↓
Docker Build + Push to GHCR
    ↓
Deploy to Staging
    ↓
Smoke Tests against Staging
    ↓
Manual Approval
    ↓
Deploy to Production
    ↓
Health Check + Notify Slack
```

## The Application
Build a simple Task Manager API with:
- `POST /tasks` — create a task
- `GET /tasks` — list tasks with pagination
- `GET /tasks/{id}` — get a task
- `PUT /tasks/{id}` — update a task
- `DELETE /tasks/{id}` — delete a task
- `GET /health` — health endpoint (returns DB status)

## Pipeline Structure
```yaml
# .github/workflows/
# ├── ci.yml          — runs on every PR
# ├── cd-staging.yml  — runs on merge to main
# └── cd-production.yml — manual trigger with approval
```

### ci.yml
```yaml
name: CI

on:
  pull_request:
    branches: [main]

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
          cache: pip
      - run: pip install -r requirements-dev.txt
      - run: ruff check .
      - run: mypy src/
      - run: black --check .

  test:
    needs: quality
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_PASSWORD: test
          POSTGRES_DB: tasks_test
        options: --health-cmd pg_isready
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
          cache: pip
      - run: pip install -r requirements.txt -r requirements-dev.txt
      - run: pytest tests/ --cov=src --cov-fail-under=80
      - uses: actions/upload-artifact@v4
        with:
          name: coverage
          path: htmlcov/

  security:
    needs: quality
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: pip install bandit pip-audit
      - run: bandit -r src/ -ll
      - run: pip-audit -r requirements.txt
      - uses: gitleaks/gitleaks-action@v2
```

### cd-staging.yml
```yaml
name: Deploy to Staging

on:
  push:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest
    outputs:
      image-tag: ${{ steps.meta.outputs.tags }}
    steps:
      - uses: actions/checkout@v4
      - uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}
      - uses: docker/metadata-action@v5
        id: meta
        with:
          images: ghcr.io/${{ github.repository }}
          tags: type=sha
      - uses: docker/build-push-action@v5
        with:
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  deploy-staging:
    needs: build
    runs-on: ubuntu-latest
    environment: staging
    steps:
      - name: Deploy to staging
        run: |
          # SSH to staging server and update the running container
          ssh -i ${{ secrets.STAGING_SSH_KEY }} \
              deploy@${{ secrets.STAGING_HOST }} \
              "docker pull ${{ needs.build.outputs.image-tag }} && \
               docker compose up -d"

  smoke-test:
    needs: deploy-staging
    runs-on: ubuntu-latest
    steps:
      - name: Wait for deployment
        run: sleep 30
      - name: Health check
        run: |
          for i in $(seq 1 10); do
            if curl -sf ${{ secrets.STAGING_URL }}/health; then
              echo "Staging is healthy"
              exit 0
            fi
            sleep 10
          done
          echo "Staging health check failed"
          exit 1
      - name: Smoke tests
        run: |
          # Create a task
          TASK=$(curl -sf -X POST ${{ secrets.STAGING_URL }}/tasks \
            -H "Content-Type: application/json" \
            -d '{"title": "Smoke test task"}')
          TASK_ID=$(echo $TASK | jq -r '.id')
          
          # Get it back
          curl -sf ${{ secrets.STAGING_URL }}/tasks/$TASK_ID
          
          # Delete it
          curl -sf -X DELETE ${{ secrets.STAGING_URL }}/tasks/$TASK_ID
          echo "Smoke tests passed"
```
