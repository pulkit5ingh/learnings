# A9.1 — GitOps: Deploy and Sync with ArgoCD

**Phase:** 9 — Advanced: GitOps, SRE & Platform Engineering  
**Topic:** GitOps principles, ArgoCD  
**Type:** Assignment

---

## Goal
Implement GitOps: your Git repository is the single source of truth for cluster state. ArgoCD watches the repo and automatically syncs the cluster to match. No manual kubectl applies in production.

---

## Task

### Part 1 — GitOps Principles
GitOps has 4 principles (from OpenGitOps):
1. **Declarative**: desired state described declaratively (YAML, Helm, Kustomize)
2. **Versioned and immutable**: desired state stored in Git, history preserved
3. **Pulled automatically**: software agents pull desired state from Git
4. **Continuously reconciled**: agents observe and correct drift

The key insight: instead of pushing deployments ("run kubectl apply"), you push changes to Git. ArgoCD pulls and applies them. The cluster always matches Git.

### Part 2 — Install ArgoCD
```bash
# Install ArgoCD in Kubernetes
kubectl create namespace argocd
kubectl apply -n argocd \
    -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml

# Wait for pods to be ready
kubectl wait --for=condition=ready pod -l app.kubernetes.io/name=argocd-server -n argocd --timeout=300s

# Get admin password
kubectl get secret argocd-initial-admin-secret -n argocd \
    -o jsonpath='{.data.password}' | base64 -d

# Port forward to access UI
kubectl port-forward svc/argocd-server -n argocd 8080:443

# Open https://localhost:8080, username: admin, password: from above

# Install ArgoCD CLI
brew install argocd

# Login via CLI
argocd login localhost:8080 --username admin --insecure
```

### Part 4 — Create an ArgoCD Application
```yaml
# apps/myapp-dev.yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: myapp-dev
  namespace: argocd
  finalizers:
    - resources-finalizer.argocd.argoproj.io  # delete resources when app is deleted
spec:
  project: default
  
  source:
    repoURL: https://github.com/myorg/gitops-repo
    targetRevision: HEAD
    path: manifests/myapp/overlays/dev
  
  destination:
    server: https://kubernetes.default.svc  # local cluster
    namespace: dev
  
  syncPolicy:
    automated:
      prune: true       # delete resources removed from Git
      selfHeal: true    # auto-fix manual changes (drift correction)
    syncOptions:
      - CreateNamespace=true
      - PrunePropagationPolicy=foreground
    retry:
      limit: 3
      backoff:
        duration: 5s
        factor: 2
```

```bash
# Apply the application
kubectl apply -f apps/myapp-dev.yaml

# Check status
argocd app get myapp-dev
argocd app list

# Manual sync (when automated sync is off)
argocd app sync myapp-dev

# View diff between Git and cluster
argocd app diff myapp-dev
```

### Part 5 — Kustomize Overlays
Kustomize lets you customize base manifests without templating:

```yaml
# manifests/myapp/base/kustomization.yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

resources:
  - deployment.yaml
  - service.yaml

commonLabels:
  app: myapp
  managed-by: kustomize
```

```yaml
# manifests/myapp/overlays/dev/kustomization.yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

bases:
  - ../../base

namespace: dev

images:
  - name: myapp
    newTag: dev-latest

patchesStrategicMerge:
  - patch-replicas.yaml

configMapGenerator:
  - name: myapp-config
    literals:
      - LOG_LEVEL=debug
      - ENVIRONMENT=dev
```

```yaml
# manifests/myapp/overlays/dev/patch-replicas.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: myapp
spec:
  replicas: 1  # dev only needs 1 replica
```

```yaml
# manifests/myapp/overlays/production/kustomization.yaml
bases:
  - ../../base

namespace: production

images:
  - name: myapp
    newTag: v1.2.3  # pinned version in production

patchesStrategicMerge:
  - patch-production.yaml
```

### Part 6 — Test Drift Detection
GitOps' superpower is drift detection:
```bash
# ArgoCD says app is "Synced"
argocd app get myapp-dev  # STATUS: Synced

# Manually change something in the cluster (simulating an unauthorized change)
kubectl scale deployment myapp -n dev --replicas=5

# ArgoCD detects drift immediately
argocd app get myapp-dev  # STATUS: OutOfSync

# With selfHeal=true, it automatically scales back to what Git says (1 replica)
# Watch it self-heal:
kubectl get deployment myapp -n dev -w
```

### Part 7 — App of Apps Pattern
```yaml
# root-app.yaml — manages all other apps
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: root
  namespace: argocd
spec:
  project: default
  source:
    repoURL: https://github.com/myorg/gitops-repo
    targetRevision: HEAD
    path: apps  # watches the apps/ folder
  destination:
    server: https://kubernetes.default.svc
    namespace: argocd
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
```

Now adding a new application = adding a YAML file to `apps/` and committing to Git. ArgoCD automatically picks it up.
