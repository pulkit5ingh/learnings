## Goal

## Task

### Part 1 — Structured Logging (The Foundation)
Never log plain strings. Log structured JSON:

```python
# Bad
logger.info(f"User {user_id} logged in from {ip_address}")

# Good — structured, searchable
logger.info("user.login", extra={
    "user_id": user_id,
    "ip_address": ip_address,
    "user_agent": request.headers.get("user-agent"),
    "session_id": session_id
})
```

Configure Python's structlog:
```python
import structlog
import logging

structlog.configure(
    processors=[
        structlog.processors.TimeStamper(fmt="iso"),
        structlog.stdlib.add_log_level,
        structlog.processors.StackInfoRenderer(),
        structlog.processors.format_exc_info,
        structlog.processors.JSONRenderer()
    ],
    wrapper_class=structlog.stdlib.BoundLogger,
    logger_factory=structlog.PrintLoggerFactory(),
)

log = structlog.get_logger()
log.info("request.received", method="GET", path="/users", user_id="123")
# Output: {"timestamp": "2025-01-01T00:00:00Z", "level": "info", "event": "request.received", "method": "GET", "path": "/users", "user_id": "123"}
```

### Part 2 — Option A: Grafana Loki Stack (Recommended)
Loki is like Prometheus but for logs — lightweight, indexing only labels not content.

```yaml
# docker-compose.loki.yml
services:
  loki:
    image: grafana/loki:latest
    ports:
      - "3100:3100"
    volumes:
      - ./loki-config.yml:/etc/loki/config.yml
      - loki_data:/data/loki

  promtail:
    image: grafana/promtail:latest
    volumes:
      - /var/log:/var/log:ro
      - /var/lib/docker/containers:/var/lib/docker/containers:ro
      - ./promtail-config.yml:/etc/promtail/config.yml
    command: -config.file=/etc/promtail/config.yml

  grafana:  # already set up in A7.1
    environment:
      - GF_AUTH_ANONYMOUS_ENABLED=true  # for demo
```

```ruby
# logstash.conf
input {
  beats { port => 5044 }
}

filter {
  json { source => "message" }
  date {
    match => ["timestamp", "ISO8601"]
    target => "@timestamp"
  }
  mutate {
    remove_field => ["message", "agent", "ecs", "input"]
  }
}

output {
  elasticsearch {
    hosts => ["elasticsearch:9200"]
    index => "app-logs-%{+YYYY.MM.dd}"
  }
}
```

### Part 4 — LogQL Queries (Loki Query Language)
```logql
# All logs from a service
{service="api"}

# Error logs only
{service="api"} |= "ERROR"

# Parse JSON and filter by field
{service="api"} | json | status_code >= 500

# Rate of error logs per minute
rate({service="api"} |= "ERROR" [1m])

# Top endpoints by error count
sum by (path) (
  count_over_time({service="api"} | json | status_code >= 500 [1h])
)

# Find slow requests (latency > 1s)
{service="api"} | json | duration_ms > 1000
```

### Part 5 — Log-Based Alerting
```yaml
# grafana alert rule
# Alert when error rate > 5 errors per minute
expr: sum(rate({service="api"} |= "ERROR" [5m])) > 0.08
for: 2m
annotations:
  summary: "High error rate in API service"
  description: "{{ $value }} errors per second"
```

### Part 6 — Application Log Best Practices
Implement these in your app:
1. **Request ID**: generate a unique ID per request, log it in every log entry
2. **Correlation ID**: pass request ID through service calls (X-Request-ID header)
3. **Log levels**: DEBUG (dev only), INFO (normal), WARNING (unexpected), ERROR (failure), CRITICAL (immediate action)
4. **Sensitive data**: never log passwords, tokens, PII — use masking
5. **Context binding**: bind common fields (user_id, tenant_id) to the logger for a request

```python
# Bind context to logger for entire request lifecycle
log = structlog.get_logger().bind(
    request_id=request_id,
    user_id=user_id,
    tenant_id=tenant_id
)
# All subsequent log calls include these fields
log.info("processing.started")
log.info("processing.completed", items=5)
```
