## Task

### Part 1 — Script Basics and Best Practices
Every script you write must start with:
```bash
#!/usr/bin/env bash
set -euo pipefail  # exit on error, undefined vars, pipe failures
IFS=$'\n\t'        # safer word splitting

# Logging function
log() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] $*"; }
error() { log "ERROR: $*" >&2; exit 1; }
```

Explain in a comment what each `set` option does.

### Part 2 — Script 1: Disk Usage Reporter
Write `disk_report.sh`:
```bash
#!/usr/bin/env bash
set -euo pipefail

THRESHOLD=${1:-80}  # default 80%
OUTPUT_FILE="/tmp/disk_report_$(date +%Y%m%d).txt"

# Check all mounted filesystems
# For each filesystem over THRESHOLD%, log a warning
# Output: filesystem, size, used, available, use%, mountpoint
# Send the report to OUTPUT_FILE
# If any filesystem is over threshold, exit with code 1

# Usage: ./disk_report.sh 75
```

Requirements:
- Uses `df -h` to get disk info
- Parses percentage correctly (strips the `%`)
- Works on both Linux and macOS

### Part 3 — Script 2: Log Rotation
Write `rotate_logs.sh`:
```bash
#!/usr/bin/env bash
set -euo pipefail

LOG_DIR=${1:?Usage: $0 <log_dir> <max_age_days>}
MAX_AGE=${2:?}

# Find all .log files older than MAX_AGE days
# Compress them with gzip
# Delete compressed files older than 30 days
# Log every action taken
# Dry-run mode: if DRY_RUN=1, only print what would happen

# Usage: ./rotate_logs.sh /var/log/myapp 7
# Usage: DRY_RUN=1 ./rotate_logs.sh /var/log/myapp 7
```

### Part 4 — Script 3: Service Health Check
Write `health_check.sh`:
```bash
#!/usr/bin/env bash
set -euo pipefail

SERVICES=("nginx" "postgresql" "redis")
WEBHOOK_URL=${SLACK_WEBHOOK:-""}

check_service() {
    local service=$1
    if systemctl is-active --quiet "$service"; then
        echo "OK: $service is running"
        return 0
    else
        echo "FAIL: $service is NOT running"
        return 1
    fi
}

# For each service in SERVICES:
# - Check if it's running
# - If not, attempt to restart it once
# - Check again
# - If still down, send a Slack webhook notification (if SLACK_WEBHOOK is set)
# - Log all actions with timestamps
```

### Part 5 — Script 4: Backup Script
Write `backup.sh` with these features:
- Takes source directory and destination (local or S3 path) as arguments
- Creates a timestamped tar.gz archive
- Verifies the archive is not corrupted after creation (`tar -tzf`)
- Keeps only the last N backups (configurable, default 7)
- Supports both local backup and S3 upload (`aws s3 cp`)
- Sends exit code 0 on success, 1 on failure
