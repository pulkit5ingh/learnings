# P2.2 — Multi-Environment Deployment Pipeline

**Phase:** 2 — Version Control & CI/CD Basics  
**Topic:** Multi-environment CI/CD, GitFlow deployment  
**Type:** Project

---

## Goal
Build a deployment pipeline that manages multiple environments (dev, staging, production) with different rules, approval gates, and configurations for each.

---

## What to Build
A pipeline that enforces:
- `feature/*` branches → auto-deploy to dev
- `main` branch → auto-deploy to staging
- `release/v*` tags → require approval → deploy to production

```
feature/add-search ──────────→ dev.myapp.com (auto)
       ↓ (merge PR)
      main ────────────────→ staging.myapp.com (auto)
       ↓ (git tag v1.2.0)
   release/v1.2.0 ─[approve]─→ prod.myapp.com (manual gate)
```

## Environment Configuration

|--------|-----|---------|-----------|
| Replicas | 1 | 2 | 5 |
| Database | SQLite/dev | Postgres | Postgres RDS |
| Log level | DEBUG | INFO | WARNING |
| Feature flags | All on | Selective | Conservative |
| Resources | Minimal | Medium | Full |

```
config/

├── dev.env            # non-secret dev config
├── staging.env        # non-secret staging config

```

Secrets stored in GitHub Secrets, referenced as `${{ secrets.PROD_DB_URL }}`.

## Pipeline Design

### Feature Branch to Dev
```yaml
# .github/workflows/deploy-dev.yml
on:
  push:
    branches:
      - 'feature/**'
      - 'fix/**'

jobs:
  deploy-dev:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Deploy to dev environment
        env:
          APP_ENV: development
          DB_URL: ${{ secrets.DEV_DB_URL }}
          FEATURE_FLAGS: all
        run: |
          BRANCH_SLUG=$(echo "${{ github.ref_name }}" | tr '/' '-' | tr '[:upper:]' '[:lower:]')
          # Deploy with branch name in URL: alice-feature-search.dev.myapp.com
          ./scripts/deploy.sh dev $BRANCH_SLUG
      
      - name: Comment PR with dev URL
        uses: actions/github-script@v7
        with:
          script: |
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: '🚀 Deployed to https://dev.myapp.com — ready for review'
            })
```

### Production Release with Approval Gate
```yaml
# .github/workflows/deploy-production.yml
on:
  push:
    tags:
      - 'v[0-9]+.[0-9]+.[0-9]+'

jobs:
  deploy-production:
    runs-on: ubuntu-latest
    environment:
      name: production        # requires 2 reviewers in GitHub settings
      url: https://myapp.com
    steps:
      - name: Deploy to production
        run: ./scripts/deploy.sh production ${{ github.ref_name }}
      
      - name: Run production smoke tests
        run: pytest tests/smoke/ -m production

      - name: Notify success
        if: success()
        run: |
          curl -X POST ${{ secrets.SLACK_WEBHOOK }} \
            -d "{\"text\": \"✅ Production deployment ${{ github.ref_name }} successful\"}"
      
      - name: Rollback on failure
        if: failure()
        run: ./scripts/rollback.sh production
```

## Rollback Strategy
```bash
# scripts/rollback.sh
#!/usr/bin/env bash
set -euo pipefail

ENVIRONMENT=$1

# Get the last stable tag from git
LAST_STABLE=$(git tag --sort=-version:refname | head -2 | tail -1)

echo "Rolling back to $LAST_STABLE"
./scripts/deploy.sh "$ENVIRONMENT" "$LAST_STABLE"
```
