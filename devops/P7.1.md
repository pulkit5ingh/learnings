# P7.1 — Full Observability Stack for a Microservices Application

**Phase:** 7 — Monitoring, Logging & Observability  
**Topic:** Prometheus, Grafana, Loki, Jaeger, alerting  
**Type:** Project

---

## Goal
Deploy a production-grade observability stack for the microservices application from Phase 4 — complete with metrics, logs, traces, and alerts.

---

## What to Build
Deploy the full observability "holy trinity":
- **Metrics**: Prometheus + Grafana
- **Logs**: Loki + Grafana
- **Traces**: Tempo + Grafana

All accessible through a single Grafana UI.

## Architecture
```
Services → OpenTelemetry Collector → Prometheus (metrics)
                                   → Loki (logs)
                                   → Tempo (traces)
                                   
Grafana reads from all three
Alertmanager → PagerDuty/Slack
```

## Deployment in Kubernetes with kube-prometheus-stack

### Install with Helm
```bash
# Add Helm repositories
helm repo add prometheus-community https://prometheus-community.github.io/helm-charts
helm repo add grafana https://grafana.github.io/helm-charts
helm repo update

# Install kube-prometheus-stack (includes Prometheus, Grafana, Alertmanager, node-exporter)
helm install monitoring prometheus-community/kube-prometheus-stack \
    --namespace monitoring \
    --create-namespace \
    -f monitoring-values.yaml

# Install Loki stack
helm install loki grafana/loki-stack \
    --namespace monitoring \
    -f loki-values.yaml

# Install Tempo
helm install tempo grafana/tempo \
    --namespace monitoring
```

### monitoring-values.yaml
```yaml
prometheus:
  prometheusSpec:
    retention: 30d
    storageSpec:
      volumeClaimTemplate:
        spec:
          storageClassName: gp2
          accessModes: [ReadWriteOnce]
          resources:
            requests:
              storage: 50Gi
    additionalScrapeConfigs:
      - job_name: 'myapp-services'
        kubernetes_sd_configs:
          - role: pod
        relabel_configs:
          - source_labels: [__meta_kubernetes_pod_annotation_prometheus_io_scrape]
            action: keep
            regex: true

grafana:
  adminPassword: admin
  dashboardProviders:
    dashboardproviders.yaml:
      providers:
        - name: 'default'
          folder: ''
          type: file
          options:
            path: /var/lib/grafana/dashboards
  datasources:
    datasources.yaml:
      datasources:
        - name: Prometheus
          type: prometheus
          url: http://monitoring-kube-prometheus-prometheus:9090
          isDefault: true
        - name: Loki
          type: loki
          url: http://loki:3100
        - name: Tempo
          type: tempo
          url: http://tempo:3100

alertmanager:
  config:
    global:
      slack_api_url: 'https://hooks.slack.com/services/...'
    route:
      receiver: slack
      routes:
        - matchers: [severity="critical"]
          receiver: pagerduty
    receivers:
      - name: slack
        slack_configs:
          - channel: '#alerts'
      - name: pagerduty
        pagerduty_configs:
          - routing_key: 'xxx'
```

## Log-to-Trace Correlation
Enable jumping from a log entry to its trace:

In your application:
```python
# Include trace ID in every log message
from opentelemetry import trace

def get_log_context():
    span = trace.get_current_span()
    ctx = span.get_span_context()
    return {
        "trace_id": format(ctx.trace_id, '032x') if ctx.is_valid else None,
        "span_id": format(ctx.span_id, '016x') if ctx.is_valid else None
    }
```

In Grafana, configure Loki datasource with derived fields:
```yaml
# Loki datasource in Grafana provisioning
jsonData:
  derivedFields:
    - matcherRegex: '"trace_id":"(\w+)"'
      name: TraceID
      url: '$${__value.raw}'
      datasourceUid: tempo
```

Now clicking a trace_id in a log entry opens the trace in Tempo.

## Alerting Rules
```yaml
# k8s-alerts.yaml — add to Prometheus
groups:
  - name: kubernetes
    rules:
      - alert: PodCrashLooping
        expr: rate(kube_pod_container_status_restarts_total[15m]) > 0
        for: 5m
        labels:
          severity: warning

      - alert: PodNotReady
        expr: kube_pod_status_ready{condition="false"} == 1
        for: 5m
        labels:
          severity: critical

  - name: services
    rules:
      - alert: ServiceHighErrorRate
        expr: |
          sum by (service) (rate(http_requests_total{status=~"5.."}[5m]))
          / sum by (service) (rate(http_requests_total[5m])) > 0.05
        for: 2m
        labels:
          severity: critical
```
