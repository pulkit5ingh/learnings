## Goal
Implement streaming SSR to send HTML progressively, avoid data-fetching waterfalls with parallel Suspense boundaries, and understand Partial Prerendering.

## Background

Traditional SSR renders the entire page on the server before sending any HTML to the browser. If one component takes 3 seconds to fetch, the user stares at a blank screen for 3 seconds. **Streaming SSR** solves this by sending HTML in chunks — the shell (layout, nav, skeleton) is sent immediately, and slow sections stream in as they complete.

React Suspense is the mechanism: `<Suspense fallback={<Skeleton />}>` creates a streaming boundary. The fallback renders immediately in the initial HTML chunk. When the async work inside completes, React streams the real content and React's client-side runtime swaps it in.

**Waterfall problem**: fetching A, then B, then C sequentially takes A + B + C time. Using `Promise.all` or independent Suspense boundaries makes them run in parallel: time = max(A, B, C). In Server Components, this means calling all async functions at the top of the component with `Promise.all`, or wrapping each data-dependent section in its own `<Suspense>` boundary.

**Partial Prerendering (PPR)** is an experimental Next.js 14 feature that combines static and dynamic rendering at the route level: the static shell is pre-rendered at build time (served from CDN), while dynamic holes stream in per-request. This gives you CDN-speed shell + fresh dynamic content.

## Tasks

### 1. Basic streaming with Suspense
```tsx
// src/app/streaming/page.tsx
import { Suspense } from "react";

// Simulate slow data fetches
async function SlowComponent({ delay, label }: { delay: number; label: string }) {
  await new Promise((r) => setTimeout(r, delay));
  return (
    <div className="p-4 border rounded-lg bg-white">
      <p className="font-medium">{label}</p>
      <p className="text-sm text-gray-500">Loaded after {delay}ms</p>
    </div>
  );
}

function ComponentSkeleton() {
  return (
    <div className="p-4 border rounded-lg animate-pulse">
      <div className="h-4 w-32 bg-gray-200 rounded mb-2" />
      <div className="h-3 w-24 bg-gray-200 rounded" />
    </div>
  );
}

export default function StreamingPage() {
  return (
    <div className="p-6 space-y-4">
      <h1 className="text-2xl font-bold">Streaming Demo</h1>
      <p className="text-gray-500">
        This page streams 3 components independently. Watch them appear one by one.
      </p>

      {/* These all start fetching simultaneously but resolve at different times */}
      <div className="grid grid-cols-3 gap-4">
        <Suspense fallback={<ComponentSkeleton />}>
          <SlowComponent delay={500}  label="Fast component" />
        </Suspense>
        <Suspense fallback={<ComponentSkeleton />}>
          <SlowComponent delay={1500} label="Medium component" />
        </Suspense>
        <Suspense fallback={<ComponentSkeleton />}>
          <SlowComponent delay={3000} label="Slow component" />
        </Suspense>
      </div>
    </div>
  );
}
```

### 2. Avoiding waterfalls — sequential vs parallel
```tsx
// WATERFALL — B cannot start until A finishes
async function WaterfallPage() {
  const user = await getUser(1);                // waits 500ms
  const posts = await getUserPosts(user.id);   // only starts AFTER user loads — waits another 800ms
  // Total: 1300ms
  return <div>{/* render */}</div>;
}

// PARALLEL — A and B start at the same time
async function ParallelPage() {
  const [user, recentPosts] = await Promise.all([
    getUser(1),          // 500ms
    getRecentPosts(),    // 800ms (these run simultaneously)
  ]);
  // Total: 800ms (the longest one)
  return <div>{/* render */}</div>;
}
```

### 3. Parallel Suspense boundaries (best pattern)
```tsx
// src/app/streaming/dashboard/page.tsx
import { Suspense } from "react";

// These async Server Components fetch independently
async function UserGreeting() {
  const user = await getUser(1); // 300ms
  return <p className="font-medium">Hello, {user.name}!</p>;
}

async function NotificationsCount() {
  const count = await getNotificationCount(1); // 800ms
  return (
    <span className="px-2 py-0.5 bg-red-100 text-red-700 rounded-full text-sm">
      {count} unread
    </span>
  );
}

async function RecentActivity() {
  const activity = await getActivity(1); // 1200ms
  return (
    <ul className="space-y-2">
      {activity.map((a: any) => (
        <li key={a.id} className="text-sm text-gray-600">{a.description}</li>
      ))}
    </ul>
  );
}

export default function DashboardPage() {
  return (
    <div className="p-6 space-y-6">
      {/* Header: shows user name as soon as it loads (300ms) */}
      <div className="flex items-center gap-3">
        <Suspense fallback={<div className="h-6 w-32 bg-gray-200 rounded animate-pulse" />}>
          <UserGreeting />
        </Suspense>
        <Suspense fallback={<div className="h-6 w-12 bg-gray-200 rounded animate-pulse" />}>
          <NotificationsCount />
        </Suspense>
      </div>

      {/* Activity: streams in later (1200ms) — user can interact with header immediately */}
      <div>
        <h2 className="text-lg font-semibold mb-3">Recent Activity</h2>
        <Suspense fallback={
          <div className="space-y-2">
            {Array.from({ length: 4 }).map((_, i) => (
              <div key={i} className="h-4 bg-gray-200 rounded animate-pulse" />
            ))}
          </div>
        }>
          <RecentActivity />
        </Suspense>
      </div>
    </div>
  );
}
```

### 5. Partial Prerendering (PPR) — experimental
```js
// next.config.js
const nextConfig = {
  experimental: {
    ppr: true, // enable Partial Prerendering
  },
};
```
```tsx
// src/app/ppr-demo/page.tsx
import { Suspense } from "react";
import { unstable_noStore as noStore } from "next/cache";

// Static part — pre-rendered at build time, served from CDN
function StaticShell() {
  return (
    <div className="p-6">
      <h1 className="text-2xl font-bold">Product Page</h1>
      <p className="text-gray-500">Static content — blazing fast from CDN</p>
    </div>
  );
}

// Dynamic part — renders per-request
async function DynamicPersonalization() {
  noStore(); // opt out of caching
  const userId = "123"; // In real app: from cookies
  const recommendations = await getRecommendations(userId);

  return (
    <div className="mt-4 p-4 bg-blue-50 rounded">
      <p className="font-medium">Recommended for you:</p>
      <ul className="mt-2 space-y-1 text-sm">
        {recommendations.map((r: any) => (
          <li key={r.id}>{r.name}</li>
        ))}
      </ul>
    </div>
  );
}

export default function PprDemoPage() {
  return (
    <div>
      {/* This renders at BUILD TIME */}
      <StaticShell />

      {/* This renders at REQUEST TIME — streams in */}
      <Suspense fallback={<div className="mt-4 h-24 bg-gray-100 rounded animate-pulse" />}>
        <DynamicPersonalization />
      </Suspense>
    </div>
  );
}
```
