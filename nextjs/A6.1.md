## Goal
Replace all `<img>` tags with `next/image` and understand every important prop for performance and Core Web Vitals.

## Background

`<img>` elements are the most common cause of poor LCP (Largest Contentful Paint) and CLS (Cumulative Layout Shift) scores. `next/image` solves both:

- **CLS**: requires `width` and `height` (or `fill` mode) so the browser reserves space before the image loads
- **LCP**: the `priority` prop removes lazy loading for above-the-fold images and adds a `<link rel="preload">` tag
- **Format**: automatically serves WebP/AVIF to supporting browsers (50–80% smaller than JPEG/PNG)
- **Sizing**: generates multiple sizes and serves the right one based on the viewport via `srcSet`
- **Lazy loading**: all images without `priority` are lazy-loaded by default

The `sizes` prop is critical for responsive images. It tells the browser what percentage of the viewport the image will occupy at each breakpoint, enabling it to pick the correct `srcSet` entry and avoid loading an oversized image.

**Remote patterns** in `next.config.js` whitelist external domains that are allowed to serve images through the Next.js image optimizer.

### 3. Hero image with priority (above the fold)
```tsx
// src/components/Hero.tsx
import Image from "next/image";

export function Hero() {
  return (
    <section className="relative h-[600px] w-full">
      {/* fill: image fills its parent container */}
      <Image
        src="https://images.unsplash.com/photo-1506905925346-21bda4d32df4?w=2000"
        alt="Mountain landscape hero"
        fill
        priority            // ← no lazy loading; adds preload link
        sizes="100vw"       // image always fills full viewport width
        className="object-cover"
        quality={85}        // default is 75; 85 for hero images
      />
      {/* Content overlaid on image */}
      <div className="relative z-10 flex items-center justify-center h-full">
        <h1 className="text-5xl font-bold text-white drop-shadow-lg">
          Hello World
        </h1>
      </div>
    </section>
  );
}
```

### 6. Avatar with fallback
```tsx
// src/components/Avatar.tsx
"use client";

import Image from "next/image";
import { useState } from "react";

type AvatarProps = {
  src?: string | null;
  name: string;
  size?: number;
};

export function Avatar({ src, name, size = 40 }: AvatarProps) {
  const [error, setError] = useState(false);
  const initials = name
    .split(" ")
    .map((n) => n[0])
    .slice(0, 2)
    .join("")
    .toUpperCase();

  if (!src || error) {
    return (
      <div
        className="rounded-full bg-blue-500 text-white flex items-center justify-center font-semibold"
        style={{ width: size, height: size, fontSize: size * 0.4 }}
      >
        {initials}
      </div>
    );
  }

  return (
    <Image
      src={src}
      alt={name}
      width={size}
      height={size}
      className="rounded-full object-cover"
      onError={() => setError(true)}
    />
  );
}
```

### 7. Image with custom loader (e.g., Cloudinary)

// src/lib/cloudinaryLoader.ts
import type { ImageLoaderProps } from "next/image";

export function cloudinaryLoader({ src, width, quality }: ImageLoaderProps) {
  return `https://res.cloudinary.com/demo/image/upload/w_${width},q_${quality ?? 75}/v1/${src}`;
}
