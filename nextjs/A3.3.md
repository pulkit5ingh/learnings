## Goal
Use CSS Modules for component-scoped styles, CSS custom properties for theming, and next/font for optimized font loading.

## Background

**CSS Modules** automatically scope class names to their component by transforming `.Card` into something like `.Card_abc123`. This eliminates naming collisions without requiring a naming convention. You import the module as an object and use `styles.Card` in your JSX.

CSS Modules support `:global()` for styles that should not be scoped (e.g., resetting third-party styles), the `composes` keyword to reuse classes from other modules, and CSS custom properties for theming.

**CSS custom properties** (also called CSS variables) are the right tool for theming. Setting `--color-brand: #2563eb` on `:root` and overriding it under `.dark` enables theme switching without JavaScript state at the CSS level.

**next/font** is a Next.js built-in module that optimises font loading: it downloads fonts at build time, generates size-adjusted fallbacks to eliminate layout shift, and self-hosts them (no network requests to Google Fonts at runtime, preserving user privacy). You can use Google Fonts or load local font files.

## Tasks

### 1. Google Fonts with next/font
```tsx
// src/app/layout.tsx
import { Inter, Fira_Code } from "next/font/google";
import "./globals.css";

const inter = Inter({
  subsets: ["latin"],
  variable: "--font-inter",
  display: "swap",
});

const firaCode = Fira_Code({
  subsets: ["latin"],
  variable: "--font-fira-code",
  display: "swap",
});

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" className={`${inter.variable} ${firaCode.variable}`}>
      <body className="font-sans antialiased">{children}</body>
    </html>
  );
}
```
```ts
// tailwind.config.ts — use CSS variables from next/font
theme: {
  extend: {
    fontFamily: {
      sans: ["var(--font-inter)", "sans-serif"],
      mono: ["var(--font-fira-code)", "monospace"],
    },
  },
},
```

### 4. Basic CSS Module
```css
/* src/components/Card/Card.module.css */
.card {
  background-color: hsl(var(--card));
  color: hsl(var(--card-foreground));
  border: 1px solid hsl(var(--border));
  border-radius: var(--radius);
  padding: 1.5rem;
  transition: box-shadow 0.2s ease;
}

.card:hover {
  box-shadow: 0 4px 6px -1px rgb(0 0 0 / 0.1);
}

.cardHeader {
  margin-bottom: 1rem;
}

.cardTitle {
  font-size: 1.125rem;
  font-weight: 600;
  line-height: 1.4;
}

.cardBody {
  font-size: 0.875rem;
  color: hsl(var(--muted-foreground));
  line-height: 1.6;
}

/* :global overrides — useful for third-party content */
.card :global(.prose) {
  max-width: none;
}
```
```tsx
// src/components/Card/Card.tsx
import styles from "./Card.module.css";
import { cn } from "@/lib/utils";

type CardProps = {
  title: string;
  children: React.ReactNode;
  className?: string;
};

export function Card({ title, children, className }: CardProps) {
  return (
    <div className={cn(styles.card, className)}>
      <div className={styles.cardHeader}>
        <h3 className={styles.cardTitle}>{title}</h3>
      </div>
      <div className={styles.cardBody}>{children}</div>
    </div>
  );
}
```

### 5. composes keyword
```css
/* src/components/Card/Card.module.css (additional) */
.featuredCard {
  composes: card; /* inherit all .card styles */
  border-color: hsl(var(--primary));
  border-width: 2px;
}
```
```tsx
// Usage
<div className={styles.featuredCard}>Featured</div>
```

### 6. Mixing CSS Modules with Tailwind
```tsx
// src/components/HeroSection/HeroSection.tsx
import styles from "./HeroSection.module.css";
import { cn } from "@/lib/utils";

export function HeroSection() {
  return (
    // Mix CSS Module class with Tailwind utilities
    <section className={cn(styles.hero, "px-4 py-24")}>
      <div className="max-w-4xl mx-auto text-center">

          Build faster

          A hero section that uses both CSS Modules and Tailwind.

}
```
