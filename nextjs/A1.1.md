**Phase:** 1 — Next.js & App Router Fundamentals  
**Topic:** app/ directory conventions — layout.tsx, page.tsx, loading.tsx, error.tsx, not-found.tsx, route segments  
**Type:** Assignment

## Goal
Understand every special file Next.js recognises inside the `app/` directory and build a small app that uses all of them.

## Background

The App Router is a file-system-based router built on top of React Server Components. Every folder inside `app/` that contains a `page.tsx` becomes a publicly accessible URL. Next.js reserves several filenames that have special meaning — they are called **co-located special files**.

`layout.tsx` wraps one or more pages. Unlike pages, layouts persist across navigations: when a user moves from `/about` to `/contact`, the root layout does not re-render. This makes it the right place for navigation bars, fonts, and theme providers. You can nest layouts: `app/dashboard/layout.tsx` wraps only dashboard routes.

`loading.tsx` is an automatic Suspense boundary. Next.js wraps the `page.tsx` in a `<Suspense>` with your `loading.tsx` as the fallback. The moment the page's async work is in flight, `loading.tsx` renders instantly. This means the shell of your UI (nav, sidebar) is interactive while content loads.

`error.tsx` must be a Client Component (`"use client"`). It receives `error` and `reset` props. It activates when an uncaught error is thrown inside any Server or Client Component in the segment. `not-found.tsx` activates when `notFound()` is called from `page.tsx` or any component it renders.

Route segments map 1-to-1 with URL segments. The folder `app/blog/[slug]/` creates the dynamic segment `/blog/:slug`. Folders wrapped in parentheses like `(marketing)` are **route groups** — they organise files without affecting the URL.

pnpm create next-app@latest a1-app-router \

### 2. Examine the default structure
Open `src/app/` and note what Next.js generated:
```
src/app/
  layout.tsx      ← root layout (wraps every page)
  page.tsx        ← the / route
  globals.css
  favicon.ico
```

### 3. Create a root layout with navigation
Replace `src/app/layout.tsx`:
```tsx
// src/app/layout.tsx
import type { Metadata } from "next";
import { Inter } from "next/font/google";
import "./globals.css";
import Link from "next/link";

const inter = Inter({ subsets: ["latin"] });

export const metadata: Metadata = {
  title: { template: "%s | My App", default: "My App" },
  description: "Learning Next.js App Router",
};

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body className={inter.className}>
        <nav className="flex gap-4 p-4 border-b">
          <Link href="/" className="font-semibold">Home</Link>
          <Link href="/about">About</Link>
          <Link href="/blog">Blog</Link>
          <Link href="/dashboard">Dashboard</Link>
        </nav>
        <main className="p-6">{children}</main>
      </body>
    </html>
  );
}
```

### 8. Add an error boundary for dashboard
```tsx
// src/app/dashboard/error.tsx
"use client";

export default function DashboardError({
  error,
  reset,
}: {
  error: Error & { digest?: string };
  reset: () => void;
}) {
  return (
    <div className="p-4 border border-red-300 rounded bg-red-50">
      <h2 className="font-bold text-red-700">Something went wrong!</h2>
      <p className="text-sm text-red-600 mt-1">{error.message}</p>
      <button
        onClick={reset}
        className="mt-3 px-4 py-1.5 bg-red-600 text-white rounded text-sm"
      >
        Try again
      </button>
    </div>
  );
}
```

### 10. Test notFound() manually
```tsx
// src/app/blog/page.tsx
import { notFound } from "next/navigation";

export default function BlogPage() {
  const posts: string[] = []; // empty — triggers notFound
  if (posts.length === 0) notFound();
  return <div>Blog</div>;
}
```
Navigate to `/blog` — you should see the not-found page.
