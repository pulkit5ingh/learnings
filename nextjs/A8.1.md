# A8.1 — Vercel Deployment

**Phase:** 8 — Deployment & CI/CD  
**Topic:** Push-to-deploy, environment variables, preview deployments, domains, Edge Network  
**Type:** Assignment

---

## Goal
Deploy a Next.js app to Vercel with production and preview environments, custom domains, and correct environment variable management.

## Background

Vercel is the company behind Next.js and provides the most optimised deployment platform for it. Their global Edge Network caches static assets, serves static pages from CDN nodes close to the user, and routes server-rendered pages to the nearest compute region.

**Deployment flow**: push to GitHub → Vercel webhook triggers → build runs → deployed. Every push to `main` deploys to production. Every pull request gets a preview deployment at a unique URL.

**Environment variables** in Vercel have three scopes: Development (injected via `vercel env pull`), Preview (all non-production branches), and Production. Variables prefixed with `NEXT_PUBLIC_` are embedded into the client bundle at build time. Server-only variables (without the prefix) are available only in Server Components, Route Handlers, and middleware.

**Vercel CLI** lets you manage deployments, environment variables, and domains from the terminal.

## Tasks

### 1. Connect repo to Vercel
```bash
# Install Vercel CLI
pnpm add -g vercel

# Authenticate
vercel login

# Link existing project (run from project root)
vercel link
# Follow prompts: create new project, choose org, confirm settings

# Or deploy immediately
vercel --prod
```

### 2. Configure project via vercel.json
```json
// vercel.json (project root)
{
  "buildCommand": "pnpm build",
  "installCommand": "pnpm install",
  "outputDirectory": ".next",
  "framework": "nextjs",
  "regions": ["iad1"],
  "headers": [
    {
      "source": "/api/(.*)",
      "headers": [
        { "key": "Cache-Control", "value": "no-store" }
      ]
    },
    {
      "source": "/(.*)\\.(?:js|css|png|jpg|svg|ico|woff2)",
      "headers": [
        { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
      ]
    }
  ],
  "redirects": [
    {
      "source": "/old-page",
      "destination": "/new-page",
      "permanent": true
    }
  ]
}
```

### 3. Environment variables via CLI
```bash
# Add production env var
vercel env add NEXT_PUBLIC_APP_URL production
# Enter value: https://myapp.com

# Add to all environments
vercel env add DATABASE_URL
# Choose: Production, Preview, Development

# List all env vars
vercel env ls

# Pull env vars for local development
vercel env pull .env.local
```

### 4. Environment variable file structure
```bash
# .env.local (never commit — local dev only)
DATABASE_URL=postgresql://localhost:5432/mydb
AUTH_SECRET=local-dev-secret-change-this

# .env (committed — safe defaults for all environments)
NEXT_PUBLIC_APP_NAME=My App
NEXT_PUBLIC_APP_URL=http://localhost:3000

# .env.production (committed — production-specific public vars)
NEXT_PUBLIC_APP_URL=https://myapp.com
```
```bash
# .gitignore (make sure these are ignored)
.env.local
.env.*.local
```

### 5. Access env vars in code
```ts
// Server-only env var (no NEXT_PUBLIC_ prefix)
// src/lib/db.ts
const databaseUrl = process.env.DATABASE_URL; // Only available server-side

// Client-accessible env var (NEXT_PUBLIC_ prefix)
// Available in Client Components, Server Components, Route Handlers
const appName = process.env.NEXT_PUBLIC_APP_NAME;

// Type-safe env var validation with Zod
// src/env.ts
import { z } from "zod";

const envSchema = z.object({
  DATABASE_URL: z.string().url(),
  AUTH_SECRET: z.string().min(32),
  NEXT_PUBLIC_APP_URL: z.string().url(),
  NEXT_PUBLIC_APP_NAME: z.string(),
});

// Throws at startup if env vars are missing or invalid
export const env = envSchema.parse(process.env);
```

### 6. Preview deployments
```yaml
# Every PR automatically gets a preview URL:
# https://my-app-git-feature-my-team.vercel.app

# To deploy a specific branch manually:
vercel --target preview

# To deploy to production manually:
vercel --prod
```

### 7. Custom domain setup
```bash
# Via CLI
vercel domains add myapp.com
vercel domains add www.myapp.com

# Then add DNS records at your registrar:
# A record:    @    →  76.76.21.21
# CNAME record: www  →  cname.vercel-dns.com
```

### 8. Vercel Analytics and Speed Insights
```bash
pnpm add @vercel/analytics @vercel/speed-insights
```
```tsx
// src/app/layout.tsx
import { Analytics } from "@vercel/analytics/react";
import { SpeedInsights } from "@vercel/speed-insights/next";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        {children}
        <Analytics />
        <SpeedInsights />
      </body>
    </html>
  );
}
```

### 9. Check deployment status
```bash
# List recent deployments
vercel ls

# Inspect a specific deployment
vercel inspect <deployment-url>

# View build logs
vercel logs <deployment-url>

# Rollback to previous deployment
vercel rollback
```

### 10. Preview deployment comment on PR (GitHub App)
Install the Vercel GitHub App at `github.com/apps/vercel`. After installation, every PR automatically gets a comment with the preview URL. Configure in Vercel project settings:
- Team Settings → Git Integration → GitHub → Connect
- Enable "Preview Comments"
