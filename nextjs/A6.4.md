**Topic:** @next/bundle-analyzer, identify large dependencies, tree shaking, font optimization, third-party script optimization  
**Type:** Assignment

---

## Goal
Analyse and reduce your JavaScript bundle size using the bundle analyzer, tree shaking, and script deferring.

## Background

The JavaScript bundle is the main bottleneck for TBT (Total Blocking Time) and TTI (Time to Interactive). A bundle analysis shows exactly what's inside your chunks, sized proportionally. It's the starting point for every performance optimization.

**Tree shaking** removes unused code from a bundle at build time. It only works with ES Modules (import/export) — not CommonJS (require). Most modern packages support it, but you have to import correctly: `import { debounce } from "lodash-es"` (tree-shakeable) vs `import _ from "lodash"` (full lodash bundle).

**next/script** controls when third-party scripts load. The `strategy` prop options:
- `beforeInteractive` — before hydration (only for critical scripts)
- `afterInteractive` (default) — after hydration
- `lazyOnload` — only after the page is fully loaded (analytics, chat widgets)
- `worker` — run in a Web Worker via Partytown (experimental)

## Tasks

### 1. Install bundle analyzer
```bash
pnpm add -D @next/bundle-analyzer
```

### 2. Configure in next.config.js
```js
// next.config.js
const withBundleAnalyzer = require("@next/bundle-analyzer")({
  enabled: process.env.ANALYZE === "true",
  openAnalyzer: true,
});

/** @type {import('next').NextConfig} */
const nextConfig = {
  // ... your config
};

module.exports = withBundleAnalyzer(nextConfig);
```

### 3. Add analyze script to package.json
```json
{
  "scripts": {
    "analyze": "ANALYZE=true pnpm build"
  }
}
```
```bash
pnpm analyze
# Opens two browser windows:
# - client bundle (what ships to the browser)
# - server bundle (what runs on the server)
```

### 4. Read the treemap
When the analyzer opens:
- Each rectangle = one module
- Size = how much it contributes to the bundle
- Hover to see the exact size and path
- Look for: `moment` (large), `lodash` (large if not using lodash-es), `@fortawesome/fontawesome-free` (huge), `draft-js`, `quill`, or any library that ends in `-full` or `-all`

### 5. Fix — replace moment.js with date-fns
```bash
# moment.js: 67KB minified+gzipped
# date-fns:  only what you import

# Remove moment
pnpm remove moment

# Add date-fns
pnpm add date-fns
```
```ts
// BEFORE (imports all of moment — 67KB)
import moment from "moment";
const formatted = moment(date).format("MMM DD, YYYY");

// AFTER (imports only format and parseISO — ~2KB)
import { format, parseISO } from "date-fns";
const formatted = format(parseISO(date), "MMM dd, yyyy");
```

### 6. Fix — named imports from lodash-es
```bash
pnpm remove lodash @types/lodash
pnpm add lodash-es @types/lodash-es
```
```ts
// BEFORE — entire lodash (73KB)
import _ from "lodash";
const sorted = _.sortBy(items, "name");

// AFTER — only sortBy from lodash-es (~2KB after tree shaking)
import { sortBy } from "lodash-es";
const sorted = sortBy(items, "name");
```

### 7. Fix — dynamic import for large optional libraries
```tsx
// BEFORE — syntax highlighter added to every page (220KB)
import { Prism as SyntaxHighlighter } from "react-syntax-highlighter";

// AFTER — loaded only when a code block is actually rendered
import dynamic from "next/dynamic";

const SyntaxHighlighter = dynamic(
  () => import("react-syntax-highlighter").then((m) => m.Prism),
  { ssr: false }
);
```

### 8. next/script for third-party scripts
```tsx
// src/app/layout.tsx
import Script from "next/script";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        {children}

        {/* Google Analytics — load after page is interactive */}
        <Script
          src={`https://www.googletagmanager.com/gtag/js?id=${process.env.NEXT_PUBLIC_GA_ID}`}
          strategy="afterInteractive"
        />
        <Script id="google-analytics" strategy="afterInteractive">
          {`
            window.dataLayer = window.dataLayer || [];
            function gtag(){dataLayer.push(arguments);}
            gtag('js', new Date());
            gtag('config', '${process.env.NEXT_PUBLIC_GA_ID}');
          `}
        </Script>

        {/* Intercom chat — only after page fully loads */}
        <Script id="intercom" strategy="lazyOnload">
          {`
            window.intercomSettings = { app_id: "${process.env.NEXT_PUBLIC_INTERCOM_APP_ID}" };
          `}
        </Script>
      </body>
    </html>
  );
}
```

### 9. Font optimization — subsetting
```tsx
// BEFORE — downloads all glyphs (can be 200KB+)
const font = new Inter();

// AFTER — only download Latin subset (~30KB)
const inter = Inter({
  subsets: ["latin"],           // only latin characters
  weight: ["400", "500", "700"], // only these weights
  display: "swap",
  preload: true,
});
```

### 10. Bundle size limits with bundlesize
```json
// package.json
{
  "bundlesize": [
    {
      "path": ".next/static/chunks/pages/*.js",
      "maxSize": "150 kB"
    },
    {
      "path": ".next/static/chunks/main*.js",
      "maxSize": "100 kB"
    }
  ],
  "scripts": {
    "bundlesize": "bundlesize"
  }
}
```
