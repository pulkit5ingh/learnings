# A1.2 — Server vs Client Components

**Phase:** 1 — Next.js & App Router Fundamentals  
**Topic:** "use client", when to use each, serialization rules, passing Server Component as prop to Client  
**Type:** Assignment

---

## Goal
Understand the boundary between Server and Client Components, why it exists, and how to compose them correctly.

## Background

In Next.js App Router, **all components are Server Components by default**. Server Components render on the server only. They can be `async`, can `await` database queries or `fetch()` calls, and their source code never ships to the browser. This dramatically reduces the JavaScript bundle.

Adding `"use client"` at the top of a file marks it as a **Client Component**. Client Components render on the server first (for SSR/hydration) and then again on the client. They have access to browser APIs, React hooks (`useState`, `useEffect`, etc.), and event handlers. The `"use client"` directive creates a **boundary** — everything imported by a Client Component that doesn't itself have `"use client"` is treated as a Client Component too. This is why the directive is contagious downward.

**Serialization rule:** Props that cross the Server → Client boundary must be serializable. You cannot pass functions, class instances, or `Date` objects (unless you serialize them to strings first). Strings, numbers, plain objects, arrays, and `null/undefined` are all safe.

The most powerful pattern is **passing Server Components as `children` props to Client Components**. A Client Component that accepts `children` does not "pull" those children into the client bundle — the Server Component stays on the server. This lets you wrap a Client Component (e.g., a context provider) around Server Components without losing their server nature.

## Tasks

### 1. Confirm a Server Component is async
```tsx
// src/app/server-demo/page.tsx
// No "use client" — this is a Server Component
async function getUser() {
  // Simulated DB call
  await new Promise((r) => setTimeout(r, 100));
  return { name: "Alice", role: "admin" };
}

export default async function ServerDemoPage() {
  const user = await getUser(); // ✅ async/await works in Server Components

  return (
    <div>
      <h1 className="text-2xl font-bold">Server Component Demo</h1>
      <p>Hello, {user.name} ({user.role})</p>
      {/* ✅ This data never exposes its fetch logic to the browser */}
    </div>
  );
}
```

### 2. Create a Client Component with hooks
```tsx
// src/components/Counter.tsx
"use client";

import { useState } from "react";

export function Counter({ initialCount = 0 }: { initialCount?: number }) {
  const [count, setCount] = useState(initialCount);

  return (
    <div className="flex items-center gap-4">
      <button
        onClick={() => setCount((c) => c - 1)}
        className="px-3 py-1 bg-gray-200 rounded"
      >
        −
      </button>
      <span className="text-xl font-mono w-10 text-center">{count}</span>
      <button
        onClick={() => setCount((c) => c + 1)}
        className="px-3 py-1 bg-gray-200 rounded"
      >
        +
      </button>
    </div>
  );
}
```

### 3. Use Client Component from a Server Component
```tsx
// src/app/server-demo/page.tsx (updated)
import { Counter } from "@/components/Counter";

export default async function ServerDemoPage() {
  const user = await getUser();

  return (
    <div className="space-y-4">
      <h1 className="text-2xl font-bold">Welcome, {user.name}</h1>
      {/* Server Component renders user data */}
      {/* Client Component handles interactivity */}
      <Counter initialCount={5} />
      {/* ✅ initialCount (number) is serializable — this is valid */}
    </div>
  );
}
```

### 4. Demonstrate the serialization constraint
```tsx
// ❌ This will cause an error — functions are not serializable
// src/app/bad-example/page.tsx
import { SomeClientComponent } from "@/components/SomeClientComponent";

export default function BadPage() {
  const handler = () => console.log("clicked");
  // return <SomeClientComponent onClick={handler} />; // ← ERROR at runtime
  return <p>Functions cannot be passed as props across the server/client boundary</p>;
}
```

```tsx
// ✅ Fix: move the handler into the Client Component itself
// src/components/SomeClientComponent.tsx
"use client";
export function SomeClientComponent() {
  const handler = () => console.log("clicked");
  return <button onClick={handler}>Click me</button>;
}
```

### 5. The children pattern — wrap a Client Provider around Server Components
```tsx
// src/components/ThemeProvider.tsx
"use client";

import { createContext, useContext, useState } from "react";

type Theme = "light" | "dark";
const ThemeContext = createContext<{ theme: Theme; toggle: () => void }>({
  theme: "light",
  toggle: () => {},
});

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setTheme] = useState<Theme>("light");
  const toggle = () => setTheme((t) => (t === "light" ? "dark" : "light"));

  return (
    <ThemeContext.Provider value={{ theme, toggle }}>
      <div data-theme={theme} className={theme === "dark" ? "bg-gray-900 text-white min-h-screen" : "min-h-screen"}>
        {children}
      </div>
    </ThemeContext.Provider>
  );
}

export const useTheme = () => useContext(ThemeContext);
```

```tsx
// src/app/layout.tsx (updated to wrap with ThemeProvider)
import { ThemeProvider } from "@/components/ThemeProvider";

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en">
      <body>
        {/* ThemeProvider is a Client Component */}
        {/* children (pages) remain Server Components */}
        <ThemeProvider>{children}</ThemeProvider>
      </body>
    </html>
  );
}
```

### 6. Add a theme toggle button (Client Component)
```tsx
// src/components/ThemeToggle.tsx
"use client";

import { useTheme } from "./ThemeProvider";

export function ThemeToggle() {
  const { theme, toggle } = useTheme();
  return (
    <button
      onClick={toggle}
      className="px-3 py-1.5 border rounded text-sm"
    >
      {theme === "light" ? "🌙 Dark" : "☀️ Light"}
    </button>
  );
}
```

### 8. Summary decision tree (add to a notes file)
```
Is this component interactive? (onClick, onChange, useState, useEffect)
  YES → "use client"
  NO  → Does it use browser APIs? (window, document, localStorage)
    YES → "use client"
    NO  → Does it fetch data or access env vars / server secrets?
      YES → Leave as Server Component (no directive needed)
      NO  → Either works; prefer Server Component
```
