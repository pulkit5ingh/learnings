## Goal
Build a small app that demonstrates every routing strategy available in the App Router.

## Background

The App Router maps filesystem folders to URL segments. Understanding the naming conventions unlocks every URL shape you will ever need.

**Dynamic segments** use square brackets: `[id]`. The segment value is available as `params.id` in the page. Multiple dynamic segments can exist in a single path: `app/[locale]/blog/[slug]/page.tsx` matches `/en/blog/hello-world`.

**Catch-all segments** use `[...slug]`. They match one or more segments: `app/docs/[...slug]/page.tsx` matches `/docs/intro`, `/docs/getting-started/install`, and so on. `slug` is an array of strings. **Optional catch-all** `[[...slug]]` also matches the root: `/docs`.

**Parallel routes** use the `@folder` convention. A layout can render multiple named "slots" at the same time — useful for dashboards where different sections load independently. Each slot gets its own `loading.tsx`, `error.tsx`, and can have its own sub-routes.

**Intercepting routes** use a `(.)` prefix to "intercept" a route and show it inside a different layout context. The classic use case is a photo feed: clicking a photo shows it in a modal (intercepted route), but navigating directly to `/photos/1` shows the full-page view. The convention: `(.)` intercepts a sibling, `(..)` intercepts a parent, `(...)` intercepts from the root.

### 2. Dynamic route — `/products/[id]`
```bash
mkdir -p src/app/products/\[id\]
```
```tsx
// src/app/products/[id]/page.tsx
type Props = {
  params: { id: string };
};

export default function ProductPage({ params }: Props) {
  return (
    <div>
      <h1 className="text-2xl font-bold">Product {params.id}</h1>
      <p className="text-gray-600">Showing details for product ID: {params.id}</p>
    </div>
  );
}
```
```tsx
// src/app/products/page.tsx
import Link from "next/link";

const PRODUCTS = [
  { id: "1", name: "Laptop" },
  { id: "2", name: "Keyboard" },
  { id: "3", name: "Monitor" },
];

export default function ProductsPage() {
  return (
    <div>
      <h1 className="text-2xl font-bold mb-4">Products</h1>
      <ul className="space-y-2">
        {PRODUCTS.map((p) => (
          <li key={p.id}>
            <Link href={`/products/${p.id}`} className="text-blue-600 hover:underline">
              {p.name}
            </Link>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

### 3. Catch-all route — `/docs/[...slug]`
```bash
mkdir -p "src/app/docs/[...slug]"
```
```tsx
// src/app/docs/[...slug]/page.tsx
type Props = {
  params: { slug: string[] };
};

export default function DocsPage({ params }: Props) {
  const path = params.slug.join("/");
  const breadcrumbs = params.slug;

  return (
    <div>
      <nav className="text-sm text-gray-500 mb-4">
        {breadcrumbs.map((crumb, i) => (
          <span key={i}>
            {i > 0 && " / "}
            <span className="capitalize">{crumb}</span>
          </span>
        ))}
      </nav>
      <h1 className="text-2xl font-bold">Docs: {path}</h1>
    </div>
  );
}
```
Visit `/docs/getting-started/installation` — `params.slug` is `["getting-started", "installation"]`.

### 4. Optional catch-all — `/blog/[[...slug]]`
```bash
mkdir -p "src/app/blog/[[...slug]]"
```
```tsx
// src/app/blog/[[...slug]]/page.tsx
type Props = {
  params: { slug?: string[] };
};

export default function BlogPage({ params }: Props) {
  if (!params.slug) {
    return <h1 className="text-2xl font-bold">All Blog Posts</h1>;
  }
  return (
    <h1 className="text-2xl font-bold">Post: {params.slug.join("/")}</h1>
  );
}
```
`/blog` → "All Blog Posts"; `/blog/hello-world` → "Post: hello-world".

### 6. Intercepting route — photo modal pattern
```bash
mkdir -p src/app/photos/\[id\]
mkdir -p "src/app/photos/(.)[\[id\]]"
# Note: actual folder name is (.)[\[id\]] — intercepting sibling [id]
```
```tsx
// src/app/photos/page.tsx
import Link from "next/link";

const PHOTOS = [{ id: "1" }, { id: "2" }, { id: "3" }];

export default function PhotosPage() {
  return (
    <div>
      <h1 className="text-2xl font-bold mb-4">Photos</h1>

}
```
