# A8.2 — Docker Deployment

**Phase:** 8 — Deployment & CI/CD  
**Topic:** Dockerfile for Next.js (standalone output), nginx reverse proxy, docker-compose  
**Type:** Assignment

---

## Goal
Package a Next.js app in a production-ready Docker image using standalone output mode and an nginx reverse proxy.

## Background

Next.js's **standalone output** (`output: "standalone"` in next.config.js) traces the minimal set of files needed to run the app and copies them into `.next/standalone/`. The result is a self-contained Node.js server that does NOT need `node_modules` installed — dramatically smaller images (300MB → 50MB).

**Multi-stage Dockerfile**: separate the build stage from the run stage. The build stage installs all dependencies (including devDependencies) and runs `next build`. The run stage copies only the production artifacts from the build stage and runs the server.

**nginx** acts as a reverse proxy in production: it handles SSL termination, compression (gzip/brotli), and serves static files from the `.next/static/` and `public/` directories directly, bypassing Node.js. This is faster than serving static files from Next.js itself.

## Tasks

### 1. Configure standalone output
```js
// next.config.js
const nextConfig = {
  output: "standalone",
  // ...
};
module.exports = nextConfig;
```

### 2. Multi-stage Dockerfile
```dockerfile
# Dockerfile
# Stage 1: Dependencies
FROM node:20-alpine AS deps
RUN apk add --no-cache libc6-compat
WORKDIR /app

# Copy package files first (layer caching)
COPY package.json pnpm-lock.yaml ./
RUN corepack enable pnpm && pnpm install --frozen-lockfile

# Stage 2: Build
FROM node:20-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .

# Set build-time env vars
ARG NEXT_PUBLIC_APP_URL
ENV NEXT_PUBLIC_APP_URL=${NEXT_PUBLIC_APP_URL}
ENV NEXT_TELEMETRY_DISABLED=1

RUN corepack enable pnpm && pnpm build

# Stage 3: Production runner
FROM node:20-alpine AS runner
WORKDIR /app

ENV NODE_ENV=production
ENV NEXT_TELEMETRY_DISABLED=1

# Create non-root user
RUN addgroup --system --gid 1001 nodejs
RUN adduser --system --uid 1001 nextjs

# Copy standalone output
COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public

# Set ownership
RUN chown -R nextjs:nodejs /app

USER nextjs
EXPOSE 3000

ENV PORT=3000
ENV HOSTNAME="0.0.0.0"

CMD ["node", "server.js"]
```

### 4. Build and test the image
```bash
# Build
docker build \
  --build-arg NEXT_PUBLIC_APP_URL=http://localhost:3000 \
  -t my-nextjs-app .

# Run
docker run -p 3000:3000 \
  -e DATABASE_URL=postgresql://localhost:5432/mydb \
  -e AUTH_SECRET=your-secret \
  my-nextjs-app

# Check size
docker images my-nextjs-app
```

### 5. nginx configuration
```nginx
# nginx/nginx.conf
events {
  worker_connections 1024;
}

http {
  include       /etc/nginx/mime.types;
  default_type  application/octet-stream;

  # Gzip compression
  gzip on;
  gzip_vary on;
  gzip_proxied any;
  gzip_comp_level 6;
  gzip_types text/plain text/css application/json application/javascript
             text/xml application/xml application/xml+rss text/javascript;

  upstream nextjs {
    server app:3000;
    keepalive 64;
  }

  server {
    listen 80;
    server_name localhost;

    # Serve Next.js static files directly (bypasses Node.js)
    location /_next/static/ {
      alias /app/.next/static/;
      expires 1y;
      add_header Cache-Control "public, immutable";
    }

    # Serve public directory
    location /public/ {
      alias /app/public/;
      expires 30d;
    }

    # All other requests go to Next.js
    location / {
      proxy_pass http://nextjs;
      proxy_http_version 1.1;
      proxy_set_header Upgrade $http_upgrade;
      proxy_set_header Connection "upgrade";
      proxy_set_header Host $host;
      proxy_set_header X-Real-IP $remote_addr;
      proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
      proxy_set_header X-Forwarded-Proto $scheme;
      proxy_cache_bypass $http_upgrade;
    }
  }
}
```

### 6. docker-compose.yml
```yaml
# docker-compose.yml
version: "3.9"

services:
  app:
    build:
      context: .
      dockerfile: Dockerfile
      args:
        NEXT_PUBLIC_APP_URL: http://localhost
    environment:
      - NODE_ENV=production
      - DATABASE_URL=${DATABASE_URL}
      - AUTH_SECRET=${AUTH_SECRET}
    restart: unless-stopped
    # Don't expose port publicly — nginx handles it
    expose:
      - "3000"

  nginx:
    image: nginx:alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      # Mount Next.js static files for direct serving
      - next_static:/app/.next/static:ro
    depends_on:
      - app
    restart: unless-stopped

volumes:
  next_static:
```

### 7. docker-compose.yml for local development with DB
```yaml
# docker-compose.dev.yml
version: "3.9"

services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: myapp
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

volumes:
  postgres_data:
```
```bash
# Start databases for local development
docker-compose -f docker-compose.dev.yml up -d

# Start full production stack
docker-compose up --build
```

### 8. Health check
```dockerfile
# Add to Dockerfile (before CMD)
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD node -e "require('http').get('http://localhost:3000/api/health', (r) => r.statusCode === 200 ? process.exit(0) : process.exit(1))"
```
