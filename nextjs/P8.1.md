# P8.1 — Deployed Next.js App

**Phase:** 8 — Deployment & CI/CD  
**Topic:** Vercel production deployment, custom domain, environment variables, preview branches  
**Type:** Project

---

## Goal
Deploy the portfolio or blog app to Vercel production, configure a custom domain, set up environment variables, and demonstrate preview deployments on pull requests.

## What to Build
- Production deployment on Vercel with automatic git integration
- Custom domain configuration with DNS records
- Environment variables for production vs preview vs development
- Preview deployments triggered by pull requests
- `vercel.json` configuration for redirects and headers
- Vercel Analytics and Speed Insights integration

## Architecture

```
vercel.json             ← redirects, headers, regions
.env.local              ← local dev secrets (gitignored)
.env.example            ← template showing required vars (committed)
.env.production         ← non-secret production config (committed)
app/
  layout.tsx            ← Vercel Analytics + Speed Insights
```

## Implementation

### Step 1: Prepare the project

```bash
# Install Vercel CLI locally
pnpm add -D vercel

# Initialize Vercel project (links to your Vercel account)
npx vercel link
```

```json
// .env.example — committed to git, no real secrets
NEXTAUTH_SECRET=
NEXTAUTH_URL=http://localhost:3000

DATABASE_URL=
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=

NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

```
# .env.local — NOT committed to git (in .gitignore)
NEXTAUTH_SECRET=your-secret-here
DATABASE_URL=postgresql://user:pass@localhost:5432/myapp
GOOGLE_CLIENT_ID=your-client-id
GOOGLE_CLIENT_SECRET=your-client-secret
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

### Step 2: Environment variables in Vercel Dashboard

Go to your Vercel project → **Settings → Environment Variables** and add:

| Variable | Production | Preview | Development |
|----------|-----------|---------|-------------|
| `NEXTAUTH_SECRET` | production-secret | preview-secret | ✗ (local only) |
| `NEXTAUTH_URL` | https://yourdomain.com | https://{{VERCEL_URL}} | http://localhost:3000 |
| `DATABASE_URL` | prod connection string | preview connection string | ✗ |
| `GOOGLE_CLIENT_ID` | ✓ | ✓ | ✗ |
| `NEXT_PUBLIC_SITE_URL` | https://yourdomain.com | https://{{VERCEL_URL}} | http://localhost:3000 |

**Important:** `NEXT_PUBLIC_*` variables are embedded at build time — they can't be different per request.

### Step 3: Use environment-aware config in code

```ts
// lib/config.ts
export const config = {
  siteUrl: process.env.NEXT_PUBLIC_SITE_URL ?? "http://localhost:3000",
  isDev: process.env.NODE_ENV === "development",
  isProd: process.env.NODE_ENV === "production",
  isPreview: process.env.VERCEL_ENV === "preview",
};
```

```ts
// auth.ts — NEXTAUTH_URL must be set correctly on Vercel
export const { handlers, signIn, signOut, auth } = NextAuth({
  // ...
  trustHost: true, // required on Vercel (HTTPS termination)
});
```

### Step 4: vercel.json

### Step 6: Custom domain

1. Go to Vercel Dashboard → Project → **Settings → Domains**
2. Add your domain: `yourdomain.com`
3. Add these DNS records at your domain registrar:

```
# If Vercel is your nameserver (recommended):
Nameservers: ns1.vercel-dns.com, ns2.vercel-dns.com

# If you manage DNS elsewhere:
Type  Name   Value
A     @      76.76.21.21
CNAME www    cname.vercel-dns.com
```

4. Vercel auto-provisions a free TLS certificate via Let's Encrypt.

### Step 7: Preview deployments workflow

Every branch/PR automatically gets a unique preview URL.

```bash
# Create a feature branch
git checkout -b feature/add-dark-mode

# Make changes, then push
git push origin feature/add-dark-mode

# Vercel creates: https://yourproject-git-feature-add-dark-mode-yourteam.vercel.app
```

**In the Vercel dashboard, you can:**
- Comment on a PR with the preview URL
- Promote a preview to production with one click
- Roll back to any previous deployment instantly

### Step 8: Deployment protection (optional)

Lock preview deployments behind a password or Vercel SSO:

```json
// vercel.json
{
  "github": {
    "deploymentProtection": "preview"
  }
}
```

### Step 9: Deployment CLI commands

```bash
# Deploy to preview (not production)
npx vercel

# Deploy to production
npx vercel --prod

# List deployments
npx vercel ls

# Rollback to a previous deployment
npx vercel rollback [deployment-url]

# View logs for current deployment
npx vercel logs
```

### Step 10: Verify deployment

```bash
# Check that all expected routes respond correctly
curl -I https://yourdomain.com
curl -I https://yourdomain.com/api/feed
curl -I https://yourdomain.com/this-page-does-not-exist

# Expected: 200, 200, 404

# Check security headers
curl -I https://yourdomain.com | grep -i "x-frame\|x-content"
```
