# A2.2 — Static vs Dynamic Rendering

**Phase:** 2 — Data Fetching & Rendering  
**Topic:** SSG, SSR, ISR, generateStaticParams, dynamic segments  
**Type:** Assignment

---

## Goal
Understand how Next.js decides between static and dynamic rendering, and control that decision explicitly using `generateStaticParams`, `revalidate`, and `dynamic` exports.

## Background

Next.js renders routes in one of three modes:

**Static (SSG):** The page is rendered at build time. The HTML is generated once and served from a CDN. This is the default for any page with no dynamic data (no `cookies()`, `headers()`, or `fetch` with `no-store`). Static pages are the fastest possible — zero server compute per request.

**Dynamic (SSR):** The page is rendered on every request. Any use of `cookies()`, `headers()`, `searchParams` (in a page), or `fetch` with `no-store` will force dynamic rendering. The HTML is generated fresh for every visitor.

**Incremental Static Regeneration (ISR):** A hybrid. The page is rendered statically at build time, but Next.js re-renders it in the background after `revalidate` seconds. Visitors always get a fast cached response, but the content eventually updates.

For dynamic routes like `[slug]`, Next.js cannot know at build time which slugs exist unless you export `generateStaticParams`. Without it, the route is rendered dynamically at request time. With it, Next.js pre-renders the listed slugs at build time.

The `dynamic` export at route level overrides automatic detection:
- `export const dynamic = "force-static"` — force static even if dynamic APIs are used
- `export const dynamic = "force-dynamic"` — always dynamic
- `export const dynamic = "error"` — throw at build time if dynamic APIs are detected

## Tasks

### 1. Check the build output
After running `pnpm build`, observe the route table:
```
○  /                    → Static
λ  /dynamic-data        → Dynamic (Server)
◐  /isr-data            → Static (ISR, revalidate: 30)
```
`○` = static, `λ` = dynamic, `◐` = ISR.

### 2. generateStaticParams — pre-render dynamic routes
```tsx
// src/app/posts/[id]/page.tsx
type Post = { id: number; title: string; body: string };

// Tells Next.js which [id] values to pre-render at build time
export async function generateStaticParams() {
  const res = await fetch("https://jsonplaceholder.typicode.com/posts?_limit=10");
  const posts: Post[] = await res.json();

  return posts.map((post) => ({
    id: String(post.id), // params are always strings
  }));
}

// Because generateStaticParams is present, this route is now Static
async function getPost(id: string): Promise<Post> {
  const res = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`, {
    cache: "force-cache",
  });
  if (!res.ok) throw new Error("Not found");
  return res.json();
}

export default async function PostPage({ params }: { params: { id: string } }) {
  const post = await getPost(params.id);

  return (
    <article className="max-w-2xl">
      <p className="text-sm text-gray-500">Post #{post.id}</p>
      <h1 className="text-2xl font-bold mt-1">{post.title}</h1>
      <p className="mt-4 text-gray-700">{post.body}</p>
    </article>
  );
}
```

### 3. Control what happens for params NOT in generateStaticParams
```tsx
// src/app/posts/[id]/page.tsx (add these exports)

// "blocking" = render on demand for unknown IDs, cache the result
// "force-static" = return 404 for unknown IDs (strict)
export const dynamicParams = true; // default: on-demand render unknown slugs
// export const dynamicParams = false; // 404 for anything not pre-rendered

export const revalidate = 3600; // re-generate all static pages every 1 hour
```

### 4. Force dynamic rendering
```tsx
// src/app/personalized/page.tsx
import { cookies } from "next/headers";

// Using cookies() forces dynamic rendering automatically
export default async function PersonalizedPage() {
  const cookieStore = cookies();
  const theme = cookieStore.get("theme")?.value ?? "light";
  const userId = cookieStore.get("userId")?.value ?? "guest";

  return (
    <div>
      <h1 className="text-2xl font-bold">Hello, {userId}</h1>
      <p className="text-gray-600">Your theme: {theme}</p>
      <p className="text-xs text-gray-400 mt-4">
        Rendered at: {new Date().toISOString()}
      </p>
    </div>
  );
}
```

### 5. ISR with revalidate export
```tsx
// src/app/products/page.tsx
type Product = { id: number; title: string; price: number };

// Page-level revalidation — all fetches in this page revalidate every 60s
export const revalidate = 60;

async function getProducts(): Promise<Product[]> {
  const res = await fetch("https://fakestoreapi.com/products?limit=6");
  if (!res.ok) return [];
  return res.json();
}

export default async function ProductsPage() {
  const products = await getProducts();

  return (
    <div>
      <h1 className="text-2xl font-bold mb-4">Products (ISR)</h1>
      <p className="text-sm text-gray-500 mb-6">
        Last generated: {new Date().toLocaleTimeString()}
      </p>
      <div className="grid grid-cols-2 md:grid-cols-3 gap-4">
        {products.map((p) => (
          <div key={p.id} className="p-4 border rounded">
            <p className="font-semibold">{p.title.slice(0, 40)}</p>
            <p className="text-green-600 mt-1">${p.price}</p>
          </div>
        ))}
      </div>
    </div>
  );
}
```

### 6. Programmatic revalidation with revalidatePath
```tsx
// src/app/api/revalidate/route.ts
import { revalidatePath } from "next/cache";
import { NextRequest, NextResponse } from "next/server";

export async function POST(request: NextRequest) {
  const secret = request.headers.get("x-revalidate-secret");

  if (secret !== process.env.REVALIDATE_SECRET) {
    return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
  }

  const { path } = await request.json();
  revalidatePath(path);

  return NextResponse.json({ revalidated: true, path });
}
```
Usage (from a CMS webhook):
```bash
curl -X POST http://localhost:3000/api/revalidate \
  -H "Content-Type: application/json" \
  -H "x-revalidate-secret: mysecret" \
  -d '{"path": "/products"}'
```

### 7. revalidateTag for granular cache invalidation
```tsx
// src/lib/products.ts
export async function getProducts() {
  const res = await fetch("https://fakestoreapi.com/products", {
    next: { tags: ["products"] }, // tag this fetch
  });
  return res.json();
}
```
```tsx
// src/app/api/revalidate-tag/route.ts
import { revalidateTag } from "next/cache";
import { NextResponse } from "next/server";

export async function POST() {
  revalidateTag("products"); // invalidates ALL fetches tagged "products"
  return NextResponse.json({ revalidated: true });
}
```
