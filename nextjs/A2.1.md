## Goal
Master the fetch() API inside Server Components and understand how the `cache` option controls static vs dynamic vs revalidated data.

## Background

In Next.js App Router, `fetch()` is extended beyond the standard Web API. Next.js intercepts `fetch()` calls in Server Components and adds a layer of caching and deduplication on top. This is one of the most important concepts to understand because it determines how your pages are rendered.

The `cache` option on `fetch()` controls behavior:
- `force-cache` (default): The response is cached indefinitely. The page becomes fully static at build time if all fetches use this option.
- `no-store`: Never cache. The response is always fresh. The page becomes dynamically rendered (runs on every request).
- `{ next: { revalidate: 60 } }`: Cache for 60 seconds, then refetch in the background when the next request arrives (Stale-While-Revalidate). This is ISR.

Next.js also **deduplicates** fetch calls: if the same URL is fetched multiple times in the same render tree, it only makes one actual HTTP request. This means you can call `getUser()` in the layout AND in the page without two round-trips.

**Request memoisation** (React cache) is the mechanism behind deduplication. You can use `import { cache } from "react"` to wrap your own data-fetching functions with the same behaviour.

## Tasks

### 1. Fetch with `force-cache` (static)
```tsx
// src/app/static-data/page.tsx
type Post = {
  id: number;
  title: string;
  body: string;
};

// force-cache is the default but being explicit is good practice
async function getPosts(): Promise<Post[]> {
  const res = await fetch("https://jsonplaceholder.typicode.com/posts?_limit=5", {
    cache: "force-cache",
  });
  if (!res.ok) throw new Error("Failed to fetch posts");
  return res.json();
}

export default async function StaticDataPage() {
  const posts = await getPosts();

  return (
    <div>
      <h1 className="text-2xl font-bold mb-1">Static Posts</h1>
      <p className="text-sm text-gray-500 mb-4">
        Fetched at build time. Timestamp: {new Date().toISOString()}
      </p>
      <ul className="space-y-4">
        {posts.map((post) => (
          <li key={post.id} className="p-4 border rounded">
            <h2 className="font-semibold">{post.title}</h2>
            <p className="text-sm text-gray-600 mt-1">{post.body.slice(0, 80)}…</p>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

### 2. Fetch with `no-store` (dynamic)
```tsx
// src/app/dynamic-data/page.tsx
type User = { id: number; name: string; email: string };

async function getRandomUser(): Promise<User> {
  const id = Math.floor(Math.random() * 10) + 1;
  const res = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`, {
    cache: "no-store",
  });
  if (!res.ok) throw new Error("Failed to fetch user");
  return res.json();
}

export default async function DynamicDataPage() {
  const user = await getRandomUser();

  return (
    <div>
      <h1 className="text-2xl font-bold mb-1">Dynamic Data</h1>
      <p className="text-sm text-gray-500 mb-4">
        Different user on every refresh. Timestamp: {new Date().toISOString()}
      </p>
      <div className="p-4 border rounded">
        <p className="font-semibold">{user.name}</p>
        <p className="text-sm text-gray-600">{user.email}</p>
      </div>
    </div>
  );
}
```
Run `pnpm build && pnpm start`. Refresh the dynamic page — the timestamp changes. Refresh the static page — it does not.

### 3. Fetch with `revalidate` (ISR)
```tsx
// src/app/isr-data/page.tsx
async function getTodos() {
  const res = await fetch(
    "https://jsonplaceholder.typicode.com/todos?_limit=5",
    { next: { revalidate: 30 } } // revalidate every 30 seconds
  );
  return res.json();
}

export default async function IsrDataPage() {
  const todos = await getTodos();

  return (
    <div>
      <h1 className="text-2xl font-bold mb-1">ISR Todos</h1>
      <p className="text-sm text-gray-500 mb-4">
        Revalidates every 30s. Rendered at: {new Date().toISOString()}
      </p>
      <ul className="space-y-2">
        {todos.map((todo: { id: number; title: string; completed: boolean }) => (
          <li key={todo.id} className="flex items-center gap-2">
            <span>{todo.completed ? "✅" : "⬜"}</span>
            <span className={todo.completed ? "line-through text-gray-400" : ""}>
              {todo.title}
            </span>
          </li>
        ))}
      </ul>
    </div>
  );
}
```

### 4. Request memoization with React cache
```tsx
// src/lib/data.ts
import { cache } from "react";

export type User = { id: number; name: string; email: string };

// This function can be called multiple times in the same render
// but will only make ONE HTTP request
export const getUser = cache(async (id: number): Promise<User | null> => {
  const res = await fetch(`https://jsonplaceholder.typicode.com/users/${id}`, {
    cache: "no-store",
  });
  if (!res.ok) return null;
  return res.json();
});
```
```tsx
// src/app/memoized/layout.tsx
import { getUser } from "@/lib/data";

export default async function MemoizedLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  const user = await getUser(1); // First call — HTTP request made
  return (
    <div>
      <p className="text-sm border-b pb-2 mb-4">
        Layout user: {user?.name}
      </p>
      {children}
    </div>
  );
}
```
```tsx
// src/app/memoized/page.tsx
import { getUser } from "@/lib/data";

export default async function MemoizedPage() {
  const user = await getUser(1); // Second call — returns cached result, NO new request
  return (
    <div>
      <h1 className="text-2xl font-bold">Page user: {user?.name}</h1>
      <p className="text-sm text-gray-500">
        Both layout and page called getUser(1) but only one fetch was made.
      </p>
    </div>
  );
}
```

### 5. Parallel data fetching
```tsx
// src/app/parallel-fetch/page.tsx
async function getPost(id: number) {
  const res = await fetch(`https://jsonplaceholder.typicode.com/posts/${id}`);
  return res.json();
}

async function getComments(postId: number) {
  const res = await fetch(
    `https://jsonplaceholder.typicode.com/posts/${postId}/comments`
  );
  return res.json();
}

export default async function ParallelFetchPage() {
  // ✅ Fire both requests at the same time
  const [post, comments] = await Promise.all([getPost(1), getComments(1)]);

  return (
    <div className="space-y-6">
      <div>
        <h1 className="text-2xl font-bold">{post.title}</h1>
        <p className="mt-2 text-gray-600">{post.body}</p>
      </div>
      <div>
        <h2 className="text-lg font-semibold mb-2">Comments ({comments.length})</h2>
        <ul className="space-y-2">
          {comments.slice(0, 3).map((c: { id: number; name: string; body: string }) => (
            <li key={c.id} className="p-3 bg-gray-50 rounded text-sm">
              <p className="font-medium">{c.name}</p>
              <p className="text-gray-600">{c.body.slice(0, 80)}…</p>
            </li>
          ))}
        </ul>
      </div>
    </div>
  );
}
```

### 6. Error handling in Server Components
```tsx
// src/app/error-fetch/page.tsx
async function getDataOrThrow() {
  const res = await fetch("https://jsonplaceholder.typicode.com/posts/9999999", {
    cache: "no-store",
  });
  if (!res.ok) {
    throw new Error(`API error: ${res.status}`);
  }
  return res.json();
}

export default async function ErrorFetchPage() {
  // Without error.tsx, this would crash the page
  // With error.tsx in the same segment, the error is caught gracefully
  const data = await getDataOrThrow();
  return <div>{data.title}</div>;
}
```

- [ ] Dynamic page (`no-store`) shows a new timestamp on every refresh
- [ ] ISR page (`revalidate: 30`) updates at most once per 30 seconds
- [ ] `getUser` wrapped in `cache()` is called from both layout and page with no duplicate network requests (verify in server logs)
- [ ] Parallel fetch uses `Promise.all` — not sequential awaits
- [ ] All fetch functions have proper TypeScript return types (no `any`)
- [ ] `pnpm build` logs show which routes are Static (○) vs Dynamic (λ)
