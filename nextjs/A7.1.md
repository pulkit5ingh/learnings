## Goal
Set up Jest and React Testing Library in a Next.js project and write reliable component tests using the right query strategies.

## Background

React Testing Library (RTL) is a testing philosophy as much as it is a library. Its core principle: **test your components the way a user would use them** — not by testing implementation details like state or component instance methods.

RTL provides queries to find elements in the DOM. Understanding when to use each query is critical:

- `getBy*` — synchronous, throws if 0 or 2+ matches
- `queryBy*` — synchronous, returns null if not found (use for asserting absence)
- `findBy*` — async, waits up to 1000ms for the element to appear (use for async operations)

The preferred query priority (most like a user):
1. `getByRole` — finds by ARIA role (button, heading, textbox, checkbox)
2. `getByLabelText` — finds form elements by their label
3. `getByPlaceholderText` — fallback for unlabeled inputs
4. `getByText` — finds by text content
5. `getByTestId` — last resort, use `data-testid` attribute

`userEvent` is preferred over `fireEvent` because it simulates the full sequence of browser events (pointerdown → mousedown → focus → keydown → keypress → input → keyup → pointerup → mouseup → click for a button click), making tests more realistic.

### 2. Test a basic component
```tsx
// src/components/ui/__tests__/Button.test.tsx
import { render, screen } from "@testing-library/react";
import userEvent from "@testing-library/user-event";
import { Button } from "../Button";

describe("Button", () => {
  it("renders with correct text", () => {
    render(<Button>Click me</Button>);
    expect(screen.getByRole("button", { name: "Click me" })).toBeInTheDocument();
  });

  it("calls onClick handler when clicked", async () => {
    const user = userEvent.setup();
    const handleClick = jest.fn();

    render(<Button onClick={handleClick}>Submit</Button>);
    await user.click(screen.getByRole("button", { name: "Submit" }));

    expect(handleClick).toHaveBeenCalledTimes(1);
  });

  it("is disabled when disabled prop is true", async () => {
    const user = userEvent.setup();
    const handleClick = jest.fn();

    render(<Button disabled onClick={handleClick}>Disabled</Button>);
    const button = screen.getByRole("button", { name: "Disabled" });

    expect(button).toBeDisabled();
    await user.click(button);
    expect(handleClick).not.toHaveBeenCalled();
  });

  it("applies danger intent styles", () => {
    render(<Button intent="danger">Delete</Button>);
    const button = screen.getByRole("button");
    expect(button).toHaveClass("bg-red-600");
  });
});
```

### 3. Query types in practice
```tsx
// src/components/__tests__/queries-demo.test.tsx
import { render, screen } from "@testing-library/react";

function DemoComponent() {
  return (
    <div>
      <h1>Page Title</h1>
      <label htmlFor="email">Email address</label>
      <input id="email" type="email" placeholder="Enter email" />
      <input type="checkbox" aria-label="Accept terms" />
      <button>Submit</button>
      <p data-testid="status">Idle</p>
    </div>
  );
}

describe("RTL queries", () => {
  beforeEach(() => render(<DemoComponent />));

  it("getByRole - find heading", () => {
    expect(screen.getByRole("heading", { name: "Page Title" })).toBeInTheDocument();
  });

  it("getByLabelText - find input by label", () => {
    expect(screen.getByLabelText("Email address")).toBeInTheDocument();
  });

  it("getByPlaceholderText - fallback query", () => {
    expect(screen.getByPlaceholderText("Enter email")).toBeInTheDocument();
  });

  it("getByRole with aria-label", () => {
    expect(screen.getByRole("checkbox", { name: "Accept terms" })).toBeInTheDocument();
  });

  it("queryBy - element absent", () => {
    expect(screen.queryByText("This does not exist")).not.toBeInTheDocument();
  });

  it("getByTestId - last resort", () => {
    expect(screen.getByTestId("status")).toHaveTextContent("Idle");
  });
});
```

### 4. Async testing with findBy
```tsx
// src/components/__tests__/async.test.tsx
import { render, screen } from "@testing-library/react";
import userEvent from "@testing-library/user-event";
import { useState } from "react";

function AsyncComponent() {
  const [status, setStatus] = useState<"idle" | "loading" | "done">("idle");

  async function handleClick() {
    setStatus("loading");
    await new Promise((r) => setTimeout(r, 100));
    setStatus("done");
  }

  return (
    <div>
      <button onClick={handleClick}>Load</button>
      {status === "loading" && <p>Loading…</p>}
      {status === "done" && <p>Data loaded!</p>}
    </div>
  );
}

describe("Async component", () => {
  it("shows loading then success", async () => {
    const user = userEvent.setup();
    render(<AsyncComponent />);

    await user.click(screen.getByRole("button", { name: "Load" }));

    // findBy waits for the element to appear
    await screen.findByText("Data loaded!");
    expect(screen.queryByText("Loading…")).not.toBeInTheDocument();
  });
});
```

### 5. Form interaction test
```tsx
// src/components/__tests__/form.test.tsx
import { render, screen } from "@testing-library/react";
import userEvent from "@testing-library/user-event";

function LoginForm({ onSubmit }: { onSubmit: (data: { email: string; password: string }) => void }) {
  return (
    <form onSubmit={(e) => {
      e.preventDefault();
      const data = new FormData(e.currentTarget);
      onSubmit({ email: data.get("email") as string, password: data.get("password") as string });
    }}>
      <label htmlFor="email">Email</label>
      <input id="email" name="email" type="email" required />
      <label htmlFor="password">Password</label>
      <input id="password" name="password" type="password" required />
      <button type="submit">Sign in</button>
    </form>
  );
}

describe("LoginForm", () => {
  it("submits email and password", async () => {
    const user = userEvent.setup();
    const handleSubmit = jest.fn();
    render(<LoginForm onSubmit={handleSubmit} />);

    await user.type(screen.getByLabelText("Email"), "alice@example.com");
    await user.type(screen.getByLabelText("Password"), "password123");
    await user.click(screen.getByRole("button", { name: "Sign in" }));

    expect(handleSubmit).toHaveBeenCalledWith({
      email: "alice@example.com",
      password: "password123",
    });
  });
});
```
