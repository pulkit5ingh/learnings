## Goal

## Background

### Security Headers with Helmet
Browsers respect certain HTTP response headers to prevent common attacks. Helmet sets these headers automatically:
- `Content-Security-Policy` — restricts which scripts, styles, and resources can load
- `X-Content-Type-Options: nosniff` — prevents MIME sniffing attacks
- `X-Frame-Options: DENY` — prevents clickjacking via iframes
- `Strict-Transport-Security` — forces HTTPS connections
- `X-XSS-Protection` — enables browser XSS filter (legacy browsers)

### Rate Limiting
Rate limiting prevents brute-force attacks (trying thousands of passwords) and DoS attacks (flooding with requests). `express-rate-limit` tracks requests by IP (or a custom key) and returns 429 Too Many Requests when a window is exceeded. For APIs behind a reverse proxy (nginx, AWS ALB), set `trustProxy: true` and use `req.ip` correctly.

For distributed systems, use a Redis-backed store so rate limits are shared across multiple Node.js instances (`rate-limit-redis`).

### CORS
Browsers enforce the Same-Origin Policy: JavaScript can only make requests to the same origin (protocol + host + port). CORS (Cross-Origin Resource Sharing) lets servers opt in to cross-origin requests by setting the `Access-Control-Allow-Origin` header. Misconfigured CORS — especially `Access-Control-Allow-Origin: *` with `Access-Control-Allow-Credentials: true` — is a serious security vulnerability.

### Input Sanitization vs Validation
Validation checks format (is this a valid email?). Sanitization modifies input to remove dangerous content (strip HTML tags, trim whitespace). For SQL injection: use parameterized queries — always. For XSS: sanitize HTML output using `DOMPurify` (client-side) or `sanitize-html` (server-side). Never trust user input.

### 2. Helmet Configuration

```js
// src/middleware/security.js
const helmet = require('helmet');

// Custom helmet config for an API (APIs don't need full CSP)
const helmetConfig = helmet({
  contentSecurityPolicy: {
    directives: {
      defaultSrc: ["'self'"],
      scriptSrc: ["'self'"],
      styleSrc: ["'self'", "'unsafe-inline'"],
      imgSrc: ["'self'", 'data:', 'https:'],
    },
  },
  crossOriginEmbedderPolicy: false, // Disable for APIs
  hsts: {
    maxAge: 31536000,     // 1 year in seconds
    includeSubDomains: true,
    preload: true,
  },
});

module.exports = helmetConfig;
```

### 3. Rate Limiter

```js
// src/middleware/rateLimiter.js
const rateLimit = require('express-rate-limit');

// General API rate limit
const apiLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 100,                  // 100 requests per window per IP
  standardHeaders: true,     // Return RateLimit-* headers
  legacyHeaders: false,
  message: {
    error: 'Too many requests, please try again after 15 minutes',
    retryAfter: '15 minutes',
  },
  // Skip rate limiting for trusted IPs (e.g., internal services)
  skip: (req) => req.ip === '127.0.0.1' && process.env.NODE_ENV === 'development',
});

// Strict limit for auth endpoints (prevent brute force)
const authLimiter = rateLimit({
  windowMs: 15 * 60 * 1000,
  max: 10, // only 10 login attempts per 15 min
  message: {
    error: 'Too many login attempts, please try again after 15 minutes',
  },
  skipSuccessfulRequests: true, // don't count successful logins against limit
});

// Strict limit for registration
const registerLimiter = rateLimit({
  windowMs: 60 * 60 * 1000, // 1 hour
  max: 5,
  message: { error: 'Too many accounts created from this IP, try again in an hour' },
});

module.exports = { apiLimiter, authLimiter, registerLimiter };
```

### 4. CORS Configuration

```js
// src/middleware/corsConfig.js
const cors = require('cors');

const ALLOWED_ORIGINS = (process.env.ALLOWED_ORIGINS ?? 'http://localhost:3000')
  .split(',')
  .map(o => o.trim());

const corsOptions = {
  origin: (origin, callback) => {
    // Allow requests with no origin (curl, Postman, mobile apps)
    if (!origin) return callback(null, true);

    if (ALLOWED_ORIGINS.includes(origin)) {
      callback(null, true);
    } else {
      callback(new Error(`CORS: Origin ${origin} not allowed`));
    }
  },
  methods: ['GET', 'POST', 'PUT', 'PATCH', 'DELETE', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization', 'X-Request-Id'],
  exposedHeaders: ['X-Request-Id', 'RateLimit-Remaining', 'RateLimit-Reset'],
  credentials: true,        // Allow cookies with cross-origin requests
  optionsSuccessStatus: 200, // Some legacy browsers (IE11) choke on 204
};

module.exports = cors(corsOptions);
```

### 5. Input Sanitization Middleware

```js
// src/middleware/sanitize.js

// Sanitize against NoSQL injection (MongoDB operator injection)
// e.g., { email: { "$gt": "" } } in req.body
const mongoSanitize = require('express-mongo-sanitize');

// Remove any keys starting with '$' or containing '.'
const sanitizeInput = mongoSanitize({
  replaceWith: '_', // replace dangerous chars instead of removing
  allowDots: false,
  onSanitize: ({ req, key }) => {
    console.warn(`[SECURITY] Sanitized field: ${key} from ${req.ip}`);
  },
});

// Prevent HTTP Parameter Pollution
// ?sort=name&sort=email → keeps only the last 'sort' value
const hpp = require('hpp');
const preventParamPollution = hpp({
  whitelist: ['tags', 'fields', 'include'], // allow these to be arrays
});

// Basic XSS sanitization for req.body string values
const xssClean = (req, res, next) => {
  if (req.body) {
    req.body = sanitizeObject(req.body);
  }
  if (req.query) {
    req.query = sanitizeObject(req.query);
  }
  next();
};

function sanitizeObject(obj) {
  if (typeof obj === 'string') {
    // Remove script tags and on* attributes
    return obj
      .replace(/<script\b[^<]*(?:(?!<\/script>)<[^<]*)*<\/script>/gi, '')
      .replace(/on\w+="[^"]*"/gi, '')
      .replace(/javascript:/gi, '');
  }
  if (typeof obj === 'object' && obj !== null) {
    return Object.fromEntries(
      Object.entries(obj).map(([k, v]) => [k, sanitizeObject(v)])
    );
  }
  return obj;
}

module.exports = { sanitizeInput, preventParamPollution, xssClean };
```

### 6. Full Security Stack

```js
// src/app.js
const express = require('express');
const helmetConfig = require('./middleware/security');
const corsConfig = require('./middleware/corsConfig');
const { apiLimiter, authLimiter, registerLimiter } = require('./middleware/rateLimiter');
const { sanitizeInput, preventParamPollution, xssClean } = require('./middleware/sanitize');

const app = express();

// 1. Security headers — first
app.use(helmetConfig);

// 2. CORS — before any routes
app.use(corsConfig);

// 3. Body parsing
app.use(express.json({ limit: '10kb' })); // Limit body size
app.use(express.urlencoded({ extended: false, limit: '10kb' }));

// 4. Input sanitization — after body parsing
app.use(sanitizeInput);
app.use(preventParamPollution);
app.use(xssClean);

// 5. Rate limiting — applied globally
app.use('/api/', apiLimiter);

// 6. Stricter limits on auth routes
app.use('/api/auth/login', authLimiter);
app.use('/api/auth/register', registerLimiter);

// --- Routes ---
app.get('/api/health', (req, res) => {
  res.json({ status: 'ok' });
});

app.post('/api/auth/login', (req, res) => {
  // Simulated login
  res.json({ message: 'Login endpoint', body: req.body });
});

// Test XSS sanitization
app.post('/api/echo', (req, res) => {
  res.json({ received: req.body });
});

module.exports = app;
```

### 7. Test Security

```bash
# Verify security headers are present
curl -I http://localhost:3000/api/health
# Should see: X-Content-Type-Options, X-Frame-Options, Strict-Transport-Security

# Test XSS sanitization
curl -X POST http://localhost:3000/api/echo \
  -H "Content-Type: application/json" \
  -d '{"comment":"<script>alert(1)</script>Hello"}'
# Script tags should be stripped from response

# Test NoSQL injection attempt
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":{"$gt":""},"password":"anything"}'
# '$gt' should be replaced/removed

# Test rate limiting — send 11 requests to auth endpoint
for i in {1..12}; do
  curl -s -o /dev/null -w "%{http_code}\n" \
    -X POST http://localhost:3000/api/auth/login \
    -H "Content-Type: application/json" \
    -d '{"email":"a@b.com","password":"wrong"}'
done
# Requests 11 and 12 should return 429

# Test CORS with disallowed origin

     -H "Access-Control-Request-Method: POST" \
     -X OPTIONS http://localhost:3000/api/health
```
