# A8.1 — Docker for Node.js

**Phase:** 8 — Deployment & DevOps
**Topic:** Multi-stage Dockerfile, non-root user, .dockerignore, docker-compose
**Type:** Assignment

---

## Goal
Containerize an Express application with a production-ready multi-stage Dockerfile and orchestrate it with docker-compose alongside MongoDB and Redis.

## Background

### Multi-Stage Builds
A multi-stage Dockerfile uses multiple `FROM` instructions. Each stage builds on the previous but starts with a fresh filesystem. This lets you install all dev dependencies and build tools in an early stage, then copy only the production artifacts into a slim final image. Result: final image is much smaller (no devDependencies, no build tools) and has a smaller attack surface.

### Non-Root User
By default, containers run as root. If an attacker exploits your app, they have root in the container. Running as a non-root user limits the blast radius. Create a dedicated user (`addgroup`, `adduser`) and switch to it before the `CMD`.

### .dockerignore
Like `.gitignore` for Docker's build context. Without it, `COPY . .` copies `node_modules` (large, wrong platform binaries), `.env` files, git history, test files — all into the image. Always have a `.dockerignore`.

### Layer Caching
Docker caches layers. `COPY package*.json ./ && RUN npm ci` installs dependencies only when `package.json` changes. `COPY . .` comes after — changing source code doesn't reinstall packages. This makes rebuilds fast.

## Tasks

### 1. Multi-Stage Dockerfile

```dockerfile
# Dockerfile
# Stage 1: Dependencies
FROM node:20-alpine AS deps
WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production && npm cache clean --force

# Stage 2: Builder (for TypeScript compile or asset builds)
FROM node:20-alpine AS builder
WORKDIR /app

COPY package*.json ./
RUN npm ci

COPY . .
# If using TypeScript: RUN npm run build
# For plain JS, just verify the app starts:
RUN node -e "require('./src/app')" && echo "App loads OK"

# Stage 3: Production image
FROM node:20-alpine AS production

# Security: non-root user
RUN addgroup -S appgroup && adduser -S appuser -G appgroup

WORKDIR /app

# Copy production dependencies from deps stage
COPY --from=deps /app/node_modules ./node_modules

# Copy source code
COPY --from=builder /app/src ./src
COPY --from=builder /app/package.json ./

# Create logs directory with correct ownership
RUN mkdir -p logs && chown -R appuser:appgroup /app

# Switch to non-root user
USER appuser

# Document exposed port
EXPOSE 3000

# Health check
HEALTHCHECK --interval=30s --timeout=10s --start-period=15s --retries=3 \
  CMD wget -qO- http://localhost:3000/health || exit 1

# Start command — use exec form (not shell form) for signal handling
CMD ["node", "src/server.js"]
```

### 3. docker-compose.yml

```yaml
# docker-compose.yml
version: '3.9'

services:
  api:
    build:
      context: .
      dockerfile: Dockerfile
      target: production
    image: my-api:latest
    container_name: api
    ports:
      - "3000:3000"
    environment:
      NODE_ENV: production
      PORT: 3000
      MONGODB_URI: mongodb://mongo:27017/myapp
      REDIS_URL: redis://redis:6379
      ACCESS_TOKEN_SECRET: ${ACCESS_TOKEN_SECRET}
      REFRESH_TOKEN_SECRET: ${REFRESH_TOKEN_SECRET}
    depends_on:
      mongo:
        condition: service_healthy
      redis:
        condition: service_healthy
    restart: unless-stopped
    networks:
      - app-network

  mongo:
    image: mongo:7
    container_name: mongo
    volumes:
      - mongo_data:/data/db
      - ./scripts/mongo-init.js:/docker-entrypoint-initdb.d/init.js:ro
    environment:
      MONGO_INITDB_DATABASE: myapp
    healthcheck:
      test: ["CMD", "mongosh", "--eval", "db.adminCommand('ping')"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 20s
    networks:
      - app-network

  redis:
    image: redis:7-alpine
    container_name: redis
    command: redis-server --appendonly yes --requirepass ${REDIS_PASSWORD:-redispass}
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "-a", "${REDIS_PASSWORD:-redispass}", "ping"]
      interval: 10s
      timeout: 5s
      retries: 3
    networks:
      - app-network

networks:
  app-network:
    driver: bridge

volumes:
  mongo_data:
  redis_data:
```

### 4. docker-compose.dev.yml (Development override)

```yaml
# docker-compose.dev.yml
version: '3.9'

services:
  api:
    build:
      target: builder  # use builder stage for dev (has devDependencies)
    volumes:
      - ./src:/app/src  # live reload: mount source
      - /app/node_modules  # don't override node_modules from container
    environment:
      NODE_ENV: development
    command: ["node", "--watch", "src/server.js"]  # Node.js 18+ --watch flag
    ports:
      - "9229:9229"  # debugger port
```

### 5. Build and Run

```bash
# Build production image
docker build -t my-api:latest .

# Check image size
docker images my-api

# Run just the app (requires external mongo/redis)
docker run -p 3000:3000 \
  -e MONGODB_URI=mongodb://host.docker.internal:27017/myapp \
  -e REDIS_URL=redis://host.docker.internal:6379 \
  my-api:latest

# Run full stack with docker-compose
docker-compose up --build

# Development mode with hot reload
docker-compose -f docker-compose.yml -f docker-compose.dev.yml up

# Run as daemon
docker-compose up -d

# View logs
docker-compose logs -f api

# Stop and remove containers
docker-compose down

# Stop and remove containers AND volumes (destroys DB data)
docker-compose down -v
```

### 6. Verify Non-Root User

```bash
# Check which user the container runs as
docker-compose up -d
docker exec api whoami
# Should print: appuser (not root)

# Check file permissions
docker exec api ls -la /app
```

### 7. Security Scan

```bash
# Scan the image for vulnerabilities
docker scout cves my-api:latest

# Or use trivy (free):
docker run --rm -v /var/run/docker.sock:/var/run/docker.sock \
  aquasec/trivy image my-api:latest
```
