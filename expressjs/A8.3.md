## Goal
Build a complete CI/CD pipeline with GitHub Actions that lints, tests, builds a Docker image, pushes to a registry, and deploys to a cloud platform.

## Background

### GitHub Actions
GitHub Actions is a CI/CD platform built into GitHub. Workflows are YAML files in `.github/workflows/`. A workflow consists of jobs; each job runs on a runner (ubuntu-latest) and contains steps. Steps run shell commands (`run`) or actions (`uses`).

Triggers: `push` to a branch, `pull_request`, `schedule` (cron), `workflow_dispatch` (manual). Using branch protection rules, you can require CI to pass before merging a PR.

### Docker Registry
After building an image, you push it to a container registry (Docker Hub, GitHub Container Registry, AWS ECR). The registry stores versioned images. Deployment pulls the specific image version.

Tag strategy: `latest` for the most recent build on main, `sha-<commit-hash>` for exact versions (enables rollback), semver tags (`v1.2.3`) for releases.

### Deployment to Railway/Render
Railway and Render are PaaS platforms that deploy Docker containers. Connect your GitHub repository; they deploy automatically on push. Alternatively, use their CLI or API to trigger a deployment from the CI pipeline.

### 2. CI Workflow (Test on PRs)

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main, develop]

jobs:
  lint-and-test:
    name: Lint & Test
    runs-on: ubuntu-latest

    services:
      redis:
        image: redis:7-alpine
        ports:
          - 6379:6379
        options: >-
          --health-cmd "redis-cli ping"
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Lint
        run: npm run lint

      - name: Run tests with coverage
        run: npm run test:coverage
        env:
          NODE_ENV: test
          ACCESS_TOKEN_SECRET: ci-access-secret-at-least-32-chars-long
          REFRESH_TOKEN_SECRET: ci-refresh-secret-at-least-32-chars-long
          REDIS_URL: redis://localhost:6379

      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v4
        with:
          file: ./coverage/lcov.info
          flags: unittests
        continue-on-error: true
```

### 3. CD Workflow (Build and Deploy on merge to main)

```yaml
# .github/workflows/cd.yml
name: CD

on:
  push:
    branches: [main]

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  build-and-push:
    name: Build & Push Docker Image
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write

    outputs:
      image-tag: ${{ steps.meta.outputs.tags }}
      image-digest: ${{ steps.build.outputs.digest }}

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Login to GitHub Container Registry
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract Docker metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}
          tags: |
            type=sha,prefix=sha-
            type=raw,value=latest,enable={{is_default_branch}}
            type=semver,pattern={{version}}

      - name: Build and push Docker image
        id: build
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          labels: ${{ steps.meta.outputs.labels }}
          cache-from: type=gha   # GitHub Actions cache
          cache-to: type=gha,mode=max
          target: production

  deploy:
    name: Deploy to Railway
    runs-on: ubuntu-latest
    needs: build-and-push
    environment: production  # requires manual approval (configure in GitHub settings)

    steps:
      - name: Install Railway CLI
        run: npm install -g @railway/cli

      - name: Deploy to Railway
        run: railway up --service api --detach
        env:
          RAILWAY_TOKEN: ${{ secrets.RAILWAY_TOKEN }}

      - name: Wait for deployment
        run: |
          echo "Waiting 30s for deployment to complete..."
          sleep 30

      - name: Health check
        run: |
          STATUS=$(curl -s -o /dev/null -w "%{http_code}" ${{ secrets.PRODUCTION_URL }}/health)
          if [ "$STATUS" != "200" ]; then
            echo "Health check failed with status $STATUS"
            exit 1
          fi
          echo "Health check passed: $STATUS"
```

### 4. Security Scanning Workflow

```yaml
# .github/workflows/security.yml
name: Security Scan

on:
  push:
    branches: [main]
  schedule:
    - cron: '0 2 * * 1'  # Every Monday at 2am

jobs:
  dependency-audit:
    name: npm audit
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
      - run: npm ci
      - run: npm audit --audit-level=high

  docker-scan:
    name: Trivy Docker scan
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Build image for scanning
        run: docker build -t scan-target:latest .
      - name: Run Trivy scan
        uses: aquasecurity/trivy-action@master
        with:
          image-ref: scan-target:latest
          format: sarif
          output: trivy-results.sarif
          exit-code: '1'
          severity: CRITICAL,HIGH

      - name: Upload scan results
        uses: github/codeql-action/upload-sarif@v3
        if: always()
        with:
          sarif_file: trivy-results.sarif
```

### 5. Branch Protection Setup

In GitHub repository settings → Branches → Add rule for `main`:
- Require status checks to pass before merging
- Select: `Lint & Test` (from ci.yml)
- Require branches to be up to date before merging
- Require pull request reviews before merging
