**Type:** Assignment

## Goal
Implement a flexible role-based access control (RBAC) system with middleware guards and resource ownership checks.

## Background

### Authentication vs Authorization
Authentication answers "who are you?" — verifying identity via password, token, or OAuth. Authorization answers "what can you do?" — checking whether an authenticated identity has permission to perform an action. These are separate concerns and should be implemented as separate middleware layers.

### Role-Based Access Control (RBAC)
In RBAC, permissions are assigned to roles, and roles are assigned to users. A user with role `admin` can do everything; a `moderator` can moderate content; a `user` can only manage their own resources. This is simpler than attribute-based access control (ABAC) and sufficient for most applications.

Design roles as a hierarchy or flat set depending on complexity. For flat sets: check if `req.user.role` is in an allowed list. For hierarchies: assign numeric levels (user=1, moderator=2, admin=3) and check `>=`.

### Resource Ownership
Role checks aren't always enough. A `user` role should be able to edit their own posts but not others'. This is ownership-based authorization. The check is typically: `resource.userId === req.user.id`. Implement this as a middleware factory that receives the model and ownership field.

### Audit Logging
Sensitive actions (admin operations, permission changes, data exports) should be logged with who performed them, when, and what the result was. This creates an audit trail for security reviews and compliance.

### 1. Role Definitions

```js
// src/config/roles.js
const ROLES = {
  USER: 'user',
  MODERATOR: 'moderator',
  ADMIN: 'admin',
};

// Role hierarchy — higher level includes lower permissions
const ROLE_LEVELS = {
  [ROLES.USER]: 1,
  [ROLES.MODERATOR]: 2,
  [ROLES.ADMIN]: 3,
};

// Fine-grained permissions per role
const PERMISSIONS = {
  [ROLES.USER]: [
    'posts:read',
    'posts:create',
    'posts:update:own',
    'posts:delete:own',
    'profile:read:own',
    'profile:update:own',
  ],
  [ROLES.MODERATOR]: [
    'posts:read',
    'posts:create',
    'posts:update:own',
    'posts:update:any',  // can edit any post
    'posts:delete:own',
    'posts:delete:any',
    'profile:read:any',
    'comments:moderate',
  ],
  [ROLES.ADMIN]: [
    'posts:read',
    'posts:create',
    'posts:update:any',
    'posts:delete:any',
    'users:read:any',
    'users:update:any',
    'users:delete:any',
    'users:ban',
    'admin:dashboard',
  ],
};

const hasPermission = (role, permission) => {
  return PERMISSIONS[role]?.includes(permission) ?? false;
};

module.exports = { ROLES, ROLE_LEVELS, PERMISSIONS, hasPermission };
```

### 2. Authorization Middleware

```js
// src/middleware/authorize.js
const { ROLE_LEVELS, hasPermission } = require('../config/roles');

// Check if user has one of the allowed roles
const requireRole = (...allowedRoles) => (req, res, next) => {
  if (!req.user) {
    return res.status(401).json({ error: 'Authentication required' });
  }

  if (!allowedRoles.includes(req.user.role)) {
    return res.status(403).json({
      error: 'Insufficient role',
      required: allowedRoles,
      current: req.user.role,
    });
  }

  next();
};

// Check if user's role level meets minimum requirement
const requireRoleLevel = (minimumRole) => (req, res, next) => {
  if (!req.user) {
    return res.status(401).json({ error: 'Authentication required' });
  }

  const userLevel = ROLE_LEVELS[req.user.role] ?? 0;
  const requiredLevel = ROLE_LEVELS[minimumRole] ?? 999;

  if (userLevel < requiredLevel) {
    return res.status(403).json({
      error: 'Insufficient permissions',
      requiredLevel: minimumRole,
      yourLevel: req.user.role,
    });
  }

  next();
};

// Check specific permission
const requirePermission = (permission) => (req, res, next) => {
  if (!req.user) {
    return res.status(401).json({ error: 'Authentication required' });
  }

  if (!hasPermission(req.user.role, permission)) {
    return res.status(403).json({
      error: `Permission denied: ${permission}`,
    });
  }

  next();
};

module.exports = { requireRole, requireRoleLevel, requirePermission };
```

### 3. Ownership Guard

```js
// src/middleware/ownershipGuard.js
const postStore = require('../data/posts');

// Generic ownership check factory
const requireOwnership = (getResource, ownerField = 'userId') => async (req, res, next) => {
  try {
    const resource = await getResource(req);

    if (!resource) {
      return res.status(404).json({ error: 'Resource not found' });
    }

    const isOwner = resource[ownerField] === req.user?.id;
    const isAdmin = req.user?.role === 'admin';
    const isModerator = req.user?.role === 'moderator';

    if (!isOwner && !isAdmin && !isModerator) {
      return res.status(403).json({
        error: 'You do not have permission to modify this resource',
      });
    }

    req.resource = resource; // attach for use in controller
    next();
  } catch (err) {
    next(err);
  }
};

// Post-specific ownership check
const requirePostOwnership = requireOwnership(
  (req) => postStore.findById(parseInt(req.params.id)),
  'userId'
);

module.exports = { requireOwnership, requirePostOwnership };
```

### 4. Audit Log Middleware

```js
// src/middleware/auditLog.js
const auditLogs = []; // In production: write to DB or logging service

const auditLog = (action) => async (req, res, next) => {
  const start = Date.now();

  // Log after response is sent
  res.on('finish', () => {
    const log = {
      timestamp: new Date().toISOString(),
      action,
      userId: req.user?.id ?? 'anonymous',
      userRole: req.user?.role ?? 'none',
      method: req.method,
      path: req.path,
      params: req.params,
      statusCode: res.statusCode,
      durationMs: Date.now() - start,
      ip: req.ip,
    };

    auditLogs.push(log);
    console.log('[AUDIT]', JSON.stringify(log));
  });

  next();
};

const getAuditLogs = () => auditLogs;

module.exports = { auditLog, getAuditLogs };
```

### 5. Protected Routes

```js
// src/routes/posts.js
const router = require('express').Router();
const authenticate = require('../middleware/authenticate');
const { requireRole, requirePermission } = require('../middleware/authorize');
const { requirePostOwnership } = require('../middleware/ownershipGuard');
const { auditLog } = require('../middleware/auditLog');
const postStore = require('../data/posts');

// Public
router.get('/', (req, res) => {
  res.json({ data: postStore.findAll() });
});

// Auth required
router.post('/',
  authenticate,
  requirePermission('posts:create'),
  async (req, res) => {
    const post = postStore.create({ ...req.body, userId: req.user.id });
    res.status(201).json(post);
  }
);

// Ownership check (owner OR moderator/admin can edit)
router.patch('/:id',
  authenticate,
  requirePostOwnership, // attaches req.resource
  async (req, res) => {
    const updated = postStore.update(req.resource.id, req.body);
    res.json(updated);
  }
);

router.delete('/:id',
  authenticate,
  requirePostOwnership,
  auditLog('post:delete'),
  async (req, res) => {
    postStore.delete(req.resource.id);
    res.json({ message: 'Post deleted' });
  }
);

module.exports = router;
```

```js
// src/routes/admin.js
const router = require('express').Router();
const authenticate = require('../middleware/authenticate');
const { requireRole } = require('../middleware/authorize');
const { auditLog, getAuditLogs } = require('../middleware/auditLog');
const userStore = require('../data/users');

// All admin routes require authentication AND admin role
router.use(authenticate);
router.use(requireRole('admin'));

router.get('/users', (req, res) => {
  res.json({ data: userStore.findAll() });
});

router.patch('/users/:id/role',
  auditLog('admin:change-role'),
  (req, res) => {
    const { role } = req.body;
    if (!['user', 'moderator', 'admin'].includes(role)) {
      return res.status(400).json({ error: 'Invalid role' });
    }
    const updated = userStore.updateRole(parseInt(req.params.id), role);
    if (!updated) return res.status(404).json({ error: 'User not found' });
    res.json(updated);
  }
);

router.delete('/users/:id',
  auditLog('admin:delete-user'),
  (req, res) => {
    const deleted = userStore.delete(parseInt(req.params.id));
    if (!deleted) return res.status(404).json({ error: 'User not found' });
    res.json({ message: 'User deleted' });
  }
);

router.get('/audit-logs', (req, res) => {
  res.json({ data: getAuditLogs() });
});

module.exports = router;
```

```bash
# Login as regular user
TOKEN_USER=$(curl -s -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@test.com","password":"password"}' | jq -r '.token')

# Login as admin
TOKEN_ADMIN=$(curl -s -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@test.com","password":"password"}' | jq -r '.token')

# Create a post as user
POST_ID=$(curl -s -X POST http://localhost:3000/api/posts \
  -H "Authorization: Bearer $TOKEN_USER" \
  -H "Content-Type: application/json" \
  -d '{"title":"My Post","body":"Content"}' | jq -r '.id')

# Edit own post (should work)
curl -X PATCH http://localhost:3000/api/posts/$POST_ID \
  -H "Authorization: Bearer $TOKEN_USER" \
  -H "Content-Type: application/json" \
  -d '{"title":"Updated"}'

# Try to access admin endpoint as user (should get 403)
curl http://localhost:3000/api/admin/users \
  -H "Authorization: Bearer $TOKEN_USER"

# Access admin endpoint as admin (should work)
curl http://localhost:3000/api/admin/users \
  -H "Authorization: Bearer $TOKEN_ADMIN"
```
