# P8.1 — Dockerized API with CI/CD

**Phase:** 8 — Deployment & DevOps  
**Topic:** Docker, GitHub Actions, deploy to Render  
**Type:** Project

---

## Goal

Containerize an Express API with a multi-stage Dockerfile, orchestrate it with docker-compose for local dev, and build a GitHub Actions pipeline that lints, tests, builds the Docker image, pushes to a registry, and deploys to Render.

## What to Build

- Multi-stage Dockerfile (builder → production; non-root user)
- `docker-compose.yml` for local dev with MongoDB and Redis
- `.dockerignore` to keep the image lean
- GitHub Actions workflow: lint → test → build & push image → deploy
- Environment variable management with Render env groups

## Architecture

```
dockerized-api/
├── src/
│   └── (your API code)
├── Dockerfile
├── .dockerignore
├── docker-compose.yml
├── docker-compose.test.yml
├── .github/
│   └── workflows/

```

## Implementation

```dockerfile
# Dockerfile
# ---- Stage 1: builder ----
FROM node:22-alpine AS builder
WORKDIR /app

# Copy package files and install ALL deps (including devDeps for build)
COPY package*.json ./
RUN npm ci

COPY . .
# If you have a build step (TypeScript compile, etc.):
# RUN npm run build

# ---- Stage 2: production ----
FROM node:22-alpine AS production
WORKDIR /app

# Create a non-root user
RUN addgroup -S appgroup && adduser -S appuser -G appgroup

# Copy only production node_modules from builder
COPY package*.json ./
RUN npm ci --omit=dev && npm cache clean --force

# Copy app source (or compiled output if using TypeScript)
COPY --from=builder /app/src ./src

# Own the files as non-root user
RUN chown -R appuser:appgroup /app
USER appuser

EXPOSE 4000

# Use exec form for proper signal handling (SIGTERM reaches Node, not shell)
CMD ["node", "src/server.js"]
```

```yaml
# docker-compose.yml  — local development
version: '3.9'

services:
  api:
    build:
      context: .
      target: production       # use the production stage
    ports:
      - '4000:4000'
    environment:
      NODE_ENV: development
      PORT: 4000
      MONGODB_URI: mongodb://mongo:27017/myapp
      REDIS_HOST: redis
      JWT_ACCESS_SECRET: localsecret
    volumes:
      - ./src:/app/src         # live reload in dev: mount src
    depends_on:
      mongo:
        condition: service_healthy
      redis:
        condition: service_healthy
    restart: unless-stopped

  mongo:
    image: mongo:7
    ports:
      - '27017:27017'
    volumes:
      - mongo_data:/data/db
    healthcheck:
      test: echo 'db.runCommand("ping").ok' | mongosh localhost:27017/test --quiet
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    ports:
      - '6379:6379'
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  mongo_data:
```

```yaml
# .github/workflows/ci.yml
name: CI / CD

on:
  push:
    branches: [main]
  pull_request:

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22', cache: npm }
      - run: npm ci
      - run: npm run lint

  test:
    runs-on: ubuntu-latest
    needs: lint
    services:
      mongo:
        image: mongo:7
        ports: ['27017:27017']
        options: --health-cmd "echo 'db.runCommand(\"ping\").ok' | mongosh localhost:27017/test --quiet" --health-interval 10s --health-timeout 5s --health-retries 5
      redis:
        image: redis:7-alpine
        ports: ['6379:6379']
        options: --health-cmd "redis-cli ping" --health-interval 10s --health-timeout 5s --health-retries 5
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '22', cache: npm }
      - run: npm ci
      - run: npm test -- --coverage
        env:
          MONGODB_URI: mongodb://localhost:27017/test
          REDIS_HOST: localhost
          JWT_ACCESS_SECRET: testsecret

  build-and-push:
    runs-on: ubuntu-latest
    needs: test
    if: github.ref == 'refs/heads/main'
    permissions:
      contents: read
      packages: write
    outputs:
      image-tag: ${{ steps.meta.outputs.tags }}
    steps:
      - uses: actions/checkout@v4

      - name: Log in to GHCR
        uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Docker metadata
        id: meta
        uses: docker/metadata-action@v5
        with:
          images: ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}
          tags: |
            type=sha,prefix=sha-
            type=raw,value=latest

      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: ${{ steps.meta.outputs.tags }}
          cache-from: type=gha
          cache-to: type=gha,mode=max

  deploy:
    runs-on: ubuntu-latest
    needs: build-and-push
    if: github.ref == 'refs/heads/main'
    steps:
      - name: Deploy to Render
        run: |
          curl -X POST "${{ secrets.RENDER_DEPLOY_HOOK_URL }}"
```

```yaml
# render.yaml  — Render Infrastructure as Code
services:
  - type: web
    name: my-express-api
    runtime: docker
    dockerfilePath: ./Dockerfile
    envVars:
      - key: NODE_ENV
        value: production
      - key: PORT
        value: 4000
      - key: MONGODB_URI
        fromService:
          type: pserv
          name: mongo
          envVarKey: MONGODB_URI
      - key: JWT_ACCESS_SECRET
        generateValue: true
    healthCheckPath: /health/live
    autoDeploy: true
```
