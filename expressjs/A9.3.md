## Goal

Decompose a monolith into focused services, communicate between them synchronously with axios and asynchronously with Bull queues and RabbitMQ, and understand message acknowledgment.

## Background

A microservice is a small, independently deployable service that owns its own data and communicates over the network. Two communication styles:
- **Synchronous** — HTTP/REST (axios): caller waits for a response. Simple but creates coupling.
- **Asynchronous** — message queues (Bull, RabbitMQ): fire and forget. More resilient, better for background work.

**Bull** uses Redis as its backing store — great for job queues within a single deployment. **RabbitMQ** is a dedicated message broker suited to multi-service fan-out and reliable delivery.

## Tasks

### 1. Service decomposition — what to separate

```
monolith/
├── users         → User Service  (auth, profiles)
├── orders        → Order Service (cart, checkout)
├── notifications → Notification Service (email, SMS push)
└── payments      → Payment Service (Stripe, invoices)

Each service:
  - Has its own database (no shared DB)
  - Exposes a REST or gRPC interface
  - Publishes events to a queue; subscribes to events it cares about
```

### 2. Inter-service HTTP with axios — timeout and retry

```js
// src/clients/userServiceClient.js
import axios from 'axios';
import axiosRetry from 'axios-retry';

const userServiceClient = axios.create({
  baseURL: process.env.USER_SERVICE_URL ?? 'http://user-service:3001',
  timeout: 3000,   // 3 s — fail fast; don't block the caller
  headers: {
    'x-internal-key': process.env.INTERNAL_API_KEY,
  },
});

// Retry up to 3 times on network errors or 5xx responses, with exponential back-off
axiosRetry(userServiceClient, {
  retries: 3,
  retryDelay: axiosRetry.exponentialDelay,
  retryCondition: (err) =>
    axiosRetry.isNetworkOrIdempotentRequestError(err) ||
    (err.response?.status >= 500 && err.response?.status < 600),
});

export const getUserById = async (userId) => {
  const { data } = await userServiceClient.get(`/users/${userId}`);
  return data;
};

export const verifyToken = async (token) => {
  const { data } = await userServiceClient.post('/auth/verify', { token });
  return data;
};
```

### 3. Bull queue — async job processing

```js
// Install: npm install bull ioredis
// src/queues/emailQueue.js
import Bull from 'bull';

export const emailQueue = new Bull('email', {
  redis: { host: process.env.REDIS_HOST ?? 'localhost', port: 6379 },
  defaultJobOptions: {
    attempts: 3,
    backoff: { type: 'exponential', delay: 2000 },
    removeOnComplete: 100,    // keep last 100 completed jobs
    removeOnFail: false,      // keep failed jobs for inspection
  },
});

// Producer — call this from order-service after checkout
export const enqueueOrderConfirmation = (payload) =>
  emailQueue.add('order-confirmation', payload, { priority: 1 });

// Consumer — runs in notification-service
emailQueue.process('order-confirmation', async (job) => {
  const { orderId, userEmail, items } = job.data;
  console.log(`[${job.id}] Sending order confirmation to ${userEmail}`);

  await sendEmail({
    to: userEmail,
    subject: `Order #${orderId} confirmed`,
    template: 'order-confirmation',
    context: { orderId, items },
  });

  // Returning resolves the job; throwing retries it (up to `attempts`)
});

emailQueue.on('completed', (job) => console.log(`Job ${job.id} done`));
emailQueue.on('failed', (job, err) => console.error(`Job ${job.id} failed:`, err.message));
```

### 4. RabbitMQ with amqplib — produce and consume

```js
// Install: npm install amqplib
// src/messaging/connection.js
import amqplib from 'amqplib';

let channel;

export const getChannel = async () => {
  if (channel) return channel;
  const conn = await amqplib.connect(process.env.RABBITMQ_URL ?? 'amqp://localhost');
  channel = await conn.createChannel();
  return channel;
};
```

```js
// src/messaging/producer.js — order-service publishes order.created
import { getChannel } from './connection.js';

const EXCHANGE = 'orders';

export const publishOrderCreated = async (order) => {
  const ch = await getChannel();

  // Declare a durable topic exchange (survives broker restart)
  await ch.assertExchange(EXCHANGE, 'topic', { durable: true });

  const routingKey = 'order.created';
  const payload = Buffer.from(JSON.stringify(order));

  // persistent: true — message survives broker restart
  ch.publish(EXCHANGE, routingKey, payload, { persistent: true });
  console.log(`Published ${routingKey}:`, order.id);
};
```

```js
// src/messaging/consumer.js — notification-service subscribes
import { getChannel } from './connection.js';

const EXCHANGE = 'orders';
const QUEUE    = 'notification-service.order.created';

export const startConsumer = async () => {
  const ch = await getChannel();

  await ch.assertExchange(EXCHANGE, 'topic', { durable: true });

  // Durable queue — survives broker restart; exclusive: false — shared across instances
  await ch.assertQueue(QUEUE, { durable: true });
  await ch.bindQueue(QUEUE, EXCHANGE, 'order.created');

  // prefetch(1) — process one message at a time; prevents overwhelming the worker
  ch.prefetch(1);

  ch.consume(QUEUE, async (msg) => {
    if (!msg) return;

    try {
      const order = JSON.parse(msg.content.toString());
      console.log('Received order.created:', order.id);

      await sendOrderConfirmationEmail(order);

      // Acknowledge — remove from queue
      ch.ack(msg);
    } catch (err) {
      console.error('Processing failed:', err.message);
      // nack + requeue: false — send to dead-letter queue instead of infinite retry
      ch.nack(msg, false, false);
    }
  });

  console.log(`Consumer listening on ${QUEUE}`);
};
```

### 5. Dead-letter queue setup

```js
// Declare main queue with a dead-letter exchange
await ch.assertExchange('orders.dlx', 'direct', { durable: true });
await ch.assertQueue('notification-service.order.created', {
  durable: true,
  arguments: {
    'x-dead-letter-exchange': 'orders.dlx',
    'x-dead-letter-routing-key': 'order.created.failed',
    'x-message-ttl': 60_000,    // also expire after 60 s if not consumed
  },
});

await ch.assertQueue('dead-letter.order.created', { durable: true });
await ch.bindQueue('dead-letter.order.created', 'orders.dlx', 'order.created.failed');
```

### 6. Health check endpoint for each service

```js
// Each service exposes this so an orchestrator (k8s, ECS) can probe it
app.get('/health', async (_req, res) => {
  const checks = {
    db: 'ok',
    queue: 'ok',
  };

  try { await mongoose.connection.db.admin().ping(); } catch { checks.db = 'error'; }

  const healthy = Object.values(checks).every((v) => v === 'ok');
  res.status(healthy ? 200 : 503).json({ status: healthy ? 'ok' : 'degraded', checks });
});
```
