# A9.1 — WebSockets with Socket.io

**Phase:** 9 — Advanced Patterns  
**Topic:** Real-time bidirectional communication with Socket.io  
**Type:** Assignment

---

## Goal

Set up Socket.io with an Express server, handle events, implement rooms and namespaces, authenticate socket connections, and scale with a Redis adapter.

## Background

HTTP is request-response: the client always initiates. WebSockets flip this — the server can push data at any time over a persistent TCP connection. Socket.io wraps raw WebSockets with fallbacks, rooms, namespaces, and event emitters, making real-time features straightforward to build.

Key concepts:
- **Socket** — represents one connected client
- **Room** — a named group; a socket can join many rooms
- **Namespace** — a virtual channel on the same connection (e.g. `/chat`, `/admin`)
- **Redis adapter** — synchronises events across multiple server processes

## Tasks

### 1. Install dependencies

```bash
npm install socket.io @socket.io/redis-adapter ioredis
```

### 2. Attach Socket.io to an HTTP server

```js
// server.js
import express from 'express';
import { createServer } from 'node:http';
import { Server } from 'socket.io';
import { createClient } from 'ioredis';
import { createAdapter } from '@socket.io/redis-adapter';

const app = express();
const httpServer = createServer(app);

const io = new Server(httpServer, {
  cors: { origin: 'http://localhost:3000', methods: ['GET', 'POST'] },
});

// Redis adapter for multi-process scaling
const pubClient = createClient({ host: 'localhost', port: 6379 });
const subClient = pubClient.duplicate();
await Promise.all([pubClient.connect(), subClient.connect()]);
io.adapter(createAdapter(pubClient, subClient));

httpServer.listen(4000, () => console.log('Listening on :4000'));
```

### 3. Handle connection and basic events

```js
io.on('connection', (socket) => {
  console.log(`Client connected: ${socket.id}`);

  // Receive an event from the client
  socket.on('message', (data) => {
    console.log('Received:', data);
    // Emit back only to the sender
    socket.emit('message:ack', { received: true, id: data.id });
  });

  // Broadcast to all connected clients except sender
  socket.on('announcement', (text) => {
    socket.broadcast.emit('announcement', { text, from: socket.id });
  });

  socket.on('disconnect', (reason) => {
    console.log(`Client disconnected: ${socket.id} — ${reason}`);
  });
});
```

### 4. Rooms — join, leave, broadcast to room

```js
io.on('connection', (socket) => {
  // Client sends: socket.emit('room:join', 'general')
  socket.on('room:join', async (roomName) => {
    await socket.join(roomName);

    // Notify everyone else in the room
    socket.to(roomName).emit('room:user-joined', {
      userId: socket.data.userId,
      room: roomName,
    });

    // Confirm to the joining socket
    socket.emit('room:joined', { room: roomName });
  });

  socket.on('room:leave', async (roomName) => {
    await socket.leave(roomName);
    socket.to(roomName).emit('room:user-left', { userId: socket.data.userId });
  });

  socket.on('room:message', ({ room, text }) => {
    // Emit to everyone in the room including sender
    io.to(room).emit('room:message', {
      from: socket.data.userId,
      text,
      timestamp: Date.now(),
    });
  });
});
```

### 5. Namespaces

```js
// /chat namespace
const chatNS = io.of('/chat');

chatNS.on('connection', (socket) => {
  console.log(`Chat namespace: ${socket.id}`);
  socket.on('msg', (data) => chatNS.emit('msg', data));
});

// /admin namespace — restricted
const adminNS = io.of('/admin');

adminNS.use((socket, next) => {
  const token = socket.handshake.auth.token;
  if (token !== process.env.ADMIN_SECRET) {
    return next(new Error('Unauthorized'));
  }
  next();
});

adminNS.on('connection', (socket) => {
  socket.emit('welcome', { msg: 'Admin connected' });
});
```

### 6. Authentication middleware using socket.handshake

```js
import jwt from 'jsonwebtoken';

// Runs before the 'connection' event fires
io.use(async (socket, next) => {
  const token = socket.handshake.auth.token;      // client passes: { auth: { token } }
  if (!token) return next(new Error('Authentication required'));

  try {
    const payload = jwt.verify(token, process.env.JWT_SECRET);
    socket.data.userId = payload.sub;
    socket.data.username = payload.username;
    next();
  } catch {
    next(new Error('Invalid token'));
  }
});

io.on('connection', (socket) => {
  // socket.data.userId is guaranteed to exist here
  console.log(`Authenticated user ${socket.data.userId} connected`);
});
```

### 7. Client-side example (browser / Node test client)

```js
// client.js (browser or node with socket.io-client)
import { io } from 'socket.io-client';

const socket = io('http://localhost:4000', {
  auth: { token: 'your-jwt-here' },
});

socket.on('connect', () => console.log('Connected:', socket.id));
socket.on('connect_error', (err) => console.error('Auth error:', err.message));

socket.emit('room:join', 'general');
socket.on('room:message', (msg) => console.log(msg));
socket.emit('room:message', { room: 'general', text: 'Hello!' });
```

### 8. Scaling: emit to all servers via Redis adapter

```js
// With the Redis adapter in place, io.to() / io.emit() automatically
// forwards events to all Socket.io server instances.

// Emit from any worker to all clients in a room
io.to('general').emit('announcement', { text: 'Server restarting in 5 min' });

// Get all socket IDs in a room (works across processes with Redis adapter)
const sockets = await io.in('general').fetchSockets();
console.log(`Users in general: ${sockets.length}`);
```
